## 一、概念讲解

### 什么是DOM？

DOM（Document Object Model）是浏览器将HTML文档解析后生成的一棵**树形结构**。每个HTML标签对应一个**节点（Node）**，JavaScript通过操作这些节点来改变页面内容、结构和样式。

```
document
  └── html
       ├── head
       │    ├── title
       │    └── meta
       └── body
            ├── header
            │    └── h1
            ├── main
            │    ├── section
            │    └── article
            └── footer
```

**核心概念：**
- **节点（Node）**：DOM树的基本单元，包括元素节点、文本节点、注释节点等
- **元素（Element）**：特指HTML标签对应的节点，是最常操作的类型
- **文档（Document）**：DOM树的根节点，提供查询和创建API

### 事件机制三阶段

事件传播经历三个阶段：

1. **捕获阶段（Capturing）**：事件从 `window` 向目标元素传播
2. **目标阶段（Target）**：事件到达触发元素
3. **冒泡阶段（Bubbling）**：事件从目标元素向上冒泡到 `window`

### 3.2 DOM创建与修改

```javascript
// v1: Basic element creation
function createElement(tag, text, className) {
  const el = document.createElement(tag);
  el.textContent = text;
  el.className = className;
  return el;
}

// v2: Fluent API with attributes and events
function h(tag, props = {}, ...children) {
  const el = document.createElement(tag);
  for (const [key, value] of Object.entries(props)) {
    if (key.startsWith('on') && typeof value === 'function') {
      el.addEventListener(key.slice(2).toLowerCase(), value);
    } else if (key === 'className') {
      el.className = value;
    } else if (key === 'style' && typeof value === 'object') {
      Object.assign(el.style, value);
    } else {
      el.setAttribute(key, value);
    }
  }
  children.flat().forEach(child => {
    el.appendChild(typeof child === 'string' ? document.createTextNode(child) : child);
  });
  return el;
}

// Usage
const card = h('div', { className: 'card', style: { color: 'red' } },
  h('h2', {}, 'Title'),
  h('p', {}, 'Description'),
  h('button', { onClick: () => alert('Clicked!') }, 'Click Me')
);
document.body.appendChild(card);
```

### 3.3 事件系统完整示例

```javascript
// Event delegation pattern
class EventManager {
  constructor(root = document) {
    this.root = root;
    this.handlers = new Map();
  }

  // Delegate event handling to root element
  on(selector, event, handler) {
    const wrapper = (e) => {
      const target = e.target.closest(selector);
      if (target && this.root.contains(target)) {
        handler.call(target, e, target);
      }
    };
    this.root.addEventListener(event, wrapper);
    const key = `${selector}:${event}`;
    this.handlers.set(key, wrapper);
    return () => this.off(selector, event);
  }

  off(selector, event) {
    const key = `${selector}:${event}`;
    const wrapper = this.handlers.get(key);
    if (wrapper) {
      this.root.removeEventListener(event, wrapper);
      this.handlers.delete(key);
    }
  }

  once(selector, event, handler) {
    const remove = this.on(selector, event, (...args) => {
      handler(...args);
      remove();
    });
    return remove;
  }
}

// Usage: efficient event delegation
const em = new EventManager(document.getElementById('todo-list'));
em.on('.delete-btn', 'click', (e, btn) => {
  const li = btn.closest('li');
  li.remove();
});
em.on('.toggle-btn', 'click', (e, btn) => {
  const li = btn.closest('li');
  li.classList.toggle('completed');
});
```

### 3.4 MutationObserver 监听DOM变化

```javascript
// Watch for DOM changes
const observer = new MutationObserver((mutations) => {
  for (const mutation of mutations) {
    if (mutation.type === 'childList') {
      console.log('Added:', mutation.addedNodes.length, 'nodes');
      console.log('Removed:', mutation.removedNodes.length, 'nodes');
    }
    if (mutation.type === 'attributes') {
      console.log(`Attribute ${mutation.attributeName} changed`);
    }
  }
});

observer.observe(document.body, {
  childList: true,     // watch for added/removed children
  subtree: true,       // watch entire subtree
  attributes: true,    // watch attribute changes
  attributeFilter: ['class', 'data-status'] // only these attributes
});

// Stop observing
// observer.disconnect();
```

## 五、注意事项

| 场景 | 注意点 | 建议 |
|------|--------|------|
| `querySelectorAll` | 返回**静态** NodeList，不是实时的 | 需要实时集合用 `getElementsByClassName` |
| `innerHTML` | 存在XSS风险，性能差 | 优先用 `textContent` 或 DOM API |
| 事件绑定 | 同一元素同一事件可绑定多个监听器 | 注意 `removeEventListener` 需同一引用 |
| 事件委托 | 依赖冒泡机制 | `focus`/`blur` 不冒泡，需用 `focusin`/`focusout` |
| `closest()` | 从自身开始向上查找 | 不会查找后代元素 |
| DocumentFragment | 批量DOM操作的最佳实践 | 减少 reflow 次数 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 | 原因 |
|------------|-----------|------|
| `el.innerHTML = userInput` | `el.textContent = userInput` | 防止XSS注入 |
| 循环内逐个 `appendChild` | 先拼 `DocumentFragment`，一次插入 | 减少reflow |
| `onclick = fn` 赋值 | `addEventListener('click', fn)` | 可绑定多个，更灵活 |
| 100个li各绑定click | 父元素事件委托1个监听器 | 内存和性能友好 |
| `e.stopPropagation()` 随意调用 | 只在确实需要时阻止 | 避免破坏其他功能 |
| `getElementById` 返回null直接用 | 先判空再操作 | 避免 TypeError |

## 九、举一反三

| 技术 | 相似概念 | 迁移思路 |
|------|---------|---------|
| DOM事件 | Node.js EventEmitter | 同为发布订阅模式，DOM多了冒泡捕获 |

## 十一、代码演进

### v1: 基础DOM操作

```javascript
// Simple direct DOM manipulation
const list = document.getElementById('myList');
items.forEach(text => {
  const li = document.createElement('li');
  li.textContent = text;
  li.addEventListener('click', () => li.classList.toggle('active'));
  list.appendChild(li);
});
```

### v2: 事件委托 + DocumentFragment

```javascript
// Event delegation + fragment for performance
const fragment = document.createDocumentFragment();
items.forEach((text, i) => {
  const li = document.createElement('li');
  li.textContent = text;
  li.dataset.index = i;
  fragment.appendChild(li);
});
list.appendChild(fragment);

list.addEventListener('click', (e) => {
  const li = e.target.closest('li');
  if (li) li.classList.toggle('active');
});
```

### v3: 封装为可复用组件

```javascript
// Reusable component with EventManager
class ListComponent {
  constructor(container, items) {
    this.container = container;
    this.em = new EventManager(container);
    this.render(items);
  }
  render(items) {
    const fragment = document.createDocumentFragment();
    items.forEach((text, i) => {
      const li = document.createElement('li');
      li.textContent = text;
      li.dataset.index = i;
      fragment.appendChild(li);
    });
    this.container.appendChild(fragment);
  }
  onItemClick(handler) {
    this.em.on('li', 'click', (e, li) => handler(li, +li.dataset.index));
  }
}
```

## 十二、总结

DOM操作是前端开发的基础功。掌握**查询、创建、修改、删除**四大操作，理解**事件捕获-冒泡机制**，善用**事件委托**减少监听器数量，利用**Observer API**实现响应式监听——这些是高效DOM编程的核心。
