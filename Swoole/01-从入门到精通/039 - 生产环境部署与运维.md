# 生产环境部署与运维

> **难度：⭐⭐⭐⭐高级** | 预计阅读时间：22分钟

## 一、概念讲解

Swoole应用是常驻内存的CLI进程，与传统PHP-FPM的"请求-销毁"模式完全不同。这意味着部署、监控、重启策略都需要重新设计。

### FPM vs Swoole部署差异

| 维度 | PHP-FPM | Swoole |
|------|---------|--------|
| 进程管理 | 系统服务（systemd） | 需要自己管理或用Supervisor |
| 代码更新 | 自动生效（每次加载） | 需要reload Worker |
| 日志 | 自动写到文件 | 需要配置log_file |
| 崩溃恢复 | FPM自动拉起 | 需要进程守护 |
| 优雅重启 | kill -USR2 | Server::reload() |
| Docker | php-fpm镜像 | 自定义Entrypoint |

### v1: Supervisor守护进程

```ini
; /etc/supervisor/conf.d/swoole-api.conf

[program:swoole-api]
command=/usr/bin/php /var/www/api/server.php
directory=/var/www/api
user=www-data
numprocs=1
autostart=true
autorestart=true
startretries=10
redirect_stderr=true
stdout_logfile=/var/log/swoole/api.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=10
stopsignal=SIGTERM
stopasgroup=true
killasgroup=true

; Health check - if process exits within 5s, something is wrong
startsecs=5

; Environment variables
environment=APP_ENV="production",APP_DEBUG="false"
```

```php
<?php
// server.php - Production-ready Swoole HTTP server

// Validate environment before starting
if (php_sapi_name() !== 'cli') {
    die("This script must be run from CLI.\n");
}

if (!extension_loaded('swoole')) {
    die("Swoole extension is required.\n");
}

// Bootstrap application (e.g., Laravel/ThinkPHP)
require __DIR__ . '/vendor/autoload.php';

$server = new Swoole\Http\Server("0.0.0.0", 9501, SWOOLE_PROCESS);

$server->set([
    'worker_num' => swoole_cpu_num() * 2,
    'max_request' => 3000,
    'max_coroutine' => 50000,
    'daemonize' => 0, // Let Supervisor manage daemonization
    'pid_file' => '/var/run/swoole-api.pid',
    'log_file' => '/var/log/swoole/server.log',
    'log_level' => SWOOLE_LOG_WARNING,
    'backlog' => 65535,
    'open_tcp_nodelay' => true,
    'enable_static_handler' => true,
    'document_root' => __DIR__ . '/public',
    'hook_flags' => SWOOLE_HOOK_ALL,
]);

$server->on('workerStart', function ($server, $workerId) {
    // Warm up connections, preload data
    echo "[Worker #{$workerId}] Started at " . date('Y-m-d H:i:s') . "\n";
});

$server->on('request', function ($request, $response) {
    // Handle request through your framework
    $response->header('Content-Type', 'application/json');
    $response->end(json_encode(['status' => 'ok', 'time' => time()]));
});

$server->on('workerError', function ($server, $workerId, $workerPid, $exitCode, $signal) {
    error_log("[Worker Error] ID:{$workerId} PID:{$workerPid} ExitCode:{$exitCode} Signal:{$signal}");
});

$server->start();
```

### v2: Systemd服务 + 部署脚本

```ini
; /etc/systemd/system/swoole-api.service

[Unit]
Description=Swoole API Server
After=network.target mysql.service redis.service
Wants=mysql.service redis.service

[Service]
Type=forking
PIDFile=/var/run/swoole-api.pid
User=www-data
Group=www-data
WorkingDirectory=/var/www/api
ExecStart=/usr/bin/php /var/www/api/server.php
ExecReload=/bin/kill -USR1 $MAINPID
ExecStop=/bin/kill -SIGTERM $MAINPID
Restart=on-failure
RestartSec=5
LimitNOFILE=1048576
LimitNPROC=65535

; Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/log/swoole /var/run /var/www/api/storage

[Install]
WantedBy=multi-user.target
```

```bash
#!/bin/bash
# deploy.sh - Zero-downtime deployment script

set -euo pipefail

APP_DIR="/var/www/api"
BACKUP_DIR="/var/www/api-backups"
CURRENT_VERSION=$(cat "$APP_DIR/version.txt" 2>/dev/null || echo "unknown")
NEW_VERSION="${1:-$(date +%Y%m%d%H%M%S)}"

echo "=== Deploying $NEW_VERSION (current: $CURRENT_VERSION) ==="

# 1. Backup current version
echo "[1/6] Backing up..."
cp -a "$APP_DIR" "$BACKUP_DIR/api-$CURRENT_VERSION"

# 2. Pull latest code
echo "[2/6] Updating code..."
cd "$APP_DIR"
git fetch origin main
git checkout "main"
git pull origin main

# 3. Install dependencies
echo "[3/6] Installing dependencies..."
composer install --no-dev --optimize-autoloader --classmap-authoritative

# 4. Run migrations
echo "[4/6] Running migrations..."
php artisan migrate --force

# 5. Warm caches
echo "[5/6] Warming caches..."
php artisan config:cache
php artisan route:cache
php artisan view:cache

# 6. Graceful reload
echo "[6/6] Reloading Swoole workers..."
if [ -f /var/run/swoole-api.pid ]; then
    # Send USR1 signal for graceful reload
    kill -USR1 "$(cat /var/run/swoole-api.pid)"
    echo "Workers reloading gracefully."
else
    echo "PID file not found, starting service..."
    systemctl start swoole-api
fi

# Save version
echo "$NEW_VERSION" > "$APP_DIR/version.txt"
echo "=== Deploy complete: $NEW_VERSION ==="
```

### v3: Docker + 监控

```dockerfile
# Dockerfile - Multi-stage build for Swoole
FROM php:8.3-cli AS builder

# Install Swoole
RUN pecl install swoole && docker-php-ext-enable swoole

# Install Composer
COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

WORKDIR /app
COPY composer.* ./
RUN composer install --no-dev --optimize-autoloader --no-scripts

COPY . .
RUN composer dump-autoload --optimize

# Production image
FROM php:8.3-cli-alpine

RUN apk add --no-cache libstdc++ && \
    pecl install swoole && docker-php-ext-enable swoole

COPY --from=builder /app /app
COPY php-production.ini /usr/local/etc/php/php.ini

WORKDIR /app
EXPOSE 9501

# Health check
HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD php -r "file_get_contents('http://localhost:9501/health')" || exit 1

# Non-root user
RUN addgroup -g 1000 app && adduser -u 1000 -G app -s /bin/sh -D app
USER app

CMD ["php", "server.php"]
```

```yaml
# docker-compose.yml
version: '3.8'

services:
  swoole-api:
    build: .
    ports:
      - "9501:9501"
    environment:
      APP_ENV: production
      DB_HOST: mysql
      REDIS_HOST: redis
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }
    deploy:
      replicas: 2
      resources:
        limits: { cpus: '2.0', memory: 512M }
        reservations: { cpus: '0.5', memory: 128M }
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "php", "-r", "file_get_contents('http://localhost:9501/health');"]
      interval: 10s
      timeout: 3s

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - swoole-api

  prometheus:
    image: prom/prometheus
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml:ro
    ports:
      - "9090:9090"
```

```php
<?php
// app/Metrics/PrometheusMetrics.php - Metrics collection

class PrometheusMetrics
{
    private array $counters = [];
    private array $gauges = [];
    private string $prefix;
    
    public function __construct(string $prefix = 'swoole_')
    {
        $this->prefix = $prefix;
    }
    
    public function incCounter(string $name, array $labels = []): void
    {
        $key = $this->labelKey($name, $labels);
        $this->counters[$key] = ($this->counters[$key] ?? 0) + 1;
    }
    
    public function setGauge(string $name, float $value, array $labels = []): void
    {
        $key = $this->labelKey($name, $labels);
        $this->gauges[$key] = $value;
    }
    
    public function render(): string
    {
        $output = "";
        foreach ($this->counters as $key => $value) {
            $output .= "# TYPE {$this->prefix}{$key} counter\n";
            $output .= "{$this->prefix}{$key} {$value}\n";
        }
        foreach ($this->gauges as $key => $value) {
            $output .= "# TYPE {$this->prefix}{$key} gauge\n";
            $output .= "{$this->prefix}{$key} {$value}\n";
        }
        return $output;
    }
    
    private function labelKey(string $name, array $labels): string
    {
        return $name . ($labels ? '{' . http_build_query($labels, '', ',') . '}' : '');
    }
}
```

## 四、注意事项

| 项目 | 说明 |
|------|------|
| 进程守护 | 必须用Supervisor或Systemd，不能裸跑 |
| 优雅重启 | 用USR1信号，不是kill -9 |
| 日志轮转 | 配置logrotate，防止日志文件无限增长 |
| 健康检查 | 实现/health端点，供负载均衡器探活 |
| 端口冲突 | 重启前确认端口已释放 |
| 权限分离 | Swoole进程用非root用户运行 |

## 五、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|-------------|-------------|
| 直接kill -9杀进程 | 用SIGTERM优雅停止 |
| 每次部署全量重启 | 用USR1 reload，只重启Worker |
| 不设进程守护 | Supervisor或Systemd守护 |
| 日志打到stdout不管 | 配置日志文件+logrotate |
| Docker中用前台进程+daemonize=1 | 设daemonize=0让Docker管理 |
| 不做健康检查 | 实现/health端点+Docker HEALTHCHECK |
