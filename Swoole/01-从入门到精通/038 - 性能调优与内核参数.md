# 性能调优与内核参数

> **难度：⭐⭐⭐⭐高级** | 预计阅读时间：22分钟

## 一、概念讲解

Swoole应用在生产环境的性能不仅取决于代码质量，还与操作系统内核参数、Swoole配置、网络栈优化密切相关。一次系统的性能调优可以让吞吐量提升**2-5倍**。

### 调优层次模型

```
┌─────────────────────────────┐
│     应用层调优               │ ← 业务逻辑、数据库查询
├─────────────────────────────┤
│     Swoole配置调优           │ ← Worker数、协程数、缓冲区
├─────────────────────────────┤
│     PHP运行时调优            │ ← OPcache、JIT、内存限制
├─────────────────────────────┤
│     Linux内核参数            │ ← fd限制、TCP栈、backlog
├─────────────────────────────┤
│     硬件层                   │ ← CPU、内存、网卡、磁盘
└─────────────────────────────┘
```

## 三、代码演进

### v1: 内核参数优化脚本

```bash
#!/bin/bash
# v1_sysctl_tuning.sh - System kernel parameter tuning for Swoole
# Run as root

echo "=== Swoole System Tuning ==="

# === File Descriptors ===
# Max file descriptors for entire system
echo "fs.file-max = 1048576" >> /etc/sysctl.conf

# Allow core dumps
echo "fs.suid_dumpable = 0" >> /etc/sysctl.conf

# === TCP Stack Optimization ===
# Maximum connection queue (listen backlog)
echo "net.core.somaxconn = 65535" >> /etc/sysctl.conf

# Socket receive/send buffer
echo "net.core.rmem_max = 16777216" >> /etc/sysctl.conf
echo "net.core.wmem_max = 16777216" >> /etc/sysctl.conf
echo "net.core.rmem_default = 262144" >> /etc/sysctl.conf
echo "net.core.wmem_default = 262144" >> /etc/sysctl.conf

# TCP connection settings
echo "net.ipv4.tcp_max_syn_backlog = 65535" >> /etc/sysctl.conf
echo "net.ipv4.tcp_syncookies = 1" >> /etc/sysctl.conf
echo "net.ipv4.tcp_tw_reuse = 1" >> /etc/sysctl.conf
echo "net.ipv4.tcp_fin_timeout = 15" >> /etc/sysctl.conf
echo "net.ipv4.tcp_keepalive_time = 300" >> /etc/sysctl.conf
echo "net.ipv4.tcp_keepalive_intvl = 15" >> /etc/sysctl.conf
echo "net.ipv4.tcp_keepalive_probes = 3" >> /etc/sysctl.conf
echo "net.ipv4.tcp_max_tw_buckets = 65535" >> /etc/sysctl.conf

# Local port range (for outgoing connections)
echo "net.ipv4.ip_local_port_range = 1024 65535" >> /etc/sysctl.conf

# TCP memory
echo "net.ipv4.tcp_rmem = 4096 87380 16777216" >> /etc/sysctl.conf
echo "net.ipv4.tcp_wmem = 4096 65536 16777216" >> /etc/sysctl.conf

# Connection tracking (if using conntrack)
echo "net.netfilter.nf_conntrack_max = 1048576" >> /etc/sysctl.conf

# === Memory ===
# Minimize swap usage
echo "vm.swappiness = 1" >> /etc/sysctl.conf

# === Apply ===
sysctl -p

# === ulimit Settings ===
cat >> /etc/security/limits.conf << 'EOF'
* soft nofile 1048576
* hard nofile 1048576
* soft nproc 65535
* hard nproc 65535
root soft nofile 1048576
root hard nofile 1048576
EOF

echo "=== Done! Reboot recommended ==="
```

### v2: Swoole配置优化

```php
<?php
// v2_swoole_config.php - Optimized Swoole server configuration

$server = new Swoole\Http\Server("0.0.0.0", 9501, SWOOLE_PROCESS);

$cpuCores = swoole_cpu_num();

$server->set([
    // === Worker Configuration ===
    // IO-intensive: 2-4x CPU cores; CPU-intensive: 1x CPU cores
    'worker_num' => $cpuCores * 2,
    
    // Max coroutines per worker (default 100000, adjust per memory)
    'max_coroutine' => 50000,
    
    // Restart worker after N requests (prevent memory leaks)
    'max_request' => 3000,
    'max_request_grace' => 100, // Grace period before restart
    
    // Task workers for async tasks
    'task_worker_num' => $cpuCores,
    'task_enable_coroutine' => true,
    
    // === Network ===
    // Listen backlog (should <= net.core.somaxconn)
    'backlog' => 65535,
    
    // TCP keepalive
    'open_tcp_keepalive' => true,
    'tcp_keepidle' => 300,    // seconds before first probe
    'tcp_keepinterval' => 15, // seconds between probes
    'tcp_keepcount' => 3,     // probes before disconnect
    
    // Nagle algorithm (disable for low latency)
    'open_tcp_nodelay' => true,
    
    // === Buffer ===
    'socket_buffer_size' => 8 * 1024 * 1024, // 8MB per connection
    'package_max_length' => 5 * 1024 * 1024,  // 5MB max package
    
    // === Static Files ===
    'enable_static_handler' => true,
    'document_root' => '/var/www/public',
    'static_handler_locations' => ['/static', '/assets'],
    
    // === Process Management ===
    'daemonize' => true,
    'pid_file' => '/var/run/swoole.pid',
    'log_file' => '/var/log/swoole.log',
    'log_level' => SWOOLE_LOG_WARNING,
    
    // === Performance ===
    'send_yield' => true,          // Yield on send buffer full
    'heartbeat_check_interval' => 30,
    'heartbeat_idle_time' => 60,
    
    // === Memory ===
    'discard_timeout_request' => true, // Discard requests during reload
    
    // === Hooks ===
    'hook_flags' => SWOOLE_HOOK_ALL,  // Hook all blocking functions
]);

// Connection pool for MySQL
class MySQLPool
{
    private Swoole\Database\PDOPool $pool;
    
    public function __construct(int $size = 64)
    {
        $this->pool = new Swoole\Database\PDOPool(
            (new Swoole\Database\PDOConfig())
                ->withHost('127.0.0.1')
                ->withPort(3306)
                ->withDbName('myapp')
                ->withCharset('utf8mb4')
                ->withUsername('root')
                ->withPassword('secret'),
            $size
        );
    }
    
    public function get(): Swoole\Database\PDOProxy
    {
        return $this->pool->get();
    }
    
    public function put(Swoole\Database\PDOProxy $pdo): void
    {
        $this->pool->put($pdo);
    }
}

// Connection pool for Redis
class RedisPool
{
    private Swoole\Database\RedisPool $pool;
    
    public function __construct(int $size = 64)
    {
        $this->pool = new Swoole\Database\RedisPool(
            (new Swoole\Database\RedisConfig())
                ->withHost('127.0.0.1')
                ->withPort(6379)
                ->withAuth('secret')
                ->withDbIndex(0),
            $size
        );
    }
    
    public function get(): Swoole\Database\RedisProxy
    {
        return $this->pool->get();
    }
    
    public function put(Swoole\Database\RedisProxy $redis): void
    {
        $this->pool->put($redis);
    }
}

// Initialize pools on worker start
$mysqlPool = null;
$redisPool = null;

$server->on('workerStart', function ($server, $workerId) use (&$mysqlPool, &$redisPool) {
    $mysqlPool = new MySQLPool(64);
    $redisPool = new RedisPool(64);
});

$server->on('request', function ($request, $response) use (&$mysqlPool, &$redisPool) {
    $pdo = $mysqlPool->get();
    $redis = $redisPool->get();
    
    try {
        $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ? LIMIT 1');
        $stmt->execute([1]);
        $user = $stmt->fetch();
        
        $redis->set('user:1', json_encode($user));
        $response->end(json_encode($user));
    } finally {
        $mysqlPool->put($pdo);
        $redisPool->put($redis);
    }
});

$server->start();
```

## 四、注意事项

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `worker_num` | CPU×2（IO密集）/ CPU（CPU密集） | 不是越多越好 |
| `max_request` | 1000-5000 | 过低浪费重启开销，过高可能内存泄漏 |
| `backlog` | ≤`somaxconn` | 超过内核限制无效 |
| `socket_buffer_size` | 2-8MB | 过大浪费内存 |
| `max_coroutine` | 10000-100000 | 取决于可用内存 |
| `ulimit -n` | ≥65535 | 否则连接数上不去 |

## 五、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|-------------|-------------|
| `worker_num`设为CPU×8 | IO密集设×2-4，CPU密集设×1 |
| 忘记开启`opcache.enable_cli` | Swoole是CLI运行，必须启用 |
| `validate_timestamps=1`生产环境 | 设为0，用reload代替自动检测 |
| 不设`max_request` | 设置合理值防止内存泄漏 |
| `tcp_tw_reuse=1`加`tcp_tw_recycle=1` | 只开reuse，recycle在NAT环境有问题 |
| 忽略`send_yield` | 开启防止发送缓冲区满时阻塞 |

## 八、举一反三

| 瓶颈 | 诊断工具 | 优化方向 |
|------|----------|----------|
| CPU 100% | top + 火焰图 | 减少计算、缓存、加Worker |
| 连接数上不去 | ss -s + ulimit | 调fd限制、somaxconn |
| 延迟高 | wrk --latency | 连接池、减少IO |
| 内存持续增长 | memory_get_usage | max_request、排查泄漏 |
| 大量TIME_WAIT | ss -ant | tcp_tw_reuse、连接复用 |
| 频繁GC | php -i opcache | 增大buffer、减少对象创建 |
