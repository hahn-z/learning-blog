# Swoole实战教程001：简介与环境搭建

> **难度标注：** 🟢 入门级 | 预计学习时间：30分钟

## 一、概念讲解

### 什么是Swoole？

Swoole是一个基于C语言编写的PHP异步、并行、高性能网络通信引擎。它不是PHP框架，而是PHP的一个扩展（Extension），为PHP提供了：

- **异步I/O**：非阻塞的网络通信
- **协程（Coroutine）**：轻量级线程，以同步方式写异步代码
- **常驻内存**：代码加载一次，常驻运行，避免重复编译
- **多进程/多线程**：充分利用多核CPU

### Swoole vs 传统PHP-FPM

| 对比维度 | PHP-FPM | Swoole |
|---------|---------|--------|
| 运行模式 | 请求来→启动→销毁 | 常驻内存 |
| 性能 | 每次重新加载框架 | 框架只加载一次 |
| 并发模型 | 多进程阻塞 | 异步事件驱动+协程 |
| 适用场景 | Web页面渲染 | API、WebSocket、TCP服务 |
| 学习曲线 | 低 | 中高 |

### 核心架构简图

```
┌─────────────────────────────────────┐
│            Master Process           │
│  ┌───────────┐    ┌──────────────┐  │
│  │  Reactor   │    │  Heartbeat   │  │
│  │  Thread    │    │  Checker     │  │
│  │ (I/O多路复用)│    │              │  │
│  └─────┬─────┘    └──────────────┘  │
│        │                            │
│  ┌─────▼─────┐                      │
│  │  Manager   │                      │
│  │  Process   │                      │
│  └─────┬─────┘                      │
│   ┌────┼────┐                        │
│   ▼    ▼    ▼                        │
│ ┌───┐┌───┐┌───┐                     │
│ │W1 ││W2 ││W3 │  Worker Processes   │
│ └───┘└───┘└───┘                     │
└─────────────────────────────────────┘
```

**Master**：主进程，负责创建和管理子进程
**Reactor**：线程级别的I/O处理，基于epoll/kqueue
**Manager**：管理Worker进程的创建和回收
**Worker**：实际处理业务逻辑的进程

## 三、环境搭建

### 方式一：PECL安装（推荐）

```php
<?php
// Installation (run in terminal):
// pecl install swoole
//
// During installation, you will be asked:
// Enable openssl support? [no] : yes
// Enable http2 support? [no] : yes
// Enable mysqlnd support? [no] : yes

// After installation, add to php.ini:
// extension=swoole.so
```

### 方式二：源码编译

```bash
#!/bin/bash
# Install Swoole from source

# Step 1: Install dependencies
sudo apt-get update
sudo apt-get install -y php-dev libssl-dev libcurl4-openssl-dev

# Step 2: Download and compile
SWOOLE_VERSION="5.1.1"
cd /tmp
wget https://github.com/swoole/swoole-src/archive/refs/tags/v${SWOOLE_VERSION}.tar.gz
tar -xzf v${SWOOLE_VERSION}.tar.gz
cd swoole-src-${SWOOLE_VERSION}

phpize
./configure --enable-openssl --enable-http2 --enable-mysqlnd
make -j$(nproc)
sudo make install

# Step 3: Enable extension
echo "extension=swoole.so" | sudo tee /etc/php/mods-available/swoole.ini
sudo phpenmod swoole
```

### 方式三：Docker（最快捷）

```dockerfile
# Dockerfile for Swoole development
FROM phpswoole/swoole:5.1-php8.3

RUN docker-php-ext-install pdo_mysql pcntl
WORKDIR /app
COPY . /app
EXPOSE 9501
CMD ["php", "/app/server.php"]
```

### 验证安装

```php
<?php
// verify_swoole.php - Verify Swoole installation

if (!extension_loaded('swoole')) {
    echo "❌ Swoole extension not loaded!\n";
    exit(1);
}

echo "✅ Swoole extension loaded!\n\n";
echo "Swoole Version: " . swoole_version() . "\n";

$features = [
    'openssl'   => SWOOLE_HAVE_SSL,
    'http2'     => SWOOLE_USE_HTTP2,
    'coroutine' => class_exists('Swoole\\Coroutine'),
];

echo "\nFeature Check:\n";
foreach ($features as $name => $enabled) {
    echo ($enabled ? '  ✅' : '  ⚠️') . " {$name}\n";
}

Swoole\\Coroutine\\run(function () {
    echo "\n✅ Coroutine test passed!\n";
});
```

## 五、注意事项

| 事项 | 说明 | 建议 |
|------|------|------|
| PHP版本 | Swoole 5.x需要PHP 8.1+ | 推荐PHP 8.2/8.3 |
| 内存 | Worker常驻内存，注意内存泄漏 | 使用memory_get_usage()监控 |
| 全局变量 | Worker进程间不共享全局变量 | 使用Table/Atomic共享数据 |
| 热更新 | 修改代码不会自动生效 | 需要reload Worker或使用热更新工具 |
| 调试 | var_dump会输出到stdout | 建议用日志系统替代 |
| Composer | 需要autoload支持常驻内存 | 在WorkerStart中加载 |

## 六、避坑指南

### ❌ 坑1：在Worker中定义全局变量期望跨请求共享
```php
<?php
// ❌ Wrong: global variable per worker process only
$counter = 0;
$server->on('request', function ($req, $resp) use (&$counter) {
    $counter++;
    $resp->end("Count: $counter");
});
```

```php
<?php
// ✅ Correct: use Swoole\Atomic for shared counter
$counter = new Swoole\Atomic(0);
$server->on('request', function ($req, $resp) use ($counter) {
    $n = $counter->add(1);
    $resp->end("Count: $n");
});
```

### ❌ 坑2：忘记添加php.ini配置
```bash
# ❌ pecl install 完成就以为结束了
pecl install swoole
# 没有在 php.ini 中添加 extension=swoole.so
```

```bash
# ✅ 安装后手动确认
php --ri swoole
# 如果报错，检查 php.ini 并添加 extension=swoole.so
```

### ❌ 坑3：在macOS上epoll不可用
```php
<?php
// ❌ macOS uses kqueue, not epoll
// Some epoll-specific features may not work
```

```php
<?php
// ✅ Use Docker (Linux) for production-like environment
// docker run -it -v $(pwd):/app -p 9501:9501 phpswoole/swoole
```

## 十一、代码演进

### v1：最简单的验证脚本
```php
<?php
// v1: Basic check only
if (extension_loaded('swoole')) {
    echo "Swoole is installed!\n";
} else {
    echo "Swoole NOT installed!\n";
}
```

### v2：增加版本和特性检测
```php
<?php
// v2: Version and feature check
if (!extension_loaded('swoole')) {
    die("❌ Swoole not installed\n");
}
echo "Swoole " . swoole_version() . "\n";
echo "Coroutine: " . (class_exists('Swoole\\Coroutine') ? 'Yes' : 'No') . "\n";
```

### v3：完整验证脚本（带协程测试）
```php
<?php
// v3: Full verification with coroutine test
if (!extension_loaded('swoole')) {
    fwrite(STDERR, "❌ Swoole extension not loaded!\n");
    exit(1);
}

printf("✅ Swoole %s loaded\n", swoole_version());

$checks = [
    'openssl'   => SWOOLE_HAVE_SSL,
    'http2'     => SWOOLE_USE_HTTP2,
    'coroutine' => class_exists('Swoole\\Coroutine'),
];

foreach ($checks as $name => $ok) {
    printf("  %s %s\n", $ok ? '✅' : '⚠️', $name);
}

Swoole\\Coroutine\\run(function () {
    $cli = new Swoole\\Coroutine\\Http\\Client('www.baidu.com', 443, true);
    $cli->get('/');
    printf("\n✅ HTTP coroutine test: %d bytes\n", strlen($cli->body));
    $cli->close();
});
```
