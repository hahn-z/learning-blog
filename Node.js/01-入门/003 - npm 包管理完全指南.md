# npm 包管理完全指南

> **难度：** ⭐⭐ 初级 | **预计阅读：** 20 分钟 | **标签：** Node.js, npm, 包管理, package.json

## 一、概念讲解

### 1.1 什么是 npm？

**npm**（Node Package Manager）是 Node.js 的默认包管理器，也是全球最大的开源软件注册表。

三个层面的含义：
- **npm CLI**：命令行工具（`npm install`、`npm run` 等）
- **npm Registry**：包仓库（npmjs.com）
- **npm 网站**：包搜索和文档平台

### 1.2 package.json 核心字段

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "A sample project",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.0",
    "jest": "^29.7.0"
  },
  "engines": {
    "node": ">=18.0.0"
  }
}
```

### 1.3 语义化版本（SemVer）

格式：`MAJOR.MINOR.PATCH`（主版本.次版本.补丁版本）

| 符号 | 含义 | 示例 | 允许范围 |
|------|------|------|---------|
| `^` | 兼容版本 | `^1.2.3` | `>=1.2.3 <2.0.0` |
| `~` | 近似版本 | `~1.2.3` | `>=1.2.3 <1.3.0` |
| 无 | 精确版本 | `1.2.3` | 仅 `1.2.3` |
| `>=` | 最低版本 | `>=1.2.0` | 任意 >= |
| `*` | 任意版本 | `*` | 所有版本 |

### 1.4 dependencies vs devDependencies

| 类型 | 用途 | 安装命令 | 生产部署 |
|------|------|---------|---------|
| `dependencies` | 运行时需要 | `npm i pkg` | ✅ 安装 |
| `devDependencies` | 开发时需要 | `npm i pkg -D` | ❌ 不安装（`npm i --production`） |

```

    ├── package.json（项目配置）
    ├── package-lock.json（版本锁定）
    └── node_modules/（依赖存放）
```

## 三、完整代码示例

### 3.1 项目初始化与依赖安装

```bash
# Initialize a new project
mkdir my-app && cd my-app
npm init -y

# Install production dependencies
npm install express
npm install cors helmet morgan

# Install dev dependencies
npm install nodemon jest eslint -D

# Install specific version
npm install lodash@4.17.21
```

### 3.2 配置 scripts

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "start": "node src/index.js",
    "dev": "nodemon src/index.js",
    "lint": "eslint src/",
    "test": "jest --coverage",
    "build": "echo 'Building...' && npm run lint && npm test",
    "prebuild": "echo 'Running pre-build checks...'"
  },
  "dependencies": {
    "cors": "^2.8.5",
    "express": "^4.18.2",
    "helmet": "^7.1.0",
    "morgan": "^1.10.0"
  },
  "devDependencies": {
    "jest": "^29.7.0",
    "nodemon": "^3.0.0"
  }
}
```

### 3.3 实际项目结构

```javascript
// src/index.js - Entry point using installed packages
const express = require('express');
const cors = require('cors');
const helmet = require('helmet');
const morgan = require('morgan');

const app = express();
const PORT = process.env.PORT || 3000;

// Middleware
app.use(helmet());          // Security headers
app.use(cors());            // CORS support
app.use(morgan('combined'));// Request logging
app.use(express.json());    // Parse JSON body

// Routes
app.get('/', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date().toISOString() });
});

app.get('/users', (req, res) => {
  res.json([
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
  ]);
});

// Start server
app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

### 3.4 自定义 npm 包

```javascript
// my-utils/index.js - A simple npm package
function formatDate(date) {
  return new Intl.DateTimeFormat('zh-CN', {
    year: 'numeric', month: '2-digit', day: '2-digit',
    hour: '2-digit', minute: '2-digit'
  }).format(new Date(date));
}

function clamp(value, min, max) {
  return Math.min(Math.max(value, min), max);
}

function retry(fn, times = 3, delay = 1000) {
  return new Promise((resolve, reject) => {
    const attempt = (n) => {
      fn().then(resolve).catch(err => {
        if (n <= 1) return reject(err);
        setTimeout(() => attempt(n - 1), delay);
      });
    };
    attempt(times);
  });
}

module.exports = { formatDate, clamp, retry };
```

```json
  "version": "1.0.0",
  "main": "index.js",
  "files": ["index.js"]
}
```

## 五、注意事项

| 事项 | 说明 | 建议 |
|------|------|------|
| package-lock.json | 锁定依赖树，确保团队一致 | **必须提交到 Git** |
| node_modules | 依赖文件，通常很大 | `.gitignore` 中忽略 |
| 全局安装 | `-g` 安装到系统路径 | 只装 CLI 工具（nodemon等），不装库 |
| 版本范围 | `^` 可能升级 minor 版本 | 锁关键依赖用精确版本 |
| npm audit | 检查安全漏洞 | 定期运行 `npm audit fix` |
| registry | 默认 registry.npmjs.org | 国内可用淘宝镜像 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 把 `node_modules` 提交到 Git | `.gitignore` 加 `node_modules/` |
| 不提交 `package-lock.json` | **必须提交**，保证团队依赖一致 |
| `npm install` 不带 `--save` | npm 5+ 自动保存，不需要 `--save` |
| 全局安装业务依赖 | 业务依赖装本地，只有 CLI 工具才全局 |
| 不看包的下载量和维护状态 | 安装前看 npmjs.com 的 weekly downloads |
| 用 `npm uninstall` 后忘了清理代码 | 卸载包后同步删除使用该包的代码 |
| 直接改 node_modules 里的文件 | 改了也白改，`npm install` 会覆盖 |
| 用 `latest` 作为版本号 | 用具体版本或 `^` 范围 |

## 七、练习题

### 🟢 入门
1. 用 `npm init -y` 创建项目，手动修改 `package.json` 的 name 和 version
2. 安装 `lodash` 和 `dayjs`，写一个脚本使用它们
3. 运行 `npm outdated` 查看项目中哪些包有更新

### 🟡 进阶
4. 配置 `scripts`，实现 `npm run dev`（热重载）、`npm run lint`（代码检查）、`npm run test`（测试）
5. 使用 `npx` 运行一个未全局安装的包（如 `npx cowsay hello`）
6. 用 `npm version patch/minor/major` 管理版本号

### 🔴 挑战
7. 创建一个自定义 npm 包，发布到 npm（或本地测试）
8. 搭建一个私有 npm registry（用 Verdaccio），发布和安装私有包
