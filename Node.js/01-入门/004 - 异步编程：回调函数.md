# 异步编程：回调函数

> **难度：** ⭐⭐（初级）
> **阅读时间：** 约 15 分钟
> **前置知识：** JavaScript 基础语法、函数作为参数传递

---

## 一、概念讲解

Node.js 最核心的设计理念就是**异步非阻塞 I/O**。当 Node.js 执行文件读取、网络请求等耗时操作时，它不会停下来等待结果，而是继续执行后面的代码。等操作完成后，通过**回调函数（Callback）**通知你结果。

**什么是回调函数？**

回调函数就是"你做完之后告诉我"的机制——把一个函数作为参数传给另一个函数，当操作完成时调用它。

**生活中的类比：**

你去餐厅点餐，服务员给你一个取餐号码牌（注册回调），你可以继续刷手机（不阻塞），等餐好了服务员叫号（触发回调）。

## 三、完整代码示例

### 3.1 基础：同步 vs 异步文件读取

```javascript
// sync-vs-async.js
const fs = require('fs');

// ---- Synchronous (blocking) ----
console.log('1. Start reading sync...');
const data = fs.readFileSync(__filename, 'utf-8');
console.log(`2. Sync read ${data.length} chars`);
console.log('3. After sync read');

console.log('---');

// ---- Asynchronous (non-blocking) ----
console.log('4. Start reading async...');
fs.readFile(__filename, 'utf-8', (err, data) => {
  if (err) throw err;
  console.log(`6. Async read ${data.length} chars`);
});
console.log('5. After async read (comes before 6!)');
```

### 3.2 错误优先回调模式

```javascript
// error-first-callback.js
const fs = require('fs');

// Error-first callback: first param is error, second is result
fs.readFile('/nonexistent/file.txt', 'utf-8', (err, data) => {
  if (err) {
    console.error('Error occurred:', err.message);
    return; // Always return early on error
  }
  console.log('File content:', data);
});

console.log('This prints first because readFile is async');
```

### 3.3 回调地狱示例及拆分

```javascript
// callback-hell.js
const fs = require('fs');
const path = require('path');

// ❌ Callback Hell - deeply nested, hard to read
fs.readFile('file1.txt', 'utf-8', (err, data1) => {
  if (err) { console.error(err); return; }
  fs.readFile('file2.txt', 'utf-8', (err, data2) => {
    if (err) { console.error(err); return; }
    fs.readFile('file3.txt', 'utf-8', (err, data3) => {
      if (err) { console.error(err); return; }
      console.log('All files read!');
      console.log(data1 + data2 + data3);
    });
  });
});

// ✅ Flatten with named functions
function readFilesSequentially(files, callback) {
  const results = [];

  function readNext(index) {
    if (index >= files.length) {
      return callback(null, results);
    }
    fs.readFile(files[index], 'utf-8', (err, data) => {
      if (err) return callback(err);
      results.push(data);
      readNext(index + 1);
    });
  }

  readNext(0);
}

// Usage
readFilesSequentially(['file1.txt', 'file2.txt', 'file3.txt'], (err, results) => {
  if (err) {
    console.error('Failed:', err.message);
    return;
  }
  console.log('All files read:', results.join('\n'));
});
```

## 五、注意事项

| 要点 | 说明 |
|------|------|
| 回调是异步的基础 | Promise 和 async/await 底层仍然依赖回调机制 |
| 始终检查错误 | Node.js 约定回调第一个参数是 error，永远不要忽略它 |
| 不要同步调用回调 | 异步回调应在下一个事件循环 tick 中调用，避免 Zalgo |
| 避免深度嵌套 | 超过 3 层嵌套就该重构，用命名函数或控制流库拆分 |
| 回调中 this 指向 | 箭头函数没有自己的 this，普通函数的 this 可能丢失 |

## 六、避坑指南

### ❌ 坑 1：忘记处理错误

```javascript
// ❌ Bad - ignoring error
fs.readFile('data.json', 'utf-8', (err, data) => {
  console.log(JSON.parse(data)); // crashes if err is set!
});

// ✅ Good - always check error first
fs.readFile('data.json', 'utf-8', (err, data) => {
  if (err) {
    console.error('Read failed:', err.message);
    return;
  }
  console.log(JSON.parse(data));
});
```

### ❌ 坑 2：在循环中用回调处理异步

```javascript
// ❌ Bad - all callbacks share the same i
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100); // prints 5,5,5,5,5
}

// ✅ Good - use let (block scoping)
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 100); // prints 0,1,2,3,4
}
```

### ❌ 坑 3：同步释放 Zalgo

```javascript
// ❌ Bad - sometimes sync, sometimes async (Zalgo!)
function getData(cache, callback) {
  if (cache) {
    callback(null, cache); // synchronous!
  } else {
    fs.readFile('data.txt', 'utf-8', callback); // async
  }
}

// ✅ Good - always async
function getData(cache, callback) {
  if (cache) {
    process.nextTick(() => callback(null, cache));
  } else {
    fs.readFile('data.txt', 'utf-8', callback);
  }
}
```

## 七、练习题

### 🟢 初级：基础回调

编写一个函数 `readPackageJson(callback)`，读取当前目录的 `package.json` 文件，如果文件不存在则返回错误，否则解析 JSON 并返回对象。

```javascript
const fs = require('fs');

function readPackageJson(callback) {
  // Your code here
}
```

<details>
<summary>参考答案</summary>

```javascript
function readPackageJson(callback) {
  fs.readFile('package.json', 'utf-8', (err, data) => {
    if (err) return callback(err);
    try {
      callback(null, JSON.parse(data));
    } catch (e) {
      callback(e);
    }
  });
}
```
</details>

### 🟡 中级：串行执行

编写函数 `readFilesSeries(files, callback)`，按顺序读取多个文件，任一文件失败则立即回调错误，全部成功则返回内容数组。

### 🔴 高级：并行 + 限制并发

实现 `parallelLimit(tasks, limit, callback)`，tasks 是回调函数数组，最多同时执行 limit 个，全部完成后回调结果数组。

## 九、举一反三

| 场景 | 回调模式 | 关键点 |
|------|---------|--------|
| 文件读取 | `fs.readFile(path, cb)` | 错误优先 |
| HTTP 请求 | `http.get(url, res => {})` | 响应流式处理 |
| 定时器 | `setTimeout(cb, ms)` | 最小延迟约 1ms |
| 数据库查询 | `db.query(sql, cb)` | 连接池管理 |
| 事件监听 | `emitter.on('event', cb)` | 可多次触发 |
| 流处理 | `stream.on('data', cb)` | 背压控制 |

## 十一、代码演进

### v1：最基础的回调

```javascript
fs.readFile('a.txt', 'utf-8', (err, data) => {
  console.log(data);
});
```

### v2：错误处理 + 嵌套操作

```javascript
fs.readFile('a.txt', 'utf-8', (err, data) => {
  if (err) { console.error(err); return; }
  const upper = data.toUpperCase();
  fs.writeFile('a-upper.txt', upper, (err) => {
    if (err) { console.error(err); return; }
    console.log('Done!');
  });
});
```

### v3：命名函数拆分（可维护）

```javascript
function handleRead(err, data) {
  if (err) return console.error(err);
  fs.writeFile('a-upper.txt', data.toUpperCase(), handleWrite);
}

function handleWrite(err) {
  if (err) return console.error(err);
  console.log('Done!');
}

fs.readFile('a.txt', 'utf-8', handleRead);
```
