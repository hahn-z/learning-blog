# 015 - Gin 错误处理与统一响应 ⭐⭐⭐

## 难度标注

| 维度 | 等级 | 说明 |
|------|------|------|
| 理解难度 | ⭐⭐ | 统一响应概念简单 |
| 实现难度 | ⭐⭐⭐ | 全局错误恢复 + 错误码体系需设计 |
| 生产就绪 | ⭐⭐⭐⭐ | 错误追踪、日志关联是生产级要求 |

## 概念讲解

### 为什么需要统一响应？

没有统一响应的 API，前端开发者不知道：
- 成功时数据在哪个字段？
- 失败时错误信息在哪？
- HTTP 状态码和业务状态码是什么关系？

**统一响应**让所有 API 返回一致的结构：

```json
// 成功
{"code": 0, "message": "success", "data": {...}}

// 业务错误
{"code": 40001, "message": "用户名已存在", "data": null}

// 系统错误
{"code": 50000, "message": "内部服务器错误", "data": null}
```

### 错误分类

```
错误
├── 业务错误 (Business Error)
│   ├── 参数验证失败 → 400
│   ├── 未认证       → 401
│   ├── 无权限       → 403
│   ├── 资源不存在   → 404
│   └── 业务规则冲突 → 409
└── 系统错误 (System Error)
    ├── 数据库错误   → 500
    ├── 第三方服务   → 502
    └── 未知异常     → 500
```

### HTTP 状态码 vs 业务状态码

| 维度 | HTTP 状态码 | 业务状态码 |
|------|------------|-----------|
| 范围 | 标准协议（200-599） | 自定义（40001, 50000） |
| 粒度 | 粗（只有几十个） | 细（按业务定义） |
| 用途 | 网关/代理/CDN 判断 | 前端业务逻辑判断 |
| 推荐 | 反映大致类别 | 反映具体原因 |

**最佳实践：** HTTP 状态码反映大致类别，业务状态码反映具体原因。两者配合使用。

### core/errors.go - 错误码与自定义错误

```go
package core

import (
    "fmt"
    "net/http"
)

// Error codes - organized by category
const (
    // Success
    CodeSuccess = 0

    // Client errors (40000-40999)
    CodeBadRequest   = 40000
    CodeUnauthorized = 40100
    CodeForbidden    = 40300
    CodeNotFound     = 40400
    CodeConflict     = 40900
    CodeValidation   = 40001
    CodeInvalidParam = 40002

    // Auth errors (40100-40199)
    CodeTokenExpired = 40101
    CodeTokenInvalid = 40102

    // Server errors (50000-50999)
    CodeInternal    = 50000
    CodeDatabase    = 50001
    CodeThirdParty  = 50002
)

// AppError is a structured business error
type AppError struct {
    Code       int    `json:"code"`
    Message    string `json:"message"`
    HTTPStatus int    `json:"-"`
    Detail     string `json:"detail,omitempty"` // Only for debug mode
}

func (e *AppError) Error() string {
    return fmt.Sprintf("[%d] %s", e.Code, e.Message)
}

// NewAppError creates a new application error
func NewAppError(code int, message string, httpStatus int) *AppError {
    return &AppError{
        Code:       code,
        Message:    message,
        HTTPStatus: httpStatus,
    }
}

// Predefined errors for common cases
var (
    ErrBadRequest   = NewAppError(CodeBadRequest, "Bad request", http.StatusBadRequest)
    ErrUnauthorized = NewAppError(CodeUnauthorized, "Unauthorized", http.StatusUnauthorized)
    ErrForbidden    = NewAppError(CodeForbidden, "Forbidden", http.StatusForbidden)
    ErrNotFound     = NewAppError(CodeNotFound, "Resource not found", http.StatusNotFound)
    ErrInternal     = NewAppError(CodeInternal, "Internal server error", http.StatusInternalServerError)
)

// WithMessage creates a new error with a custom message based on existing error
func WithMessage(base *AppError, message string) *AppError {
    return &AppError{
        Code:       base.Code,
        Message:    message,
        HTTPStatus: base.HTTPStatus,
    }
}

// WithDetail adds detail information for debugging
func WithDetail(base *AppError, message, detail string) *AppError {
    return &AppError{
        Code:       base.Code,
        Message:    message,
        HTTPStatus: base.HTTPStatus,
        Detail:     detail,
    }
}
```

### core/response.go - 统一响应

```go
package core

import (
    "github.com/gin-gonic/gin"
)

// APIResponse is the standard response structure
type APIResponse struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data,omitempty"`
    Meta    interface{} `json:"meta,omitempty"`
}

// PaginatedMeta holds pagination metadata
type PaginatedMeta struct {
    Page  int `json:"page"`
    Size  int `json:"size"`
    Total int `json:"total"`
}

// Success sends a successful response with data
func Success(c *gin.Context, data interface{}) {
    c.JSON(200, APIResponse{
        Code:    CodeSuccess,
        Message: "success",
        Data:    data,
    })
}

// SuccessWithMeta sends a successful response with pagination info
func SuccessWithMeta(c *gin.Context, data interface{}, meta interface{}) {
    c.JSON(200, APIResponse{
        Code:    CodeSuccess,
        Message: "success",
        Data:    data,
        Meta:    meta,
    })
}

// SuccessMessage sends a successful response with only a message
func SuccessMessage(c *gin.Context, message string) {
    c.JSON(200, APIResponse{
        Code:    CodeSuccess,
        Message: message,
    })
}

// Created sends a 201 response for resource creation
func Created(c *gin.Context, data interface{}) {
    c.JSON(201, APIResponse{
        Code:    CodeSuccess,
        Message: "created",
        Data:    data,
    })
}

// HandleError handles both AppError and generic errors
func HandleError(c *gin.Context, err error) {
    if appErr, ok := err.(*AppError); ok {
        c.JSON(appErr.HTTPStatus, APIResponse{
            Code:    appErr.Code,
            Message: appErr.Message,
        })
        return
    }

    // Unknown error → treat as internal server error
    c.JSON(500, APIResponse{
        Code:    CodeInternal,
        Message: "Internal server error",
    })
}
```

### core/middleware.go - 错误处理中间件

```go
package core

import (
    "fmt"
    "log"
    "runtime/debug"
    "time"

    "github.com/gin-gonic/gin"
    "github.com/google/uuid"
)

// RequestID injects a unique request ID into context
func RequestID() gin.HandlerFunc {
    return func(c *gin.Context) {
        requestID := c.GetHeader("X-Request-ID")
        if requestID == "" {
            requestID = uuid.New().String()
        }
        c.Set("request_id", requestID)
        c.Header("X-Request-ID", requestID)
        c.Next()
    }
}

// Recovery is a custom recovery middleware that returns JSON on panic
func Recovery() gin.HandlerFunc {
    return func(c *gin.Context) {
        defer func() {
            if r := recover(); r != nil {
                requestID, _ := c.Get("request_id")

                // Log the panic with stack trace
                log.Printf("[PANIC] request_id=%s error=%v\n%s",
                    requestID, r, debug.Stack())

                // Return unified error response
                c.AbortWithStatusJSON(500, APIResponse{
                    Code:    CodeInternal,
                    Message: "Internal server error",
                })
            }
        }()
        c.Next()
    }
}

// ErrorLogger logs errors that occur during request processing
func ErrorLogger() gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        path := c.Request.URL.Path
        method := c.Request.Method

        c.Next()

        // Log if there were errors
        if len(c.Errors) > 0 {
            requestID, _ := c.Get("request_id")
            for _, err := range c.Errors {
                log.Printf("[ERROR] request_id=%s method=%s path=%s latency=%v error=%v",
                    requestID, method, path, time.Since(start), err.Err)
            }
        }
    }
}

// Cors adds basic CORS headers
func Cors() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Header("Access-Control-Allow-Origin", "*")
        c.Header("Access-Control-Allow-Methods", "GET,POST,PUT,DELETE,PATCH,OPTIONS")
        c.Header("Access-Control-Allow-Headers", "Content-Type,Authorization")
        if c.Request.Method == "OPTIONS" {
            c.AbortWithStatus(204)
            return
        }
        c.Next()
    }
}
```

### handlers/demo.go - 演示各种错误场景

```go
package handlers

import (
    "errors"
    "strconv"

    "gin-error-demo/core"

    "github.com/gin-gonic/gin"
)

// User represents a demo user
type User struct {
    ID       uint   `json:"id"`
    Username string `json:"username"`
    Email    string `json:"email"`
}

// Mock database
var users = map[uint]*User{
    1: {ID: 1, Username: "admin", Email: "admin@example.com"},
    2: {ID: 2, Username: "editor", Email: "editor@example.com"},
}

// GetUser demonstrates successful response
// GET /api/users/:id
func GetUser(c *gin.Context) {
    id, err := strconv.ParseUint(c.Param("id"), 10, 32)
    if err != nil {
        core.HandleError(c, core.WithMessage(core.ErrBadRequest, "Invalid user ID"))
        return
    }

    user, ok := users[uint(id)]
    if !ok {
        core.HandleError(c, core.WithMessage(core.ErrNotFound, "User not found"))
        return
    }

    core.Success(c, user)
}

// CreateUser demonstrates validation and creation response
// POST /api/users
func CreateUser(c *gin.Context) {
    var req struct {
        Username string `json:"username" binding:"required,min=3"`
        Email    string `json:"email" binding:"required,email"`
    }

    if err := c.ShouldBindJSON(&req); err != nil {
        core.HandleError(c, core.WithDetail(
            core.NewAppError(core.CodeValidation, "Validation failed", 400),
            err.Error(),
            "Check required fields and formats",
        ))
        return
    }

    // Check uniqueness
    for _, u := range users {
        if u.Username == req.Username {
            core.HandleError(c, core.NewAppError(
                core.CodeConflict,
                "Username already exists",
                409,
            ))
            return
        }
    }

    user := &User{
        ID:       uint(len(users) + 1),
        Username: req.Username,
        Email:    req.Email,
    }
    users[user.ID] = user

    core.Created(c, user)
}

// ListUsers demonstrates paginated response
// GET /api/users?page=1&size=10
func ListUsers(c *gin.Context) {
    page, _ := strconv.Atoi(c.DefaultQuery("page", "1"))
    size, _ := strconv.Atoi(c.DefaultQuery("size", "10"))

    var list []User
    for _, u := range users {
        list = append(list, *u)
    }

    core.SuccessWithMeta(c, list, core.PaginatedMeta{
        Page:  page,
        Size:  size,
        Total: len(list),
    })
}

// TriggerPanic demonstrates panic recovery
// GET /api/panic
func TriggerPanic(c *gin.Context) {
    // This will be caught by Recovery middleware
    panic("something went terribly wrong!")
}

// TriggerError demonstrates generic error handling
// GET /api/error
func TriggerError(c *gin.Context) {
    err := errors.New("database connection lost")
    core.HandleError(c, err) // Generic error → 500
}

// DeleteMessage demonstrates simple message response
// DELETE /api/users/:id
func DeleteUser(c *gin.Context) {
    id, _ := strconv.ParseUint(c.Param("id"), 10, 32)
    delete(users, uint(id))
    core.SuccessMessage(c, "User deleted")
}
```

### main.go

```go
package main

import (
    "gin-error-demo/core"
    "gin-error-demo/handlers"

    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.New() // Use gin.New() instead of Default() for custom middleware

    // Global middleware chain
    r.Use(
        core.RequestID(),  // Inject request ID
        core.Cors(),       // CORS headers
        core.ErrorLogger(),// Log errors
        core.Recovery(),   // Panic recovery → JSON
        gin.Logger(),      // Access log
    )

    api := r.Group("/api")
    {
        api.GET("/users", handlers.ListUsers)
        api.GET("/users/:id", handlers.GetUser)
        api.POST("/users", handlers.CreateUser)
        api.DELETE("/users/:id", handlers.DeleteUser)

        // Demo routes
        api.GET("/panic", handlers.TriggerPanic)
        api.GET("/error", handlers.TriggerError)
    }

    r.Run(":8080")
}
```

## 注意事项

| 事项 | 说明 | 严重度 |
|------|------|--------|
| 不暴露堆栈 | 生产环境不返回 stack trace | 🔴 严重 |
| 请求追踪 | 每个请求有唯一 ID，方便日志排查 | 🟡 重要 |
| 错误日志 | 记录完整上下文（请求ID、路径、参数） | 🟡 重要 |
| 错误码文档 | 维护错误码清单，前后端共享 | 🟡 重要 |
| panic 处理 | 必须有 Recovery 中间件，否则进程崩溃 | 🔴 严重 |
| 一致性 | 所有接口统一用 `HandleError`，不要 `c.JSON(500, ...)` 散落各处 | 🟡 重要 |

## 避坑指南

### ❌ 直接用 c.JSON 返回错误
```go
// BAD: Inconsistent error responses across handlers
func GetUser(c *gin.Context) {
    if user == nil {
        c.JSON(404, gin.H{"error": "not found"}) // One format
    }
}
func CreateUser(c *gin.Context) {
    if err != nil {
        c.JSON(400, map[string]string{"msg": err.Error()}) // Different format!
    }
}
```
✅ **正确做法：** 统一用 `core.HandleError(c, err)` 或 `core.Success(c, data)`。

### ❌ 向客户端暴露内部错误
```go
// BAD: Leaks database error details
c.JSON(500, gin.H{"error": err.Error()})
// "pq: duplicate key value violates unique constraint \"users_username_key\""
```
✅ **正确做法：** 返回通用消息，详细错误只记日志。

### ❌ 不处理 panic
```go
// BAD: No recovery middleware, panic crashes the server
r := gin.New()
r.GET("/api", func(c *gin.Context) {
    panic("boom") // Server crashes!
})
```
✅ **正确做法：** 使用 `gin.Recovery()` 或自定义 Recovery 中间件。

### ❌ HTTP 状态码当业务码
```go
// BAD: Only HTTP status code, no business code
c.JSON(404, gin.H{"error": "not found"})
// Frontend can't distinguish: user not found vs article not found
```
✅ **正确做法：** HTTP 状态码 + 业务状态码双重标识。

### ❌ gin.Default() 不够用
```go
// BAD: Default() doesn't give you control over middleware order
r := gin.Default() // Logger + Recovery only
```
✅ **正确做法：** 用 `gin.New()` + 手动注册中间件，控制顺序和自定义行为。

## 练习题

### 🟢 基础题

1. **统一响应封装：** 创建 `Success()` 和 `Error()` 函数，让所有 handler 返回统一 JSON 格式。

2. **自定义 404：** 替换 Gin 默认的 404 响应为 JSON 格式。

### 🟡 进阶题

3. **错误码管理：** 设计一套完整的错误码体系（至少 20 个），包含客户端错误、认证错误、服务端错误，并生成文档。

4. **请求追踪：** 实现 Request-ID 中间件，错误日志中包含请求 ID，能通过 ID 追踪完整请求链路。

### 🔴 挑战题

5. **Sentry 集成：** 将系统错误（panic、5xx）自动上报到 Sentry，包含请求上下文、用户信息、堆栈信息。

```go
// v2: Centralized response functions
func GetUser(c *gin.Context) {
    user, err := userService.Find(id)
    if err != nil {
        RespondError(c, err) // Unified error handling
        return
    }
```

### v3 - 类型安全错误 + 中间件链

```go
// v3: AppError type, middleware chain, request tracking
func GetUser(c *gin.Context) error {
    user, err := userService.Find(id)
    if err != nil {
        return core.NotFound("User not found") // Typed error
    }
    core.Success(c, user)
    return nil
}

// Middleware handles the returned error
func ErrorHandler() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Next()
        if len(c.Errors) > 0 {
            core.HandleError(c, c.Errors.Last().Err)
        }
    }
}
```
