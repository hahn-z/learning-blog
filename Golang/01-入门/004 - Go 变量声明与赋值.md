# 004 - Go 变量声明与赋值

⭐⭐☆☆☆

**预估用时：** 30-45 分钟

---

## 概念讲解

**一句话定义：** 变量是程序中用于存储和引用数据的命名内存空间，Go 通过 `var` 关键字或 `:=` 短变量声明来创建变量。

**现实类比：** 变量就像一个贴了标签的收纳盒——盒子上写着名字（变量名），盒子规定了能装什么类型的东西（类型），里面装的内容就是值。你可以随时更换盒子里的东西，但必须符合盒子的规定。

**技术场景：** 在 Web 服务中，你用变量存储请求参数、计算中间结果、追踪用户状态。变量是程序运行时状态管理的基础单元。

## 完整代码

```go
package main

import (
	"fmt"
	"strings"
)

// Package-level variables
var (
	version   = "1.0.0"
	buildTime string // zero value: ""
	maxRetry  int   // zero value: 0
)

func main() {
	// === 1. var declaration with explicit type ===
	var name string = "Go Learner"
	var age int = 25
	var score float64 = 98.5
	fmt.Printf("name=%s age=%d score=%.1f\n", name, age, score)

	// === 2. var with type inference ===
	var city = "Shanghai" // compiler infers string
	var count = 42        // compiler infers int
	fmt.Printf("city=%s count=%d\n", city, count)

	// === 3. var block declaration ===
	var (
		x     = 10
		y     = 20
		delta string
	)
	delta = fmt.Sprintf("delta=%d", y-x)
	fmt.Println(delta)

	// === 4. Short variable declaration (:=) ===
	// Only inside functions!
	lang := "Go"
	year := 2009
	fmt.Printf("%s was released in %d\n", lang, year)

	// === 5. Multiple variable declaration ===
	width, height := 100, 200
	fmt.Printf("area=%d\n", width*height)

	// === 6. Blank identifier ===
	fullName := "John Doe"
	// Only need the last name
	_, lastName := splitName(fullName)
	fmt.Println("Last name:", lastName)

	// === 7. Zero values ===
	var n int
	var f float64
	var b bool
	var s string
	var ptr *int
	fmt.Printf("zero values: n=%d f=%v b=%v s=%q ptr=%v\n", n, f, b, s, ptr)

	// === 8. Redeclaration with := (at least one new variable) ===
	val, err := 10, "ok"
	fmt.Println(val, err)
	val, err = 20, "still ok" // reassign with =
	// Note: := only if at least one variable on left is new
	fmt.Println(val, err)

	// === 9. Scope demonstration ===
	demonstrateScope()
}

// splitName splits a full name into first and last name
func splitName(full string) (string, string) {
	parts := strings.SplitN(full, " ", 2)
	if len(parts) < 2 {
		return parts[0], ""
	}
	return parts[0], parts[1]
}

// demonstrateScope shows variable scope and lifetime
func demonstrateScope() {
	x := "outer"

	if true {
		x := "inner" // new variable, shadows outer x
		y := "block-only"
		fmt.Println("inside block:", x, y)
	}

	// y is not accessible here
	fmt.Println("outside block:", x) // prints "outer"

	for i := 0; i < 3; i++ {
		msg := fmt.Sprintf("iteration %d", i) // msg created each iteration
		fmt.Println(msg)
	}
	// i and msg not accessible here
}
```

## 注意事项

| 事项 | 说明 |
|------|------|
| `:=` 只能在函数内 | 包级变量必须用 `var` 声明 |
| 声明未使用编译报错 | Go 不允许声明但未使用的变量（`_` 除外） |
| `:=` 至少一个新变量 | 左侧至少有一个未声明过的变量才能用 `:=` |
| 零值机制 | 声明未赋值的变量自动获得零值，不会是 undefined |
| 类型不可改变 | 变量一旦声明类型就固定了，不能赋其他类型的值 |
| 短变量不总是新建 | `:=` 在同一作用域中，若变量已存在且类型相同，会复用 |

## 避坑指南

**❌ 在函数外使用 `:=`**
```go
// compile error
x := 10  // outside function body
```
**✅ 包级用 `var`**
```go
var x = 10
```

**❌ 声明了不用**
```go
func main() {
    unused := 42  // compile error: unused variable
}
```
**✅ 用 `_` 丢弃或直接使用**
```go
_ = 42           // use blank identifier
// or actually use it
```

**❌ 忽略变量遮蔽**
```go
x := "outer"
{
    x := "inner"  // shadows outer x, may cause confusion
    fmt.Println(x)
}
fmt.Println(x) // still "outer", but developer may forget
```
**✅ 使用 `go vet` 或工具检测遮蔽**
```bash
go vet ./...   # basic checks
# Use shadow detector: go install golang.org/x/tools/go/analysis/passes/shadow/cmd/shadow@latest
```

**❌ `:=` 当赋值用**
```go
x := 10
x := 20  // error: no new variables on left side of :=
```
**✅ 已声明用 `=` 赋值**
```go
x := 10
x = 20   // correct: use = for reassignment
```

## 知识点总结

```
变量声明与赋值
├── 声明方式
│   ├── var name type = value    # 完整声明
│   ├── var name = value         # 类型推断
│   ├── var (块声明)             # 批量声明
│   └── name := value            # 短声明(函数内)
├── 零值
│   ├── int → 0
│   ├── float64 → 0.0
│   ├── bool → false
│   ├── string → ""
│   └── pointer/slice/map → nil
├── 作用域
│   ├── 包级
│   ├── 函数级
│   └── 块级(if/for/switch)
├── 特殊
│   ├── _ 空白标识符
│   └── 变量遮蔽(shadowing)
└── 规则
    ├── 声明必须使用
    ├── 类型不可变
    └── := 至少一新变量
```

## 举一反三

| 概念 | Go 写法 | 对比语言 | 主要差异 |
|------|---------|----------|----------|
| 声明变量 | `var x int` 或 `x := 0` | Python `x = 0` | Go 静态类型，Python 动态类型 |
| 常量 | `const x = 10` | JS `const x = 10` | Go const 必须编译期可确定 |
| 未使用变量 | 编译错误 | Python/JS 不报错 | Go 强制代码整洁 |
| 多返回值 | `a, b := f()` | Python `a, b = f()` | Go 原生支持，需处理每个返回值 |
| 空白标识符 | `_, b := f()` | Python `_` | 相同理念，Go 更广泛使用 |
