# 016-切片进阶：容量与底层原理

> **难度：⭐⭐⭐** | 适合已掌握基础切片用法，想深入理解底层机制的开发者

## 一、概念讲解

Go 的切片（slice）是对底层数组的一层封装，它本身是一个**轻量级的数据结构**，包含三个字段：

- **指针（pointer）**：指向底层数组的起始地址
- **长度（len）**：当前切片中元素的个数
- **容量（cap）**：从切片起始位置到底层数组末尾的元素总数

```
slice header {
    pointer: *array  →  [x x x x x x x x x x]
    len:     3          ^--- 切片可见范围 ---^
    cap:     7          ^------ 容量范围 --------^
}
```

理解 len 和 cap 的关系是掌握切片的关键。`len <= cap` 永远成立，切片只能在容量范围内通过 `s[i]` 访问元素。

## 三、完整 Go 代码

```go
package main

import "fmt"

// DemoSliceStruct shows the internal structure of a slice
func DemoSliceStruct() {
	// Create a slice with make: len=3, cap=5
	s := make([]int, 3, 5)
	fmt.Printf("slice: %v, len: %d, cap: %d\n", s, len(s), cap(s))

	// Append beyond capacity triggers reallocation
	s = append(s, 1, 2, 3)
	fmt.Printf("after append: %v, len: %d, cap: %d\n", s, len(s), cap(s))
}

// DemoGrowthPolicy demonstrates slice growth behavior
func DemoGrowthPolicy() {
	var s []int
	prevCap := 0
	for i := 0; i < 1000; i++ {
		s = append(s, i)
		if cap(s) != prevCap {
			fmt.Printf("len=%4d  old_cap=%4d  new_cap=%4d\n", len(s), prevCap, cap(s))
			prevCap = cap(s)
		}
	}
}

// DemoMemoryLeak shows the classic memory leak with large array slicing
func DemoMemoryLeak() {
	// Simulate a large data source
	bigData := make([]byte, 1<<20) // 1MB
	// Only need a small piece - but the whole 1MB stays in memory!
	small := bigData[:10]

	// Fix: copy to a new slice to allow GC to collect bigData
	fixed := make([]byte, 10)
	copy(fixed, small)
	// Now bigData can be garbage collected
	_ = fixed
	fmt.Println("Memory leak demo: copy small slices from large arrays")
}

// DemoSharedBackingArray shows the shared backing array trap
func DemoSharedBackingArray() {
	original := []int{1, 2, 3, 4, 5}
	slice1 := original[1:3] // [2, 3], shares backing array

	fmt.Printf("Before: slice1=%v, original=%v\n", slice1, original)
	slice1[0] = 99
	fmt.Printf("After:  slice1=%v, original=%v\n", slice1, original)
	// original is now [1, 99, 3, 4, 5] — surprise!

	// Fix: use full slice expression or copy
	safe := make([]int, 2)
	copy(safe, original[1:3])
	_ = safe
}

// DemoPreallocation shows performance benefit of preallocation
func DemoPreallocation() {
	// Without preallocation
	s1 := make([]int, 0)
	for i := 0; i < 10000; i++ {
		s1 = append(s1, i)
	}

	// With preallocation — avoids multiple reallocations
	s2 := make([]int, 0, 10000)
	for i := 0; i < 10000; i++ {
		s2 = append(s2, i)
	}
	fmt.Printf("Without prealloc: len=%d, cap=%d\n", len(s1), cap(s1))
	fmt.Printf("With prealloc:    len=%d, cap=%d\n", len(s2), cap(s2))
}

// DemoDeleteElement shows how to delete an element from a slice
func DemoDeleteElement() {
	s := []int{1, 2, 3, 4, 5}
	i := 2 // remove index 2 (value 3)

	// Method: shift elements left and reslice
	s = append(s[:i], s[i+1:]...)
	fmt.Printf("After delete index %d: %v\n", i, s)
}

// DemoInsertElement shows how to insert an element into a slice
func DemoInsertElement() {
	s := []int{1, 2, 4, 5}
	i := 2      // insert position
	val := 3    // value to insert

	// Expand slice, shift right, then assign
	s = append(s, 0)       // make room
	copy(s[i+1:], s[i:])   // shift right
	s[i] = val             // insert

	fmt.Printf("After insert %d at index %d: %v\n", val, i, s)
}

// DemoDeduplicate shows deduplication for sorted slices
func DemoDeduplicate() {
	s := []int{1, 1, 2, 3, 3, 3, 4, 5, 5}

	result := s[:1]
	for _, v := range s[1:] {
		if v != result[len(result)-1] {
			result = append(result, v)
		}
	}
	fmt.Printf("Deduplicated: %v\n", result)
}

func main() {
	fmt.Println("=== Slice Structure ===")
	DemoSliceStruct()

	fmt.Println("\n=== Growth Policy ===")
	DemoGrowthPolicy()

	fmt.Println("\n=== Memory Leak ===")
	DemoMemoryLeak()

	fmt.Println("\n=== Shared Backing Array ===")
	DemoSharedBackingArray()

	fmt.Println("\n=== Preallocation ===")
	DemoPreallocation()

	fmt.Println("\n=== Delete Element ===")
	DemoDeleteElement()

	fmt.Println("\n=== Insert Element ===")
	DemoInsertElement()

	fmt.Println("\n=== Deduplicate ===")
	DemoDeduplicate()
}
```

## 五、注意事项（表格）

| 注意点 | 说明 | 影响 |
|--------|------|------|
| 切片是引用类型 | 赋值和传参只复制 header | 修改会影响原切片 |
| nil 切片 vs 空切片 | `var s []int` vs `s := []int{}` | JSON 序列化结果不同 |
| 扩容会复制数据 | 触发扩容时底层数组会重新分配 | 旧切片不受影响 |
| `s[low:high]` 不会复制 | 只是新建 header 指向同个数组 | 共享底层数组陷阱 |
| `copy` 的返回值 | 返回实际复制的元素数 | 取 min(dst, src) 的 len |

## 六、避坑指南

| ❌ 错误写法 | ✅ 正确写法 | 说明 |
|------------|-----------|------|
| `s := bigData[:10]` 保留大数组引用 | `copy` 到新切片 | 内存泄漏 |
| `newSlice := oldSlice[1:3]` 直接修改 | 用 `copy` 或 full slice `arr[1:3:3]` | 共享底层数组 |
| `append` 后不接收返回值 | `s = append(s, x)` | 扩容后旧地址失效 |
| 在循环中不断 `append` 不预分配 | `make([]T, 0, n)` 预分配 | 多次扩容影响性能 |
| `s = s[:cap(s)]` 盲目扩展 | 检查元素是否合法 | 访问未初始化的内存 |

## 七、练习题

🟢 **基础题：** 编写函数 `Remove(s []int, i int) []int`，从切片中删除指定索引的元素，保持顺序不变。

🟡 **进阶题：** 实现一个 `RingBuffer` 结构体，使用固定容量的切片实现环形缓冲区，支持 `Push` 和 `Pop` 操作。

🔴 **挑战题：** 编写函数 `Chunk(s []int, size int) [][]int`，将切片按 size 分块。要求：不预分配结果切片的情况下，每个 chunk 不共享底层数组。

## 八、知识点总结（树状）

```
切片（Slice）
├── 底层结构
│   ├── pointer → 底层数组
│   ├── len → 元素数量
│   └── cap → 容量
├── 扩容机制
│   ├── Go 1.18 前：≤1024 翻倍，>1024 ×1.25
│   └── Go 1.18+：更平滑的增长策略
├── 常见陷阱
│   ├── 内存泄漏（大数组小切片）
│   └── 共享底层数组
└── 实用技巧
    ├── 预分配 make([]T, 0, n)
    ├── 删除 append(s[:i], s[i+1:]...)
    ├── 插入 append + copy
    └── 去重（排序后遍历）
```

## 九、举一反三（表格）

| 场景 | 方案 | 关键点 |
|------|------|--------|
| 大文件读取后只保留部分 | `copy` 到新切片 | 释放原底层数组 |
| 函数返回子切片 | 使用 full slice expression `a[low:high:max]` | 限制 cap 防误操作 |
| 高频 append | 预分配 cap | 减少扩容次数 |
| 多协程读切片 | 安全（只读） | 写操作需要加锁 |
| 去重 | map 或排序后遍历 | 根据数据规模选择 |

## 十一、代码演进

### v1 — 基础：append 不关心容量

```go
func addItems(s []int, items ...int) []int {
    return append(s, items...)
}
// 问题：多次扩容，性能差
```

### v2 — 预分配容量

```go
func addItems(s []int, items ...int) []int {
    if cap(s)-len(s) < len(items) {
        newSlice := make([]int, len(s), len(s)+len(items))
        copy(newSlice, s)
        s = newSlice
    }
    return append(s, items...)
}
// 改进：手动控制扩容，减少分配次数
```

### v3 — 泛型版本（Go 1.18+）

```go
func addItems[T any](s []T, items ...T) []T {
    if cap(s)-len(s) < len(items) {
        newSlice := make([]T, len(s), len(s)+len(items))
        copy(newSlice, s)
        s = newSlice
    }
    return append(s, items...)
}
// 改进：支持任意类型，更通用
```
