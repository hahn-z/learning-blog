一、概念讲解

什么是指针？

指针是一个变量，它存储的值是另一个变量的内存地址。

想象一下：你家有一个门牌号（地址），快递员不需要知道你家里面长什么样，只需要知道门牌号就能找到你。指针就是那个"门牌号"。

普通变量：  值 → [42]
指针变量：  地址 → [0xc0000b2008] → [42]

两个核心操作符

操作符 名称 作用 示例
& 取地址 获取变量的内存地址 &x 返回 x 的地址
* 解引用 获取指针指向的值 *p 返回 p 指向的值

零值 nil

指针的零值是 nil，表示"不指向任何地址"。

var p *int   // p == nil, 类型是 *int

指针传参：避免大拷贝

当一个结构体很大时，传值会复制整个结构体，传指针只复制一个地址（8字节）。

Go 不支持指针运算

与 C 语言不同，Go 不支持指针加减运算。这是为了安全——你无法通过指针越界访问内存。

三、完整 Go 代码

package main

import "fmt"

// Demo: basic pointer operations
func basicPointer() {
    x := 42
    p := &x // p is a pointer to x

    fmt.Printf("x 的值: %d\n", x)
    fmt.Printf("x 的地址: %p\n", &x)
    fmt.Printf("p 的值(地址): %p\n", p)
    fmt.Printf("p 指向的值: %d\n", *p)

    // Modify value through pointer
    *p = 100
    fmt.Printf("修改后 x = %d\n", x) // x is now 100
}

// Demo: pointer as function parameter
func doubleValue(val int) {
    val *= 2 // Only modifies the copy, original unchanged
}

func doublePointer(p *int) {
    *p *= 2 // Modifies the original value
}

func pointerParam() {
    x := 5
    doubleValue(x)
    fmt.Printf("doubleValue 后 x = %d\n", x) // Still 5

    doublePointer(&x)
    fmt.Printf("doublePointer 后 x = %d\n", x) // Now 10
}

// Demo: pointer with struct
type User struct {
    Name string
    Age  int
}

func updateAge(u *User, newAge int) {
    u.Age = newAge // No need to write (*u).Age in Go
}

func structPointer() {
    u := User{Name: "Alice", Age: 25}
    fmt.Printf("更新前: %+v\n", u)

    updateAge(&u, 30)
    fmt.Printf("更新后: %+v\n", u)
}

// Demo: new function
func newDemo() {
    // new(T) allocates zero-valued memory and returns *T
    p := new(int)
    fmt.Printf("new(int): value=%d, isNil=%v\n", *p, p == nil) // 0, false

    *p = 42
    fmt.Printf("赋值后: %d\n", *p)
}

// Demo: nil pointer
func nilPointer() {
    var p *int // nil pointer
    fmt.Printf("nil 指针: %v\n", p)

    // Uncommenting this will cause panic
    // fmt.Println(*p) // panic: runtime error: invalid memory address
}

// Demo: returning pointer to local variable is SAFE in Go
func createPointer() *int {
    x := 42
    return &x // Safe! Go compiler does escape analysis
}

func returnPointerDemo() {
    p := createPointer()
    fmt.Printf("返回的局部变量指针: %d\n", *p) // Works fine
}

// Demo: pointer to pointer
func doublePointerDemo() {
    x := 42
    p := &x
    pp := &p // **int

    fmt.Printf("x=%d, *p=%d, **pp=%d\n", x, *p, **pp)
    **pp = 999
    fmt.Printf("修改后 x=%d\n", x)
}

func main() {
    fmt.Println("=== 基本指针操作 ===")
    basicPointer()

    fmt.Println("\n=== 指针传参 ===")
    pointerParam()

    fmt.Println("\n=== 结构体指针 ===")
    structPointer()

    fmt.Println("\n=== new 函数 ===")
    newDemo()

    fmt.Println("\n=== nil 指针 ===")
    nilPointer()

    fmt.Println("\n=== 返回局部变量指针 ===")
    returnPointerDemo()

    fmt.Println("\n=== 指针的指针 ===")
    doublePointerDemo()
}

五、注意事项

项目 说明 示例
空指针解引用 对 nil 指针解引用会 panic var p int; p → panic
指针零值 指针的零值是 nil，不是 0 var p *int → nil
函数返回局部指针 Go 中安全（逃逸分析） return &x 合法
结构体指针访问字段 Go 自动解引用 p.Name 等价 (*p).Name
指针比较 可用 == 比较（含 nil） p == nil 合法
不支持运算 不能 p++ 或 p+1 编译错误
指针默认值 new 返回的是零值的指针 new(int) → *int, 值为 0

六、避坑指南

❌ 空指针解引用

// ❌ 直接解引用 nil 指针
var p *int
fmt.Println(*p) // panic!

// ✅ 先检查 nil
var p *int
if p != nil {
    fmt.Println(*p)
}

❌ 传值期望修改原值

// ❌ 传值无法修改原值
func modify(x int) { x = 10 }

// ✅ 传指针才能修改
func modify(x *int) { *x = 10 }

❌ 用 new 创建切片/映射

// ❌ new 返回的是零值指针，不是已初始化的切片
s := *new([]int) // s is nil slice, not ready to use

// ✅ 用 make 初始化
s := make([]int, 0)

❌ 迷信指针一定更快

// ❌ 小类型（int等）传指针不一定更快（指针本身有间接访问开销）
func add(a, b *int) int { return *a + *b }

// ✅ 小类型直接传值
func add(a, b int) int { return a + b }
