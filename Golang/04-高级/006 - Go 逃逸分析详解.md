# Go 逃逸分析详解

**难度：⭐⭐⭐⭐⭐（专家）**

逃逸分析（Escape Analysis）是 Go 编译器的核心优化。它决定了变量分配在栈还是堆上——直接影响程序性能。理解逃逸分析，是写出高性能 Go 代码的关键。

## 📖 概念讲解

### 栈 vs 堆

| 特性 | 栈 (Stack) | 堆 (Heap) |
|------|-----------|-----------|
| 分配速度 | 极快（移动指针） | 较慢（找空闲块） |
| 回收方式 | 函数返回自动回收 | GC 扫描回收 |
| 大小限制 | 较小（默认 1MB → 可增长） | 灵活 |
| 碎片 | 无 | 有 |
| 缓存友好 | 极好 | 较差 |

### 什么是"逃逸"

当一个变量的生命周期超出了创建它的函数作用域，编译器就必须把它分配到堆上——这就是"逃逸"。

**Go 编译器的逃逸分析原则**：
1. 尽可能分配在栈上（零成本回收）
2. 只在必要时逃逸到堆上
3. 分析是在编译期完成的，不是运行时

### 分析命令

```bash
# 基本逃逸分析
go build -gcflags='-m' ./...

# 详细逃逸分析（包含决策理由）
go build -gcflags='-m -m' ./...

# 查看汇编（确认分配位置）
go tool compile -S main.go
```

## 💻 完整代码示例

### v1：基本逃逸场景识别

```go
package main

import "fmt"

// Scenario 1: Return pointer to local variable -> escapes
func createInt() *int {
    x := 42      // x escapes to heap
    return &x    // &x causes escape
}

// Scenario 2: Value stays on stack
func noEscape() int {
    x := 42      // stays on stack
    return x     // copy, no escape
}

// Scenario 3: Slice causes escape (size unknown at compile time)
func makeSlice(n int) []int {
    // n is not constant -> escapes
    return make([]int, n)
}

// Scenario 4: Fixed-size array stays on stack
func fixedArray() [4]int {
    // size known at compile time -> may stay on stack
    return [4]int{1, 2, 3, 4}
}

// Scenario 5: Interface causes escape
func printValue(v interface{}) {
    // v is interface{} -> value escapes to heap
    fmt.Println(v)
}

// Scenario 6: Closure captures variable
func newCounter() func() int {
    count := 0            // escapes: captured by closure
    return func() int {
        count++
        return count
    }
}

// Scenario 7: String literal does NOT escape (read-only data segment)
func getString() string {
    return "hello" // string literal in read-only segment
}

func main() {
    // Scenario 1: pointer escape
    p := createInt()
    fmt.Println(*p) // 42

    // Scenario 2: no escape
    v := noEscape()
    fmt.Println(v) // 42

    // Scenario 3: slice escape
    s := makeSlice(10)
    fmt.Println(len(s)) // 10

    // Scenario 4: no escape (fixed size)
    a := fixedArray()
    fmt.Println(a[0]) // 1

    // Scenario 5: interface escape
    printValue("world")

    // Scenario 6: closure escape
    counter := newCounter()
    fmt.Println(counter()) // 1
    fmt.Println(counter()) // 2

    // Scenario 7: string literal
    str := getString()
    fmt.Println(str) // hello
}
```

**编译并查看逃逸分析**：
```bash
$ go build -gcflags='-m' main.go
# command-line-arguments
./main.go:6:6: &x escapes to heap
./main.go:7:9: &x ~r0 does not escape
./main.go:24:12: make([]int, n) escapes to heap
./main.go:46:15: v escapes to heap
./main.go:53:2: count escapes to heap
```

**执行预览**：
```

42
10
1
world
1
2
hello
```

### v2：优化逃逸——减少堆分配

```go
package main

import (
    "fmt"
    "strings"
)

// BAD: causes escape due to interface
func badPrint(items ...interface{}) {
    for _, item := range items {
        fmt.Println(item) // each item escapes
    }
}

// GOOD: type-specific, no interface
func goodPrint(items []string) {
    for _, item := range items {
        fmt.Println(item) // string may still escape to heap for fmt
    }
}

// BAD: unnecessary pointer return
func badCreate() *User {
    u := User{Name: "Alice", Age: 30}
    return &u // escape!
}

// GOOD: return by value
func goodCreate() User {
    u := User{Name: "Alice", Age: 30}
    return u // no escape
}

type User struct {
    Name string
    Age  int
}

// BAD: growing slice in loop causes repeated allocation
func badAccumulate(items []string) []string {
    var result []string
    for _, item := range items {
        result = append(result, strings.ToUpper(item)) // may grow & escape
    }
    return result
}

// GOOD: pre-allocate slice
func goodAccumulate(items []string) []string {
    result := make([]string, len(items)) // pre-allocate
    for i, item := range items {
        result[i] = strings.ToUpper(item)
    }
    return result
}

// BAD: method with pointer receiver on small struct
type Point struct{ X, Y float64 }

func (p *Point) DistanceBad() float64 {
    // pointer receiver may cause escape when called through interface
    return p.X*p.X + p.Y*p.Y
}

// GOOD: value receiver for small structs
func (p Point) DistanceGood() float64 {
    return p.X*p.X + p.Y*p.Y
}

// Inline buffer to avoid allocation
func formatGreeting(name string) string {
    var buf strings.Builder
    buf.Grow(64) // pre-allocate buffer
    buf.WriteString("Hello, ")
    buf.WriteString(name)
    buf.WriteString("!")
    return buf.String()
}

func main() {
    // Value vs pointer return
    u1 := badCreate()
    u2 := goodCreate()
    fmt.Println(u1.Name, u2.Name)

    // Pre-allocated slice
    items := []string{"hello", "world", "foo"}
    r1 := badAccumulate(items)
    r2 := goodAccumulate(items)
    fmt.Println(r1)
    fmt.Println(r2)

    // Value receiver
    p := Point{3, 4}
    fmt.Println(p.DistanceBad())  // 25
    fmt.Println(p.DistanceGood()) // 25

    // Inline buffer
    fmt.Println(formatGreeting("Go")) // Hello, Go!
}
```

**执行预览**：
```
Alice Alice
[HELLO WORLD FOO]
[HELLO WORLD FOO]
25
25
Hello, Go!
```

### v3：sync.Pool 与高级优化

```go
package main

import (
    "fmt"
    "runtime"
    "sync"
    "time"
)

// sync.Pool: reuse objects to reduce GC pressure
var bufPool = sync.Pool{
    New: func() interface{} {
        buf := make([]byte, 0, 1024)
        return &buf
    },
}

func processData(data string) string {
    // Get buffer from pool
    bufPtr := bufPool.Get().(*[]byte)
    buf := (*bufPtr)[:0] // reset length, keep capacity

    // Use buffer
    buf = append(buf, "processed: "...)
    buf = append(buf, data...)

    result := string(buf)

    // Return buffer to pool
    *bufPtr = buf
    bufPool.Put(bufPtr)

    return result
}

// Benchmark: measure allocation difference
func benchmarkWithoutPool(n int) {
    for i := 0; i < n; i++ {
        _ = make([]byte, 1024)
    }
}

func benchmarkWithPool(n int) {
    for i := 0; i < n; i++ {
        buf := bufPool.Get().(*[]byte)
        *buf = (*buf)[:0]
        _ = *buf
        bufPool.Put(buf)
    }
}

// Escape-friendly struct design
type Buffer struct {
    data [64]byte // inline array: stays on stack for small sizes
    pos  int
}

func (b *Buffer) Write(p []byte) int {
    n := copy(b.data[b.pos:], p)
    b.pos += n
    return n
}

func (b *Buffer) String() string {
    return string(b.data[:b.pos])
}

// GC stats monitoring
func printGCStats(label string) {
    var stats debug.GCStats
    stats.Pause = make([]time.Duration, 10)
    runtime.ReadGCStats(&stats)
    fmt.Printf("[%s] GC runs: %d, last pause: %v\n",
        label, stats.NumGC, stats.Pause[0])
}

var debug = struct{ GCStats struct{ NumGC int; Pause []time.Duration } }{}

func main() {
    // sync.Pool usage
    fmt.Println(processData("hello")) // processed: hello
    fmt.Println(processData("world")) // processed: world

    // Compare allocation counts
    var m1, m2 runtime.MemStats

    // Without pool
    runtime.GC()
    runtime.ReadMemStats(&m1)
    benchmarkWithoutPool(10000)
    runtime.ReadMemStats(&m2)
    allocsWithout := m2.TotalAlloc - m1.TotalAlloc

    // With pool
    runtime.GC()
    runtime.ReadMemStats(&m1)
    benchmarkWithPool(10000)
    runtime.ReadMemStats(&m2)
    allocsWithPool := m2.TotalAlloc - m1.TotalAlloc

    fmt.Printf("Without pool: %d bytes\n", allocsWithout)
    fmt.Printf("With pool:    %d bytes\n", allocsWithPool)

    // Stack-friendly buffer
    var buf Buffer
    buf.Write([]byte("hello"))
    buf.Write([]byte(" "))
    buf.Write([]byte("world"))
    fmt.Println(buf.String()) // hello world
}
```

**执行预览**：
```
processed: hello
processed: world
Without pool: 10240000 bytes
With pool:    1024 bytes
hello world
```

## ⚠️ 注意事项

| 序号 | 注意点 | 说明 |
|------|--------|------|
| 1 | `fmt.Println` 参数逃逸 | `interface{}` 参数导致值逃逸到堆 |
| 2 | 闭包捕获的变量必逃逸 | 变量生命周期超出函数作用域 |
| 3 | `reflect` 使用导致逃逸 | 反射需要堆上分配 |
| 4 | `sync.Pool` 不保证持久 | GC 时 pool 内容可能被清除 |
| 5 | 编译器版本影响分析 | 不同 Go 版本逃逸行为可能不同 |
| 6 | 小结构体用值接收者 | 减少指针逃逸 |

## 🚫 避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|------------|
| 不看逃逸分析就优化 | 先 `-gcflags='-m'` 确认逃逸点 |
| 所有参数都用指针传递 | 小结构体用值传递，避免逃逸 |
| `append` 不断扩容 | 预分配 `make([]T, 0, cap)` |
| 频繁创建临时对象 | 用 `sync.Pool` 复用 |
| 在热路径上用 `interface{}` | 用具体类型或泛型 |
| 过度关注微优化 | 先 profile，再优化热点 |

## 🔄 举一反三

| 场景 | 逃逸原因 | 优化方案 |
|------|----------|----------|
| `fmt.Println(x)` | `interface{}` 参数 | 用 `log.Println` 或直接输出 |
| `return &local` | 指针逃逸 | 返回值类型 |
| `append` 扩容 | 新底层数组 | `make([]T, 0, cap)` 预分配 |
| 闭包变量 | 跨栈引用 | 改用参数传递 |
| `json.Marshal` | 反射 + 接口 | 用代码生成（easyjson） |
| HTTP handler | `interface{}` 路由 | 泛型路由或类型断言 |
| `sync.Mutex` 嵌入 | 可能逃逸 | 按值嵌入小锁 |
