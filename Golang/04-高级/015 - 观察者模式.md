# 095-观察者模式

> **难度：🟡 中高级** | 预计阅读：25 分钟 | 2024 年更新

## 一、概念讲解

**观察者模式（Observer Pattern）** 定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。

**核心角色：**
- **Subject（主题）**：维护观察者列表，状态变更时通知所有观察者
- **Observer（观察者）**：定义更新接口，接收主题通知

**Go 实现要点：**
- 用接口定义 Observer，避免继承
- 用 channel 可实现异步通知
- 注意 goroutine 泄漏和并发安全
- 支持取消订阅，防止内存泄漏

**典型场景：**
- 事件总线（Event Bus）
- 股票价格推送
- 配置变更通知
- 日志系统
- 消息广播

## 三、完整 Go 代码

```go
package main

import (
	"context"
	"fmt"
	"sync"
	"time"
)

// ============================================
// V1: Basic synchronous observer
// ============================================

// Observer defines the update interface
type Observer interface {
	OnUpdate(data string)
}

// Subject maintains a list of observers
type Subject struct {
	observers []Observer
	mu        sync.RWMutex
}

func (s *Subject) Attach(o Observer) {
	s.mu.Lock()
	defer s.mu.Unlock()
	s.observers = append(s.observers, o)
}

func (s *Subject) Detach(o Observer) {
	s.mu.Lock()
	defer s.mu.Unlock()
	for i, obs := range s.observers {
		if obs == o {
			s.observers = append(s.observers[:i], s.observers[i+1:]...)
			return
		}
	}
}

func (s *Subject) Notify(data string) {
	s.mu.RLock()
	defer s.mu.RUnlock()
	for _, o := range s.observers {
		o.OnUpdate(data) // synchronous call
	}
}

// Concrete observers
type LogObserver struct{ Name string }
func (l *LogObserver) OnUpdate(data string) { fmt.Printf("[%s] Log: %s\n", l.Name, data) }

type AlertObserver struct{ Threshold int }
func (a *AlertObserver) OnUpdate(data string) { fmt.Printf("[Alert] Received: %s\n", data) }

// ============================================
// V2: Async channel-based event bus
// ============================================

// Event represents a typed event
type Event struct {
	Topic string
	Data  interface{}
}

// Handler processes events
type Handler func(event Event)

// EventBus provides async pub/sub
type EventBus struct {
	subscribers map[string][]chan Event
	mu          sync.RWMutex
	buffer      int
}

func NewEventBus(bufferSize int) *EventBus {
	return &EventBus{
		subscribers: make(map[string][]chan Event),
		buffer:      bufferSize,
	}
}

// Subscribe returns a channel that receives events for the topic
func (eb *EventBus) Subscribe(topic string) <-chan Event {
	eb.mu.Lock()
	defer eb.mu.Unlock()
	ch := make(chan Event, eb.buffer)
	eb.subscribers[topic] = append(eb.subscribers[topic], ch)
	return ch
}

// Publish sends an event to all subscribers of the topic
func (eb *EventBus) Publish(topic string, data interface{}) {
	eb.mu.RLock()
	defer eb.mu.RUnlock()
	for _, ch := range eb.subscribers[topic] {
		select {
		case ch <- Event{Topic: topic, Data: data}:
		default:
			fmt.Printf("[WARN] subscriber channel full, dropping event on %s\n", topic)
		}
	}
}

// Close closes all subscriber channels
func (eb *EventBus) Close() {
	eb.mu.Lock()
	defer eb.mu.Unlock()
	for _, subs := range eb.subscribers {
		for _, ch := range subs {
			close(ch)
		}
	}
	eb.subscribers = make(map[string][]chan Event)
}

// ============================================
// V3: Generic event bus with context & error handling
// ============================================

// ObserverFunc is a generic event handler function
type ObserverFunc[T any] func(ctx context.Context, data T) error

// GenericEventBus provides type-safe pub/sub
type GenericEventBus[T any] struct {
	handlers map[string][]ObserverFunc[T]
	mu       sync.RWMutex
	wg       sync.WaitGroup
}

func NewGenericEventBus[T any]() *GenericEventBus[T] {
	return &GenericEventBus[T]{
		handlers: make(map[string][]ObserverFunc[T]),
	}
}

// On registers a handler for a topic
func (g *GenericEventBus[T]) On(topic string, handler ObserverFunc[T]) {
	g.mu.Lock()
	defer g.mu.Unlock()
	g.handlers[topic] = append(g.handlers[topic], handler)
}

// Emit publishes an event asynchronously to all handlers
func (g *GenericEventBus[T]) Emit(ctx context.Context, topic string, data T) {
	g.mu.RLock()
	handlers := make([]ObserverFunc[T], len(g.handlers[topic]))
	copy(handlers, g.handlers[topic])
	g.mu.RUnlock()

	for _, h := range handlers {
		g.wg.Add(1)
		go func(handler ObserverFunc[T]) {
			defer g.wg.Done()
			defer func() {
				if r := recover(); r != nil {
					fmt.Printf("[RECOVER] handler panic: %v\n", r)
				}
			}()
			if err := handler(ctx, data); err != nil {
				fmt.Printf("[ERROR] handler failed: %v\n", err)
			}
		}(h)
	}
}

// Wait blocks until all async handlers complete
func (g *GenericEventBus[T]) Wait() { g.wg.Wait() }

// ============================================
// Demo
// ============================================

func main() {
	// V1: Synchronous
	fmt.Println("=== V1: Synchronous ===")
	sub := &Subject{}
	logger := &LogObserver{Name: "main"}
	alert := &AlertObserver{}
	sub.Attach(logger)
	sub.Attach(alert)
	sub.Notify("server started")
	sub.Detach(alert)
	sub.Notify("alert removed")

	// V2: Async EventBus
	fmt.Println("\n=== V2: Async EventBus ===")
	bus := NewEventBus(10)
	orderCh := bus.Subscribe("order")
	go func() {
		for evt := range orderCh {
			fmt.Printf("[OrderHandler] %v\n", evt.Data)
		}
	}()
	bus.Publish("order", map[string]interface{}{"id": "A123", "amount": 99.9})
	time.Sleep(100 * time.Millisecond)
	bus.Close()

	// V3: Generic EventBus
	fmt.Println("\n=== V3: Generic EventBus ===")
	type OrderEvent struct {
		ID     string
		Amount float64
	}

	orderBus := NewGenericEventBus[OrderEvent]()
	orderBus.On("created", func(ctx context.Context, o OrderEvent) error {
		fmt.Printf("[Email] New order %s: $%.2f\n", o.ID, o.Amount)
		return nil
	})
	orderBus.On("created", func(ctx context.Context, o OrderEvent) error {
		fmt.Printf("[Inventory] Reserve stock for %s\n", o.ID)
		return nil
	})

	ctx := context.Background()
	orderBus.Emit(ctx, "created", OrderEvent{ID: "B456", Amount: 199.9})
	orderBus.Wait()
}
```

## 五、注意事项

| 维度 | 注意点 | 说明 |
|------|--------|------|
| 并发安全 | 通知时持有读锁，增删时持有写锁 | 避免死锁：不要在 OnUpdate 中调 Attach |
| goroutine 泄漏 | Close 时关闭所有 channel | 确保 subscriber 的 goroutine 能退出 |
| 背压控制 | channel 满时选择丢弃或阻塞 | 根据业务选择策略 |
| Panic 隔离 | handler 中 recover | 一个 observer panic 不影响其他 |
| 顺序性 | 异步模式不保证顺序 | 需要顺序用同步模式 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| Notify 时遍历修改 slice | 用读写锁保护，或复制一份再遍历 |
| handler 中阻塞操作拖慢通知 | 异步通知，用 channel 或 goroutine |
| 只 Subscribe 不 Unsubscribe | 提供 Detach/Close 清理机制 |
| channel 无缓冲导致 Publish 阻塞 | 带缓冲 channel + select default 丢弃 |
| handler panic 导致整个系统崩溃 | 每个 handler 加 recover |
| 忽略 context 取消 | 传递 context，支持优雅关闭 |

## 七、练习题

**🟢 初级：** 实现一个温度传感器（Subject）和两个观察者（显示器、报警器），温度超过阈值时报警器触发。

**🟡 中级：** 实现一个支持通配符 topic 的 EventBus（如 `order.*` 匹配 `order.created` 和 `order.cancelled`）。

**🔴 高级：** 实现一个分布式事件总线，支持跨进程通信（通过 Redis Pub/Sub 或 NATS），本地接口保持不变。

## 十一、代码演进

**V1 — 同步通知（基础版）：**
- Subject 持有 Observer 接口切片
- Notify 同步调用所有观察者
- 简单直观，但 handler 阻塞会拖慢通知

**V2 — 异步 Channel EventBus：**
- 基于 channel 的发布订阅
- 带缓冲防止 Publish 阻塞
- 支持 topic 路由和背压丢弃
- Close 优雅关闭所有 channel

**V3 — 泛型事件总线 + 错误处理：**
- Go 1.18 泛型支持，类型安全
- context 传播，支持超时取消
- 每个 handler 独立 goroutine + panic recover
- Wait() 等待所有 handler 完成
- 生产级错误隔离
