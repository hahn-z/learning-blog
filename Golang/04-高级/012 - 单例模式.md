# 092-单例模式

> **难度：🟡 中高级** | 预计阅读：20 分钟 | 2024 年更新

## 一、概念讲解

**单例模式（Singleton）** 确保一个类型只有一个实例，并提供全局访问点。在 Go 中，单例常用于：
- 全局配置管理
- 数据库连接池
- 日志实例
- 缓存服务

Go 没有类和构造函数的访问修饰符，实现单例需要依靠包级别的可见性规则和同步原语。**核心挑战：并发安全 + 延迟初始化。**

## 三、完整 Go 代码

```go
package main

import (
	"fmt"
	"sync"
	"sync/atomic"
)

// ============================================
// V1: Basic — NOT thread-safe
// ============================================

var instanceV1 *ConfigV1

type ConfigV1 struct {
	AppName string
	Port    int
}

func GetConfigV1() *ConfigV1 {
	if instanceV1 == nil {
		instanceV1 = &ConfigV1{AppName: "myapp", Port: 8080}
	}
	return instanceV1
}

// ============================================
// V2: sync.Once — recommended approach
// ============================================

var (
	once     sync.Once
	instance *Config
)

// Config holds global application configuration
type Config struct {
	AppName string
	Port    int
	Version string
}

func GetConfig() *Config {
	once.Do(func() {
		instance = &Config{
			AppName: "myapp",
			Port:    8080,
			Version: "1.0.0",
		}
		fmt.Println("Config initialized once")
	})
	return instance
}

// ============================================
// V3: Generic singleton with reload support
// ============================================

// Singleton provides a generic thread-safe singleton holder
type Singleton[T any] struct {
	value  atomic.Pointer[T]
	once   sync.Once
	build  func() *T
	mu     sync.Mutex // for reload
}

// NewSingleton creates a singleton holder with a builder function
func NewSingleton[T any](build func() *T) *Singleton[T] {
	return &Singleton[T]{build: build}
}

// Get returns the singleton instance, initializing on first call
func (s *Singleton[T]) Get() *T {
	s.once.Do(func() {
		s.value.Store(s.build())
	})
	return s.value.Load()
}

// Reload recreates the singleton instance (thread-safe)
func (s *Singleton[T]) Reload() *T {
	s.mu.Lock()
	defer s.mu.Unlock()
	newVal := s.build()
	s.value.Store(newVal)
	return newVal
}

// ============================================
// Demo
// ============================================

func main() {
	// V2: sync.Once
	var wg sync.WaitGroup
	for i := 0; i < 10; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()
			cfg := GetConfig()
			fmt.Printf("goroutine got config: %s:%d\n", cfg.AppName, cfg.Port)
		}()
	}
	wg.Wait()

	// V3: Generic singleton with reload
	cfgSingleton := NewSingleton(func() *Config {
		return &Config{AppName: "reloaded-app", Port: 9090, Version: "2.0.0"}
	})

	cfg := cfgSingleton.Get()
	fmt.Printf("Initial: %s v%s\n", cfg.AppName, cfg.Version)

	cfg.Reload()
	cfg2 := cfgSingleton.Get()
	fmt.Printf("After reload: %s v%s\n", cfg2.AppName, cfg2.Version)
}
```

## 五、注意事项

| 维度 | 注意点 | 说明 |
|------|--------|------|
| sync.Once | 只执行一次，即使 panic 也算执行过 | Go 1.21+ `sync.OnceFunc` 更安全 |
| init() | 包加载时执行，无法延迟 | 适合必须初始化的场景 |
| 并发安全 | 多 goroutine 同时调用必须加锁 | sync.Once 内部已处理 |
| 可测试性 | 单例使测试难以隔离 | 考虑依赖注入替代 |
| 泛型 | Go 1.18+ 支持 | Singleton[T] 需要 Go 1.18+ |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 裸 if nil 判断创建单例 | 用 `sync.Once` 保证并发安全 |
| 单例中引用可变全局状态 | 单例内部状态用 Mutex 保护 |
| init() 中初始化数据库连接 | 用 `sync.Once` 延迟到首次使用 |
| 到处硬编码 `GetConfig()` | 通过接口注入，方便测试 |
| 单例持有 goroutine 不释放 | 提供 Close/Shutdown 方法 |

## 七、练习题

**🟢 初级：** 用 `sync.Once` 实现一个全局日志实例，支持 `Log(msg string)` 方法。

**🟡 中级：** 实现泛型单例 `Singleton[T]`，支持 `Reload()` 方法重新创建实例。

**🔴 高级：** 设计一个支持依赖注入的配置管理器，在测试中可以替换为 mock 实现，生产中使用单例。

## 九、举一反三

| 场景 | 实现方式 | 适用条件 |
|------|----------|----------|
| 配置管理 | sync.Once | 启动时加载，运行时只读 |
| 数据库连接池 | sync.Once + context | 需要优雅关闭 |
| 日志实例 | sync.Once | 全局共享，很少改变 |
| 分布式唯一 ID | 原子计数器 | 需要严格递增 |
| 缓存实例 | sync.Once + 定时刷新 | 需要后台更新 |

## 十一、代码演进

**V1 — 裸 if nil 判断：**
- 简单直接，但存在竞态条件
- 多 goroutine 可能创建多个实例

**V2 — sync.Once 标准实现：**
- 利用 `sync.Once` 保证只执行一次
- 并发安全，推荐用法
- 适合大多数单例场景

**V3 — 泛型 Singleton[T] + Reload：**
- Go 1.18 泛型支持，类型安全
- 支持 Reload 热更新
- 可复用的通用单例容器
