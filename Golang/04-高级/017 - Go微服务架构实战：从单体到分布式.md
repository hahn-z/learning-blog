# Go微服务架构实战：从单体到分布式

> **难度标注：** ⭐⭐⭐⭐⭐ 高级
> **前置知识：** Go并发、HTTP、数据库、Docker基础
> **预计用时：** 90分钟

---

## 一、概念讲解

微服务不是目的，而是解决规模问题的手段。本文从实际出发，展示如何将一个单体Go应用逐步拆分为微服务。

### 何时拆分？

| 信号 | 描述 |
|------|------|
| 团队 > 8人 | 代码冲突频繁，沟通成本高 |
| 模块独立部署需求 | 用户模块要一天更新3次，订单模块要稳定 |
| 不同模块性能需求不同 | 搜索要Elasticsearch，支付要强一致 |
| 单库瓶颈 | 数据量或QPS超过单机上限 |

### 核心组件

```
单体应用 → API网关 + 用户服务 + 订单服务 + 通知服务
              ↓           ↓          ↓          ↓
           gRPC       gRPC       gRPC      消息队列
```

## 三、完整Go代码

### v1 单体应用

```go
package main

import (
	"database/sql"
	"encoding/json"
	"fmt"
	"log"
	"net/http"

	_ "github.com/go-sql-driver/mysql"
)

// User represents a user entity
type User struct {
	ID    int    `json:"id"`
	Name  string `json:"name"`
	Email string `json:"email"`
}

// Order represents an order entity
type Order struct {
	ID     int     `json:"id"`
	UserID int     `json:"user_id"`
	Amount float64 `json:"amount"`
	Status string  `json:"status"`
}

// MonolithApp holds all dependencies
type MonolithApp struct {
	db *sql.DB
}

// GetUser fetches a user by ID
func (app *MonolithApp) GetUser(w http.ResponseWriter, r *http.Request) {
	id := r.URL.Query().Get("id")
	var u User
	err := app.db.QueryRow("SELECT id, name, email FROM users WHERE id = ?", id).Scan(&u.ID, &u.Name, &u.Email)
	if err != nil {
		http.Error(w, "user not found", 404)
		return
	}
	json.NewEncoder(w).Encode(u)
}

// CreateOrder creates a new order
func (app *MonolithApp) CreateOrder(w http.ResponseWriter, r *http.Request) {
	var o Order
	if err := json.NewDecoder(r.Body).Decode(&o); err != nil {
		http.Error(w, "bad request", 400)
		return
	}
	result, err := app.db.Exec("INSERT INTO orders (user_id, amount, status) VALUES (?, ?, ?)", o.UserID, o.Amount, "pending")
	if err != nil {
		http.Error(w, "db error", 500)
		return
	}
	o.ID, _ = result.LastInsertId()
	o.Status = "pending"
	// Send notification inline (tight coupling!)
	log.Printf("Sending email to user %d: order %d created", o.UserID, o.ID)
	json.NewEncoder(w).Encode(o)
}

func main() {
	db, err := sql.Open("mysql", "blog:password@tcp(127.0.0.1:3306)/monolith")
	if err != nil {
		log.Fatal(err)
	}
	defer db.Close()

	app := &MonolithApp{db: db}
	http.HandleFunc("/user", app.GetUser)
	http.HandleFunc("/order", app.CreateOrder)
	log.Fatal(http.ListenAndServe(":8080", nil))
}
```

### v2 微服务拆分：gRPC服务

**用户服务 proto（user.proto）：**

```protobuf
syntax = "proto3";
package user;
option go_package = "./pb";

service UserService {
  rpc GetUser(GetUserRequest) returns (GetUserResponse);
}

message GetUserRequest { int32 id = 1; }
message GetUserResponse {
  int32 id = 1;
  string name = 2;
  string email = 3;
}
```

**用户服务实现：**

```go
package main

import (
	"context"
	"database/sql"
	"log"
	"net"

	_ "github.com/go-sql-driver/mysql"
	"google.golang.org/grpc"
	pb "user-service/pb"
)

type server struct {
	pb.UnimplementedUserServiceServer
	db *sql.DB
}

// GetUser returns user info by ID
func (s *server) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.GetUserResponse, error) {
	var resp pb.GetUserResponse
	err := s.db.QueryRow("SELECT id, name, email FROM users WHERE id = ?", req.Id).
		Scan(&resp.Id, &resp.Name, &resp.Email)
	if err != nil {
		return nil, err
	}
	return &resp, nil
}

func main() {
	db, _ := sql.Open("mysql", "blog:password@tcp(127.0.0.1:3306)/users")
	defer db.Close()

	lis, _ := net.Listen("tcp", ":50051")
	s := grpc.NewServer()
	pb.RegisterUserServiceServer(s, &server{db: db})
	log.Println("User service on :50051")
	s.Serve(lis)
}
```

**API网关：**

```go
package main

import (
	"context"
	"encoding/json"
	"log"
	"net/http"
	"time"

	pb "user-service/pb"
	"google.golang.org/grpc"
	"google.golang.org/grpc/credentials/insecure"
)

// Gateway routes HTTP to gRPC services
type Gateway struct {
	userClient pb.UserServiceClient
}

func (g *Gateway) HandleGetUser(w http.ResponseWriter, r *http.Request) {
	ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
	defer cancel()

	id := r.URL.Query().Get("id")
	var idInt int32
	fmt.Sscanf(id, "%d", &idInt)

	resp, err := g.userClient.GetUser(ctx, &pb.GetUserRequest{Id: idInt})
	if err != nil {
		http.Error(w, "user service error", 502)
		return
	}
	json.NewEncoder(w).Encode(resp)
}

func main() {
	conn, err := grpc.Dial("localhost:50051", grpc.WithTransportCredentials(insecure.NewCredentials()))
	if err != nil {
		log.Fatal(err)
	}
	defer conn.Close()

	gw := &Gateway{userClient: pb.NewUserServiceClient(conn)}
	http.HandleFunc("/user", gw.HandleGetUser)
	log.Println("Gateway on :8080")
	log.Fatal(http.ListenAndServe(":8080", nil))
}
```

### v3 生产版本：服务发现 + 健康检查 + 重试

```go
// Service registry with TTL-based health checking
type Registry struct {
	mu      sync.RWMutex
	services map[string][]string // serviceName -> []addr
}

func (r *Registry) Register(name, addr string) {
	r.mu.Lock()
	r.services[name] = append(r.services[name], addr)
	r.mu.Unlock()
}

func (r *Registry) Discover(name string) (string, error) {
	r.mu.RLock()
	defer r.mu.RUnlock()
	addrs := r.services[name]
	if len(addrs) == 0 {
		return "", fmt.Errorf("service not found: %s", name)
	}
	return addrs[0], nil // simple round-robin in production
}

// Retry wrapper for gRPC calls
func withRetry(fn func() error, maxRetries int) error {
	var err error
	for i := 0; i < maxRetries; i++ {
		if err = fn(); err == nil {
			return nil
		}
		time.Sleep(time.Duration(i+1) * 100 * time.Millisecond)
	}
	return err
}
```

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| 数据一致性 | 跨服务事务用Saga模式，不要分布式锁 |
| 服务粒度 | 先粗后细，不要过度拆分 |
| gRPC版本 | proto定义要向后兼容 |
| 超时设置 | 每层都要设超时，避免级联阻塞 |
| 日志追踪 | 用traceID串联跨服务请求 |
| 网络分区 | 考虑服务不可用时的降级策略 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 一上来就微服务 | 先单体后拆分，有明确需求再拆 |
| 共享数据库 | 每个服务独立数据库 |
| 同步调用一切 | 非核心路径用消息队列异步 |
| 忽略服务发现 | 引入Consul/etcd或自建注册中心 |
| 不设超时 | 每个gRPC/HTTP调用都要设超时 |
| 微服务=Kubernetes | 小团队用Docker Compose就够了 |

## 十一、代码演进总结

| 版本 | 内容 | 适用场景 |
|------|------|---------|
| v1 | 单体应用 | 初创期，快速验证 |
| v2 | gRPC微服务 + 网关 | 成长期，团队分工 |
| v3 | 服务发现 + 重试 + 健康检查 | 成熟期，高可用 |

> **核心思想：** 微服务是演进而非设计出来的。先让单体跑起来，有了明确的拆分理由再动手。
