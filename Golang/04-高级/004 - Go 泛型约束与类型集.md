# Go 泛型约束与类型集

**难度：⭐⭐⭐⭐（高级）**

泛型约束是 Go 泛型系统的核心设计。约束不仅仅是"限制"，更是"能力声明"——它告诉编译器类型参数能做什么操作。本文深入类型集（Type Set）、近似元素（`~`）、接口约束演进以及 `cmp` 标准库包。

├── 近似元素 ~

## 📖 概念讲解

### 类型集（Type Set）

Go 1.18 对接口做了根本性扩展：接口不仅可以包含方法，还可以包含**类型集**。类型集定义了一组允许的具体类型，是泛型约束的基础。

**核心规则**：
- `A | B`：并集（union），类型属于 A 或 B
- 嵌入多个接口：交集（intersection），类型必须同时满足所有接口
- 空接口 `interface{}` = 所有类型的集合

### 近似元素 `~`

`~T` 表示"底层类型为 T 的所有类型"。这是约束系统的精髓——它让泛型函数既能匹配原生类型，也能匹配用户自定义的类型别名。

> **规则**：`~` 只能用于底层类型为基本类型的类型，不能写 `~MyStruct`。

### `comparable` 与 `cmp.Ordered` 的区别

| 特性 | `comparable` | `cmp.Ordered` |
|------|-------------|---------------|
| 支持 `==` `!=` | ✅ | ✅ |
| 支持 `<` `>` `<=` `>=` | ❌ | ✅ |
| 包含 `bool` | ✅ | ❌ |
| 包含 字符串 | ✅ | ✅ |
| 包含 结构体 | ✅（字段全 comparable） | ❌ |

## 💻 完整代码示例

### v1：基础类型集与近似元素

```go
package main

import "fmt"

// Type set: declare allowed concrete types
type Signed interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64
}

type Unsigned interface {
    ~uint | ~uint8 | ~uint16 | ~uint32 | ~uint64 | ~uintptr
}

// Combined constraint: union of Signed and Unsigned
type Integer interface {
    Signed | Unsigned
}

// Generic function using Integer constraint
func Abs[T Integer](n T) T {
    if n < 0 {
        return -n
    }
    return n
}

// Custom type with underlying type int
type MyInt int

func main() {
    // Works with built-in types
    fmt.Println(Abs(-42))     // 42
    fmt.Println(Abs(int8(-5))) // 5

    // Works with custom types (thanks to ~)
    fmt.Println(Abs(MyInt(-100))) // 100
}
```

**执行预览**：
```
42
5
100
```

### v2：comparable 与 cmp 包实战

```go
package main

import (
    "cmp"
    "fmt"
    "slices"
)

// comparable: supports == and !=
func IndexOf[T comparable](slice []T, target T) int {
    for i, v := range slice {
        if v == target {
            return i
        }
    }
    return -1
}

// cmp.Ordered: supports < <= >= >
func Clamp[T cmp.Ordered](value, min, max T) T {
    if value < min {
        return min
    }
    if value > max {
        return max
    }
    return value
}

// Generic Min/Max using cmp.Ordered
func Min[T cmp.Ordered](a, b T) T {
    if a < b {
        return a
    }
    return b
}

func Max[T cmp.Ordered](a, b T) T {
    if a > b {
        return a
    }
    return b
}

func main() {
    // IndexOf with different types
    fmt.Println(IndexOf([]string{"a", "b", "c"}, "b"))    // 1
    fmt.Println(IndexOf([]int{10, 20, 30}, 20))           // 1

    // Clamp
    fmt.Println(Clamp(15, 0, 10))     // 10
    fmt.Println(Clamp(-3.5, 0.0, 1.0)) // 0

    // Min/Max
    fmt.Println(Min(3, 7))            // 3
    fmt.Println(Max("apple", "banana")) // banana

    // cmp.Compare: returns -1, 0, 1
    fmt.Println(cmp.Compare(3, 5))    // -1
    fmt.Println(cmp.Compare("b", "a")) // 1

    // cmp.Or: returns first non-zero value
    fmt.Println(cmp.Or("", "fallback"))  // fallback
    fmt.Println(cmp.Or("hello", "fallback")) // hello

    // Sort with cmp
    nums := []int{3, 1, 4, 1, 5, 9}
    slices.SortFunc(nums, cmp.Compare)
    fmt.Println(nums) // [1 1 3 4 5 9]
}
```

**执行预览**：
```
1
1
10
0
3
banana
-1
1
fallback
hello
[1 1 3 4 5 9]
```

### v3：组合约束与高级模式

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
)

// Pattern 1: Behavior constraint (method set)
type Stringer interface {
    String() string
}

func Join[T Stringer](items []T, sep string) string {
    parts := make([]string, len(items))
    for i, item := range items {
        parts[i] = item.String()
    }
    return strings.Join(parts, sep)
}

// Pattern 2: Type constraint (type set)
type Numeric interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~float32 | ~float64
}

func Sum[T Numeric](nums []T) T {
    var total T
    for _, n := range nums {
        total += n
    }
    return total
}

// Pattern 3: Combined constraint (behavior + type)
type OrderedStringer interface {
    ~int | ~float64
    String() string
}

// Intersection constraint: must be int AND implement String()
type StringableInt interface {
    ~int
    String() string
}

type RichInt int

func (r RichInt) String() string {
    return strconv.Itoa(int(r))
}

func Describe[T StringableInt](v T) string {
    return fmt.Sprintf("value=%s", v.String())
}

func main() {
    // Join with Stringer constraint
    nums := []RichInt{1, 2, 3}
    fmt.Println(Join(nums, "-")) // 1-2-3

    // Sum with Numeric constraint
    fmt.Println(Sum([]int{1, 2, 3}))       // 6
    fmt.Println(Sum([]float64{1.5, 2.5}))  // 4

    // Combined constraint
    fmt.Println(Describe(RichInt(42))) // value=42

    // Generic map function using interface
    doubles := Map([]int{1, 2, 3}, func(n int) int { return n * 2 })
    fmt.Println(doubles) // [2 4 6]
}

// Generic Map (no method type params needed, use function param)
func Map[T any, U any](slice []T, fn func(T) U) []U {
    result := make([]U, len(slice))
    for i, v := range slice {
        result[i] = fn(v)
    }
    return result
}
```

**执行预览**：
```
1-2-3
6
4
value=42
[2 4 6]
```

## ⚠️ 注意事项

| 序号 | 注意点 | 说明 |
|------|--------|------|
| 1 | `~` 不适用于接口类型 | 只能用于基本类型的底层类型 |
| 2 | 约束越宽泛，能做的操作越少 | `any` 约束只能赋值，不能运算 |
| 3 | 类型集交集是强大工具 | 可组合多个约束实现精确控制 |
| 4 | `comparable` ≠ `cmp.Ordered` | 前者 `==`，后者 `<` |
| 5 | 约束设计要考虑扩展性 | 过窄的约束会限制复用 |
| 6 | 方法不能引入新类型参数 | 用函数参数代替 |

## 🚫 避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|------------|
| 约束写 `int` 不带 `~`，导致自定义类型不匹配 | 用 `~int` 匹配底层类型为 int 的所有类型 |
| 用 `any` 约束后尝试 `+` 运算 | 定义 `Numeric` 约束包含数值类型集 |
| 对泛型参数做类型断言 `x.(T)` | 使用类型开关 `switch v := x.(type)` 在 `any` 上 |
| 方法上定义类型参数 `func (c C[T]) Map[U]()` | 定义包级函数 `func Map[T, U any]()` |
| 混淆 `comparable` 和 `cmp.Ordered` | 需要排序用 `cmp.Ordered`，只需判等用 `comparable` |

## 🎯 练习题

**🟢 基础题**：定义约束 `FloatOnly interface{ ~float32 | ~float64 }`，实现泛型函数 `Sqrt[T FloatOnly](n T) T`（使用 Newton 迭代法）。

**🟡 进阶题**：实现泛型函数 `Clamp[T cmp.Ordered](value, min, max T) T`，并在此基础上实现泛型 `BoundedSlice[T cmp.Ordered]` 类型，自动将追加的值限制在范围内。

**🔴 挑战题**：设计一个完整的 `OrderedMap[K comparable, V cmp.Ordered]` 泛型结构体，支持 `Set`/`Get`/`Delete`/`Min`/`Max`/`Range` 操作，并与标准库 `maps` 包对比 API 设计。

## 📝 知识点总结

```
泛型约束体系
├── 核心概念
│   ├── 类型集 → 控制允许的类型范围
│   ├── 近似元素 ~ → 匹配底层类型
│   └── 接口演化 → Go 1.18 接口 = 方法集 + 类型集
├── 内置约束
│   ├── any → 无约束（所有类型）
│   └── comparable → 支持 == !=
├── 标准库 (cmp)
│   ├── Ordered → 支持 < <= >= >
│   ├── Compare(a,b) → -1/0/1
│   └── Or(vals...) → 第一个非零值
├── 约束设计三模式
│   ├── 行为约束 → 方法集（做什么）
│   ├── 类型约束 → 类型集（是什么）
│   └── 组合约束 → 行为 + 类型（精确控制）
└── 限制与边界
    ├── 方法无类型参数 → 用函数参数绕过
    ├── 无特化 → 用 if/switch 在运行时分派
    └── 无运算符重载 → 用类型集预定义
```

## 🔄 举一反三

| 场景 | 约束选择 | 适用场景 |
|------|----------|----------|
| 通用容器（Map/Set） | `comparable` | 需要 key 去重 |
| 数值计算（Sum/Avg） | 自定义 `Numeric` | 需要 `+` `-` `*` 运算 |
| 排序/比较 | `cmp.Ordered` | 需要 `<` `>` 比较 |
| 序列化/展示 | 自定义 `Stringer` | 需要 `String()` 方法 |
| 范围限制 | `cmp.Ordered` | 需要 min/max 边界 |
| 函数式变换 | `any`（用函数参数） | Map/Filter/Reduce |
