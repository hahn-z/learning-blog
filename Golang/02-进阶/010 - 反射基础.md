# 040 - 反射基础

> **难度：** ⭐⭐⭐⭐
> **关键词：** reflect、Type、Value、Kind、FieldByName、MethodByName

## 一、概念讲解

反射（Reflection）是 Go 语言在运行时检查类型信息、读取和修改值的能力。Go 的 `reflect` 包提供了两大核心类型：

- **reflect.Type**：类型的元信息（名称、种类、字段、方法等）
- **reflect.Value**：运行时的值（可以读取、修改、调用方法）

反射的本质是：**在编译期不知道具体类型的情况下，在运行时动态获取类型信息和操作值**。

### 为什么需要反射？

- `fmt.Println` 能打印任意类型 → 靠反射
- `encoding/json` 能序列化任意结构体 → 靠反射
- ORM 框架能映射数据库行到结构体 → 靠反射

### Kind — 类型种类

Go 把类型分为底层的"种类"（Kind），如 `Struct`、`Slice`、`Map`、`Func` 等。注意 Type 和 Kind 的区别：

```go
type User struct{ Name string }
// Type.Name() = "User", Kind() = Struct
```

## 二、脑图

```
反射基础 (reflect)
├── reflect.Type (类型元信息)
│   ├── Name()      → 类型名
│   ├── Kind()      → 种类 (Struct/Slice/Map/...)
│   ├── NumField()  → 结构体字段数
│   ├── Field(i)    → 按索引取字段
│   ├── FieldByName()  → 按名称取字段
│   ├── NumMethod()    → 方法数量
│   └── Method(i)      → 按索引取方法
├── reflect.Value (运行时值)
│   ├── Interface()    → 转回 interface{}
│   ├── Kind()         → 种类
│   ├── Type()         → 获取 Type
│   ├── FieldByName()  → 读取结构体字段
│   ├── MethodByName() → 获取方法
│   └── Call()         → 调用方法
└── 性能影响
    └── 比静态调用慢 10-100 倍
```

## 三、完整代码

```go
package main

import (
	"fmt"
	"reflect"
)

// User demonstrates a struct with fields and methods
type User struct {
	Name string `json:"name" db:"user_name"`
	Age  int    `json:"age"`
	City string `json:"city"`
}

// Greet is a method bound to User
func (u User) Greet() string {
	return fmt.Sprintf("Hello, I'm %s, age %d, from %s", u.Name, u.Age, u.City)
}

// HasBirthday is a pointer receiver method
func (u *User) HasBirthday() {
	u.Age++
}

func main() {
	u := User{Name: "Alice", Age: 30, City: "Shanghai"}

	// === 1. Basic Type and Value ===
	t := reflect.TypeOf(u)
	v := reflect.ValueOf(u)

	fmt.Println("=== Type Info ===")
	fmt.Printf("Type Name: %s\n", t.Name())   // User
	fmt.Printf("Type Kind: %s\n", t.Kind())    // struct
	fmt.Printf("Package Path: %s\n", t.PkgPath())

	fmt.Println("\n=== Value Info ===")
	fmt.Printf("Value Kind: %s\n", v.Kind())   // struct
	fmt.Printf("Value Type: %s\n", v.Type())   // User
	fmt.Printf("Interface: %v\n", v.Interface()) // {Alice 30 Shanghai}

	// === 2. Iterate struct fields ===
	fmt.Println("\n=== Fields ===")
	for i := 0; i < t.NumField(); i++ {
		field := t.Field(i)
		fieldValue := v.Field(i)
		fmt.Printf("  [%d] %s (%s) = %v, tag=%s\n",
			i, field.Name, field.Type, fieldValue.Interface(), field.Tag)
	}

	// === 3. FieldByName ===
	fmt.Println("\n=== FieldByName ===")
	if f, ok := t.FieldByName("Name"); ok {
		fmt.Printf("  Found field: %s, type: %s\n", f.Name, f.Type)
	}
	if fv := v.FieldByName("Age"); fv.IsValid() {
		fmt.Printf("  Age value: %v\n", fv.Int())
	}

	// === 4. Methods ===
	fmt.Println("\n=== Methods ===")
	fmt.Printf("NumMethod on User value: %d\n", v.NumMethod())
	fmt.Printf("NumMethod on User type: %d\n", t.NumMethod())

	// MethodByName + Call
	greetMethod := v.MethodByName("Greet")
	if greetMethod.IsValid() {
		results := greetMethod.Call(nil) // no args
		fmt.Printf("  Greet result: %s\n", results[0].String())
	}

	// === 5. Pointer receiver methods ===
	fmt.Println("\n=== Pointer Methods ===")
	pv := reflect.ValueOf(&u)
	fmt.Printf("NumMethod on *User: %d\n", pv.NumMethod())
	// Note: pointer Value includes both value and pointer receiver methods

	hasBirthday := pv.MethodByName("HasBirthday")
	if hasBirthday.IsValid() {
		hasBirthday.Call(nil)
		fmt.Printf("  After HasBirthday, Age: %d\n", u.Age)
	}

	// === 6. Kind switch ===
	fmt.Println("\n=== Kind Examples ===")
	examine("hello")
	examine(42)
	examine([]int{1, 2, 3})
	examine(map[string]int{"a": 1})
}

// examine prints type info using Kind switch
func examine(v interface{}) {
	t := reflect.TypeOf(v)
	switch t.Kind() {
	case reflect.String:
		fmt.Printf("  String: %s (len=%d)\n", v, len(v.(string)))
	case reflect.Int:
		fmt.Printf("  Int: %d\n", v)
	case reflect.Slice:
		fmt.Printf("  Slice: len=%d, elem=%s\n",
			reflect.ValueOf(v).Len(), t.Elem())
	case reflect.Map:
		fmt.Printf("  Map: len=%d, key=%s, val=%s\n",
			reflect.ValueOf(v).Len(), t.Key(), t.Elem())
	default:
		fmt.Printf("  Other: %s (%s)\n", t.Name(), t.Kind())
	}
}
```

## 五、注意事项

| 项目 | 说明 |
|------|------|
| ValueOf 传值 vs 传指针 | `ValueOf(u)` 只能读取，`ValueOf(&u)` 才能 CanSet |
| 值接收者 vs 指针接收者 | 值类型的 NumMethod 不包含指针接收者方法 |
| FieldByName 查找 | 大小写敏感，未导出字段也能读到但不建议依赖 |
| Kind vs Type | Kind 是底层种类（Struct），Type 是具体名称（User） |
| Nil 检查 | Value.IsValid() 检查是否为零值，不要直接判 nil |

## 六、避坑指南

❌ **直接修改不可寻址的值**
```go
v := reflect.ValueOf(42)
v.SetInt(100) // panic: call of reflect.Value.SetInt on unaddressable value
```
✅ **使用指针 + Elem 获取可寻址值**
```go
x := 42
v := reflect.ValueOf(&x).Elem()
v.SetInt(100) // OK
```

❌ **忽略 IsValid 检查**
```go
v := reflect.ValueOf(u).FieldByName("NotExist")
fmt.Println(v.Int()) // panic: Method on nil Interface
```
✅ **先检查 IsValid**
```go
v := reflect.ValueOf(u).FieldByName("NotExist")
if v.IsValid() {
    fmt.Println(v.Int())
}
```

❌ **在性能敏感路径使用反射**
```go
for i := 0; i < 1000000; i++ {
    v := reflect.ValueOf(obj)
    name := v.FieldByName("Name").String() // slow!
}
```
✅ **缓存反射结果或用代码生成替代**
```go
// Use code generation (like easyjson) for hot paths
// Or cache reflect.Type/Field offsets
```

## 七、练习题

🟢 **基础：** 写一个函数 `PrintFields(x interface{})`，遍历任意结构体的字段名和值并打印。

## 八、知识点总结

```
反射基础
├── 核心入口
│   ├── reflect.TypeOf(x) → Type
│   └── reflect.ValueOf(x) → Value
├── Type 操作
│   ├── Name() / Kind()
│   ├── NumField() / Field(i) / FieldByName()
│   ├── NumMethod() / Method(i)
│   ├── Key() / Elem() (Map/Slice/Chan/Ptr)
│   └── Field(i).Tag.Get("json")
├── Value 操作
│   ├── Interface() → 转回 interface{}
│   ├── Int() / String() / Float() 等
│   ├── FieldByName() → 取字段 Value
│   ├── MethodByName() → 取方法 Value
│   └── Call(in []Value) → 调用方法
├── 可寻址性
│   ├── CanAddr() / CanSet()
│   ├── Elem() 解引用指针
│   └── 传指针才能修改
└── 性能
    ├── 反射调用比静态调用慢 10-100x
    ├── 避免在热路径使用
    └── 考虑代码生成替代
```

## 十一、代码演进

### v1 — 基本类型检查

```go
func printType(x interface{}) {
    fmt.Println(reflect.TypeOf(x).Name())
}
// Only prints type name, no field/method inspection
```

### v2 — 结构体字段遍历

```go
func printFields(x interface{}) {
    v := reflect.ValueOf(x)
    t := v.Type()
    for i := 0; i < t.NumField(); i++ {
        fmt.Printf("%s: %v\n", t.Field(i).Name, v.Field(i).Interface())
    }
}
// Can inspect fields but cannot modify or call methods
```

### v3 — 完整反射工具（含方法调用和 Kind switch）

```go
func inspect(x interface{}) {
    v := reflect.ValueOf(x)
    t := v.Type()

    // Print all fields
    for i := 0; i < t.NumField(); i++ {
        f := t.Field(i)
        fmt.Printf("Field %s: %v (tag: %s)\n",
            f.Name, v.Field(i).Interface(), f.Tag.Get("json"))
    }

    // Print all methods
    for i := 0; i < v.NumMethod(); i++ {
        m := t.Method(i)
        fmt.Printf("Method %s: %v\n", m.Name, m.Type)
    }

    // Try to call a method
    if greet := v.MethodByName("Greet"); greet.IsValid() {
        results := greet.Call(nil)
        fmt.Println("Greet:", results[0])
    }
}
// Full inspection: fields, tags, methods, and invocation
```

> **小结：** 反射是 Go 运行时类型系统的基石。掌握 Type/Value 的基本操作，理解可寻址性规则，避免在热路径滥用反射，是 Go 工程师的必修课。
