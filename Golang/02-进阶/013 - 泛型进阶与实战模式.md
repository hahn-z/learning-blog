# 043 - 泛型进阶与实战模式

> **难度：** ⭐⭐⭐⭐（进阶）
>
> **前提：** 已掌握 Go 泛型基础语法（类型参数、基础约束），希望深入实战应用。

---

## 一、概念讲解

Go 1.18 引入泛型后，社区经历了从狂热到理性的过程。泛型不是银弹，但在数据结构和工具函数场景中确实能大幅减少重复代码。本文聚焦**泛型结构体、约束进阶、泛型与接口的配合**三大实战方向。

### 1.1 泛型结构体

泛型最自然的应用场景是**容器类型**：栈、集合、有序映射等。这些结构体对元素类型无关，但对行为有统一要求。

### 1.2 泛型方法约束

Go 的泛型方法不能引入新的类型参数（这是语言限制），但可以利用结构体的类型参数实现通用方法。

### 1.3 泛型与接口配合

泛型约束（constraint）本质上是一个接口，可以包含方法签名。这意味着你可以要求类型参数既满足底层类型约束，又实现特定方法。

### 1.4 泛型 Channel

Channel 天然支持泛型——`chan T` 中的 `T` 可以是任意类型参数。

### 1.5 约束进阶

- **联合类型（Union）：** `int | int32 | int64`
- **近似类型（Approximation）：** `~int` 匹配所有底层类型为 int 的自定义类型
- **组合约束：** 将联合与接口方法组合

## 三、完整 Go 代码

### 3.1 泛型 Stack

```go
package main

import "fmt"

// Stack is a generic LIFO data structure.
type Stack[T any] struct {
    items []T
}

// Push adds an element to the top of the stack.
func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

// Pop removes and returns the top element. Returns zero value and false if empty.
func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    top := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return top, true
}

// Peek returns the top element without removing it.
func (s *Stack[T]) Peek() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    return s.items[len(s.items)-1], true
}

// Len returns the number of elements.
func (s *Stack[T]) Len() int {
    return len(s.items)
}

func main() {
    // String stack
    s := Stack[string]{}
    s.Push("hello")
    s.Push("world")
    
    for v, ok := s.Pop(); ok; v, ok = s.Pop() {
        fmt.Println(v)
    }
}
```

### 3.2 泛型 Set

```go
package main

import "fmt"

// Set is a generic collection with unique elements.
// K must be comparable (supports == and !=).
type Set[K comparable] struct {
    m map[K]struct{}
}

// NewSet creates a new Set.
func NewSet[K comparable]() *Set[K] {
    return &Set[K]{m: make(map[K]struct{})}
}

// Add inserts an element. Returns false if already exists.
func (s *Set[K]) Add(k K) bool {
    if _, ok := s.m[k]; ok {
        return false
    }
    s.m[k] = struct{}{}
    return true
}

// Contains checks if an element exists.
func (s *Set[K]) Contains(k K) bool {
    _, ok := s.m[k]
    return ok
}

// Remove deletes an element.
func (s *Set[K]) Remove(k K) {
    delete(s.m, k)
}

// Union returns a new Set with elements from both sets.
func (s *Set[K]) Union(other *Set[K]) *Set[K] {
    result := NewSet[K]()
    for k := range s.m {
        result.Add(k)
    }
    for k := range other.m {
        result.Add(k)
    }
    return result
}

// Slice returns all elements as a slice (order not guaranteed).
func (s *Set[K]) Slice() []K {
    result := make([]K, 0, len(s.m))
    for k := range s.m {
        result = append(result, k)
    }
    return result
}

func main() {
    s := NewSet[int]()
    s.Add(1)
    s.Add(2)
    s.Add(1) // duplicate
    
    fmt.Printf("Contains 1: %v\n", s.Contains(1))
    fmt.Printf("Elements: %v\n", s.Slice())
}
```

### 3.3 泛型 OrderedMap

```go
package main

import "fmt"

// OrderedMap preserves insertion order, unlike built-in map.
type OrderedMap[K comparable, V any] struct {
    keys   []K
    entries map[K]V
}

func NewOrderedMap[K comparable, V any]() *OrderedMap[K, V] {
    return &OrderedMap[K, V]{
        entries: make(map[K]V),
    }
}

// Set stores a key-value pair. Updates value if key exists.
func (m *OrderedMap[K, V]) Set(k K, v V) {
    if _, exists := m.entries[k]; !exists {
        m.keys = append(m.keys, k)
    }
    m.entries[k] = v
}

// Get retrieves a value by key.
func (m *OrderedMap[K, V]) Get(k K) (V, bool) {
    v, ok := m.entries[k]
    return v, ok
}

// Range iterates in insertion order.
func (m *OrderedMap[K, V]) Range(fn func(k K, v V) bool) {
    for _, k := range m.keys {
        if !fn(k, m.entries[k]) {
            break
        }
    }
}

func main() {
    om := NewOrderedMap[string, int]()
    om.Set("banana", 2)
    om.Set("apple", 5)
    om.Set("cherry", 3)
    
    om.Range(func(k string, v int) bool {
        fmt.Printf("%s: %d\n", k, v)
        return true
    })
}
```

### 3.4 约束进阶：通用排序

```go
package main

import (
    "cmp"
    "fmt"
    "slices"
)

// Ordered is a constraint for types that can be ordered.
// cmp.Ordered is built-in since Go 1.21.
type Number interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~float32 | ~float64
}

// Sum adds all numbers in a slice.
func Sum[T Number](nums []T) T {
    var total T
    for _, n := range nums {
        total += n
    }
    return total
}

// Sortable constraint: must be ordered.
type Sortable interface {
    cmp.Ordered
}

// GenericSort sorts any ordered slice.
func GenericSort[T Sortable](s []T) {
    slices.Sort(s)
}

// Clamp restricts a value to [min, max].
func Clamp[T Number](val, min, max T) T {
    if val < min {
        return min
    }
    if val > max {
        return max
    }
    return val
}

type Celsius float64 // ~float64, satisfies Number constraint

func main() {
    nums := []int{5, 3, 8, 1, 9}
    GenericSort(nums)
    fmt.Printf("Sorted: %v\n", nums)
    fmt.Printf("Sum: %d\n", Sum(nums))
    fmt.Printf("Clamp(15, 0, 10) = %d\n", Clamp(15, 0, 10))
    
    // Custom type with ~float64
    temps := []Celsius{36.5, 38.2, 37.0}
    fmt.Printf("Temp sum: %.1f\n", Sum(temps))
}
```

### 3.5 泛型与接口配合

```go
package main

import "fmt"

// Stringer constraint: type must have a String() method.
type Stringer interface {
    String() string
}

// Join concatenates Stringer elements.
func Join[T Stringer](items []T, sep string) string {
    if len(items) == 0 {
        return ""
    }
    result := items[0].String()
    for _, item := range items[1:] {
        result += sep + item.String()
    }
    return result
}

// Person implements Stringer.
type Person struct {
    Name string
    Age  int
}

func (p Person) String() string {
    return fmt.Sprintf("%s(%d)", p.Name, p.Age)
}

// Combined constraint: ordered AND has a String method
type OrderedStringer interface {
    ~int | ~float64 | ~string
    String() string
}

func main() {
    people := []Person{
        {"Alice", 30},
        {"Bob", 25},
        {"Charlie", 35},
    }
    fmt.Println(Join(people, ", "))
}
```

### 3.6 泛型 Channel：Fan-In

```go
package main

import (
    "fmt"
    "sync"
)

// FanIn merges multiple channels into one.
func FanIn[T any](channels ...<-chan T) <-chan T {
    out := make(chan T)
    var wg sync.WaitGroup
    
    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan T) {
            defer wg.Done()
            for v := range c {
                out <- v
            }
        }(ch)
    }
    
    go func() {
        wg.Wait()
        close(out)
    }()
    
    return out
}

func main() {
    ch1 := make(chan string, 3)
    ch2 := make(chan string, 3)
    
    ch1 <- "A1"
    ch1 <- "A2"
    ch2 <- "B1"
    ch2 <- "B2"
    close(ch1)
    close(ch2)
    
    merged := FanIn(ch1, ch2)
    for v := range merged {
        fmt.Println(v)
    }
}
```

### 3.7 通用堆（Min-Heap）

```go
package main

import "fmt"

// Heap is a generic min-heap implementation.
type Heap[T any] struct {
    data []T
    less func(a, b T) bool // comparator function
}

// NewHeap creates a min-heap with a custom comparator.
func NewHeap[T any](less func(a, b T) bool) *Heap[T] {
    return &Heap[T]{less: less}
}

// Push inserts an element.
func (h *Heap[T]) Push(v T) {
    h.data = append(h.data, v)
    h.siftUp(len(h.data) - 1)
}

// Pop removes and returns the minimum element.
func (h *Heap[T]) Pop() (T, bool) {
    if len(h.data) == 0 {
        var zero T
        return zero, false
    }
    root := h.data[0]
    last := h.data[len(h.data)-1]
    h.data = h.data[:len(h.data)-1]
    if len(h.data) > 0 {
        h.data[0] = last
        h.siftDown(0)
    }
    return root, true
}

func (h *Heap[T]) siftUp(i int) {
    for i > 0 {
        parent := (i - 1) / 2
        if h.less(h.data[i], h.data[parent]) {
            h.data[i], h.data[parent] = h.data[parent], h.data[i]
            i = parent
        } else {
            break
        }
    }
}

func (h *Heap[T]) siftDown(i int) {
    n := len(h.data)
    for {
        smallest := i
        left := 2*i + 1
        right := 2*i + 2
        if left < n && h.less(h.data[left], h.data[smallest]) {
            smallest = left
        }
        if right < n && h.less(h.data[right], h.data[smallest]) {
            smallest = right
        }
        if smallest == i {
            break
        }
        h.data[i], h.data[smallest] = h.data[smallest], h.data[i]
        i = smallest
    }
}

func (h *Heap[T]) Len() int {
    return len(h.data)
}

func main() {
    // Min-heap of integers
    h := NewHeap[int](func(a, b int) bool { return a < b })
    h.Push(5)
    h.Push(3)
    h.Push(8)
    h.Push(1)
    
    fmt.Println("Min-Heap pop order:")
    for h.Len() > 0 {
        v, _ := h.Pop()
        fmt.Printf("%d ", v)
    }
    fmt.Println()
    
    // Max-heap (just flip the comparator)
    maxH := NewHeap[int](func(a, b int) bool { return a > b })
    maxH.Push(5)
    maxH.Push(3)
    maxH.Push(8)
    maxH.Push(1)
    
    fmt.Println("Max-Heap pop order:")
    for maxH.Len() > 0 {
        v, _ := maxH.Pop()
        fmt.Printf("%d ", v)
    }
    fmt.Println()
}
```

## 五、注意事项

| 项目 | 说明 |
|------|------|
| 性能 | 泛型在 Go 中通过单态化（monomorphization）实现，每种具体类型生成一份代码，无运行时开销 |
| 方法限制 | 泛型方法不能引入新的类型参数，只能使用结构体上的类型参数 |
| 接口 vs 泛型 | 接口适合运行时多态；泛型适合编译时多态和类型安全 |
| 约束设计 | 约束要尽量宽松（`any` > `comparable` > 自定义），避免过度约束 |
| 代码膨胀 | 过多泛型特化会增加二进制体积，对于热路径需关注 |
| `any` vs `interface{}` | 泛型中 `any` 是类型约束，`interface{}` 是具体类型，语义不同 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 泛型方法添加新类型参数 `func (s S[T]) Foo[U any]()` | 用包级函数 `func Foo[T, U any](s S[T])` |
| 用 `interface{}` 做泛型约束 | 使用 `any` 或自定义约束接口 |
| 忘记 `~` 导致自定义类型不满足约束 | 约束中使用近似类型 `~int` 而非 `int` |
| 泛型结构体做 map key | 泛型结构体本身不可比较，需用具体字段 |
| 在约束中写联合类型时遗漏类型 | 使用 `cmp.Ordered` 等标准库约束 |
| 过度使用泛型导致代码可读性下降 | 简单场景用接口，复杂容器用泛型 |

## 十一、代码演进

### v1：接口方案（泛型前）

```go
// Type-unsafe, requires assertions
type Stack struct {
    items []interface{}
}

func (s *Stack) Push(v interface{}) {
    s.items = append(s.items, v)
}

func (s *Stack) Pop() (interface{}, bool) {
    if len(s.items) == 0 {
        return nil, false
    }
    top := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return top, true
}

// Usage: need type assertion everywhere
// val, ok := s.Pop().(string) // risky!
```

### v2：基础泛型（类型安全）

```go
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    top := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return top, true
}

// Usage: type-safe, no assertions needed
```

### v3：约束 + 标准库集成（生产级）

```go
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(v T)          { s.items = append(s.items, v) }
func (s *Stack[T]) Pop() (T, bool)    { /* same */ }
func (s *Stack[T]) Len() int          { return len(s.items) }
func (s *Stack[T]) IsEmpty() bool     { return len(s.items) == 0 }
func (s *Stack[T]) ToSlice() []T      { return append([]T{}, s.items...) }

// Integration with standard library constraints
func (s *Stack[T]) Contains(eq func(a, b T) bool, target T) bool {
    for _, item := range s.items {
        if eq(item, target) {
            return true
        }
    }
    return false
}
```

## 十二、总结

泛型在 Go 中的定位很明确：**减少重复代码，提供编译时类型安全**。它不会取代接口，而是与接口互补。实战中，泛型最适合数据结构（Stack/Set/Map）、工具函数（Sort/Filter/Map）和类型安全的管道模式。掌握约束的设计（联合、近似、接口方法）是泛型进阶的关键。
