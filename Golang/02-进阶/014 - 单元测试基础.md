# 044 - 单元测试基础

> **难度：** ⭐⭐⭐（中级）
>
> **前提：** 已掌握 Go 基础语法，能编写简单的 Go 程序。

---

## 一、概念讲解

测试是工程质量的生命线。Go 语言把测试作为一等公民——`go test` 命令、`testing` 包、命名约定构成了完整的测试体系。与 Python 的 pytest 或 Java 的 JUnit 不同，Go 的测试不需要第三方框架就能用。

### 1.1 testing 包与 TestXxx

Go 测试的核心规则：

1. 测试文件以 `_test.go` 结尾
2. 测试函数以 `Test` 开头，参数为 `*testing.T`
3. `go test` 自动发现并运行

### 1.2 t.Run 子测试

`t.Run(name, func(t *testing.T))` 创建子测试，支持：
- 逻辑分组
- 独立执行/过滤
- 并行控制

### 1.3 表驱动测试

Table-Driven Testing 是 Go 社区最推崇的测试模式——用结构体切片定义测试用例，循环执行。

### 1.4 TestMain

`TestMain(m *testing.M)` 是测试入口，适合做全局 setup/teardown。

### 1.5 测试覆盖率

`go test -cover` 统计代码覆盖率，`-coverprofile` 导出详细报告。

## 三、完整 Go 代码

### 3.1 被测代码 math.go

```go
package mathutil

import "errors"

// Add returns the sum of two integers.
func Add(a, b int) int {
    return a + b
}

// Divide returns the quotient of two floats.
// Returns error if divisor is zero.
func Divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// IsPalindrome checks if a string reads the same forwards and backwards.
func IsPalindrome(s string) bool {
    runes := []rune(s)
    n := len(runes)
    for i := 0; i < n/2; i++ {
        if runes[i] != runes[n-1-i] {
            return false
        }
    }
    return true
}

// Fibonacci returns the nth Fibonacci number.
func Fibonacci(n int) int {
    if n <= 1 {
        return n
    }
    a, b := 0, 1
    for i := 2; i <= n; i++ {
        a, b = b, a+b
    }
    return b
}

// Max returns the larger of two integers.
func Max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

### 3.2 基础测试 math_test.go

```go
package mathutil

import "testing"

// Basic test function
func TestAdd(t *testing.T) {
    result := Add(2, 3)
    if result != 5 {
        t.Errorf("Add(2, 3) = %d, want 5", result)
    }
}

// Test with multiple cases using t.Run
func TestDivide(t *testing.T) {
    t.Run("normal division", func(t *testing.T) {
        result, err := Divide(10.0, 2.0)
        if err != nil {
            t.Fatalf("unexpected error: %v", err)
        }
        if result != 5.0 {
            t.Errorf("Divide(10, 2) = %f, want 5.0", result)
        }
    })

    t.Run("division by zero", func(t *testing.T) {
        _, err := Divide(10.0, 0)
        if err == nil {
            t.Fatal("expected error for division by zero")
        }
    })

    t.Run("negative division", func(t *testing.T) {
        result, err := Divide(-10.0, 2.0)
        if err != nil {
            t.Fatalf("unexpected error: %v", err)
        }
        if result != -5.0 {
            t.Errorf("Divide(-10, 2) = %f, want -5.0", result)
        }
    })
}
```

### 3.3 表驱动测试

```go
package mathutil

import "testing"

// Table-driven test for IsPalindrome
func TestIsPalindrome_TableDriven(t *testing.T) {
    tests := []struct {
        name  string
        input string
        want  bool
    }{
        {"empty string", "", true},
        {"single char", "a", true},
        {"even palindrome", "abba", true},
        {"odd palindrome", "abcba", true},
        {"not palindrome", "hello", false},
        {"chinese palindrome", "上海自来水来自海上", true},
        {"mixed case", "Aba", false}, // case-sensitive
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := IsPalindrome(tt.input)
            if got != tt.want {
                t.Errorf("IsPalindrome(%q) = %v, want %v", tt.input, got, tt.want)
            }
        })
    }
}

// Table-driven test for Fibonacci
func TestFibonacci_TableDriven(t *testing.T) {
    tests := []struct {
        n    int
        want int
    }{
        {0, 0},
        {1, 1},
        {2, 1},
        {3, 2},
        {5, 5},
        {10, 55},
    }

    for _, tt := range tests {
        got := Fibonacci(tt.n)
        if got != tt.want {
            t.Errorf("Fibonacci(%d) = %d, want %d", tt.n, got, tt.want)
        }
    }
}
```

### 3.4 TestMain 示例

```go
package mathutil

import (
    "fmt"
    "os"
    "testing"
)

// TestMain runs before all tests. Use for global setup/teardown.
func TestMain(m *testing.M) {
    // Setup
    fmt.Println("=== Test Suite Setup ===")

    // Run all tests
    code := m.Run()

    // Teardown
    fmt.Println("=== Test Suite Teardown ===")

    os.Exit(code)
}
```

### 3.5 t.Helper / t.Skip / t.Parallel

```go
package mathutil

import (
    "os"
    "testing"
)

// assertEqual is a test helper. t.Helper() makes error
// report point to the caller's line, not this function.
func assertEqual(t *testing.T, got, want int) {
    t.Helper() // Important: marks this as a helper function
    if got != want {
        t.Errorf("got %d, want %d", got, want)
    }
}

func TestMax_WithHelper(t *testing.T) {
    assertEqual(t, Max(3, 5), 5)
    assertEqual(t, Max(10, 2), 10)
    assertEqual(t, Max(-1, -5), -1)
}

// Skip test in short mode (go test -short)
func TestFibonacci_Large_SkipShort(t *testing.T) {
    if testing.Short() {
        t.Skip("skipping large Fibonacci test in short mode")
    }
    // This test takes longer
    result := Fibonacci(50)
    if result <= 0 {
        t.Error("Fibonacci(50) should be positive")
    }
}

// Parallel test execution
func TestAdd_Parallel(t *testing.T) {
    cases := []struct {
        a, b, want int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {-1, 1, 0},
        {100, 200, 300},
    }

    for _, tt := range cases {
        tt := tt // capture range variable
        t.Run(fmt.Sprintf("%d+%d", tt.a, tt.b), func(t *testing.T) {
            t.Parallel()
            got := Add(tt.a, tt.b)
            if got != tt.want {
                t.Errorf("Add(%d, %d) = %d, want %d", tt.a, tt.b, got, tt.want)
            }
        })
    }
}

// Environment-dependent test
func TestRequiresCI(t *testing.T) {
    if os.Getenv("CI") == "" {
        t.Skip("skipping: set CI=1 to run this test")
    }
    // CI-specific tests...
}
```

### 3.6 测试文件组织

```
mathutil/
├── math.go              # 被测代码
```

```go

// Export for testing only. This file's _test.go suffix
// means it's only compiled during tests.
```

## 五、注意事项

| 项目 | 说明 |
|------|------|
| 文件命名 | 必须以 `_test.go` 结尾，否则 `go test` 不识别 |
| 函数签名 | 必须是 `func TestXxx(t *testing.T)`，参数类型不对会跳过 |
| 包名选择 | 同包可测私有函数；`_test` 后缀包（外部测试）只能测公开 API |
| 并发安全 | `t.Parallel()` 的子测试之间不要共享可变状态 |
| 测试缓存 | `go test` 会缓存通过的测试结果，用 `-count=1` 强制重跑 |
| testdata | `testdata/` 目录被 `go test` 特殊处理，用于存放测试fixture |
| 测试顺序 | 测试执行顺序不确定，不要依赖测试间的顺序 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 用 `t.Log` 判断测试通过 | 用 `t.Error`/`t.Fatal` 报告失败 |
| 在测试间共享全局状态 | 每个测试独立初始化，使用 `t.Cleanup` 清理 |
| `t.Run` 中忘记 `tt := tt` 捕获变量 | Go 1.22+ 已修复此问题，旧版本需手动捕获 |
| 用 `os.Exit()` 在测试中退出 | 用 `t.Fatal`/`t.Skip` 代替 |
| 测试函数名用小写开头 `testAdd` | 必须大写开头 `TestAdd`，否则不执行 |
| 不检查错误返回值 `_, _ = Divide(1, 0)` | 始终检查 error，这是测试的意义所在 |

## 七、练习题

### 🟢 简单

为以下函数编写表驱动测试：

```go
func Abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}
```

### 🟡 中等

实现 `StringReverse(s string) string` 并编写测试，要求覆盖：空串、单字符、奇数长度、偶数长度、含中文、含 emoji。

### 🔴 困难

编写 `TestMain` 实现以下功能：
1. 启动时创建临时目录
2. 所有测试共享该目录
3. 测试结束后清理（即使测试失败也要清理）
4. 使用 `t.Cleanup` 在个别测试中注册清理函数

## 九、举一反三

| 测试场景 | Go 方案 | 其他语言对比 |
|---------|--------|------------|
| 基本断言 | `if got != want { t.Errorf() }` | pytest: `assert got == want` |
| 子测试分组 | `t.Run(name, fn)` | JUnit 5: `@Nested` |
| 参数化测试 | 表驱动测试 | JUnit 5: `@ParameterizedTest` |
| setup/teardown | `TestMain` + `t.Cleanup` | pytest: `fixture` |
| 跳过测试 | `t.Skip()` / `testing.Short()` | pytest: `@pytest.mark.skip` |
| 并行测试 | `t.Parallel()` | pytest: `pytest-xdist` |

## 十一、代码演进

### v1：最简单的测试

```go
func TestAdd(t *testing.T) {
    if Add(1, 2) != 3 {
        t.Error("1 + 2 should be 3")
    }
}
```

### v2：表驱动测试

```go
func TestAdd(t *testing.T) {
    tests := []struct{ a, b, want int }{
        {1, 2, 3},
        {0, 0, 0},
        {-1, 1, 0},
    }
    for _, tt := range tests {
        got := Add(tt.a, tt.b)
        if got != tt.want {
            t.Errorf("Add(%d, %d) = %d, want %d", tt.a, tt.b, got, tt.want)
        }
    }
}
```

### v3：子测试 + 并行 + Helper

```go
func TestAdd(t *testing.T) {
    tests := []struct {
        name      string
        a, b, want int
    }{
        {"positive", 1, 2, 3},
        {"zeros", 0, 0, 0},
        {"negative", -1, 1, 0},
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            t.Parallel()
            assertEqual(t, Add(tt.a, tt.b), tt.want)
        })
    }
}

func assertEqual(t *testing.T, got, want int) {
    t.Helper()
    if got != want {
        t.Errorf("got %d, want %d", got, want)
    }
}
```

## 十二、总结

Go 的测试哲学是**简单、统一、内置**。不需要选框架，不需要学 DSL，`testing` 包 + `go test` 就够了。表驱动测试是 Go 社区的事实标准，掌握它是写好 Go 测试的第一步。`t.Helper()`、`t.Parallel()`、`t.Cleanup()` 等辅助函数让测试代码更清晰、更高效。下一篇文章将深入基准测试和 Mock，让测试覆盖更多场景。
