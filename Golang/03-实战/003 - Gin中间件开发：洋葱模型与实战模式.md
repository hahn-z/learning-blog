## 概念讲解

中间件（Middleware）是 Web 框架中的**洋葱模型**：请求从外层进入，逐层穿过中间件到达处理函数，响应再逐层返回。

**Gin 中间件本质：** 一个返回 `gin.HandlerFunc` 的函数，通过 `c.Next()` 将控制权交给下一个中间件或最终 handler。

**执行顺序：**
```
请求 → Middleware1(前) → Middleware2(前) → Handler → Middleware2(后) → Middleware1(后) → 响应
```

**中间件分类：**
- **全局中间件**：所有路由生效（如日志、Recovery）
- **路由组中间件**：特定路由组生效（如认证）
- **单路由中间件**：仅某个路由生效

**核心 API：**
- `c.Next()` — 调用下一个 handler
- `c.Abort()` — 终止链式调用
- `c.Set()` / `c.Get()` — 在中间件间传递数据

## 完整 Go 代码

```go
package main

import (
	"log"
	"net/http"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
)

func main() {
	r := gin.New() // No default middleware

	// ============================================
	// 1. Global middleware
	// ============================================
	r.Use(RequestID())
	r.Use(gin.Logger())
	r.Use(gin.Recovery())
	r.Use(CORS())

	// ============================================
	// 2. Rate limiter middleware (closure pattern)
	// ============================================
	rateLimit := RateLimit(100, time.Minute) // 100 req/min

	// ============================================
	// 3. Public routes
	// ============================================
	r.GET("/ping", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"message":    "pong",
			"request_id": c.GetString("request_id"),
		})
	})

	// ============================================
	// 4. Protected routes with middleware
	// ============================================
	api := r.Group("/api")
	api.Use(rateLimit)
	{
		// Public API
		api.POST("/login", func(c *gin.Context) {
			c.JSON(http.StatusOK, gin.H{"token": "fake-jwt-token"})
		})

		// Protected API - requires auth
		protected := api.Group("")
		protected.Use(AuthRequired())
		{
			protected.GET("/profile", func(c *gin.Context) {
				user := c.GetString("user")
				c.JSON(http.StatusOK, gin.H{"user": user})
			})

			protected.GET("/admin", AdminOnly(), func(c *gin.Context) {
				c.JSON(http.StatusOK, gin.H{"message": "welcome admin"})
			})
		}
	}

	r.Run(":8080")
}

// ============================================
// Middleware implementations
// ============================================

// RequestID adds a unique request ID to each request.
func RequestID() gin.HandlerFunc {
	var counter uint64
	var mu sync.Mutex
	return func(c *gin.Context) {
		mu.Lock()
		counter++
		id := counter
		mu.Unlock()
		c.Set("request_id", fmt.Sprintf("req-%d", id))
		c.Header("X-Request-ID", fmt.Sprintf("req-%d", id))
		c.Next()
	}
}

// CORS adds Cross-Origin Resource Sharing headers.
func CORS() gin.HandlerFunc {
	return func(c *gin.Context) {
		c.Header("Access-Control-Allow-Origin", "*")
		c.Header("Access-Control-Allow-Methods", "GET,POST,PUT,DELETE,OPTIONS")
		c.Header("Access-Control-Allow-Headers", "Content-Type,Authorization")

		if c.Request.Method == http.MethodOptions {
			c.AbortWithStatus(http.StatusNoContent)
			return
		}
		c.Next()
	}
}

// RateLimit creates a rate limiting middleware using a token bucket approach.
// maxRequests: maximum requests allowed in the time window.
// window: the time window duration.
func RateLimit(maxRequests int, window time.Duration) gin.HandlerFunc {
	type visitor struct {
		count   int
		expires time.Time
	}
	var (
		mu       sync.Mutex
		visitors = make(map[string]*visitor)
	)

	// Cleanup expired entries periodically
	go func() {
		for {
			time.Sleep(window)
			mu.Lock()
			for ip, v := range visitors {
				if time.Now().After(v.expires) {
					delete(visitors, ip)
				}
			}
			mu.Unlock()
		}
	}()

	return func(c *gin.Context) {
		ip := c.ClientIP()
		mu.Lock()
		v, exists := visitors[ip]
		if !exists || time.Now().After(v.expires) {
			visitors[ip] = &visitor{
				count:   1,
				expires: time.Now().Add(window),
			}
			mu.Unlock()
			c.Next()
			return
		}
		v.count++
		if v.count > maxRequests {
			mu.Unlock()
			c.AbortWithStatusJSON(http.StatusTooManyRequests, gin.H{
				"error": "rate limit exceeded",
			})
			return
		}
		mu.Unlock()
		c.Next()
	}
}

// AuthRequired checks for a valid Authorization header.
func AuthRequired() gin.HandlerFunc {
	return func(c *gin.Context) {
		token := c.GetHeader("Authorization")
		if token == "" {
			c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
				"error": "missing authorization header",
			})
			return
		}
		if token != "Bearer fake-jwt-token" {
			c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
				"error": "invalid token",
			})
			return
		}
		c.Set("user", "demo_user")
		c.Next()
	}
}

// AdminOnly checks if the authenticated user is an admin.
func AdminOnly() gin.HandlerFunc {
	return func(c *gin.Context) {
		user := c.GetString("user")
		if user != "admin" {
			c.AbortWithStatusJSON(http.StatusForbidden, gin.H{
				"error": "admin access required",
			})
			return
		}
		c.Next()
	}
}
```

> ⚠️ 注意：上面代码需要 `import "fmt"` 才能编译。实际项目中请确保 import 完整。

## 注意事项

| 项目 | 说明 |
|------|------|
| `gin.New()` vs `gin.Default()` | `New()` 不含任何中间件，`Default()` 自动加 Logger + Recovery |
| `c.Next()` | 阻塞等待后续 handler 执行完，后面的代码是"后置逻辑" |
| `c.Abort()` | 阻止后续 handler 执行，但当前函数会继续执行完毕 |
| goroutine 安全 | `gin.Context` 不能直接传入 goroutine，用 `c.Copy()` |
| 中间件顺序 | `r.Use()` 的调用顺序即为执行顺序 |

## 避坑指南

- ❌ 在中间件中 `c.Next()` 之前就返回响应
- ✅ 先 `c.Next()` 或 `c.Abort()`，确保链式逻辑完整

- ❌ 用全局变量存请求相关数据
- ✅ 使用 `c.Set()` / `c.Get()` 在 Context 中传递

- ❌ 限流中间件用 sync.Map 但不清理过期数据
- ✅ 定期清理过期条目，或使用 `golang.org/x/time/rate`

- ❌ 在 goroutine 中直接使用 `c *gin.Context`
- ✅ 使用 `cCopy := c.Copy()` 传递副本
