# 068: 微服务架构设计

> **难度：⭐⭐⭐⭐⭐ 专家级** | 从单体到微服务的架构演进，服务拆分、通信、治理的全链路设计。
>
> **前置知识：** gRPC 基础（066-067）、数据库设计、基本架构概念
>
> **学习目标：** 掌握微服务拆分原则、同步/异步通信、API 网关、熔断器和分布式追踪的完整设计。

## 概念讲解

微服务架构将一个大应用拆分为多个独立部署的小服务。核心原则：

- **服务拆分（DDD 限界上下文）：** 按业务域划分服务边界。每个服务有自己的聚合根（Aggregate Root），对外暴露 API，内部数据隔离。
- **同步通信（gRPC/HTTP）：** 服务间直接调用，适合需要即时响应的场景。缺点是强耦合、级联故障。
- **异步通信（消息队列）：** 通过事件解耦，适合不需要即时响应的场景。优点是松耦合、削峰填谷。
- **API 网关：** 统一入口，负责路由、认证、限流。客户端只跟网关通信。
- **可观测性三件套：** 日志（Logging）、指标（Metrics）、追踪（Tracing）。
- **熔断器（Circuit Breaker）：** 当下游服务故障时快速失败，避免级联雪崩。

### 单体 vs 微服务判断标准

| 因素 | 适合单体 | 适合微服务 |
|------|---------|-----------|
| 团队规模 | 1-5 人 | 10+ 人 |
| 业务复杂度 | 低 | 高（多业务域） |
| 部署频率 | 周/月级 | 天/小时级 |
| 扩展需求 | 统一扩展 | 差异化扩展 |
| 技术栈 | 统一 | 可混合 |

package order

import (
    "context"
    "fmt"

    pb "github.com/example/proto/product"
    "google.golang.org/grpc"
)

// OrderService depends on Product service client
type OrderService struct {
    productClient pb.ProductServiceClient
}

func NewOrderService(productConn *grpc.ClientConn) *OrderService {
    return &OrderService{
        productClient: pb.NewProductServiceClient(productConn),
    }
}

func (s *OrderService) CreateOrder(ctx context.Context, req *CreateOrderReq) error {
    // Synchronous call to Product service for stock check
    product, err := s.productClient.GetProduct(ctx, &pb.GetProductReq{
        Id: req.ProductID,
    })
    if err != nil {
        return fmt.Errorf("get product failed: %w", err)
    }
    if product.Stock < req.Quantity {
        return fmt.Errorf("insufficient stock: have %d, want %d", product.Stock, req.Quantity)
    }
    // Create order in local DB
    // ...
    return nil
}

### v2：异步通信 + 消息队列

```go
// messaging/rabbitmq.go - Event-driven communication
package messaging

import (
    "context"
    "encoding/json"
    "log"

    "github.com/rabbitmq/amqp091-go"
)

// EventBus wraps RabbitMQ connection
type EventBus struct {
    conn *amqp091.Connection
    ch   *amqp091.Channel
}

func NewEventBus(url string) (*EventBus, error) {
    conn, err := amqp091.Dial(url)
    if err != nil {
        return nil, err
    }
    ch, err := conn.Channel()
    if err != nil {
        return nil, err
    }
    // Declare exchange for pub/sub pattern
    ch.ExchangeDeclare("events", "topic", true, false, false, false, nil)
    return &EventBus{conn: conn, ch: ch}, nil
}

// Publish sends an event to the message broker
func (b *EventBus) Publish(ctx context.Context, routingKey string, event interface{}) error {
    body, _ := json.Marshal(event)
    return b.ch.PublishWithContext(ctx, "events", routingKey, false, false,
        amqp091.Publishing{
            ContentType: "application/json",
            Body:       body,
        })
}

// Subscribe registers a handler for a routing key
func (b *EventBus) Subscribe(queue, routingKey string, handler func([]byte) error) error {
    q, _ := b.ch.QueueDeclare(queue, true, false, false, false, nil)
    b.ch.QueueBind(q.Name, routingKey, "events", false, nil)

    msgs, _ := b.ch.Consume(q.Name, "", false, false, false, false, nil)
    go func() {
        for msg := range msgs {
            if err := handler(msg.Body); err != nil {
                msg.Nack(false, true) // Requeue on error
            } else {
                msg.Ack(false) // Acknowledge on success
            }
        }
    }()
    return nil
}
```

### v3：API 网关 + 熔断器 + 分布式追踪

```go
// gateway/main.go - API Gateway with reverse proxy
package main

import (
    "log"
    "net/http"
    "net/http/httputil"
    "net/url"
    "strings"
)

type Gateway struct {
    routes map[string]*httputil.ReverseProxy
}

func NewGateway() *Gateway {
    gw := &Gateway{routes: make(map[string]*httputil.ReverseProxy)}

    userURL, _ := url.Parse("http://user-service:8081")
    orderURL, _ := url.Parse("http://order-service:8082")

    gw.routes["/api/users"] = httputil.NewSingleHostReverseProxy(userURL)
    gw.routes["/api/orders"] = httputil.NewSingleHostReverseProxy(orderURL)
    return gw
}

func (gw *Gateway) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    for prefix, proxy := range gw.routes {
        if strings.HasPrefix(r.URL.Path, prefix) {
            proxy.ServeHTTP(w, r)
            return
        }
    }
    http.NotFound(w, r)
}

func main() {
    log.Fatal(http.ListenAndServe(":8080", NewGateway()))
}
```

**熔断器：**

```go
// circuitbreaker/breaker.go - Circuit breaker using gobreaker
package circuitbreaker

import (
    "time"
    "github.com/sony/gobreaker"
)

func NewBreaker(name string) *gobreaker.CircuitBreaker {
    return gobreaker.NewCircuitBreaker(gobreaker.Settings{
        Name:        name,
        MaxRequests: 3,                // Requests allowed in half-open state
        Interval:    10 * time.Second, // Statistical period
        Timeout:     30 * time.Second, // Open → Half-open wait time
        ReadyToTrip: func(counts gobreaker.Counts) bool {
            return counts.ConsecutiveFailures > 5 // Trip after 5 consecutive failures
        },
    })
}

// Execute wraps a function call with circuit breaker protection
func callWithBreaker(breaker *gobreaker.CircuitBreaker, fn func() (interface{}, error)) (interface{}, error) {
    return breaker.Execute(func() (interface{}, error) {
        return fn()
    })
}
```

**分布式追踪（OpenTelemetry）：**

```go
// tracing/setup.go - Initialize OpenTelemetry tracer
package tracing

import (
    "context"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/exporters/otlp/otlptrace/otlptracegrpc"
    "go.opentelemetry.io/otel/sdk/resource"
    sdktrace "go.opentelemetry.io/otel/sdk/trace"
    semconv "go.opentelemetry.io/otel/semconv/v1.24.0"
)

func InitTracer(serviceName, collectorAddr string) (func(context.Context) error, error) {
    exporter, err := otlptracegrpc.New(context.Background(),
        otlptracegrpc.WithEndpoint(collectorAddr),
        otlptracegrpc.WithInsecure(),
    )
    if err != nil {
        return nil, err
    }

    provider := sdktrace.NewTracerProvider(
        sdktrace.WithBatcher(exporter),
        sdktrace.WithResource(resource.NewWithAttributes(
            semconv.SchemaURL,
            semconv.ServiceName(serviceName),
        )),
    )
    otel.SetTracerProvider(provider)
    return provider.Shutdown, nil
}
```

## 注意事项

| 项目 | 说明 | 严重度 |
|------|------|--------|
| 不要过早拆分 | 先写好单体，业务稳定后再拆。拆分成本很高 | 🔴 严重 |
| 分布式事务 | 尽量用 Saga 模式（最终一致性）替代强一致性 | ⚠️ 重要 |
| 网络不可靠 | 服务间调用必须考虑超时、重试、熔断 | 🔴 严重 |
| 可观测性 | 日志+指标+追踪三件套缺一不可，否则排障是噩梦 | ⚠️ 重要 |
| 数据隔离 | 每个服务独立数据库，禁止跨服务直接查库 | 🔴 严重 |
| 幂等性 | 异步消息可能重复投递，消费者必须幂等 | ⚠️ 重要 |

## 避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 一上来就微服务 | 先模块化单体，有明确痛点再拆 |
| 服务间共享数据库 | 每个服务独立数据库，通过 API 通信 |
| 同步调用链路过长（>3层） | 引入消息队列异步解耦 |
| 不设超时直接调用下游 | 每次调用都带 context.WithTimeout |
| 熔断器阈值设太低 | 根据实际 QPS 和延迟设置，5-10 次连续失败再熔断 |
| 消息消费不幂等 | 消息带唯一 ID，消费前检查是否已处理 |

**🟡 进阶：RabbitMQ 异步订单流程**
