# 008-biz业务层开发

> 难度：⭐⭐⭐⭐（高级）
> 前置知识：DDD领域驱动设计、Go接口、Kratos项目结构

## 一、概念讲解

### biz 层是什么？

biz 层是 Kratos 项目中**纯业务逻辑**所在的层，对应 DDD 中的 Domain Layer。它的核心原则：

1. **不依赖任何外部框架**（不直接操作数据库、不处理 HTTP/gRPC）
2. **通过接口定义依赖**（Repository 接口由 biz 层定义，data 层实现）
3. **封装业务规则**（所有业务判断在 biz 层完成）

### Kratos 分层架构中 biz 的位置

```
HTTP/gRPC  →  Service（薄层，协议适配）  →  biz（业务逻辑）  →  data（数据访问）
                                                    ↑
                                            Repository 接口（biz定义）
                                            Repository 实现（data层提供）
```

**关键设计**：biz 层定义 Repository 接口，data 层实现它。这是**依赖倒置原则（DIP）**的体现。

## 三、完整代码示例

### 3.1 领域模型

```go
// internal/biz/order.go
package biz

import (
    "context"
    "time"

    "github.com/go-kratos/kratos/v2/log"
)

// OrderStatus defines order status type
type OrderStatus string

const (
    StatusPending   OrderStatus = "pending"
    StatusPaid      OrderStatus = "paid"
    StatusShipped   OrderStatus = "shipped"
    StatusCompleted OrderStatus = "completed"
    StatusCancelled OrderStatus = "cancelled"
)

// Order is the domain model (Business Object)
type Order struct {
    ID              int64
    UserID          int64
    Status          OrderStatus
    TotalAmount     float64
    Items           []*OrderItem
    ShippingAddress *Address
    Remark          string
    PaidAt          *time.Time
    ShippedAt       *time.Time
    CreatedAt       time.Time
    UpdatedAt       time.Time
}

// OrderItem is a domain sub-entity
type OrderItem struct {
    ID          int64
    ProductID   int64
    ProductName string
    Quantity    int32
    UnitPrice   float64
}

// Address is a value object
type Address struct {
    Name     string
    Phone    string
    FullAddr string
}

// CanPay checks if the order can be paid (business rule)
func (o *Order) CanPay() bool {
    return o.Status == StatusPending
}

// CanCancel checks if the order can be cancelled
func (o *Order) CanCancel() bool {
    return o.Status == StatusPending || o.Status == StatusPaid
}

// CanShip checks if the order can be shipped
func (o *Order) CanShip() bool {
    return o.Status == StatusPaid
}

// CalculateTotal computes the total amount from items
func (o *Order) CalculateTotal() float64 {
    total := 0.0
    for _, item := range o.Items {
        total += float64(item.Quantity) * item.UnitPrice
    }
    return total
}
```

### 3.2 Repository 接口定义

```go
// internal/biz/repository.go
package biz

import "context"

// OrderRepo defines the repository interface for order operations.
// This interface is OWNED by biz layer and IMPLEMENTED by data layer.
type OrderRepo interface {
    // Save persists an order (create or update)
    Save(ctx context.Context, order *Order) (*Order, error)

    // FindByID retrieves an order by ID
    FindByID(ctx context.Context, id int64) (*Order, error)

    // FindByUser retrieves orders by user ID with pagination
    FindByUser(ctx context.Context, userID int64, page, size int) ([]*Order, int64, error)

    // UpdateStatus updates order status
    UpdateStatus(ctx context.Context, id int64, status OrderStatus) error
}

// Transaction defines the transaction interface for biz layer
type Transaction interface {
    // ExecTx executes a function within a database transaction
    ExecTx(ctx context.Context, fn func(ctx context.Context) error) error
}
```

### 3.3 Usecase 实现

```go
// internal/biz/order_usecase.go
package biz

import (
    "context"
    "fmt"

    "github.com/go-kratos/kratos/v2/errors"
    "github.com/go-kratos/kratos/v2/log"
)

// OrderUsecase implements order business logic
type OrderUsecase struct {
    repo OrderRepo
    tx   Transaction
    log  *log.Helper
}

// NewOrderUsecase creates a new OrderUsecase
func NewOrderUsecase(repo OrderRepo, tx Transaction, logger log.Logger) *OrderUsecase {
    return &OrderUsecase{repo: repo, tx: tx, log: log.NewHelper(logger)}
}

// CreateOrder handles order creation business logic
func (uc *OrderUsecase) CreateOrder(ctx context.Context, order *Order) (*Order, error) {
    // Business rule: validate order items
    if len(order.Items) == 0 {
        return nil, errors.BadRequest("ORDER_EMPTY", "order must have at least one item")
    }

    // Business rule: calculate total amount
    order.TotalAmount = order.CalculateTotal()

    // Business rule: validate total amount > 0
    if order.TotalAmount <= 0 {
        return nil, errors.BadRequest("INVALID_AMOUNT", "order total must be positive")
    }

    // Set initial status
    order.Status = StatusPending

    // Persist within transaction
    var saved *Order
    err := uc.tx.ExecTx(ctx, func(ctx context.Context) error {
        var err error
        saved, err = uc.repo.Save(ctx, order)
        return err
    })
    if err != nil {
        return nil, fmt.Errorf("create order failed: %w", err)
    }

    uc.log.WithContext(ctx).Infof("Order created: id=%d, user=%d, amount=%.2f",
        saved.ID, saved.UserID, saved.TotalAmount)
    return saved, nil
}

// GetOrder retrieves an order with business validation
func (uc *OrderUsecase) GetOrder(ctx context.Context, id int64) (*Order, error) {
    order, err := uc.repo.FindByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("get order failed: %w", err)
    }
    if order == nil {
        return nil, errors.NotFound("ORDER_NOT_FOUND", fmt.Sprintf("order %d not found", id))
    }
    return order, nil
}

// PayOrder handles payment business logic with status transition
func (uc *OrderUsecase) PayOrder(ctx context.Context, id int64) (*Order, error) {
    order, err := uc.GetOrder(ctx, id)
    if err != nil {
        return nil, err
    }

    // Business rule: only pending orders can be paid
    if !order.CanPay() {
        return nil, errors.Conflict("ORDER_STATUS_INVALID",
            fmt.Sprintf("order %d cannot be paid, current status: %s", id, order.Status))
    }

    // Transition status
    err = uc.repo.UpdateStatus(ctx, id, StatusPaid)
    if err != nil {
        return nil, fmt.Errorf("pay order failed: %w", err)
    }

    order.Status = StatusPaid
    uc.log.WithContext(ctx).Infof("Order paid: id=%d", id)
    return order, nil
}

// CancelOrder handles cancellation with business rules
func (uc *OrderUsecase) CancelOrder(ctx context.Context, id int64, reason string) (*Order, error) {
    order, err := uc.GetOrder(ctx, id)
    if err != nil {
        return nil, err
    }

    // Business rule: only pending/paid orders can be cancelled
    if !order.CanCancel() {
        return nil, errors.Conflict("ORDER_CANNOT_CANCEL",
            fmt.Sprintf("order %d in status %s cannot be cancelled", id, order.Status))
    }

    err = uc.repo.UpdateStatus(ctx, id, StatusCancelled)
    if err != nil {
        return nil, fmt.Errorf("cancel order failed: %w", err)
    }

    // Business rule: if order was paid, trigger refund process
    if order.Status == StatusPaid {
        uc.log.WithContext(ctx).Infof("Refund triggered for cancelled order: id=%d, reason=%s", id, reason)
        // Could publish a domain event here
    }

    order.Status = StatusCancelled
    return order, nil
}

// ListOrders lists orders for a user
func (uc *OrderUsecase) ListOrders(ctx context.Context, userID int64, page, size int) ([]*Order, int64, error) {
    if page < 1 {
        page = 1
    }
    if size < 1 || size > 100 {
        size = 20
    }
    return uc.repo.FindByUser(ctx, userID, page, size)
}
```

// internal/biz/order_usecase_test.go
package biz

import (
    "context"
    "testing"
    "time"

    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/mock"
)

// MockOrderRepo is a mock implementation of OrderRepo
type MockOrderRepo struct {
    mock.Mock
}

func (m *MockOrderRepo) Save(ctx context.Context, order *Order) (*Order, error) {
    args := m.Called(ctx, order)
    if args.Get(0) == nil {
        return nil, args.Error(1)
    }
    return args.Get(0).(*Order), args.Error(1)
}

func (m *MockOrderRepo) FindByID(ctx context.Context, id int64) (*Order, error) {
    args := m.Called(ctx, id)
    if args.Get(0) == nil {
        return nil, args.Error(1)
    }
    return args.Get(0).(*Order), args.Error(1)
}

func (m *MockOrderRepo) FindByUser(ctx context.Context, userID int64, page, size int) ([]*Order, int64, error) {
    args := m.Called(ctx, userID, page, size)
    return args.Get(0).([]*Order), args.Get(1).(int64), args.Error(2)
}

func (m *MockOrderRepo) UpdateStatus(ctx context.Context, id int64, status OrderStatus) error {
    args := m.Called(ctx, id, status)
    return args.Error(0)
}

// MockTransaction is a mock transaction implementation
type MockTransaction struct {
    mock.Mock
}

func (m *MockTransaction) ExecTx(ctx context.Context, fn func(ctx context.Context) error) error {
    args := m.Called(ctx, fn)
    // Actually execute the function to simulate transaction
    if fn != nil && args.Error(0) == nil {
        return fn(ctx)
    }
    return args.Error(0)
}

func TestCreateOrder_Success(t *testing.T) {
    repo := new(MockOrderRepo)
    tx := new(MockTransaction)
    uc := NewOrderUsecase(repo, tx, log.DefaultLogger)

    order := &Order{
        UserID: 1,
        Items: []*OrderItem{
            {ProductID: 100, Quantity: 2, UnitPrice: 50.0},
        },
    }

    // Mock: transaction wraps the save
    tx.On("ExecTx", mock.Anything, mock.AnythingOfType("func(context.Context) error")).
        Return(nil)
    repo.On("Save", mock.Anything, mock.AnythingOfType("*biz.Order")).
        Return(&Order{ID: 1, UserID: 1, Status: StatusPending, TotalAmount: 100.0}, nil)

    result, err := uc.CreateOrder(context.Background(), order)

    assert.NoError(t, err)
    assert.Equal(t, float64(100.0), order.TotalAmount)
    assert.Equal(t, StatusPending, order.Status)
    repo.AssertExpectations(t)
}

func TestCreateOrder_EmptyItems(t *testing.T) {
    repo := new(MockOrderRepo)
    tx := new(MockTransaction)
    uc := NewOrderUsecase(repo, tx, log.DefaultLogger)

    order := &Order{UserID: 1, Items: []*OrderItem{}}

    result, err := uc.CreateOrder(context.Background(), order)

    assert.Nil(t, result)
    assert.Error(t, err)
}

func TestPayOrder_NotPending(t *testing.T) {
    repo := new(MockOrderRepo)
    tx := new(MockTransaction)
    uc := NewOrderUsecase(repo, tx, log.DefaultLogger)

    // Order already paid
    repo.On("FindByID", mock.Anything, int64(1)).
        Return(&Order{ID: 1, Status: StatusPaid}, nil)

    result, err := uc.PayOrder(context.Background(), 1)

    assert.Nil(t, result)
    assert.Error(t, err)
    assert.Contains(t, err.Error(), "cannot be paid")
}

## 五、注意事项

| 事项 | 说明 | 影响 |
|------|------|------|
| biz 层不 import data 包 | 保持依赖方向正确 | 否则循环依赖或耦合 |
| Repository 接口在 biz 定义 | 依赖倒置原则 | data 层实现 biz 层接口 |
| Usecase 方法是事务边界 | 一个 Usecase 方法 = 一个事务 | 跨表操作需要事务包装 |
| 领域模型不含 ORM 标签 | BO 纯粹，DO 在 data 层 | 分离关注点 |
| 业务错误用 kratos/errors | 统一错误格式 | Service 层可直接返回 |
| 避免 God Usecase | 一个领域一个 Usecase | 过大的 Usecase 需拆分 |

## 六、避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| biz 层直接操作 *gorm.DB | 定义 Repository 接口，data 层注入实现 |
| Repository 返回 data 层的 ORM 模型 | 返回 biz 层的领域模型（BO） |
| Service 层写业务逻辑 | Service 层只做协议转换，逻辑在 biz |
| biz 层 import Proto 生成的类型 | biz 用自己的领域模型，不依赖 Proto |
| 一个 Usecase 方法做太多事 | 单一职责，一个方法一个业务操作 |
| 事务在 data 层管理 | 事务边界在 biz 层通过 Transaction 接口控制 |
| 忽略单元测试 | biz 层逻辑最复杂，必须有单元测试 |

## 十一、代码演进

### v1: 贫血模型（逻辑在 Service 层）

```go
// Service 层直接操作 repo，业务逻辑散落
func (s *OrderService) CreateOrder(ctx context.Context, req *pb.CreateReq) (*pb.CreateResp, error) {
    order := &Order{UserID: req.UserId, Status: "pending"}
    for _, item := range req.Items {
        order.TotalAmount += float64(item.Quantity) * item.Price
    }
    return s.repo.Save(ctx, order)  // 业务逻辑泄漏到 service
}
```

**问题**：业务逻辑分散在 Service 层，不可复用、不可测试。

### v2: 充血模型（Usecase + 领域方法）

```go
// biz 层 Usecase 封装逻辑
func (uc *OrderUsecase) CreateOrder(ctx context.Context, order *Order) (*Order, error) {
    order.TotalAmount = order.CalculateTotal()  // 领域方法
    order.Status = StatusPending
    return uc.repo.Save(ctx, order)
}
```

**改进**：逻辑集中在 biz 层。**但**没有事务控制。

### v3: 生产级（事务 + 事件 + 规则引擎）

```go
func (uc *OrderUsecase) CreateOrder(ctx context.Context, order *Order) (*Order, error) {
    // Business rules
    if err := uc.validateOrder(order); err != nil {
        return nil, err
    }
    order.TotalAmount = order.CalculateTotal()
    order.Status = StatusPending

    // Transaction boundary
    var saved *Order
    err := uc.tx.ExecTx(ctx, func(ctx context.Context) error {
        var err error
        saved, err = uc.repo.Save(ctx, order)
        if err != nil {
            return err
        }
        // Domain event
        return uc.event.Publish(ctx, &OrderCreatedEvent{OrderID: saved.ID})
    })
    return saved, err
}
```

**最终形态**：完整的事务控制、领域事件、规则验证。
