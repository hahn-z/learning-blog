# HTML5语义化标签完全指南

> **难度：** 🟡 中级 | **阅读时间：** 25分钟 | **前置知识：** HTML基础、CSS布局

---

## 一、概念讲解

### 1.1 语义化的本质

HTML5语义化不仅是"用正确的标签"，更是**让机器（搜索引擎、辅助工具、爬虫）像人类一样理解页面结构**。

语义化的三个层次：

| 层次 | 说明 | 示例 |
|------|------|------|
| **标签语义** | 用有意义的HTML元素 | `<nav>` 代替 `<div class="nav">` |
| **属性语义** | 用ARIA和微数据补充 | `role="banner"`, `aria-label` |
| **结构语义** | 标签之间的嵌套关系 | `article > section > h2` |

### 1.2 完整语义标签分类

**结构标签（Sectioning）：**

| 标签 | 语义 | 大纲算法 | 注意事项 |
|------|------|----------|----------|
| `<header>` | 最近的节或根元素的头部 | 不创建新节 | 可在article内使用 |
| `<footer>` | 最近的节或根元素的尾部 | 不创建新节 | 可在article内使用 |
| `<main>` | 页面主体内容 | 不创建新节 | 每页唯一 |
| `<nav>` | 导航链接区域 | 创建新节 | 主要导航才用 |
| `<aside>` | 间接相关内容 | 创建新节 | 侧边栏、广告 |
| `<section>` | 主题性分组 | 创建新节 | 需要heading |
| `<article>` | 独立可分发内容 | 创建新节 | 博文、评论 |

**文本级语义标签：**

| 标签 | 语义 | 默认样式 |
|------|------|----------|
| `<time>` | 日期/时间 | 无特殊样式 |
| `<mark>` | 高亮/标记 | 黄色背景 |
| `<abbr>` | 缩写 | 下划线虚线 |
| `<cite>` | 作品标题 | 斜体 |
| `<details>` | 可展开详情 | 折叠控件 |
| `<summary>` | details标题 | 可点击 |
| `<figure>` | 自包含内容 | 有margin |
| `<figcaption>` | figure标题 | 居中 |
| `<address>` | 联系信息 | 斜体 |

## 四、注意事项

| 注意点 | 详细说明 | 建议 |
|--------|----------|------|
| section必须有heading | section创建新大纲节点 | 没有heading就用div |
| header/footer可嵌套 | article内也可有header/footer | 不要与页面级混淆 |
| details兼容性 | IE不支持 | 提供polyfill或降级方案 |
| time的datetime | 机器可读日期写在datetime属性中 | 标签内容可以是人类可读格式 |
| figure不限于图片 | 代码块、表格、引用都可以 | 配合figcaption使用 |

## 五、避坑指南

### ❌ nav包裹所有链接
```html
<nav>
    <a href="#">链接1</a>
    <p>段落中的<a href="#">链接2</a></p>
</nav>
```
### ✅ nav仅用于主要导航块
```html
<nav aria-label="主导航">
    <ul><li><a href="/">首页</a></li></ul>
</nav>
```

### ❌ article内没有heading
```html
<article>
    <p>一些内容...</p>
</article>
```
### ✅ article内应有标题
```html
<article>
    <h2>文章标题</h2>
    <p>一些内容...</p>
</article>
```

### ❌ 滥用h1
```html
<section><h1>标题1</h1></section>
<section><h1>标题2</h1></section>
```
### ✅ 按层级使用heading
```html
<section>
    <h2>分区标题</h2>
    <section><h3>子分区标题</h3></section>
</section>
```

## 十、代码演进

### v1 — 非语义化
```html
<div class="page">
    <div class="top-bar">
        <div class="logo">博客</div>
        <div class="links"><a href="/">首页</a></div>
    </div>
    <div class="content-area">
        <div class="post">
            <div class="post-title">文章标题</div>
            <div class="post-body">内容</div>
        </div>
    </div>
    <div class="bottom">版权信息</div>
</div>
```

### v2 — 语义化标签
```html
<header>
    <h1>博客</h1>
    <nav><a href="/">首页</a></nav>
</header>
<main>
    <article>
        <h2>文章标题</h2>
        <p>内容</p>
    </article>
</main>
<footer>版权信息</footer>
```

### v3 — 语义化 + 微数据 + ARIA
```html
<header role="banner">
    <h1>博客</h1>
    <nav aria-label="主导航"><a href="/">首页</a></nav>
</header>
<main id="content" role="main">
    <article itemscope itemtype="https://schema.org/BlogPosting">
        <h2 itemprop="headline">文章标题</h2>
        <time datetime="2026-04-28" itemprop="datePublished">2026-04-28</time>
        <div itemprop="articleBody"><p>内容</p></div>
    </article>
</main>
<footer role="contentinfo">
    <p>&copy; 2026 博客</p>
</footer>
```

> **演进要点：** div → 语义标签 → 加Schema.org微数据 + ARIA角色，实现三层层语义增强。
