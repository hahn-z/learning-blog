## 一、概念讲解

### 什么是响应式设计？

响应式设计（Responsive Web Design, RWD）是一种让网页在不同设备和屏幕尺寸下都能良好展示的设计理念。2010年由 Ethan Marcotte 提出，核心思想是：**一个页面，适应所有设备**。

响应式设计的三大技术支柱：

1. **流式网格（Fluid Grid）** — 用百分比替代固定像素
2. **弹性图片（Flexible Images）** — 图片随容器缩放
3. **媒体查询（Media Queries）** — 根据设备特征应用不同样式

### 媒体查询是什么？

媒体查询是CSS3引入的功能，允许根据设备特征（如屏幕宽度、分辨率、方向）有条件地应用CSS规则。

```css
/* Basic syntax */
@media (condition) {
  /* CSS rules */
}
```

### 断点策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 移动优先（Mobile First） | 从小屏样式开始，用 `min-width` 向上扩展 | 主流推荐 |
| 桌面优先（Desktop First） | 从大屏样式开始，用 `max-width` 向下兼容 | 遗留项目改造 |

常见断点参考（以移动优先为例）：

```css
/* Mobile: < 576px (default) */

/* Tablet: >= 576px */
@media (min-width: 576px) { }

/* Tablet landscape: >= 768px */
@media (min-width: 768px) { }

/* Desktop: >= 992px */
@media (min-width: 992px) { }

/* Large desktop: >= 1200px */
@media (min-width: 1200px) { }
```

### 逻辑操作符

媒体查询支持 `and`、`or`（逗号）、`not`、`only` 逻辑组合：

```css
/* AND: both conditions must be true */
@media (min-width: 768px) and (max-width: 991px) { }

/* OR: either condition */
@media (max-width: 575px), (min-width: 1200px) { }

/* NOT: negate a query */
@media not screen and (max-width: 575px) { }
```

### 常用媒体特性

| 特性 | 说明 | 示例 |
|------|------|------|
| `width` / `height` | 视口宽高 | `(min-width: 768px)` |
| `device-width` | 设备屏幕宽（已废弃） | — |
| `orientation` | 屏幕方向 | `(orientation: landscape)` |
| `resolution` | 设备像素密度 | `(min-resolution: 2dppx)` |
| `prefers-color-scheme` | 用户主题偏好 | `(prefers-color-scheme: dark)` |
| `prefers-reduced-motion` | 减少动画偏好 | `(prefers-reduced-motion: reduce)` |
| `hover` | 是否支持悬停 | `(hover: hover)` |

## 三、完整CSS代码 — 响应式卡片布局

```css
/* ===== Reset & Base ===== */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

:root {
  --color-primary: #4a90d9;
  --color-bg: #f5f5f5;
  --color-card: #ffffff;
  --color-text: #333333;
  --radius: 8px;
  --gap: 16px;
}

body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
  background: var(--color-bg);
  color: var(--color-text);
  line-height: 1.6;
}

/* ===== Container ===== */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: var(--gap);
}

/* ===== Card Grid — Mobile First ===== */
.card-grid {
  display: grid;
  /* Single column by default (mobile) */
  grid-template-columns: 1fr;
  gap: var(--gap);
}

/* Tablet: 2 columns */
@media (min-width: 576px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop: 3 columns */
@media (min-width: 992px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
  }
}

/* Large desktop: 4 columns */
@media (min-width: 1200px) {
  .card-grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

/* ===== Card Component ===== */
.card {
  background: var(--color-card);
  border-radius: var(--radius);
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
  overflow: hidden;
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.15);
}

.card__image {
  width: 100%;
  height: 200px;
  object-fit: cover;
}

.card__body {
  padding: var(--gap);
}

.card__title {
  font-size: 1.1rem;
  margin-bottom: 8px;
}

.card__text {
  font-size: 0.9rem;
  color: #666;
}

/* ===== Responsive Typography ===== */
html {
  /* Base: 16px on mobile */
  font-size: 100%;
}

@media (min-width: 768px) {
  html {
    /* ~18px on tablet */
    font-size: 112.5%;
  }
}

@media (min-width: 1200px) {
  html {
    /* ~19px on desktop */
    font-size: 118.75%;
  }
}

/* ===== Dark Mode Support ===== */
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary: #6ab0ff;
    --color-bg: #1a1a2e;
    --color-card: #16213e;
    --color-text: #e0e0e0;
  }
}

/* ===== Reduced Motion ===== */
@media (prefers-reduced-motion: reduce) {
  .card {
    transition: none;
  }
}

/* ===== Responsive Navigation ===== */
.nav {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-between;
  align-items: center;
  padding: 12px var(--gap);
  background: var(--color-primary);
  color: white;
}

.nav__logo {
  font-size: 1.2rem;
  font-weight: bold;
}

/* Hide hamburger on desktop */
.nav__toggle {
  display: block;
  background: none;
  border: none;
  color: white;
  font-size: 1.5rem;
  cursor: pointer;
}

.nav__menu {
  display: none;
  width: 100%;
  list-style: none;
  padding-top: 12px;
}

.nav__menu.active {
  display: block;
}

.nav__menu a {
  display: block;
  padding: 10px 0;
  color: white;
  text-decoration: none;
}

/* Show horizontal menu on tablet+ */
@media (min-width: 768px) {
  .nav__toggle {
    display: none;
  }

  .nav__menu {
    display: flex;
    width: auto;
    padding-top: 0;
    gap: 24px;
  }

  .nav__menu a {
    padding: 0;
  }
}
```

## 五、注意事项

| 项目 | 说明 | 推荐做法 |
|------|------|----------|
| Viewport meta | 必须添加，否则媒体查询在移动端无效 | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| 断点选择 | 不要针对具体设备，按内容需求设置 | 内容在哪断就在哪设断点 |
| 单位选择 | 响应式排版避免固定 px | 使用 `rem`、`em`、`vw` 等相对单位 |
| 图片性能 | 移动端加载大图浪费流量 | 使用 `srcset` 提供多分辨率 |
| max-width | 图片等元素不要超过容器 | `max-width: 100%; height: auto;` |
| 测试方式 | 不要只靠浏览器缩放 | 用 DevTools 设备模拟 + 真机测试 |

## 六、避坑指南

### ❌ 用固定设备宽度做断点
```css
/* ❌ Bad: targeting specific device */
@media (width: 375px) { /* iPhone 6/7/8 only */ }
```
```css
/* ✅ Good: use range breakpoints */
@media (min-width: 576px) and (max-width: 767px) { }
```

### ❌ 桌面优先导致移动端覆盖样式过多
```css
/* ❌ Bad: desktop first, tons of overrides */
.nav { display: flex; gap: 24px; }
@media (max-width: 767px) {
  .nav { flex-direction: column; gap: 0; }
  /* ... more overrides */
}
```
```css
/* ✅ Good: mobile first, progressive enhancement */
.nav { display: flex; flex-direction: column; }
@media (min-width: 768px) {
  .nav { flex-direction: row; gap: 24px; }
}
```

### ❌ 忘记 viewport meta 标签
```html
<!-- ❌ Bad: missing viewport -->
<head>
  <meta charset="UTF-8">
  <title>My Page</title>
</head>
```
```html
<!-- ✅ Good: always include viewport -->
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Page</title>
</head>
```

### ❌ 媒体查询中用 `!important`
```css
/* ❌ Bad: !important in media queries is a code smell */
@media (max-width: 575px) {
  .title { font-size: 24px !important; }
}
```
```css
/* ✅ Good: adjust specificity or reorder */
@media (max-width: 575px) {
  .hero .title { font-size: 24px; }
}
```

## 九、举一反三

| 技术 | 用途 | 示例场景 |
|------|------|----------|
| Container Queries | 组件级响应式，基于父容器而非视口 | 侧边栏中的卡片组件自适应 |
| `clamp()` | 响应式字体一行搞定 | `font-size: clamp(1rem, 2.5vw, 1.5rem)` |
| `aspect-ratio` | 固定宽高比 | 视频容器 `aspect-ratio: 16/9` |
| CSS Grid `auto-fill` | 自动填充列数 | `grid-template-columns: repeat(auto-fill, minmax(250px, 1fr))` |
| `min()` / `max()` | 响应式尺寸限制 | `width: min(100%, 600px)` |
| `@layer` | 媒体查询层叠管理 | 在 cascade layer 中组织响应式样式 |

## 十一、代码演进

### v1 — 最基本的媒体查询

```css
.container {
  width: 100%;
  padding: 16px;
}

@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

@media (min-width: 992px) {
  .container {
    width: 970px;
  }
}
```

**问题：** 固定宽度不够灵活，大屏与小屏之间体验断裂。

### v2 — 引入流式布局 + CSS变量

```css
:root {
  --max-width: 1200px;
  --gap: 16px;
}

.container {
  width: 100%;
  max-width: var(--max-width);
  margin: 0 auto;
  padding: var(--gap);
}

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: var(--gap);
}

img {
  max-width: 100%;
  height: auto;
}
```

**改进：** 流式布局 + `auto-fill` 自动适配列数，减少手动断点。但缺少暗色模式和无障碍支持。

```css
/* Key improvements:
   1. Mobile-first with clear breakpoints
   2. CSS custom properties for theming
   3. Dark mode via prefers-color-scheme
   4. Reduced motion support
   5. Responsive navigation pattern
   6. clamp() for fluid typography
*/
html {
  font-size: clamp(100%, 2vw + 0.5rem, 118.75%);
}

/* ... (see full code above) */
```

**改进：** `clamp()` 流式字体、暗色模式、减少动画、完整组件级响应式。生产级别。

## 十二、总结

响应式设计不是可选项，而是现代Web开发的基本功。掌握媒体查询是第一步，但更重要的是理解**移动优先**的思维方式和**内容决定断点**的策略。

关键要点：
- ✅ 始终添加 viewport meta 标签
- ✅ 优先使用移动优先（min-width）策略
- ✅ 用相对单位（rem/em/vw）替代固定 px
- ✅ 善用 CSS 自定义属性管理主题
- ✅ 关注 `prefers-*` 媒体特性，提升可访问性
- ✅ 用 `clamp()` 和 Container Queries 减少手动断点
