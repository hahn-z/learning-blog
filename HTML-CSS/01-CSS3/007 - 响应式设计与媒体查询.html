一、概念讲解

什么是响应式设计？

响应式设计（Responsive Web Design, RWD）是一种网页设计理念，使网站能够自动适配不同屏幕尺寸的设备，从手机到桌面显示器都能提供良好的浏览体验。

Ethan Marcotte 在 2010 年提出响应式设计的三大支柱：

  1. 弹性网格（Fluid Grid）：使用百分比而非固定像素布局
    1. 弹性图片（Flexible Images）：图片随容器缩放
      1. 媒体查询（Media Queries）：根据设备特征应用不同样式

        2. 核心概念：Viewport

        <!-- 必须设置 viewport meta 标签 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
        属性 说明
        width=device-width 视口宽度等于设备宽度
        initial-scale=1.0 初始缩放比例为 1
        maximum-scale=5.0 最大缩放比例（可访问性考虑）
        user-scalable=yes 允许用户缩放

        媒体查询语法详解

        /* 基础语法 */
@media screen and (max-width: 768px) {
  /* 当屏幕宽度 ≤ 768px 时生效 */
  .container {
    flex-direction: column;
  }
}

/* 多条件组合 */
@media screen and (min-width: 769px) and (max-width: 1024px) {
  /* 平板尺寸 */
}

/* 使用特性查询 */
@media (hover: hover) {
  /* 支持悬停的设备（非触摸屏） */
}

/* 横屏/竖屏 */
@media (orientation: landscape) {
  /* 横屏模式 */
}

        移动优先 vs 桌面优先

        策略 使用关键字 思路
        移动优先 ✅ min-width 先写移动端样式，逐步增强
        桌面优先 max-width 先写桌面端样式，逐步降级

        推荐移动优先，因为移动端样式更简单，渐进增强更符合开发习惯。

        五、注意事项

        项目 说明 重要性
        Viewport Meta 必须添加，否则移动端会缩放到桌面宽度 ⭐⭐⭐⭐⭐
        box-sizing 建议全局设为 border-box，避免宽度计算问题 ⭐⭐⭐⭐
        断点选择 基于内容变化选择断点，而非具体设备型号 ⭐⭐⭐⭐
        图片优化 使用 max-width: 100% 防止图片溢出 ⭐⭐⭐⭐
        触摸目标 按钮最小 44×44px，方便手指点击 ⭐⭐⭐
        测试 必须在真机上测试，模拟器不完全可靠 ⭐⭐⭐⭐

        六、避坑指南

        ❌ 用固定宽度布局

        /* 错误：不同屏幕会溢出 */
.container {
  width: 960px;
}

        ✅ 用 max-width + 百分比

        /* 正确：限制最大宽度，小屏自动缩放 */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
}

        ❌ 媒体查询写死设备宽度

        /* 错误：只针对 iPhone 14 Pro */
@media (width: 393px) { ... }

        ✅ 基于内容变化设断点

        /* 正确：当内容排不下时切换布局 */
@media (max-width: 768px) {
  .sidebar { display: none; }
}

        ❌ 使用 !important 覆盖媒体查询

        /* 错误：优先级混乱 */
.sidebar {
  width: 300px !important;
}
@media (max-width: 768px) {
  .sidebar {
    width: 100% !important; /* 恶性循环 */
  }
}

        ✅ 合理组织 CSS 层级

        /* 正确：利用选择器特异性，不用 !important */
.layout .sidebar { width: 300px; }
@media (max-width: 768px) {
  .layout .sidebar { width: 100%; }
}

        ❌ 忽视 clamp() 函数

        /* 错误：字体大小跳变 */
h1 { font-size: 2rem; }
@media (max-width: 768px) {
  h1 { font-size: 1.2rem; }
}

        ✅ 使用流体排版

        /* 正确：平滑缩放，无跳变 */
h1 {
  font-size: clamp(1.2rem, 0.5rem + 3vw, 2.5rem);
}

        七、练习题

        🟢 入门级

        题目 1： 编写媒体查询，当屏幕宽度小于 480px 时，将 .box 的宽度从 50% 变为 100%。

        /* 你的答案 */

        参考答案

        .box {
  width: 50%;
}

@media (max-width: 480px) {
  .box {
    width: 100%;
  }
}

        题目 2： 让所有图片自适应容器宽度，且不变形。

        参考答案

        img {
  max-width: 100%;
  height: auto;
  display: block;
}

        🟡 中级

        题目 3： 使用 CSS Grid 实现响应式卡片网格：桌面3列、平板2列、手机1列，不使用媒体查询。

        参考答案

        .card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 1.5rem;
}

        题目 4： 使用 clamp() 实现标题字体在手机端 1.5rem、桌面端 2.5rem 之间平滑过渡。

        参考答案

        h1 {
  font-size: clamp(1.5rem, 0.5rem + 2.5vw, 2.5rem);
}

        🔴 高级

        题目 5： 实现一个容器查询（Container Query）：当卡片容器宽度大于 400px 时水平排列，否则垂直排列。

        参考答案

        .card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
  .card__image {
    width: 200px;
    flex-shrink: 0;
  }
}

          • 十一、代码演进

          v1：固定宽度 + 媒体查询跳变

          /* 早期做法：固定宽度 + 断点跳变 */
.container { width: 960px; margin: 0 auto; }
@media (max-width: 768px) { .container { width: 100%; } }

h1 { font-size: 2rem; }
@media (max-width: 768px) { h1 { font-size: 1.2rem; } }

.grid { display: grid; grid-template-columns: 1fr 1fr 1fr; }
@media (max-width: 768px) { .grid { grid-template-columns: 1fr 1fr; } }
@media (max-width: 480px) { .grid { grid-template-columns: 1fr; } }

          v2：弹性布局 + 移动优先 + clamp

          /* 进阶：弹性单位 + 流体排版 */
.container { width: 100%; max-width: 1200px; margin: 0 auto; padding: 0 1rem; }

h1 { font-size: clamp(1.2rem, 0.5rem + 3vw, 2.5rem); }

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: clamp(1rem, 2vw, 2rem);
}

          v3：容器查询 + 自定义属性 + 完整设计系统

          /* 现代：容器查询 + 设计 Token */
:root {
  --space-sm: clamp(0.5rem, 1vw, 1rem);
  --space-md: clamp(1rem, 2vw, 1.5rem);
  --fs-h1: clamp(1.5rem, 0.5rem + 3vw, 3rem);
}

.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card { display: grid; grid-template-columns: 200px 1fr; }
}

/* 自适应配色 */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #0f172a;
    --color-text: #e2e8f0;
  }
}

          演进总结： v1 依赖固定断点和硬编码值 → v2 引入流体单位和 auto-fit → v3 加入容器查询和设计系统，真正实现组件级响应式。
