# CSS3 动画与过渡

> **难度：⭐⭐⭐ 中级** | 预计阅读：18 分钟 | 需要基础 CSS 知识

## 一、概念讲解

### 过渡（Transition）vs 动画（Animation）

| 特性 | Transition | Animation |
|------|-----------|-----------|
| 触发方式 | 需要状态变化（hover、class切换） | 自动播放或手动控制 |
| 关键帧 | 只能起点→终点 | 支持多个关键帧 |
| 循环 | 不支持 | 支持 infinite 循环 |
| 暂停 | 不支持 | 支持 `animation-play-state` |
| 复杂度 | 简单，2个状态 | 复杂，多阶段控制 |
| 适用场景 | 按钮悬停、面板展开 | 加载动画、持续动效 |

### Transition 详解

```css
/* 完整语法 */
.element {
  transition: property duration timing-function delay;
}

/* 等价于：
  transition-property: all;
  transition-duration: 0.3s;
  transition-timing-function: ease;
  transition-delay: 0s;
*/

/* 多属性过渡 */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease, opacity 0.2s ease-in;
}
```

**时序函数（Timing Function）：**

| 函数 | 效果 | 使用场景 |
|------|------|---------|
| `ease` | 慢→快→慢（默认） | 通用 |
| `linear` | 匀速 | 进度条、旋转 |
| `ease-in` | 慢→快 | 元素退出 |
| `ease-out` | 快→慢 | 元素进入 |
| `ease-in-out` | 慢→快→慢 | 淡入淡出 |
| `cubic-bezier()` | 自定义曲线 | 精确控制 |

### Animation 详解

```css
/* 定义关键帧 */
@keyframes slideIn {
  0% {
    transform: translateX(-100%);
    opacity: 0;
  }
  60% {
    transform: translateX(10%);
    /* opacity 继承上一帧的值 */
  }
  100% {
    transform: translateX(0);
    opacity: 1;
  }
}

/* 应用动画 */
.element {
  animation: slideIn 0.6s ease-out forwards;
}

/* 完整属性 */
.element {
  animation-name: slideIn;
  animation-duration: 0.6s;
  animation-timing-function: ease-out;
  animation-delay: 0.1s;
  animation-iteration-count: 1;      /* or infinite */
  animation-direction: normal;        /* alternate = 来回 */
  animation-fill-mode: forwards;      /* 保留最后一帧 */
  animation-play-state: running;      /* paused = 暂停 */
}
```

### Transform：动画的基石

Transform 是 GPU 加速的属性，**不影响文档流**，是动画的首选属性。

```css
/* 常用变换 */
.box {
  /* 平移 */
  transform: translateX(50px) translateY(20px);
  transform: translate(50px, 20px);    /* 简写 */

  /* 缩放 */
  transform: scale(1.2);
  transform: scaleX(1.5);

  /* 旋转 */
  transform: rotate(45deg);
  transform: rotateX(180deg);          /* 3D 旋转 */

  /* 倾斜 */
  transform: skewX(10deg);

  /* 组合使用 */
  transform: translateX(50px) rotate(45deg) scale(1.1);
}
```

## 五、注意事项

| 项目 | 说明 | 重要性 |
|------|------|--------|
| 性能属性 | 优先使用 `transform` 和 `opacity`，GPU 加速 | ⭐⭐⭐⭐⭐ |
| will-change | 提前告知浏览器要变化的属性，但别滥用 | ⭐⭐⭐⭐ |
| animation-fill-mode | `forwards` 保留最后一帧，避免回弹 | ⭐⭐⭐⭐ |
| prefers-reduced-motion | 尊重用户的无障碍偏好设置 | ⭐⭐⭐⭐⭐ |
| transform 顺序 | 多个变换的顺序影响结果（矩阵乘法不可交换） | ⭐⭐⭐ |
| 3D 性能 | `perspective` 和 3D 变换消耗更大，谨慎使用 | ⭐⭐⭐ |

## 六、避坑指南

### ❌ 动画 width / height / margin / padding
```css
/* 错误：触发重排（Layout），性能差 */
.box:hover {
  width: 300px;  /* 每帧重新计算布局 */
}
```

### ✅ 使用 transform: scale()
```css
/* 正确：GPU 加速，不触发重排 */
.box:hover {
  transform: scale(1.5);
}
```

### ❌ 忘记 animation-fill-mode
```css
/* 错误：动画结束后回到初始状态（闪一下） */
.element {
  animation: slideIn 0.5s ease;
}
```

### ✅ 加上 forwards
```css
/* 正确：动画结束后保持最后一帧 */
.element {
  animation: slideIn 0.5s ease forwards;
}
```

### ❌ 所有元素同时动画
```css
/* 错误：同时出现，像页面闪烁 */
.item { animation: fadeIn 0.3s ease; }
```

### ✅ 使用交错动画（Stagger）
```css
/* 正确：错落有致，体验更好 */
.item { animation: fadeIn 0.3s ease both; }
.item:nth-child(1) { animation-delay: 0s; }
.item:nth-child(2) { animation-delay: 0.1s; }
.item:nth-child(3) { animation-delay: 0.2s; }
```

### ❌ 忽视无障碍偏好
```css
/* 错误：对所有用户都播放动画 */
.spinner { animation: spin 1s infinite; }
```

### ✅ 尊重 prefers-reduced-motion
```css
/* 正确：为敏感用户减少动画 */
.spinner { animation: spin 1s infinite; }

@media (prefers-reduced-motion: reduce) {
  .spinner { animation: none; }
  /* 或者降低动画频率 */
}
```

## 八、知识点总结

```
CSS3 动画系统
├── Transition 过渡
│   ├── 四个属性 (property/duration/timing/delay)
│   ├── 状态触发 (hover/focus/class)
│   └── 时序函数 (ease/linear/cubic-bezier)
├── Animation 动画
│   ├── @keyframes 定义
│   ├── 动画属性简写
│   ├── fill-mode (forwards/backwards/both)
│   ├── direction (normal/reverse/alternate)
│   ├── play-state (running/paused)
│   └── 多动画组合
├── Transform 变换
│   ├── 2D (translate/scale/rotate/skew)
│   ├── 3D (perspective/rotateX/rotateY)
│   ├── transform-origin 变换原点
│   └── 组合变换顺序
├── 性能优化
│   ├── 仅动画 transform + opacity
│   ├── will-change 提示
│   ├── contain: layout paint
│   └── GPU 图层提升
└── 无障碍
    ├── prefers-reduced-motion
    └── 渐进增强策略
```

## 十一、代码演进

### v1：简单 transition 直接动画

```css
/* 早期：直接动画所有属性 */
.box {
  width: 100px;
  height: 100px;
  background: blue;
  transition: all 0.3s;
}

.box:hover {
  width: 200px;    /* 触发 Layout */
  height: 200px;   /* 触发 Layout */
  background: red;
}
```

### v2：使用 transform + 性能优化

```css
/* 进阶：GPU 加速属性 */
.box {
  width: 100px;
  height: 100px;
  background: blue;
  will-change: transform;         /* 提前创建 GPU 图层 */
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.box:hover {
  transform: scale(2);            /* 不触发 Layout */
  box-shadow: 0 10px 30px rgba(0, 0, 0, 0.3);
}

/* 响应无障碍需求 */
@media (prefers-reduced-motion: reduce) {
  .box {
    transition: none;
  }
}
```

### v3：CSS 自定义属性动画 + @property

```css
/* 现代：注册自定义属性实现渐变动画 */
@property --angle {
  syntax: '<angle>';
  initial-value: 0deg;
  inherits: false;
}

@property --hue {
  syntax: '<number>';
  initial-value: 0;
  inherits: false;
}

.gradient-box {
  --angle: 0deg;
  background: conic-gradient(from var(--angle), #6366f1, #c084fc, #6366f1);
  animation: rotateGradient 3s linear infinite;
  border-radius: 12px;
  padding: 2rem;
}

@keyframes rotateGradient {
  to { --angle: 360deg; }
}

/* 交错入场系统 */
.stagger > * {
  animation: fadeSlideUp 0.6s ease-out both;
  animation-delay: calc(var(--i, 0) * 0.1s);
}

@keyframes fadeSlideUp {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}

/* HTML: <div class="stagger">
   <div style="--i: 0">Item 1</div>
   <div style="--i: 1">Item 2</div>
   <div style="--i: 2">Item 3</div>
</div> */
```

> **演进总结：** v1 动画触发重排的性能差 → v2 用 transform + will-change 实现 GPU 加速 → v3 利用 @property 实现自定义属性动画 + CSS 变量驱动的交错系统。
