概念讲解

为什么选择器和盒模型如此重要？

CSS的两个核心问题：

  1. 选谁？ → CSS选择器
    1. 它多大？ → 盒模型

      2. 这两个概念是所有CSS开发的地基。无论你用Flexbox、Grid还是定位，都需要先用选择器锁定目标元素，再用盒模型控制它的尺寸和间距。

      CSS选择器分类

      选择器类型：
├── 基础选择器
│   ├── 元素选择器: p, div, span
│   ├── 类选择器: .className
│   ├── ID选择器: #idName
│   └── 通配符: *
├── 组合选择器
│   ├── 后代: A B （所有后代）
│   ├── 子代: A > B （直接子元素）
│   ├── 相邻兄弟: A + B （紧邻的下一个）
│   └── 通用兄弟: A ~ B （后面的所有兄弟）
├── 属性选择器
│   ├── [attr] （有此属性）
│   ├── [attr=val] （等于）
│   ├── [attr^=val] （开头匹配）
│   ├── [attr$=val] （结尾匹配）
│   └── [attr*=val] （包含匹配）
└── 伪类/伪元素
    ├── :hover, :focus, :active
    ├── :first-child, :last-child, :nth-child(n)
    ├── ::before, ::after
    └── ::placeholder

      盒模型（Box Model）

      每个HTML元素都是一个"盒子"，由四层组成：

      ┌────────────── Margin ──────────────┐
│  ┌──────── Border ──────────────┐  │
│  │  ┌───── Padding ──────────┐  │  │
│  │  │                        │  │  │
│  │  │       Content          │  │  │
│  │  │     (width × height)   │  │  │
│  │  │                        │  │  │
│  │  └────────────────────────┘  │  │
│  └──────────────────────────────┘  │
└────────────────────────────────────┘

      两种盒模型：

      模型 width包含 box-sizing值
      content-box（默认） 仅内容 content-box
      border-box（推荐） 内容+内边距+边框 border-box 
      /* Content-box: width = content only */
.box-content {
  box-sizing: content-box;
  width: 200px;          /* content area = 200px */
  padding: 20px;         /* total = 200 + 20*2 = 240px */
  border: 2px solid;     /* total = 240 + 2*2 = 244px */
}

/* Border-box: width = content + padding + border */
.box-border {
  box-sizing: border-box;
  width: 200px;          /* total = 200px! */
  padding: 20px;         /* content = 200 - 20*2 - 2*2 = 156px */
  border: 2px solid;
}

      完整CSS代码

      选择器实战

      /* ===== Basic Selectors ===== */
h1 { font-size: 2rem; }              /* element selector */
.card-title { font-weight: 600; }    /* class selector */
#main-content { padding: 24px; }     /* ID selector */
* { margin: 0; padding: 0; }         /* universal selector */

/* ===== Combinator Selectors ===== */
/* Descendant: any nested .link inside .nav */
.nav .link { color: #3b82f6; }

/* Child: direct children only */
.menu > li { border-bottom: 1px solid #e2e8f0; }

/* Adjacent sibling: the element right after */
h2 + p { margin-top: 8px; font-size: 1.125rem; }

/* General sibling: all following siblings */
h2 ~ p { color: #475569; }

/* ===== Attribute Selectors ===== */
/* Links that open in new tab */
a[target="_blank"]::after {
  content: " ↗";
  font-size: 0.75em;
}

/* Images with alt text containing "logo" */
img[alt*="logo"] { height: 40px; }

/* Input types */
input[type="email"] { border-color: #3b82f6; }
input[type="password"] { letter-spacing: 0.15em; }

/* ===== Pseudo-classes ===== */
/* Interactive states */
.btn:hover { background: #2563eb; }
.btn:active { transform: scale(0.98); }
input:focus { outline: 2px solid #3b82f6; outline-offset: 2px; }
a:visited { color: #7c3aed; }

/* Structural pseudo-classes */
/* First and last item in list */
.list-item:first-child { border-radius: 8px 8px 0 0; }
.list-item:last-child { border-radius: 0 0 8px 8px; }

/* Odd/even rows for zebra striping */
.table-row:nth-child(odd) { background: #f8fafc; }
.table-row:nth-child(even) { background: white; }

/* Every 3rd item */
.grid-item:nth-child(3n) { margin-right: 0; }

/* Form states */
input:required { border-left: 3px solid #ef4444; }
input:valid { border-color: #22c55e; }
input:invalid { border-color: #ef4444; }
input:disabled { opacity: 0.5; cursor: not-allowed; }

/* Negation */
input:not([type="submit"]) { padding: 10px 14px; }

/* ===== Pseudo-elements ===== */
/* Decorative quote marks */
blockquote::before {
  content: "\201C";   /* left double quotation mark */
  font-size: 3rem;
  color: #93c5fd;
  position: absolute;
  top: -10px;
  left: -10px;
}

/* Custom list bullets */
.feature-list li::before {
  content: "✓ ";
  color: #22c55e;
  font-weight: bold;
}

/* Placeholder styling */
input::placeholder {
  color: #94a3b8;
  font-style: italic;
}

/* Text selection color */
::selection {
  background: #3b82f6;
  color: white;
}

      盒模型实战

      /* ===== Global box-sizing reset ===== */
/* Apply border-box to everything */
*,
*::before,
*::after {
  box-sizing: border-box;
}

/* ===== Card component demonstrating box model ===== */
.card {
  width: 100%;
  max-width: 400px;
  margin: 16px auto;           /* center horizontally */
  padding: 24px;               /* inner spacing */
  border: 1px solid #e2e8f0;
  border-radius: 8px;
  background: white;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.06);
}

.card-title {
  margin: 0 0 12px;            /* bottom margin only */
  padding-bottom: 12px;
  border-bottom: 1px solid #f1f5f9;
  font-size: 1.25rem;
}

.card-body {
  margin: 0;
  padding: 0;                  /* inherit from parent card */
  color: #475569;
  line-height: 1.6;
}

.card-footer {
  margin-top: 16px;
  padding-top: 12px;
  border-top: 1px solid #f1f5f9;
  display: flex;
  justify-content: flex-end;
  gap: 8px;
}

/* ===== Margin auto centering trick ===== */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;              /* horizontal center */
  padding: 0 24px;
}

/* ===== Prevent margin collapsing with BFC ===== */
.parent {
  overflow: hidden;            /* creates BFC, prevents margin collapse */
}

.child {
  margin-top: 20px;            /* won't collapse with parent margin */
}

      选择器优先级计算实战

      /* Specificity: (inline, id, class, element) */

p                        { color: black; }        /* 0,0,0,1 */
.text                    { color: blue; }         /* 0,0,1,0 */
p.text                   { color: green; }        /* 0,0,1,1 */
#special                 { color: red; }          /* 0,1,0,0 */
p#special.highlighted    { color: purple; }       /* 0,1,1,1 */
style="color: orange"                             /* 1,0,0,0 */
!important               { color: pink !important; } /* overrides all */

/* Specificity comparison (low → high):
   element < class < id < inline < !important */

      注意事项

      概念 注意点 建议
      box-sizing 默认content-box 全局设 border-box
      margin合并 垂直margin会合并（取较大值） 用padding或BFC避免
      ID选择器 优先级过高，难以覆盖 优先用class选择器
      !important 破坏优先级规则 仅在覆盖第三方样式时使用
      * 选择器 影响所有元素 仅用于全局reset
      nth-child(n) n从1开始，不是0 nth-child(1) = first-child
      ::before/::after 必须有content属性 不设content则不显示
      属性选择器 性能略低于类选择器 高频操作场景避免使用

      避坑指南

      ❌ 不设置全局box-sizing

      /* ❌ Bad: default content-box makes width calculations painful */
.card {
  width: 33.33%;
  padding: 20px;    /* actual width = 33.33% + 40px! */
  border: 1px solid; /* actual width = 33.33% + 40px + 2px! */
}
      /* ✅ Good: border-box everywhere */
*, *::before, *::after {
  box-sizing: border-box;
}
.card {
  width: 33.33%;    /* total width = exactly 33.33% */
  padding: 20px;
  border: 1px solid;
}

      ❌ 滥用ID选择器

      /* ❌ Bad: ID has high specificity, hard to override */
#header { background: navy; }
#header .nav { color: white; }  /* need to match specificity to override */
      /* ✅ Good: use class selectors, easy to cascade */
.header { background: navy; }
.header .nav { color: white; }

      ❌ 忽略margin合并

      /* ❌ Bad: unexpected spacing due to margin collapse */
h2 { margin-bottom: 24px; }
p { margin-top: 16px; }
/* Actual gap = max(24px, 16px) = 24px, NOT 40px */
      /* ✅ Good: use only one direction margin */
h2 { margin-bottom: 24px; }
p { margin-top: 0; }    /* rely on h2's margin-bottom */

      ❌ 混淆:nth-child和:nth-of-type

      /* ❌ Bad: nth-child counts ALL siblings */
/* If a <p> is the 2nd child but 1st <p>, nth-child(2) won't match it as first p */
p:nth-child(1) { }  /* matches only if <p> is the 1st child of parent */
      /* ✅ Good: nth-of-type counts only elements of that type */
p:nth-of-type(1) { }  /* matches the 1st <p> regardless of other siblings */

      4. 计算选择器优先级

      以下选择器优先级从高到低排列：

      • #nav .link:hover
      • div.nav a
      • .header .nav .link
      • a.link

        • 参考答案

        1. #nav .link:hover  → 0,1,2,1  (1 ID + 2 class + 1 element)
2. .header .nav .link → 0,0,3,0 (3 classes)
3. div.nav a          → 0,0,1,2 (1 class + 2 elements)
4. a.link             → 0,0,1,1 (1 class + 1 element)

        🔴 高级

        5. 实现纯CSS tooltip

        要求：用 ::after 伪元素在按钮悬停时显示提示文字，提示文字来自 data-tooltip 属性。

        <button data-tooltip="点击保存">保存</button>

        参考答案

        [data-tooltip] {
  position: relative;
}
[data-tooltip]::after {
  content: attr(data-tooltip);
  position: absolute;
  bottom: 100%;
  left: 50%;
  transform: translateX(-50%);
  padding: 6px 12px;
  background: #1e293b;
  color: white;
  font-size: 0.75rem;
  border-radius: 4px;
  white-space: nowrap;
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.2s;
}
[data-tooltip]:hover::after {
  opacity: 1;
}

        知识点总结

        CSS 选择器与盒模型
├── 选择器体系
│   ├── 基础: * / element / .class / #id
│   ├── 组合: 后代(A B) / 子代(A>B) / 相邻(A+B) / 兄弟(A~B)
│   ├── 属性: [attr] [=] [^=] [$=] [*=]
│   ├── 伪类
│   │   ├── 交互: :hover :focus :active
│   │   ├── 结构: :first-child :nth-child() :last-child
│   │   ├── 表单: :checked :disabled :valid
│   │   └── 逻辑: :not() :is() :where()
│   └── 伪元素: ::before ::after ::placeholder
├── 优先级(Specificity)
│   ├── !important > inline > id > class > element
│   └── 同级按书写顺序，后者覆盖前者
├── 盒模型
│   ├── content-box: width = content
│   ├── border-box: width = content + padding + border ✅
│   └── 推荐: 全局 border-box
└── 尺寸控制
    ├── width/height + min/max
    ├── padding: 内边距
    ├── margin: 外边距（注意合并）
    └── margin: auto 水平居中

        举一反三

        需求场景 推荐选择器 关键技巧
        表单验证样式 :valid / :invalid 配合HTML5验证属性
        斑马纹表格 :nth-child(odd/even) 性能优于JS添加类
        清除浮动 ::after 伪元素 content: ""; display: table; clear: both
        外部链接标记 a[target="_blank"] 属性选择器
        文本省略 盒模型+overflow white-space: nowrap; overflow: hidden; text-overflow: ellipsis
        水平居中 margin: 0 auto 配合固定/max-width
        最后一项无margin :last-child 或用 :not(:last-child)
        自定义checkbox :checked + label 相邻兄弟选择器
