## 概念讲解

### 性能优化三大维度

```
性能优化
├── 加载性能 → 用户看到内容的时间
├── 运行时性能 → 交互流畅度
└── 内存性能 → 长时间运行的稳定性
```

### Vue 3 自身的性能提升

| 特性 | 说明 |
|------|------|
| Proxy代理 | 惰性响应式，只在访问时创建依赖 |
| 静态提升 | 模板中的静态节点只创建一次 |
| Patch Flag | 编译时标记动态节点，diff时跳过静态部分 |
| Block Tree | 扁平化虚拟DOM树，减少递递 |
| Tree-shaking | 未使用的API不打包 |

## 完整代码示例

```vue
<!-- PerformanceDemo.vue -->
<template>
  <div class="perf-demo">
    <h2>Vue 3 性能优化演示</h2>

    <section>
      <h3>1. v-memo 条件缓存</h3>
      <div
        v-for="item in items"
        :key="item.id"
        v-memo="[item.id, selectedId === item.id]"
        :class="{ selected: selectedId === item.id }"
        @click="selectedId = item.id"
      >
        {{ item.name }}
      </div>
      <button @click="shuffleItems">打乱顺序</button>
    </section>

    <section>
      <h3>2. shallowRef vs ref 性能对比</h3>
      <button @click="updateDeep">深更新 (ref)</button>
      <button @click="updateShallow">浅更新 (shallowRef)</button>
      <p>ref更新耗时: {{ refTime }}ms</p>
      <p>shallowRef更新耗时: {{ shallowTime }}ms</p>
    </section>
  </div>
</template>

<script setup>
import { ref, shallowRef } from 'vue'

const items = ref(
  Array.from({ length: 20 }, (_, i) => ({ id: i, name: `项目${i}` }))
)
const selectedId = ref(-1)

function shuffleItems() {
  items.value = [...items.value].sort(() => Math.random() - 0.5)
}

const refTime = ref(0)
const shallowTime = ref(0)

const deepData = ref(Array.from({ length: 1000 }, (_, i) => ({ id: i, value: i })))
const shallowData = shallowRef(Array.from({ length: 1000 }, (_, i) => ({ id: i, value: i })))

function updateDeep() {
  const start = performance.now()
  for (let i = 0; i < 1000; i++) { deepData.value[i].value = Math.random() }
  refTime.value = Math.round(performance.now() - start)
}

function updateShallow() {
  const start = performance.now()
  const arr = [...shallowData.value]
  for (let i = 0; i < 1000; i++) { arr[i] = { ...arr[i], value: Math.random() } }
  shallowData.value = arr
  shallowTime.value = Math.round(performance.now() - start)
}
</script>
```

## 注意事项

| 优化策略 | 适用场景 | 过度使用的风险 |
|---------|---------|--------------|
| shallowRef | 大型不可变数据 | 失去深层响应式 |
| v-memo | 纯展示型列表项 | 增加内存开销 |
| 虚拟滚动 | 1000+条列表 | 50条以下没必要 |
