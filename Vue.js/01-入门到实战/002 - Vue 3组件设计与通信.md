# Vue 3组件设计与通信

> 📊 **难度：中级** | 🏷️ Vue.js 3.x | ⏱️ 阅读约 25 分钟

---

## 📖 概念讲解

组件通信是Vue应用架构的核心。不同关系层级需要不同的通信模式。

**通信模式总览：**

| 模式 | 方向 | 适用场景 | 复杂度 |
|------|------|---------|--------|
| Props | 父→子 | 传递数据 | ⭐ |
| Events | 子→父 | 子组件通知 | ⭐ |
| v-model | 双向 | 表单类组件 | ⭐⭐ |
| Provide/Inject | 跨层级 | 深层传递 | ⭐⭐ |
| EventBus | 任意 | 简单全局通信 | ⭐⭐ |
| Pinia | 全局 | 复杂状态管理 | ⭐⭐⭐ |
| $refs | 父→子 | 直接操作子组件 | ⭐⭐ |

**Props传递类型：**

- **基础类型**：String, Number, Boolean（直接传递）
- **引用类型**：Object, Array（注意引用传递）
- **函数类型**：Function（作为回调传递）
- **v-bind批量传递**：`v-bind="obj"` 将对象所有属性作为props传递

**Provide/Inject：**

解决"Props逐层透传"问题。祖先组件provide数据，后代组件inject使用。

```js
// Ancestor: provide
import { provide, ref } from 'vue'
const theme = ref('dark')
provide('theme', theme)       // 提供响应式数据
provide('theme', 'readonly')  // 只读

// Descendant: inject
import { inject } from 'vue'
const theme = inject('theme', 'light')  // 第二参数是默认值
```

**组件设计原则：**

1. **Smart vs Dumb组件**：容器组件（有逻辑）vs 展示组件（纯UI）
2. **单一职责**：一个组件只做一件事
3. **接口最小化**：Props尽量少，Emits尽量少
4. **插槽优先**：灵活的内容分发优于复杂的Props

## 💻 完整代码

```vue
<!-- ComponentCommunicationDemo.vue -->
<template>
  <div class="demo">
    <h2>1. Props + Events (Parent ↔ Child)</h2>
    <ChildForm
      :model-value="formData"
      @update:model-value="formData = $event"
      :disabled="isSubmitting"
      @submit="handleSubmit"
    />

    <h2>2. Provide/Inject (Deep nesting)</h2>
    <p>Current theme: {{ theme }} <button @click="toggleTheme">Toggle</button></p>
    <DeepParent />

    <h2>3. $attrs (Attribute fallthrough)</h2>
    <WrappedInput
      v-model="inputValue"
      class="custom-input"
      placeholder="Attrs fallthrough demo"
      data-testid="wrapped"
    />
    <p>Value: {{ inputValue }}</p>

    <h2>4. Template Refs ($refs)</h2>
    <RefChild ref="childRef" />
    <button @click="callChildMethod">Call Child Method</button>
    <p>Child state: {{ childState }}</p>
  </div>
</template>

<script setup>
import { ref, provide, readonly } from 'vue'

// --- 1. Props + Events ---
const formData = ref({ name: '', email: '' })
const isSubmitting = ref(false)

function handleSubmit(data) {
  isSubmitting.value = true
  console.log('Submitted:', data)
  setTimeout(() => { isSubmitting.value = false }, 1000)
}

// --- 2. Provide/Inject ---
const theme = ref('light')
provide('theme', readonly(theme))  // Readonly to prevent direct mutation
provide('toggleTheme', toggleTheme)

function toggleTheme() {
  theme.value = theme.value === 'light' ? 'dark' : 'light'
}

// Deep nesting simulation (inline for demo)
import { defineComponent, inject, h } from 'vue'

const DeepChild = defineComponent({
  setup() {
    const theme = inject('theme')
    const toggle = inject('toggleTheme')
    return () => h('div', {
      style: { padding: '0.5rem', background: theme.value === 'dark' ? '#333' : '#eee', color: theme.value === 'dark' ? '#fff' : '#000' }
    }, [
      h('p', `DeepChild theme: ${theme.value}`),
      h('button', { onClick: toggle }, 'Toggle from deep child')
    ])
  }
})

const DeepParent = defineComponent({
  setup() {
    const theme = inject('theme')
    return () => h('div', { style: { border: '1px solid #42b883', padding: '0.5rem' } }, [
      h('p', `Middle component, theme: ${theme.value}`),
      h(DeepChild)
    ])
  }
})

// --- 3. $attrs ---
const WrappedInput = defineComponent({
  inheritAttrs: false,  // Prevent automatic attribute fallthrough
  emits: ['update:modelValue'],
  props: ['modelValue'],
  setup(props, { attrs, emit }) {
    return () => h('div', { class: 'input-wrapper' }, [
      h('label', 'Wrapped: '),
      h('input', {
        ...attrs,  // Forward all attributes to inner input
        value: props.modelValue,
        onInput: (e) => emit('update:modelValue', e.target.value)
      })
    ])
  }
})
const inputValue = ref('')

// --- 4. Template Refs ---
const childRef = ref(null)
const childState = ref('')

const RefChild = defineComponent({
  setup() {
    const secret = ref('🤫 hidden data')
    const getSecret = () => secret.value
    return { secret, getSecret }
  },
  template: '<div>RefChild component</div>'
})

function callChildMethod() {
  if (childRef.value) {
    childState.value = childRef.value.getSecret()
  }
}
</script>
```

## ⚠️ 注意事项

| 注意点 | 说明 | 建议 |
|--------|------|------|
| Props引用传递 | Object/Array是引用传递 | 子组件修改会影响父组件 |
| Provide响应性 | 直接provide非响应式值 | provide(ref)或provide(readonly(ref)) |
| $attrs透传 | 默认自动透传到根元素 | inheritAttrs:false手动控制 |
| $refs时机 | 模板ref在mounted后才可用 | onMounted中访问ref |
| 事件命名 | 推荐kebab-case | `@update:model-value` |
| readonly | 保护provide数据不被直接修改 | 配合provide使用 |

## 🚫 避坑指南

### 1. 直接修改对象类型的Props
❌ **错误：** `props.user.name = 'Bob'`（引用传递，父组件也被改了！）
✅ **正确：** emit事件让父组件修改，或深拷贝后修改本地副本

### 2. Provide非响应式值
❌ **错误：** `provide('count', 0)` → inject后永远不变
✅ **正确：** `provide('count', ref(0))` → inject得到响应式ref

### 3. $refs滥用
❌ **错误：** 用$refs做组件间通信（命令式）
✅ **正确：** 优先用Props/Events（声明式），$refs只在必要时用

### 4. 循环依赖的Provide/Inject
❌ **错误：** 子组件provide，父组件inject（方向反了）
✅ **正确：** 祖先provide，后代inject（自上而下）

### 5. 忘记readonly
❌ **错误：** inject组件直接修改provide的数据
✅ **正确：** provide时用readonly包装，inject方通过事件/方法修改

## 🔄 举一反三

| 概念 | 生活类比 | 说明 |
|------|---------|------|
| Props | 快递包裹 | 父组件打包，子组件签收 |
| Events | 快递回执 | 子组件签收后发回执给父 |
| v-model | 双向快递 | 来回都能寄 |
| Provide/Inject | 广播电台 | 祖先广播，后代接收 |
| $refs | 远程控制 | 父组件直接操作子组件 |
| $attrs | 转发邮件 | 收到后原封不动转发给内部元素 |
| readonly | 阅览室书刊 | 可以看，不能带走或修改 |

const theme = ref('dark')

const props = defineProps(['modelValue'])
