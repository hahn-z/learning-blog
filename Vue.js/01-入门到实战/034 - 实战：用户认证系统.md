## 概念讲解

### 前端认证架构

用户认证是 Web 应用的核心安全模块。前端需要处理：

1. **登录/注册流程**：表单提交、验证、错误处理
2. **Token 管理**：存储、自动注入、刷新、过期处理
3. **路由守卫**：未登录跳转、权限验证
4. **权限控制**：角色/权限级别的 UI 控制

### JWT 认证流程

```
登录 → POST /auth/login (email+password)
     ← { accessToken, refreshToken }
     → 存储 Token (localStorage/cookie)
     → 后续请求 Authorization: Bearer <token>
     → Token 过期 → POST /auth/refresh → 新 Token
```

### Token 存储对比

| 方案 | 安全性 | 持久性 | XSS 风险 | 适用场景 |
|------|--------|--------|----------|----------|
| localStorage | 中 | 浏览器关闭保留 | 有 | SPA 普通项目 |
| sessionStorage | 中 | 标签页关闭清除 | 有 | 敏感操作 |
| Cookie (HttpOnly) | 高 | 可设过期时间 | 无 | 安全优先 |
| Pinia (内存) | 高 | 刷新丢失 | 无 | 需配合持久化 |

## 完整代码

### Auth Store

```ts
// src/stores/auth.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { authApi, type User, type LoginParams, type RegisterParams } from '@/api/auth'
import router from '@/router'

export const useAuthStore = defineStore('auth', () => {
  const user = ref<User | null>(null)
  const token = ref<string>(localStorage.getItem('token') || '')
  const refreshToken = ref<string>(localStorage.getItem('refreshToken') || '')

  // Computed auth state
  const isAuthenticated = computed(() => !!token.value)
  const userRole = computed(() => user.value?.role || 'guest')

  // Login action
  async function login(params: LoginParams) {
    const res = await authApi.login(params)
    token.value = res.accessToken
    refreshToken.value = res.refreshToken
    user.value = res.user
    localStorage.setItem('token', res.accessToken)
    localStorage.setItem('refreshToken', res.refreshToken)
  }

  // Register action
  async function register(params: RegisterParams) {
    const res = await authApi.register(params)
    token.value = res.accessToken
    refreshToken.value = res.refreshToken
    user.value = res.user
    localStorage.setItem('token', res.accessToken)
    localStorage.setItem('refreshToken', res.refreshToken)
  }

  // Get current user profile
  async function fetchUser() {
    if (!token.value) return
    try {
      user.value = await authApi.getProfile()
    } catch {
      logout()
    }
  }

  // Refresh expired token
  async function refreshAccessToken() {
    const res = await authApi.refresh(refreshToken.value)
    token.value = res.accessToken
    localStorage.setItem('token', res.accessToken)
    return res.accessToken
  }

  // Logout: clear all auth state
  function logout() {
    user.value = null
    token.value = ''
    refreshToken.value = ''
    localStorage.removeItem('token')
    localStorage.removeItem('refreshToken')
    router.push('/login')
  }

  return {
    user, token, isAuthenticated, userRole,
    login, register, fetchUser, refreshAccessToken, logout
  }
})
```

### Axios 拦截器（Token 自动注入 + 刷新）

```ts
// src/utils/http.ts (auth-related part)
import axios from 'axios'
import { useAuthStore } from '@/stores/auth'

let isRefreshing = false
let pendingRequests: ((token: string) => void)[] = []

http.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config

    // If 401 and not already retrying
    if (error.response?.status === 401 && !originalRequest._retry) {
      const auth = useAuthStore()

      if (isRefreshing) {
        // Queue pending requests while refreshing
        return new Promise((resolve) => {
          pendingRequests.push((newToken: string) => {
            originalRequest.headers.Authorization = `Bearer ${newToken}`
            resolve(http(originalRequest))
          })
        })
      }

      originalRequest._retry = true
      isRefreshing = true

      try {
        const newToken = await auth.refreshAccessToken()
        // Replay all pending requests
        pendingRequests.forEach(cb => cb(newToken))
        pendingRequests = []
        originalRequest.headers.Authorization = `Bearer ${newToken}`
        return http(originalRequest)
      } catch {
        auth.logout()
        return Promise.reject(error)
      } finally {
        isRefreshing = false
      }
    }

    return Promise.reject(error)
  }
)
```

### 路由守卫

```ts
// src/router/guards.ts
import type { Router } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

// Routes that don't require authentication
const WHITE_LIST = ['/login', '/register', '/forgot-password', '/']

export function setupRouterGuards(router: Router) {
  router.beforeEach(async (to, from, next) => {
    const auth = useAuthStore()

    // White list routes: always allow
    if (WHITE_LIST.includes(to.path)) {
      // If already logged in, redirect to dashboard
      if (auth.isAuthenticated && to.path === '/login') {
        next('/dashboard')
        return
      }
      next()
      return
    }

    // Not authenticated: redirect to login
    if (!auth.isAuthenticated) {
      next({ path: '/login', query: { redirect: to.fullPath } })
      return
    }

    // Fetch user profile if not loaded
    if (!auth.user) {
      try {
        await auth.fetchUser()
      } catch {
        next('/login')
        return
      }
    }

    // Check role-based permissions
    const requiredRole = to.meta.role as string | undefined
    if (requiredRole && auth.userRole !== requiredRole) {
      next('/403') // Forbidden
      return
    }

    next()
  })
}
```

### 登录页面

```vue
<!-- src/views/Login.vue -->
<template>
  <div class="login-page">
    <el-card class="login-card">
      <h2>Sign In</h2>
      <el-form ref="formRef" :model="form" :rules="rules" @submit.prevent="handleLogin">
        <el-form-item prop="email">
          <el-input v-model="form.email" placeholder="Email" prefix-icon="Message" />
        </el-form-item>
        <el-form-item prop="password">
          <el-input v-model="form.password" type="password" placeholder="Password"
            prefix-icon="Lock" show-password />
        </el-form-item>
        <el-form-item>
          <el-button type="primary" native-type="submit" :loading="loading" style="width: 100%">
            Sign In
          </el-button>
        </el-form-item>
      </el-form>
      <p class="hint">
        Don't have an account?
        <router-link to="/register">Register</router-link>
      </p>
    </el-card>
  </div>
</template>

<script setup lang="ts">
import { ref, reactive } from 'vue'
import { useRouter, useRoute } from 'vue-router'
import { useAuthStore } from '@/stores/auth'
import type { FormInstance } from 'element-plus'

const router = useRouter()
const route = useRoute()
const auth = useAuthStore()
const formRef = ref<FormInstance>()
const loading = ref(false)

const form = reactive({ email: '', password: '' })
const rules = {
  email: [
    { required: true, message: 'Please enter email', trigger: 'blur' },
    { type: 'email', message: 'Invalid email format', trigger: 'blur' }
  ],
  password: [
    { required: true, message: 'Please enter password', trigger: 'blur' },
    { min: 6, message: 'Password must be at least 6 characters', trigger: 'blur' }
  ]
}

async function handleLogin() {
  await formRef.value?.validate()
  loading.value = true
  try {
    await auth.login(form)
    // Redirect to original page or dashboard
    const redirect = (route.query.redirect as string) || '/dashboard'
    router.push(redirect)
  } catch (err: any) {
    // Error handled by interceptor
  } finally {
    loading.value = false
  }
}
</script>
```

### 权限指令

```ts
// src/directives/permission.ts
import type { Directive } from 'vue'
import { useAuthStore } from '@/stores/auth'

// v-permission="'admin'" — only show for admin users
export const vPermission: Directive<HTMLElement, string> = {
  mounted(el, binding) {
    const auth = useAuthStore()
    const requiredRole = binding.value
    if (auth.userRole !== requiredRole) {
      el.parentNode?.removeChild(el)
    }
  }
}

// Usage in template:
// <button v-permission="'admin'">Admin Only Button</button>
```

## 执行预览

```
访问 /dashboard → 未登录 → 跳转 /login?redirect=/dashboard
输入 email + password → POST /auth/login → 200 { accessToken, refreshToken }
→ 存储 Token → 跳转 /dashboard

Token 过期后请求 → 401 → 自动 refresh → 重发原请求 → 200
→ 用户无感知

退出登录 → 清除 Token → 跳转 /login
```

## 注意事项

| 事项 | 说明 | 建议 |
|------|------|------|
| 密码安全 | 前端不明文存储密码 | 后端 bcrypt 加密 |
| Token 刷新 | 并发请求时只刷新一次 | 队列机制处理 |
| 路由回跳 | 登录后跳回原页面 | redirect query 参数 |
| 首次加载 | 刷新页面后需获取用户信息 | 路由守卫中 fetchUser |
| 记住登录 | localStorage vs sessionStorage | 根据安全需求选择 |
| XSS 防护 | Token 可被窃取 | HttpOnly Cookie 更安全 |

## 避坑指南

- ❌ Token 放在 sessionStorage → 刷新丢失 → ✅ localStorage + 过期时间检查
- ❌ 并发请求都触发 refresh → ✅ 锁机制 + 队列，只刷新一次
- ❌ 路由守卫只检查 Token 存在 → ✅ 还需验证 Token 有效性（fetchUser）
- ❌ 登录后直接 `router.push('/dashboard')` → ✅ 读取 redirect 参数回跳
- ❌ 权限检查只在前端 → ✅ 后端必须同步做权限验证
- ❌ logout 不清 localStorage → ✅ 清除所有 Token 和用户数据
