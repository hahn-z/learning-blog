## 概念讲解

### 什么是 Vue 组件测试？

Vue 组件测试是指通过自动化工具验证组件的**渲染输出、用户交互、状态变化**是否符合预期。测试分为三个层级：

- **单元测试（Unit Test）**：测试单个函数或组合式逻辑
- **组件测试（Component Test）**：测试组件的渲染和交互
- **E2E 测试（End-to-End Test）**：测试完整用户流程

### 核心工具链

| 工具 | 用途 | 特点 |
|------|------|------|
| Vitest | 测试运行器 | Vite 原生集成，速度极快 |
| Vue Test Utils | 组件挂载工具 | 官方维护，API 稳定 |
| happy-dom / jsdom | DOM 模拟 | happy-dom 更快，jsdom 更兼容 |
| Cypress / Playwright | E2E 测试 | 真实浏览器环境 |

### 待测组件

```vue
<!-- src/components/Counter.vue -->
<template>
  <div class="counter">
    <h2>{{ title }}</h2>
    <p class="count" data-testid="count">{{ count }}</p>
    <p class="status">{{ status }}</p>
    <button data-testid="increment" @click="increment">+1</button>
    <button data-testid="decrement" @click="decrement">-1</button>
    <button data-testid="reset" @click="reset">Reset</button>
    <slot name="footer" :count="count" />
  </div>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'

const props = withDefaults(defineProps<{
  title?: string
  initialValue?: number
  min?: number
  max?: number
}>(), {
  title: 'Counter',
  initialValue: 0,
  min: -10,
  max: 10
})

const emit = defineEmits<{
  change: [value: number]
  limit: [direction: 'min' | 'max']
}>()

const count = ref(props.initialValue)

const status = computed(() => {
  if (count.value >= props.max) return 'At maximum'
  if (count.value <= props.min) return 'At minimum'
  return 'Normal'
})

function increment() {
  if (count.value < props.max) {
    count.value++
    emit('change', count.value)
  } else {
    emit('limit', 'max')
  }
}

function decrement() {
  if (count.value > props.min) {
    count.value--
    emit('change', count.value)
  } else {
    emit('limit', 'min')
  }
}

function reset() {
  count.value = props.initialValue
  emit('change', count.value)
}
</script>
```

### 单元测试

```ts
// src/composables/useCounter.test.ts
import { describe, it, expect } from 'vitest'
import { useCounter } from './useCounter'

describe('useCounter', () => {
  it('initializes with default value', () => {
    const { count } = useCounter()
    expect(count.value).toBe(0)
  })

  it('initializes with custom value', () => {
    const { count } = useCounter(10)
    expect(count.value).toBe(10)
  })

  it('increments correctly', () => {
    const { count, increment } = useCounter(5)
    increment()
    expect(count.value).toBe(6)
  })

  it('respects max limit', () => {
    const { count, increment } = useCounter(9, { max: 10 })
    increment()
    expect(count.value).toBe(10)
    increment()
    expect(count.value).toBe(10) // no change beyond max
  })
})
```

### 组件测试

```ts
// src/components/Counter.test.ts
import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import Counter from './Counter.vue'

describe('Counter.vue', () => {
  // Helper to create wrapper with default props
  const createWrapper = (props = {}) =>
    mount(Counter, { props: { ...props } })

  it('renders title prop', () => {
    const wrapper = createWrapper({ title: 'My Counter' })
    expect(wrapper.find('h2').text()).toBe('My Counter')
  })

  it('displays initial count value', () => {
    const wrapper = createWrapper({ initialValue: 5 })
    expect(wrapper.find('[data-testid="count"]').text()).toBe('5')
  })

  it('increments count on button click', async () => {
    const wrapper = createWrapper({ initialValue: 0 })
    await wrapper.find('[data-testid="increment"]').trigger('click')
    expect(wrapper.find('[data-testid="count"]').text()).toBe('1')
  })

  it('decrements count on button click', async () => {
    const wrapper = createWrapper({ initialValue: 5 })
    await wrapper.find('[data-testid="decrement"]').trigger('click')
    expect(wrapper.find('[data-testid="count"]').text()).toBe('4')
  })

  it('resets count to initial value', async () => {
    const wrapper = createWrapper({ initialValue: 3 })
    await wrapper.find('[data-testid="increment"]').trigger('click')
    await wrapper.find('[data-testid="reset"]').trigger('click')
    expect(wrapper.find('[data-testid="count"]').text()).toBe('3')
  })

  it('emits change event on increment', async () => {
    const wrapper = createWrapper()
    await wrapper.find('[data-testid="increment"]').trigger('click')
    expect(wrapper.emitted('change')).toBeTruthy()
    expect(wrapper.emitted('change')![0]).toEqual([1])
  })

  it('emits limit event when hitting max', async () => {
    const wrapper = createWrapper({ initialValue: 10, max: 10 })
    await wrapper.find('[data-testid="increment"]').trigger('click')
    expect(wrapper.emitted('limit')).toBeTruthy()
    expect(wrapper.emitted('limit')![0]).toEqual(['max'])
  })

  it('renders scoped slot content', () => {
    const wrapper = mount(Counter, {
      props: { initialValue: 7 },
      slots: {
        footer: `<template #footer="{ count }">
          <span data-testid="slot">Count: {{ count }}</span>
        </template>`
      }
    })
    expect(wrapper.find('[data-testid="slot"]').text()).toBe('Count: 7')
  })

  it('updates status text based on count', async () => {
    const wrapper = createWrapper({ initialValue: 10, max: 10 })
    expect(wrapper.find('.status').text()).toBe('At maximum')
  })
})
```

### 异步组件测试

```ts
// src/components/UserList.test.ts
import { describe, it, expect, vi } from 'vitest'
import { mount, flushPromises } from '@vue/test-utils'
import UserList from './UserList.vue'

// Mock the API module
vi.mock('@/api/users', () => ({
  fetchUsers: vi.fn(() =>
    Promise.resolve([
      { id: 1, name: 'Alice' },
      { id: 2, name: 'Bob' }
    ])
  )
}))

describe('UserList.vue (async)', () => {
  it('shows loading state then renders users', async () => {
    const wrapper = mount(UserList)
    // Loading state
    expect(wrapper.find('[data-testid="loading"]').exists()).toBe(true)
    // Wait for API call to resolve
    await flushPromises()
    // Users rendered
    expect(wrapper.find('[data-testid="loading"]').exists()).toBe(false)
    expect(wrapper.findAll('[data-testid="user"]')).toHaveLength(2)
    expect(wrapper.findAll('[data-testid="user"]')[0].text()).toBe('Alice')
  })
})
```

## 注意事项

| 事项 | 说明 | 建议 |
|------|------|------|
| 测试环境 | happy-dom vs jsdom | 默认 happy-dom，有问题切 jsdom |
| 异步更新 | DOM 不会同步更新 | 用 `await wrapper.vm.$nextTick()` 或 `flushPromises()` |
| 全局组件 | mount 不注册全局组件 | 需手动传入 `global.components` |
| Pinia | 测试中需要安装 | 用 `createTestingPinia()` |
| 路由 | 需要 `vue-router` mock | 用 `createRouter` + `createWebHistory` 或 mock |
| 定时器 | `setTimeout` 导致测试慢 | `vi.useFakeTimers()` + `vi.advanceTimersByTime()` |
| data-testid | 使用测试标记 | 比用 CSS 类名更稳定 |

## 避坑指南

- ❌ `wrapper.find('.btn')` 依赖 CSS 类名 → ✅ `wrapper.find('[data-testid="submit"]')` 用测试标记
- ❌ 测试组件内部实现细节 → ✅ 测试用户可见的行为和输出
- ❌ `expect(wrapper.vm.count).toBe(1)` 直接访问内部状态 → ✅ `expect(wrapper.text()).toContain('1')` 验证渲染结果
- ❌ 忘记 `await` 异步操作 → ✅ 所有 `trigger`、`setValue` 后都 `await`
- ❌ 在 `beforeEach` 中做复杂设置 → ✅ 工厂函数 `createWrapper()` 更灵活
- ❌ 覆盖率 100% 执念 → ✅ 关键路径覆盖 > 追求数字

```ts
// Factory function + edge cases + async + coverage
const createWrapper = (props = {}, slots = {}) =>
  mount(Counter, { props: { ...props }, slots, global: { plugins: [] } })

describe('Counter', () => {
  it.each([[-10, 'min'], [10, 'max']])(
    'emits limit at boundary %i',
    async (value, direction) => {
      const wrapper = createWrapper({ initialValue: value, [direction]: value })
      const btn = direction === 'max' ? 'increment' : 'decrement'
      await wrapper.find(`[data-testid="${btn}"]`).trigger('click')
      expect(wrapper.emitted('limit')![0]).toEqual([direction])
    }
  )
})
```
