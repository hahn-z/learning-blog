# 计算属性与侦听器

> 📊 **难度：初级** | 🏷️ Vue.js 3.x | ⏱️ 阅读约 15 分钟

---

## 📖 概念讲解

计算属性和侦听器是Vue中处理数据变化的两个核心机制，但它们的使用场景截然不同。

**计算属性 (computed)：**

计算属性是基于响应式数据自动缓存的派生值。只有当依赖数据变化时才会重新计算。

特点：
- **自动缓存**：依赖不变，不重新计算，直接返回缓存值
- **惰性求值**：只有在被访问时才计算
- **可写**：默认只读，但可以提供getter和setter
- **返回值**：一个ref，在模板中自动解包

**侦听器 (watch / watchEffect)：**

侦听器用于在数据变化时执行副作用（异步操作、DOM操作等）。

| 特性 | watch | watchEffect |
|------|-------|-------------|
| 指定监听 | 显式指定数据源 | 自动追踪依赖 |
| 旧值 | 可获取oldValue | 无法获取 |
| 首次执行 | 默认不执行 | 立即执行 |
| 精确控制 | 支持deep/immediate | 不支持选项 |
| 适用场景 | 对比新旧值 | 简单副作用 |

**何时用computed，何时用watch？**

- **computed**：需要根据现有数据派生新数据 → "算"
- **watch**：数据变化时需要做某事（请求API、操作DOM、定时器等） → "做"
- **原则**：能用computed就不用watch


## 🧠 知识脑图

```
计算属性与侦听器
├── computed（计算属性）
│   ├── 自动缓存
│   ├── 只读 vs 可写(getter+setter)
│   ├── vs methods（缓存区别）
│   └── 适用：派生数据
├── watch（侦听器）
│   ├── 监听ref / reactive / getter
│   ├── deep: true（深度监听）
│   ├── immediate: true（立即执行）
│   └── 适用：副作用（API/DOM/定时器）
├── watchEffect
│   ├── 自动追踪依赖
│   ├── 立即执行
│   ├── 返回stop函数
│   └── 适用：简单副作用
└── 最佳实践
    ├── computed优先
    ├── 避免watch中修改自身
    └── 清理副作用(onCleanup)
```

## 💻 完整代码

```vue
<!-- ComputedWatchDemo.vue -->
<template>
  <div class="demo">
    <h2>Computed Demo - Shopping Cart</h2>
    <div>
      <label>Price: <input v-model.number="price" type="number" /></label>
      <label>Quantity: <input v-model.number="quantity" type="number" /></label>
      <label>Discount: <input v-model.number="discount" type="number" step="0.1" /></label>
    </div>
    <!-- Computed properties displayed -->
    <p>Subtotal: ¥{{ subtotal }}</p>
    <p>Discount: -¥{{ discountAmount }}</p>
    <p><strong>Total: ¥{{ total }}</strong></p>

    <h2>Writable Computed - Full Name</h2>
    <input v-model="fullName" placeholder="Full Name" />
    <p>First: {{ firstName }}, Last: {{ lastName }}</p>

    <h2>Watch Demo - Search</h2>
    <input v-model="searchQuery" placeholder="Search (debounced)..." />
    <p>Searching: {{ searchQuery }}</p>
    <p>Results: {{ searchResults }}</p>
    <p>Search count: {{ searchCount }}</p>

    <h2>watchEffect Demo</h2>
    <input v-model="theme" placeholder="Theme color" />
    <p>Theme applied: {{ currentTheme }}</p>
  </div>
</template>

<script setup>
import { ref, computed, watch, watchEffect } from 'vue'

// === Computed Demo ===
const price = ref(100)
const quantity = ref(2)
const discount = ref(0.1)

// Read-only computed: calculate subtotal
const subtotal = computed(() => price.value * quantity.value)

// Computed chaining: discount based on subtotal
const discountAmount = computed(() => subtotal.value * discount.value)

// Final total
const total = computed(() => subtotal.value - discountAmount.value)

// === Writable Computed ===
const firstName = ref('San')
const lastName = ref('Zhang')

// Computed with getter and setter
const fullName = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (val) => {
    const parts = val.split(' ')
    firstName.value = parts[0] || ''
    lastName.value = parts.slice(1).join(' ') || ''
  }
})

// === Watch Demo ===
const searchQuery = ref('')
const searchResults = ref([])
const searchCount = ref(0)

// Watch with debounce simulation
watch(searchQuery, (newVal, oldVal, onCleanup) => {
  console.log(`Search: "${oldVal}" → "${newVal}"`)
  // Simulate debounce with cleanup
  const timer = setTimeout(() => {
    // Simulated search results
    searchResults.value = newVal
      ? [`Result 1 for "${newVal}"`, `Result 2 for "${newVal}"`]
      : []
    searchCount.value++
  }, 500)
  // Cleanup previous timer on next trigger
  onCleanup(() => clearTimeout(timer))
})

// Watch multiple sources
watch([price, quantity], ([newPrice, newQty], [oldPrice, oldQty]) => {
  console.log(`Cart changed: ${oldPrice}x${oldQty} → ${newPrice}x${newQty}`)
})

// === watchEffect Demo ===
const theme = ref('#42b883')
const currentTheme = ref('')

// watchEffect: auto-tracks dependencies
const stopEffect = watchEffect(() => {
  // Automatically tracks `theme.value` as dependency
  currentTheme.value = theme.value
  document.title = `Theme: ${theme.value}`
})

// Can stop the effect later if needed
// stopEffect()
</script>
```

## ⚠️ 注意事项

| 注意点 | 说明 | 建议 |
|--------|------|------|
| computed缓存 | 依赖不变则不重算 | 适合模板中多次使用的计算 |
| computed vs method | method每次渲染都执行 | 需要缓存用computed |
| watch的deep | 监听对象需要deep:true | 性能有开销，按需使用 |
| watch的immediate | 默认不立即执行 | 需要初始值时设immediate:true |
| watchEffect清理 | 组件卸载时自动停止 | 异步操作用onCleanup |
| 副作用位置 | 不要在computed中做异步操作 | 异步操作放watch中 |

## 🚫 避坑指南

### 1. computed中执行异步操作
❌ **错误：** `const data = computed(() => fetch('/api'))`（computed不能是异步的）
✅ **正确：** 用watch监听变化，在回调中调用API

### 2. 修改自己监听的数据
❌ **错误：** `watch(count, () => { count.value++ })`（无限循环！）
✅ **正确：** 避免在watch中修改被监听的源数据

### 3. 忘记deep选项
❌ **错误：** `watch(obj, handler)` → 监听reactive对象内部变化不触发
✅ **正确：** `watch(obj, handler, { deep: true })` 或监听getter函数

### 4. watch监听reactive对象的属性
❌ **错误：** `watch(() => obj.count, handler)` 用getter形式
✅ **正确：** 直接监听ref或用`watch(() => obj.count, handler)`

### 5. 滥用watch做计算
❌ **错误：** 用watch把a+b的结果赋给c
✅ **正确：** 用computed: `const c = computed(() => a.value + b.value)`

## 📝 知识点总结

```
计算属性与侦听器
├── computed
│   ├── 只读 computed(() => expr)
│   ├── 可写 computed({ get, set })
│   ├── 自动缓存 → 性能优化
│   └── 适用：派生数据
├── watch
│   ├── watch(source, callback, options)
│   ├── deep: true → 深度监听
│   ├── immediate: true → 首次执行
│   ├── onCleanup → 清理副作用
│   └── 适用：异步操作/副作用
├── watchEffect
│   ├── 自动追踪依赖
│   ├── 立即执行
│   └── 适用：简单副作用
└── 选择原则
    ├── 需要值 → computed
    ├── 需要做 → watch
    └── 能computed就不watch
```


## 🔄 举一反三

| 概念 | 生活类比 | 说明 |
|------|---------|------|
| computed | 计算器 | 输入不变，结果不变（缓存） |
| computed缓存 | 自动售货机 | 同一按钮不会重新调配已有饮料 |
| watch | 监控摄像头 | 目标变化时触发录像 |
| watchEffect | 哮喘预警器 | 自动感知环境变化并报警 |
| deep:true | 透视镜 | 看透表面，监控深层变化 |
| onCleanup | 垃圾回收 | 上一次的残留要清理干净 |

## 🚀 代码演进

### v1 - Options API中computed和watch

```vue
<script>
export default {
  data() { return { price: 100, qty: 2 } },
  computed: {
    total() { return this.price * this.qty }
  },
  watch: {
    total(newVal) { console.log('Total changed:', newVal) }
  }
}
</script>
```

### v2 - Composition API（setup函数）

```vue
<script>
import { ref, computed, watch } from 'vue'
export default {
  setup() {
    const price = ref(100)
    const qty = ref(2)
    const total = computed(() => price.value * qty.value)
    watch(total, (val) => console.log('Total:', val))
    return { price, qty, total }
  }
}
</script>
```

### v3 - script setup（推荐）

```vue
<script setup>
import { ref, computed, watchEffect } from 'vue'
const price = ref(100)
const qty = ref(2)
const total = computed(() => price.value * qty.value)
watchEffect(() => console.log('Total:', total.value))
</script>
```
