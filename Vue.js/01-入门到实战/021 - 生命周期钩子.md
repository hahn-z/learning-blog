> 🟢 **初级** — Vue 3 基础，了解 Composition API

## 概念讲解

### 什么是生命周期？

Vue 组件从**创建、挂载、更新到销毁**的整个过程称为生命周期。Vue 在每个阶段提供了**钩子函数**，让你能在特定时机执行自定义逻辑。

### Vue 3 生命周期钩子

| 钩子 | 时机 | 常见用途 |
|------|------|----------|
| `onMounted` | DOM 挂载完成 | 获取 DOM、发请求、初始化第三方库 |
| `onUpdated` | 响应式数据更新后 | 操作更新后的 DOM |
| `onUnmounted` | 组件卸载 | 清除定时器、取消订阅、销毁实例 |
| `onBeforeMount` | 挂载前 | 很少使用 |
| `onBeforeUpdate` | 更新前 | 获取更新前的 DOM 状态 |
| `onBeforeUnmount` | 卸载前 | 最后的清理机会 |

### 与 Vue 2 的区别

| Vue 2 | Vue 3 Composition API |
|-------|----------------------|
| `created` | `setup()` 本身 |
| `mounted` | `onMounted()` |
| `destroyed` | `onUnmounted()` |
| `beforeDestroy` | `onBeforeUnmount()` |

## 脑图

```
生命周期钩子
├── 执行顺序
│   ├── setup() ≈ created
│   ├── onBeforeMount
│   ├── onMounted ← 最常用
│   ├── onBeforeUpdate
│   ├── onUpdated
│   ├── onBeforeUnmount
│   └── onUnmounted ← 清理资源
├── 常见场景
│   ├── onMounted: API请求、DOM操作
│   ├── onUpdated: 依赖DOM的操作
│   └── onUnmounted: 清除副作用
├── 父子组件顺序
│   ├── 挂载: 父beforeMount → 子mounted → 父mounted
│   └── 卸载: 父beforeUnmount → 子unmounted → 父unmounted
└── 特殊场景
    ├── keep-alive: onActivated/onDeactivated
    ├── 异步组件: Suspense
    └── SSR: only onMounted在客户端执行
```

## 完整代码

```vue
<!-- components/LifecycleDemo.vue — Complete lifecycle demonstration -->
<script setup>
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted,
  ref
} from 'vue'

const count = ref(0)
let timer = null

onBeforeMount(() => {
  console.log('1. onBeforeMount — template about to render')
})

onMounted(() => {
  console.log('2. onMounted — DOM is ready')

  // Safe to access DOM
  const el = document.getElementById('counter-display')
  console.log('DOM element:', el?.textContent)

  // Start a timer (will be cleaned up in onUnmounted)
  timer = setInterval(() => {
    count.value++
  }, 1000)
})

onBeforeUpdate(() => {
  console.log('3. onBeforeUpdate — data changed, DOM not yet updated')
})

onUpdated(() => {
  console.log('4. onUpdated — DOM updated, count:', count.value)
})

onBeforeUnmount(() => {
  console.log('5. onBeforeUnmount — component about to destroy')
})

onUnmounted(() => {
  console.log('6. onUnmounted — component destroyed, cleaning up')
  // Critical: clean up timer to prevent memory leaks
  if (timer) {
    clearInterval(timer)
    timer = null
  }
})
</script>

<template>
  <div>
    <p id="counter-display">Count: {{ count }}</p>
    <button @click="count++">Manual +1</button>
  </div>
</template>
```

```vue
<!-- composables/useEventListener.js — Cleanup pattern -->
<script setup>
import { onUnmounted, ref } from 'vue'

// Composable that auto-cleans up
function useEventListener(target, event, callback) {
  target.addEventListener(event, callback)

  onUnmounted(() => {
    target.removeEventListener(event, callback)
  })
}

// Usage
const mouseX = ref(0)
const mouseY = ref(0)

useEventListener(window, 'mousemove', (e) => {
  mouseX.value = e.clientX
  mouseY.value = e.clientY
})
</script>
```

```vue
<!-- components/AutoRefresh.vue — Real-world pattern -->
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const data = ref(null)
const loading = ref(false)
let intervalId = null

async function fetchData() {
  loading.value = true
  try {
    const res = await fetch('/api/dashboard')
    data.value = await res.json()
  } catch (e) {
    console.error('Fetch failed:', e)
  } finally {
    loading.value = false
  }
}

// Mount: initial fetch + auto refresh
onMounted(() => {
  fetchData()
  intervalId = setInterval(fetchData, 30000) // Refresh every 30s
})

// Unmount: cleanup
onUnmounted(() => {
  if (intervalId) {
    clearInterval(intervalId)
    intervalId = null
  }
})
</script>

<template>
  <div v-if="loading">Loading...</div>
  <div v-else-if="data">
    <pre>{{ data }}</pre>
  </div>
</template>
```

## 执行预览

```
组件创建:
  1. setup() 执行（≈ created）
  2. onBeforeMount 触发
  3. onMounted 触发 → 开始定时器

count 变化:
  4. onBeforeUpdate 触发
  5. DOM 更新
  6. onUpdated 触发

组件销毁:
  7. onBeforeUnmount 触发
  8. onUnmounted 触发 → clearInterval(timer)
```

## 注意事项

| 注意点 | 说明 |
|--------|------|
| setup ≈ created | `setup()` 本身相当于 `created`，此时 DOM 不可用 |
| onMounted 发请求 | 大多数数据获取放在 onMounted 中 |
| 必须清理副作用 | 定时器、事件监听、WebSocket 必须在 onUnmounted 清除 |
| onUpdated 小心死循环 | 在 onUpdated 中修改响应式数据可能导致无限循环 |
| 钩子注册位置 | 必须在 `setup()` 同步代码中调用，不能放在 async 函数里 |
| SSR 场景 | 服务端只执行 setup 和 onBeforeMount，onMounted 只在客户端 |

## 避坑指南

| ❌ 错误做法 | ✅ 正确做法 |
|------------|-----------|
| 在 setup 中直接操作 DOM | 在 onMounted 中操作 DOM |
| 定时器/事件不在 onUnmounted 清理 | onUnmounted 中 clearInterval/removeEventListener |
| 在 onUpdated 中修改响应式数据 | onUpdated 只做读取，避免无限循环 |
| 钩子写在 async 函数内部 | 钩子必须在 setup 同步上下文中注册 |
| 忘记父 → 子的挂载顺序 | 子组件 onMounted 先于父组件 onMounted |

## 知识点总结

```
生命周期钩子
├── 核心
│   ├── onMounted — DOM就绪，发请求
│   ├── onUpdated — 更新后操作
│   └── onUnmounted — 清理资源
├── 模式
│   ├── 数据获取 → onMounted
│   ├── 事件监听 → onMounted + onUnmounted
│   ├── 定时器 → onMounted + onUnmounted
│   └── 第三方库 → onMounted init + onUnmounted destroy
├── 父子顺序
│   ├── 挂载: 子先 父后
│   └── 卸载: 子先 父后
└── 特殊
    ├── keep-alive: onActivated/onDeactivated
    └── setup ≈ created
```

## 举一反三

| 场景 | 钩子 | 模式 |
|------|------|------|
| 首次数据加载 | onMounted | fetch + render |
| 窗口 resize | onMounted + onUnmounted | addEventListener + cleanup |
| 轮询数据 | onMounted + onUnmounted | setInterval + cleanup |
| 图表初始化 | onMounted + onUnmounted | new Chart() + chart.destroy() |
| WebSocket | onMounted + onUnmounted | ws.connect() + ws.close() |
