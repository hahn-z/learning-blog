# 条件渲染与列表渲染

> 📊 **难度：初级** | 🏷️ Vue.js 3.x | ⏱️ 阅读约 15 分钟

---

## 📖 概念讲解

条件渲染和列表渲染是Vue模板中最常用的两大功能，控制"显示什么"和"显示多少"。

**条件渲染：v-if vs v-show**

| 特性 | v-if | v-show |
|------|------|--------|
| 切换方式 | 销毁/重建DOM元素 | CSS `display:none` |
| 初始渲染 | 条件为false时不渲染 | 始终渲染 |
| 切换开销 | 高（涉及DOM操作） | 低（仅CSS切换） |
| 初始开销 | 低（惰性渲染） | 高（总是渲染） |
| 配合使用 | v-else / v-else-if | 无 |
| 适用场景 | 条件很少变化时 | 频繁切换显示状态 |

**列表渲染核心要点：**

- `v-for`支持遍历数组、对象、数字范围
- **必须提供key属性**：帮助Vue高效更新列表
- key应该是**唯一且稳定**的标识符（如id），不要用index
- `v-for`和`v-if`不应在同一元素上使用（v-for优先级更高在Vue 3中）

**v-for的几种形式：**
```html
<!-- 数组 -->
<li v-for="item in items" :key="item.id">{{ item.name }}</li>
<li v-for="(item, index) in items" :key="item.id">{{ index }}: {{ item.name }}</li>
<!-- 对象 -->
<div v-for="(value, key) in obj" :key="key">{{ key }}: {{ value }}</div>
<!-- 数字范围 -->
<span v-for="n in 10" :key="n">{{ n }}</span>
```


## 🧠 知识脑图

```
条件渲染与列表渲染
├── 条件渲染
│   ├── v-if → 真正销毁/创建
│   ├── v-else-if → 多条件分支
│   ├── v-else → 兜底分支
│   ├── v-show → CSS显示隐藏
│   ├── template + v-if → 多元素条件
│   └── 选择：v-if(低频切换) vs v-show(高频切换)
├── 列表渲染
│   ├── 数组 v-for="item in list"
│   ├── 索引 v-for="(item, i) in list"
│   ├── 对象 v-for="(val, key) in obj"
│   ├── 范围 v-for="n in count"
│   └── template + v-for → 多元素循环
├── Key管理
│   ├── 必须唯一稳定
│   ├── 用id不用index
│   └── 影响diff算法效率
└── 性能优化
    ├── computed过滤
    ├── 虚拟滚动(大数据)
    └── v-once / v-memo
```

## 💻 完整代码

```vue
<!-- ConditionalListDemo.vue -->
<template>
  <div class="demo">
    <!-- 1. v-if / v-else-if / v-else -->
    <h2>Conditional Rendering</h2>
    <input v-model.number="score" type="number" min="0" max="100" />
    <p v-if="score >= 90">🏆 Excellent! Grade: A</p>
    <p v-else-if="score >= 80">👍 Good! Grade: B</p>
    <p v-else-if="score >= 60">✅ Pass! Grade: C</p>
    <p v-else>❌ Fail. Keep trying!</p>

    <!-- v-show for frequent toggle -->
    <button @click="showDetails = !showDetails">
      {{ showDetails ? 'Hide' : 'Show' }} Details
    </button>
    <div v-show="showDetails" class="details">
      This content uses v-show (CSS toggle, frequent switching)
    </div>

    <!-- template + v-if for multiple elements -->
    <template v-if="showDetails">
      <p>Detail line 1</p>
      <p>Detail line 2</p>
    </template>

    <!-- 2. v-for list rendering -->
    <h2>List Rendering - Task Manager</h2>
    <input v-model="newTask" @keyup.enter="addTask" placeholder="Add a task" />
    <button @click="addTask">Add</button>

    <!-- Filter buttons using computed -->
    <div class="filters">
      <button v-for="f in filters" :key="f.value"
        :class="{ active: currentFilter === f.value }"
        @click="currentFilter = f.value">
        {{ f.label }}
      </button>
    </div>

    <!-- Task list with v-for -->
    <ul>
      <li v-for="task in filteredTasks" :key="task.id"
        :class="{ done: task.done }">
        <input type="checkbox" v-model="task.done" />
        <span>{{ task.text }}</span>
        <button @click="removeTask(task.id)">✕</button>
      </li>
    </ul>
    <p>Total: {{ tasks.length }}, Done: {{ doneCount }}, Left: {{ leftCount }}</p>

    <!-- 3. v-for with objects -->
    <h2>Object Iteration</h2>
    <div v-for="(value, key) in userInfo" :key="key">
      <strong>{{ key }}:</strong> {{ value }}
    </div>

    <!-- 4. v-for range -->
    <h2>Star Rating</h2>
    <div>
      <span v-for="n in 5" :key="n"
        @click="rating = n"
        :style="{ cursor: 'pointer', opacity: n <= rating ? 1 : 0.3 }">
        ⭐
      </span>
      <span> {{ rating }}/5</span>
    </div>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

// --- Conditional rendering ---
const score = ref(85)
const showDetails = ref(false)

// --- List rendering ---
let nextId = 4
const newTask = ref('')
const tasks = ref([
  { id: 1, text: 'Learn Vue 3 basics', done: true },
  { id: 2, text: 'Build a project', done: false },
  { id: 3, text: 'Deploy to production', done: false }
])

const filters = [
  { value: 'all', label: 'All' },
  { value: 'active', label: 'Active' },
  { value: 'done', label: 'Done' }
]
const currentFilter = ref('all')

// Computed: filter tasks based on current filter
const filteredTasks = computed(() => {
  switch (currentFilter.value) {
    case 'active': return tasks.value.filter(t => !t.done)
    case 'done': return tasks.value.filter(t => t.done)
    default: return tasks.value
  }
})

const doneCount = computed(() => tasks.value.filter(t => t.done).length)
const leftCount = computed(() => tasks.value.filter(t => !t.done).length)

function addTask() {
  const text = newTask.value.trim()
  if (!text) return
  tasks.value.push({ id: nextId++, text, done: false })
  newTask.value = ''
}

function removeTask(id) {
  tasks.value = tasks.value.filter(t => t.id !== id)
}

// --- Object iteration ---
const userInfo = ref({
  name: 'Vue Developer',
  level: 'Intermediate',
  framework: 'Vue 3',
  experience: '2 years'
})

// --- Star rating ---
const rating = ref(3)
</script>

<style scoped>
.done span { text-decoration: line-through; color: #999; }
.filters .active { background: #42b883; color: white; }
.details { padding: 0.5rem; background: #f0f0f0; margin: 0.5rem 0; }
</style>
```

## ⚠️ 注意事项

| 注意点 | 说明 | 建议 |
|--------|------|------|
| key必须唯一 | 重复key导致渲染异常 | 使用数据的唯一id |
| 不要用index作key | 列表增删时index变化 | 用item.id等稳定标识 |
| v-for+v-if | Vue 3中v-if优先级更高 | 先computed过滤再v-for |
| v-show不配合template | template不是真实DOM | 多元素用v-if+template |
| 数组响应性 | Vue 3中数组方法全支持 | 直接赋值也支持（Proxy） |
| 大列表性能 | 超过1000条考虑虚拟滚动 | 使用vue-virtual-scroller |

## 🚫 避坑指南

### 1. 用index作为key
❌ **错误：** `:key="index"`（列表增删时状态错乱）
✅ **正确：** `:key="item.id"`（使用唯一稳定标识）

### 2. v-for和v-if同一元素
❌ **错误：** `<li v-for="item in list" v-if="item.active">`
✅ **正确：** 用computed先过滤：`<li v-for="item in activeList">`

### 3. 直接修改数组长度
❌ **错误：** `arr.length = 0`（Vue 2不响应，Vue 3可以但不推荐）
✅ **正确：** `arr.splice(0)` 或 `arr.value = []`

### 4. v-show在组件上
❌ **错误：** 在`<template>`上用`v-show`（template不是真实DOM）
✅ **正确：** 用`v-if`或包裹在真实DOM元素中

### 5. 忘记template可以当不可见容器
❌ **错误：** 用div包裹导致多余DOM节点
✅ **正确：** `<template v-for="item in list">` 不会产生额外DOM

## 🔄 举一反三

| 概念 | 生活类比 | 说明 |
|------|---------|------|
| v-if | 拆房/建房 | 需要时建，不需要时拆（开销大） |
| v-show | 百叶窗 | 房子一直在，只是遮住/露出 |
| v-for | 复印机 | 一个模板复印出多份 |
| key | 身份证号 | 唯一标识，帮Vue找到"谁是谁" |
| index作key | 用排队号做名字 | 有人插队后号码全错 |
| computed过滤 | 筛子 | 先筛再展示，而不是展示后再挑 |
