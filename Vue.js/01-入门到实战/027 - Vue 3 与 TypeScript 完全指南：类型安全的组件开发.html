概念讲解

为什么Vue 3 + TypeScript？

Vue 3的源码本身就是用TypeScript重写的，从底层提供了完整的类型支持。与Vue 2 + vue-class-component 的"外挂"方式不同，Vue 3的TS集成是原生级别的。

核心类型工具

工具 用途
defineProps() Props类型定义
defineEmits() Emits类型定义
ref() 指定ref的值类型
computed() 指定计算属性返回类型
PropType 复杂Props类型

脑图

Vue 3 + TypeScript
├── 组件类型
│   ├── defineProps<{}>()
│   ├── defineEmits<{}>()
│   └── withDefaults()
├── 响应式类型
│   ├── Ref<T>

│   └── computed<T>()
└── 常见类型
    ├── PropType<T> 复杂props
    └── ComponentPublicInstance

完整代码示例

<!-- TypeScriptDemo.vue -->
<template>
  <div class="ts-demo">
    <h2>Vue 3 + TypeScript 类型演示</h2>

    <section>
      <h3>1. Props 类型系统</h3>
      <p>用户: {{ user.name }} ({{ user.role }})</p>
    </section>

    <section>
      <h3>2. Emits 类型约束</h3>
      <button @click="handleUpdate">触发更新</button>
      <p>{{ emitLog }}</p>
    </section>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

interface User {
  name: string
  age: number
  role: 'admin' | 'editor' | 'viewer'
}

const props = withDefaults(defineProps<{
  user: User
  tags?: string[]
}>(), {
  tags: () => ['Vue', 'TypeScript']
})

const emit = defineEmits<{
  update: [id: number, data: Partial<User>]
  delete: [id: number]
}>()

const emitLog = ref('等待操作...')

function handleUpdate() {
  emit('update', 1, { name: 'Updated' })
  emitLog.value = '已触发 update 事件'
}
</script>

注意事项

场景 注意点 解决方案
reactive类型 自动推导可能不够精确 显式标注 reactive(...)
ref初始null ref(null) 类型为 Ref 用 `ref null>(null)`
Props默认值 defineProps 泛型形式无默认值 用 withDefaults() 包装

避坑指南

❌ Props 不用 withDefaults

// ❌ 泛型props没有默认值
defineProps<{ count?: number }>()
// ✅ 用 withDefaults 设置默认值
withDefaults(defineProps<{ count?: number }>(), {
  count: 0
})
