018 - 动态路由与嵌套路由

📂 Vue.js入门到实战 🏷️ Vue.js

056-动态路由与嵌套路由

📌 难度标注：⭐⭐⭐ 中级（需要理解 Vue Router 基础配置）

一、概念讲解

动态路由

动态路由通过在路径中使用动态参数（:param）匹配多个相似路由。例如 /user/:id 可以匹配 /user/1、/user/42 等。

关键特性：

  • 参数获取：通过 useRoute().params 访问
  • 参数变化监听：同组件间跳转时组件会复用，需要手动监听参数变化
  • 可选参数：/user/:id? 中 ? 表示参数可选
  • 正则约束：/user/:id(\d+) 限制 id 只能是数字
  • 通配符：/:pathMatch(.) 匹配所有未命中的路径（404页面）

    • 嵌套路由

    嵌套路由允许在组件内部嵌套 ，实现页面布局的层级嵌套。例如后台管理系统中，侧边栏固定、内容区根据子路由变化。

    命名视图

    同一层级同时展示多个 ，通过 name 属性区分，适合复杂布局。

    二、脑图

    动态路由与嵌套路由
├── 动态路由
│   ├── 基础参数  /user/:id
│   ├── 多参数    /post/:category/:id
│   ├── 可选参数  /user/:id?
│   ├── 正则约束  /user/:id(\d+)
│   ├── 通配符    /:pathMatch(.*)*
│   └── 参数获取  useRoute().params
├── 嵌套路由
│   ├── children 配置
│   ├── 嵌套 <router-view>
│   ├── 默认子路由
│   └── 深层嵌套 (3+ levels)
├── 命名视图
│   ├── <router-view name="header">
│   ├── components 配置 (复数)
│   └── 嵌套命名视图
└── 编程式导航
    ├── router.push()
    ├── router.replace()
    └── router.go() / back() / forward()

    三、完整代码

    // router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const AdminLayout = () => import('../views/admin/Layout.vue')
const DashboardHome = () => import('../views/admin/Dashboard.vue')
const UserList = () => import('../views/admin/UserList.vue')
const UserDetail = () => import('../views/admin/UserDetail.vue')
const UserEdit = () => import('../views/admin/UserEdit.vue')
const PostDetail = () => import('../views/admin/PostDetail.vue')
const Settings = () => import('../views/admin/Settings.vue')
const NotFound = () => import('../views/NotFound.vue')

const routes = [
  {
    path: '/',
    component: () => import('../views/Home.vue')
  },
  {
    path: '/admin',
    component: AdminLayout,
    children: [
      {
        path: '',           // /admin
        name: 'AdminHome',
        component: DashboardHome
      },
      {
        path: 'users',      // /admin/users
        name: 'UserList',
        component: UserList
      },
      {
        path: 'users/:id',  // /admin/users/:id
        name: 'UserDetail',
        component: UserDetail,
        children: [
          {
            path: '',        // /admin/users/:id
            name: 'UserProfile',
            component: () => import('../views/admin/UserProfile.vue')
          },
          {
            path: 'edit',   // /admin/users/:id/edit
            name: 'UserEdit',
            component: UserEdit
          }
        ]
      },
      {
        path: 'posts/:category/:id', // Multiple dynamic params
        name: 'PostDetail',
        component: PostDetail
      },
      {
        path: 'settings',
        name: 'Settings',
        component: Settings
      }
    ]
  },
  {
    // Regex constraint: id must be numeric
    path: '/article/:id(\d+)',
    name: 'Article',
    component: () => import('../views/Article.vue')
  },
  {
    // Named views example
    path: '/blog/:id',
    name: 'BlogPost',
    components: {
      default: () => import('../views/BlogContent.vue'),
      sidebar: () => import('../views/BlogSidebar.vue'),
      footer: () => import('../views/BlogFooter.vue')
    }
  },
  {
    // Catch-all 404
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    component: NotFound
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router

    四、执行预览

    URL结构示例：

/admin                     -> Layout + DashboardHome
/admin/users               -> Layout + UserList
/admin/users/42            -> Layout + UserDetail + UserProfile
/admin/users/42/edit       -> Layout + UserDetail + UserEdit
/admin/posts/vue/123       -> Layout + PostDetail (category=vue, id=123)
/article/999               -> Article (id must be numeric)
/article/abc               -> NotFound (regex fails)
/anything/else             -> NotFound (catch-all)

    五、注意事项

    场景 注意事项 推荐做法
    组件复用 /user/1->/user/2 组件不重建 watch(() => route.params)
    404路由 必须放在路由配置最后 使用 /:pathMatch(.)
    嵌套层级 避免超过3层 过深时考虑扁平化
    参数类型 params 拿到的都是字符串 手动 parseInt()
    正则转义 字符串中需要双重转义 仔细测试正则
    命名视图 components 是复数 components: { default: A, sidebar: B }

    六、避坑指南

    ❌ 坑1：组件复用时数据不更新

    <!-- ❌ onMounted 只执行一次 -->
<script setup>
import { onMounted } from 'vue'
import { useRoute } from 'vue-router'
const route = useRoute()
onMounted(() => {
  fetchUser(route.params.id) // /user/1->/user/2 won't trigger
})
</script>

<!-- ✅ 使用 watch 监听参数变化 -->
<script setup>
import { watch } from 'vue'
import { useRoute } from 'vue-router'
const route = useRoute()
watch(() => route.params.id, (newId) => {
  fetchUser(newId)
}, { immediate: true })
</script>

    ❌ 坑2：子路由 path 加了前导斜杠

    // ❌ 绝对路径，不再是嵌套子路由
children: [
  { path: '/users', component: UserList } // matches /users, not /admin/users
]

// ✅ 相对路径
children: [
  { path: 'users', component: UserList } // matches /admin/users
]

    ❌ 坑3：编程式导航 params 被忽略

    // ❌ path + params 无效（Vue Router 4）
router.push({ path: '/user', params: { id: 1 } })

// ✅ 使用 name 传递 params
router.push({ name: 'UserDetail', params: { id: 1 } })

    八、知识点总结

    动态路由与嵌套路由
├── 动态路由参数
│   ├── :param        -> 单个参数
│   ├── :param?       -> 可选参数
│   ├── :param(\d+)   -> 正则约束
│   └── :pathMatch(.*)* -> 通配符
├── 嵌套路由
│   ├── children 数组
│   ├── 父组件 <router-view>
│   └── 默认子路由 path: ''
├── 命名视图
│   ├── components (复数)
│   └── <router-view name="xxx">
└── 关键API
    ├── useRoute()  -> 当前路由信息
    ├── useRouter() -> 编程式导航
    └── watch(route.params) -> 参数变化监听

    九、举一反三

    技术 场景 实际应用
    动态路由 RESTful URL /api/post/:id
    嵌套路由 后台管理菜单 系统->用户->详情
    命名视图 复杂布局 CMS：头部+主体+侧边栏
    正则约束 参数校验 /article/:id(\d+)
    通配符404 兜底页面 未匹配URL显示404
    watch params 组件复用刷新 列表点击不同项
