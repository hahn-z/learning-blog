# 055-路由守卫与元信息

> 📌 **难度标注**：⭐⭐⭐ 中级（需要理解Vue Router基础与异步操作）

## 一、概念讲解

路由守卫（Navigation Guards）是 Vue Router 提供的路由跳转控制机制，允许你在路由发生跳转前、后执行自定义逻辑。常见的使用场景包括：

- **登录鉴权**：未登录用户不能访问特定页面
- **权限控制**：不同角色访问不同路由
- **数据预加载**：在进入页面前获取必要数据
- **页面标题管理**：根据路由动态修改 document.title
- **离开确认**：表单未保存时提醒用户

### 三类守卫

| 类型 | 定义位置 | 说明 |
|------|---------|------|
| 全局守卫 | router 实例上 | 所有路由跳转都会触发 |
| 路由独享守卫 | 路由配置中 | 只在进入特定路由时触发 |
| 组件内守卫 | 组件选项中 | 在组件内部定义，与组件生命周期相关 |

### 路由元信息（Meta）

每个路由可以配置 `meta` 字段，存放自定义数据，如权限标识、页面标题等。守卫中通过 `to.meta` 读取。

## 二、脑图

```
路由守卫与元信息
├── 全局守卫
│   ├── beforeEach  -- 每次跳转前
│   ├── beforeResolve -- 导航确认前
│   └── afterEach   -- 每次跳转后
├── 路由独享守卫
│   └── beforeEnter -- 进入该路由前
├── 组件内守卫
│   ├── onBeforeRouteEnter  -- 进入组件前
│   ├── onBeforeRouteUpdate -- 路由参数变化时
│   └── onBeforeRouteLeave  -- 离开组件前
└── 路由元信息 meta
    ├── requiresAuth  -- 是否需要登录
    ├── title         -- 页面标题
    ├── roles         -- 允许的角色列表
    └── keepAlive     -- 是否缓存组件
```

## 三、完整代码

```javascript
// router/index.js
import { createRouter, createWebHistory } from 'vue-router'

// Simulated auth store
const authStore = {
  isLoggedIn: false,
  userRole: 'guest', // 'guest' | 'user' | 'admin'
  login(role = 'user') {
    this.isLoggedIn = true
    this.userRole = role
  },
  logout() {
    this.isLoggedIn = false
    this.userRole = 'guest'
  }
}

// Lazy-loaded route components
const Home = () => import('../views/Home.vue')
const About = () => import('../views/About.vue')
const Login = () => import('../views/Login.vue')
const Dashboard = () => import('../views/Dashboard.vue')
const Admin = () => import('../views/Admin.vue')
const Editor = () => import('../views/Editor.vue')
const Forbidden = () => import('../views/Forbidden.vue')
const UserProfile = () => import('../views/UserProfile.vue')

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home,
    meta: { title: '首页' }
  },
  {
    path: '/about',
    name: 'About',
    component: About,
    meta: { title: '关于我们' }
  },
  {
    path: '/login',
    name: 'Login',
    component: Login,
    meta: { title: '登录', guestOnly: true }
  },
  {
    path: '/dashboard',
    name: 'Dashboard',
    component: Dashboard,
    meta: { title: '控制台', requiresAuth: true }
  },
  {
    path: '/admin',
    name: 'Admin',
    component: Admin,
    meta: { title: '管理后台', requiresAuth: true, roles: ['admin'] }
  },
  {
    path: '/editor',
    name: 'Editor',
    component: Editor,
    meta: { title: '编辑器', requiresAuth: true }
  },
  {
    path: '/user/:id',
    name: 'UserProfile',
    component: UserProfile,
    meta: { title: '用户主页', requiresAuth: true },
    // Route-level guard: validate user ID format
    beforeEnter: (to, from) => {
      const id = parseInt(to.params.id)
      if (isNaN(id) || id < 1) {
        return { name: 'Home' }
      }
    }
  },
  {
    path: '/403',
    name: 'Forbidden',
    component: Forbidden,
    meta: { title: '无权限' }
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// --- Global beforeEach guard ---
router.beforeEach((to, from) => {
  // Update page title
  document.title = to.meta.title || 'Vue App'

  // Check if route requires authentication
  if (to.meta.requiresAuth && !authStore.isLoggedIn) {
    return { name: 'Login', query: { redirect: to.fullPath } }
  }

  // Check role-based access
  if (to.meta.roles && !to.meta.roles.includes(authStore.userRole)) {
    return { name: 'Forbidden' }
  }

  // Already logged in? Redirect away from guest-only pages
  if (to.meta.guestOnly && authStore.isLoggedIn) {
    return { name: 'Dashboard' }
  }
})

// --- Global afterEach guard ---
router.afterEach((to, from) => {
  console.log(`Navigated: ${from.path} -> ${to.path}`)
  window.scrollTo(0, 0)
})

export { router, authStore }
```

```vue
<!-- views/Editor.vue - Component-level guard -->
<template>
  <div class="editor">
    <h1>文章编辑器</h1>
    <textarea v-model="content" placeholder="写点什么..." rows="10" cols="50" />
    <div class="actions">
      <button @click="handleSave">保存</button>
      <button @click="$router.back()">返回</button>
    </div>
    <p v-if="saved" class="saved-msg">已保存</p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { onBeforeRouteLeave } from 'vue-router'

const content = ref('')
const saved = ref(false)

const handleSave = () => { saved.value = true }

// Warn user before leaving with unsaved changes
onBeforeRouteLeave((to, from) => {
  if (content.value && !saved.value) {
    if (!window.confirm('你有未保存的内容，确定要离开吗？')) {
      return false // Cancel navigation
    }
  }
})
</script>
```

```vue
<!-- views/UserProfile.vue - Route update guard -->
<template>
  <div class="profile">
    <h1>用户 {{ userId }} 的主页</h1>
    <p>用户名：{{ userName }}</p>
    <nav>
      <router-link :to="'/user/' + (Number(userId) - 1)">上一个</router-link>
      <router-link :to="'/user/' + (Number(userId) + 1)">下一个</router-link>
    </nav>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useRoute, onBeforeRouteUpdate } from 'vue-router'

const route = useRoute()
const userId = ref(route.params.id)
const userName = ref('')

const fetchUser = async (id) => {
  userName.value = 'user_' + id
}

// Triggered when same component, different params
onBeforeRouteUpdate(async (to) => {
  if (to.params.id !== userId.value) {
    userId.value = to.params.id
    await fetchUser(to.params.id)
  }
})

fetchUser(userId.value)
</script>
```

## 四、执行预览

```
用户操作流程：

1. 访问 /admin（未登录）
   -> beforeEach 拦截 -> 跳转 /login?redirect=/admin
   -> 登录后自动跳回 /admin

2. 以普通用户访问 /admin
   -> beforeEach 检测角色不符 -> 跳转 /403

3. 在编辑器页面输入内容后点返回
   -> onBeforeRouteLeave 弹出确认框
   -> 确认离开 or 取消留下

4. 在 /user/1 页面点击"下一个用户"
   -> onBeforeRouteUpdate 触发，组件不销毁
   -> 数据更新为 user/2 的信息
```

## 五、注意事项

| 场景 | 注意事项 | 推荐做法 |
|------|---------|---------|
| 多个全局守卫 | 按注册顺序执行 | 集中在一个 beforeEach 中处理 |
| 守卫中异步操作 | 必须返回 Promise | 使用 async 函数 |
| next() 函数 | Vue Router 4 推荐 return 替代 | return false 取消，return 路由跳转 |
| onBeforeRouteEnter | 无法访问组件实例 | 在 onMounted 中处理初始化 |
| 循环重定向 | A守卫跳B，B守卫跳A | 确保守卫逻辑有终止条件 |
| meta 字段合并 | 嵌套路由 meta 不自动合并 | 遍历 matched 数组合并 |

## 六、避坑指南

### ❌ 坑1：忘记 return 导致导航卡住

```javascript
// ❌ 什么都没返回，导航行为不可预期
router.beforeEach((to, from) => {
  if (to.meta.requiresAuth && !isLoggedIn) {
    router.push('/login')  // Direct push while navigation is pending
  }
})

// ✅ return 路由对象来中断当前导航
router.beforeEach((to, from) => {
  if (to.meta.requiresAuth && !isLoggedIn) {
    return { name: 'Login' }
  }
})
```

### ❌ 坑2：onBeforeRouteEnter 中访问 this

```javascript
// ❌ 组件还没创建，this 是 undefined
onBeforeRouteEnter((to, from) => {
  this.loadData() // TypeError!
})

// ✅ 在 onMounted 中处理初始化逻辑
onMounted(() => {
  loadData()
})
```

### ❌ 坑3：嵌套路由 meta 没有合并

```javascript
// ❌ 只检查 to.meta，子路由覆盖父路由
if (to.meta.requiresAuth) { ... }

// ✅ 遍历所有匹配的路由记录
if (to.matched.some(record => record.meta.requiresAuth)) { ... }
```

## 七、练习题

### 🟢 入门：基础登录拦截
实现一个全局前置守卫，未登录时所有页面跳转到 `/login`，登录后可正常访问。

### 🔴 挑战：数据预加载 + 进度条

## 八、知识点总结

```
路由守卫
├── 执行顺序
│   ├── beforeEach -> beforeEnter -> beforeRouteEnter
│   ├── beforeResolve -> afterEach
│   └── 离开时：beforeRouteLeave -> beforeEach -> ...
├── 导航解析流程（完整10步）
│   1. 导航被触发
│   2. 失活组件 beforeRouteLeave
│   3. 全局 beforeEach
│   4. 重用组件 beforeRouteUpdate
│   5. 路由配置 beforeEnter
│   6. 解析异步路由组件
│   7. 激活组件 beforeRouteEnter
│   8. 全局 beforeResolve
│   9. 导航确认
│   10. 全局 afterEach + DOM更新
├── 返回值
│   ├── undefined/true -> 放行
│   ├── false -> 取消
│   ├── 路由对象 -> 重定向
│   └── Promise -> 等待解析
└── meta 元信息
    ├── 定义在 routes 配置中
    ├── 嵌套路由需手动合并 matched
    └── 常用于权限、标题、缓存标识
```

## 九、举一反三

| 守卫类型 | 适用场景 | 示例 |
|---------|---------|------|
| beforeEach | 登录检查、权限验证 | 未登录跳转登录页 |
| beforeResolve | 确保所有数据预加载完成 | 等待异步钩子完成 |
| afterEach | 页面统计、滚动复位 | 发送 PV 统计 |
| beforeEnter | 路由级独立验证 | 验证参数格式 |
| onBeforeRouteLeave | 离开确认 | 表单未保存提醒 |
| onBeforeRouteUpdate | 同组件参数变化 | /user/1 -> /user/2 |

### v1：基础登录拦截
```javascript
router.beforeEach((to, from, next) => {
  if (to.path !== '/login' && !isLoggedIn) {
    next('/login')
  } else {
    next()
  }
})
```

### v3：完整方案 + 嵌套路由 + 异步
```javascript
import NProgress from 'nprogress'

router.beforeEach(async (to) => {
  NProgress.start()
  const requiresAuth = to.matched.some(r => r.meta.requiresAuth)
  const roles = to.matched.flatMap(r => r.meta.roles || [])
  if (requiresAuth) {
    const user = await getCurrentUser()
    if (!user) return { name: 'Login', query: { redirect: to.fullPath } }
    if (roles.length && !roles.includes(user.role)) return { name: 'Forbidden' }
  }
})
router.afterEach(() => NProgress.done())
```

> 💡 **总结**：路由守卫是 Vue Router 的核心安全机制。掌握三类守卫的执行顺序、正确使用 return 替代 next()、以及通过 meta 实现声明式权限控制，是构建健壮前端应用的基础。
