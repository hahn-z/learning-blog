一、概念讲解

Interface（接口）

接口是TypeScript中描述对象形状的主要方式。它定义了一个对象应该有哪些属性、什么类型。

interface User {
  id: number;
  name: string;
  email: string;
}

Type Alias（类型别名）

类型别名可以给任何类型起一个新名字，不仅限于对象。

type UserID = number | string;
type Point = { x: number; y: number };

核心区别

特性 interface type
对象形状 ✅ ✅
联合/交叉类型 ❌ ✅
声明合并 ✅ 自动合并 ❌
extends继承 ✅ 用 & 交叉
基本类型别名 ❌ ✅
计算属性 ❌ ✅

选择建议： 对象形状用interface，其他场景用type。

三、完整代码示例

// ============================================
// Interfaces and Type Aliases - Complete Guide
// ============================================

// --- 1. Basic Interface ---
interface User {
  id: number;
  name: string;
  email: string;
  avatar?: string;           // optional property
  readonly createdAt: Date;  // immutable after creation
}

function createUser(user: User): User {
  return { ...user, createdAt: user.createdAt ?? new Date() };
}

const alice = createUser({
  id: 1,
  name: 'Alice',
  email: 'alice@example.com',
  createdAt: new Date()
});

// --- 2. Interface Inheritance ---
interface Animal {
  name: string;
  age: number;
}

interface Pet extends Animal {
  owner: string;
}

interface Dog extends Pet {
  breed: string;
  bark(): void;
}

const myDog: Dog = {
  name: 'Rex',
  age: 3,
  owner: 'Alice',
  breed: 'Labrador',
  bark() { console.log('Woof!'); }
};

// Multiple inheritance
interface Flyable {
  fly(): void;
}

interface Swimmable {
  swim(): void;
}

interface Duck extends Animal, Flyable, Swimmable {
  quack(): void;
}

// --- 3. Declaration Merging (interface only) ---
interface Window {
  myCustomProp: string;
}

// This MERGES with the above declaration
interface Window {
  anotherProp: number;
}

// Window now has both myCustomProp and anotherProp

// --- 4. Index Signatures ---
interface Dictionary<T> {
  [key: string]: T;
}

const wordCount: Dictionary<number> = {
  hello: 5,
  world: 10
};

interface Config {
  [key: string]: string | number | boolean;
  version: string;  // specific prop must match index signature type
  debug: boolean;
}

// --- 5. Type Alias ---

// Type for primitives
type ID = string | number;

// Type for objects (similar to interface)
type Point = {
  x: number;
  y: number;
  distance(): number;
};

const origin: Point = {
  x: 0,
  y: 0,
  distance() { return Math.sqrt(this.x ** 2 + this.y ** 2); }
};

// Union types (only type can do this)
type Status = 'active' | 'inactive' | 'pending';
type Result<T> = { success: true; data: T } | { success: false; error: string };

function fetchUser(): Result<User> {
  return { success: true, data: alice };
}

// Intersection types (equivalent to extends)
type Timestamped = {
  createdAt: Date;
  updatedAt: Date;
};

type Entity = User & Timestamped;

// Mapped types (only type can do this)
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

type Partial<T> = {
  [K in keyof T]?: T[K];
};

// Computed property keys (only type)
type EventMap<T extends string> = {
  [K in `on${Capitalize<T>}`]: () => void;
};

type MouseEvents = EventMap<'click' | 'hover'>;
// { onClick: () => void; onHover: () => void }

// --- 6. Callable & Constructable ---
interface SearchFunc {
  (source: string, term: string): boolean;
}

const mySearch: SearchFunc = (src, term) => src.includes(term);
console.log(mySearch('hello world', 'world')); // true

// --- 7. Practical Pattern: Branded Types ---
type UserId = string & { __brand: 'UserId' };
type OrderId = string & { __brand: 'OrderId' };

function createUserId(id: string): UserId {
  return id as UserId;
}

function getUser(id: UserId): void { console.log(`User: ${id}`); }
function getOrder(id: OrderId): void { console.log(`Order: ${id}`); }

const uid = createUserId('u123');
getUser(uid);     // OK
// getOrder(uid); // Error! Type safety between IDs

console.log('All interface & type examples completed!');

五、注意事项

注意点 说明
interface可以被class implements type也可以，但如果包含联合类型则不行
声明合并是把双刃剑 库作者用它可以扩展类型，但也可能意外合并
索引签名会限制所有属性 所有已知属性的类型必须是签名类型的子集
readonly是浅层只读 readonly T[]不阻止修改数组内部对象
type不能重新声明 同名type会报错，interface会合并

六、避坑指南

❌ 常见错误 ✅ 正确做法
用interface定义联合类型 用type定义联合/交叉/条件类型
忽略声明合并 扩展第三方类型时用interface，避免意外合并
readonly嵌套不深 用Readonly工具类型做深层只读
?和undefined混用 prop?: T等于`prop: T \ undefined`
索引签名太宽 尽量用具体属性，索引签名作为补充

            2. 八、知识点总结

            接口与类型别名
├── Interface
│   ├── 属性签名
│   ├── 可选属性 (?:)
│   ├── 只读 (readonly)
│   ├── 索引签名 [key: K]: T
│   ├── 函数签名
│   ├── extends 继承
│   └── 声明合并
├── Type Alias
│   ├── 基本类型别名
│   ├── 联合类型 (A | B)
│   ├── 交叉类型 (A & B)
│   ├── 条件类型 (T extends U ? X : Y)
│   ├── 映射类型 ({[K in keyof T]: V})
│   └── 模板字面量类型
├── 共同能力
│   ├── 描述对象形状
│   ├── 泛型
│   └── class implements
└── 选择策略
    ├── 对象/API → interface
    ├── 联合/工具/计算 → type
    └── 不确定 → interface（更灵活）

            九、举一反三

            场景 interface type
            React组件Props ✅ 可扩展 也可以
            API响应类型 ✅ 可合并 用联合更灵活
            Redux Action ❌ 需要联合 ✅ type更合适
            事件类型映射 ❌ ✅ 映射类型
            类implements ✅ 更自然 也可以
            工具类型 ❌ ✅ 映射/条件

              • 十一、代码演进

              v1 — 松散的对象定义

              function processUser(user: any) {
  console.log(user.name);
}

              v2 — 添加interface

              interface User {
  name: string;
  email: string;
}
function processUser(user: User) {
  console.log(user.name);
}

              v3 — 泛型+高级类型

              interface Entity {
  id: string;
  createdAt: Date;
  updatedAt: Date;
}

type CreateInput<T extends Entity> = Omit<T, 'id' | 'createdAt' | 'updatedAt'>;
type UpdateInput<T extends Entity> = Partial<CreateInput<T>> & { id: string };

interface User extends Entity {
  name: string;
  email: string;
  role: 'admin' | 'user';
}

function createUser(input: CreateInput<User>): User {
  return { ...input, id: crypto.randomUUID(), createdAt: new Date(), updatedAt: new Date() };
}

              总结： interface和type各有所长。对象形状优先用interface（可扩展），联合/工具类型用type。掌握两者区别，在合适的场景选合适的工具。
