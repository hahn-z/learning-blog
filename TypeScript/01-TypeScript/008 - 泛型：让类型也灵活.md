# 035-泛型：让类型也灵活

> **难度：** 🟡 进阶 | **阅读时间：** 22分钟 | **前置知识：** 函数类型与重载

---

## 一、概念讲解

### 什么是泛型（Generics）？

泛型让类型变成**参数**——就像函数接受值参数一样，泛型接受**类型参数**。

```typescript
// Without generics: one function per type
function identityString(val: string): string { return val; }
function identityNumber(val: number): number { return val; }

// With generics: one function for all types
function identity<T>(val: T): T { return val; }
```

### 泛型的三个使用场景

1. **泛型函数** — `function fn<T>(arg: T): T`
2. **泛型接口** — `interface Container<T> { value: T }`
3. **泛型类** — `class Stack<T> { ... }`

### 泛型约束（Constraints）

```typescript
// T must have a 'length' property
function logLength<T extends { length: number }>(arg: T): number {
  console.log(arg.length);
  return arg.length;
}

logLength('hello');     // OK, string has .length
logLength([1, 2, 3]);   // OK, array has .length
// logLength(123);      // Error, number has no .length
```

### keyof 和泛型

```typescript
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: 'Alice', age: 30 };
getProperty(user, 'name'); // type: string
// getProperty(user, 'job'); // Error: 'job' not in keyof user
```

## 三、完整代码示例

```typescript
// ============================================
// Generics - Complete Guide
// ============================================

// --- 1. Generic Functions ---
function identity<T>(value: T): T {
  return value;
}

const num = identity(42);        // type: number
const str = identity('hello');   // type: string

// Multiple type parameters
function merge<T, U>(a: T, b: U): T & U {
  return { ...a, ...b };
}

const merged = merge({ name: 'Alice' }, { age: 30 });
// type: { name: string } & { age: number }

// --- 2. Generic Constraints ---
interface HasId {
  id: string | number;
}

function findById<T extends HasId>(items: T[], id: T['id']): T | undefined {
  return items.find(item => item.id === id);
}

// keyof constraint
function pluck<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

// --- 3. Generic Interfaces ---
interface Repository<T> {
  findAll(): T[];
  findById(id: string): T | undefined;
  create(item: Omit<T, 'id'>): T;
  update(id: string, item: Partial<T>): T;
  delete(id: string): boolean;
}

// API Response pattern
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
  timestamp: Date;
}

function success<T>(data: T): ApiResponse<T> {
  return { data, status: 200, message: 'OK', timestamp: new Date() };
}

// --- 4. Generic Classes ---
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  get size(): number {
    return this.items.length;
  }

  get isEmpty(): boolean {
    return this.items.length === 0;
  }
}

const numberStack = new Stack<number>();
numberStack.push(1);
numberStack.push(2);
console.log(numberStack.pop());  // 2
console.log(numberStack.peek()); // 1

// --- 5. Default Type Parameters ---
interface PaginatedResult<T, Meta = { total: number; page: number }> {
  items: T[];
  meta: Meta;
}

// --- 6. Conditional Types ---
type IsString<T> = T extends string ? 'yes' : 'no';
type A = IsString<string>;  // 'yes'
type B = IsString<number>;  // 'no'

// infer keyword
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
type Unpacked = UnwrapPromise<Promise<string>>; // string

type GetReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type R = GetReturnType<() => string>; // string

// --- 7. Mapped Types ---
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Key remapping (TS 4.1+)
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface User {
  name: string;
  age: number;
}

type UserGetters = Getters<User>;
// { getName: () => string; getAge: () => number }

// --- 8. Built-in Utility Types in Action ---
interface Todo {
  title: string;
  description: string;
  done: boolean;
}

// Pick: choose specific fields
type TodoPreview = Pick<Todo, 'title' | 'done'>;

// Omit: exclude specific fields
type TodoInfo = Omit<Todo, 'done'>;

// Record: construct a type with keys and value type
type PageInfo = Record<'home' | 'about' | 'contact', { title: string }>;

const pages: PageInfo = {
  home: { title: 'Home' },
  about: { title: 'About' },
  contact: { title: 'Contact' }
};

console.log('All generics examples completed!');
```

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| 泛型参数要有意义 | `T`、`K`、`V`是约定，但`TItem`、`TKey`更清晰 |
| 别过度泛化 | 只有一个调用场景的函数不需要泛型 |
| `keyof`在泛型中极常用 | 访问对象属性时几乎必用 |
| 条件类型分布式 | 联合类型会**分发**条件检查 |
| `infer`只能在条件类型的extends中使用 | 不能在别处使用 |

---

## 六、避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-------------|-----------|
| 泛型参数从来不约束 | 用`extends`约束，缩小类型范围 |
| `T extends any`无意义约束 | 约束到具体接口如`T extends Entity` |
| 忘记`keyof`直接用字符串访问 | `obj[key: K]`配合`K extends keyof T` |
| 泛型类的方法返回any | 用`T`作为方法返回类型保持类型链 |
| 条件类型内不用`infer` | 利用`infer`提取嵌套类型 |

## 十一、代码演进

### v1 — 重复函数
```typescript
function firstString(arr: string[]): string | undefined { return arr[0]; }
function firstNumber(arr: number[]): number | undefined { return arr[0]; }
```

### v2 — 泛型函数
```typescript
function first<T>(arr: T[]): T | undefined { return arr[0]; }
```

### v3 — 约束+工具类型
```typescript
interface Entity { id: string; }

function findById<T extends Entity>(items: T[], id: string): T | undefined {
  return items.find(item => item.id === id);
}

function updateEntity<T extends Entity>(
  items: T[], id: string, patch: Partial<Omit<T, 'id'>>
): T[] {
  return items.map(item =>
    item.id === id ? { ...item, ...patch } : item
  );
}
```
