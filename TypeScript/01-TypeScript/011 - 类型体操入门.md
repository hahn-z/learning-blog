# 038-类型体操入门

> **难度：** 🔴 高级 | **阅读时间：** 25分钟 | **前置知识：** 泛型、条件类型基础

---

## 一、概念讲解

### 什么是类型体操？

类型体操（Type Gymnastics）指利用TypeScript类型系统实现**编译时计算**——在类型层面做逻辑判断、字符串操作、递归等。

### 核心工具

1. **条件类型** — `T extends U ? X : Y`
2. **infer** — 在条件类型中**提取**类型
3. **映射类型** — `{[K in keyof T]: V}`
4. **递归类型** — 类型引用自身
5. **模板字面量类型** — `` `prefix${T}` ``

### 条件类型的分发特性

```typescript
// Distributive: applied to each member of union
type ToArray<T> = T extends unknown ? T[] : never;
type Result = ToArray<string | number>; // string[] | number[]

// Non-distributive: wrap in tuple to prevent
type ToArrayND<T> = [T] extends [unknown] ? T[] : never;
type ResultND = ToArrayND<string | number>; // (string | number)[]
```

## 三、完整代码示例

```typescript
// ============================================
// Type Gymnastics - From Beginner to Practical
// ============================================

// --- 1. Basic Conditional Types ---

// Is it a string?
type IsString<T> = T extends string ? 'yes' : 'no';
type T1 = IsString<'hello'>;  // 'yes'
type T2 = IsString<42>;        // 'no'

// --- 2. infer: Extract Types ---

// Extract return type of a function
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type FnReturn = MyReturnType<() => string>; // string

// Extract first element of tuple
type Head<T extends any[]> = T extends [infer H, ...any[]] ? H : never;
type H = Head<[1, 2, 3]>; // 1

// Extract last element
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never;
type L = Last<[1, 2, 3]>; // 3

// --- 3. Recursive Types ---

// DeepPartial: make all nested properties optional
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepPartial<T[K]>
    : T[K];
};

// DeepReadonly: recursively readonly
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? T[K] extends Function
      ? T[K]
      : DeepReadonly<T[K]>
    : T[K];
};

interface Config {
  db: { host: string; port: number };
  cache: { enabled: boolean; ttl: number };
}

type PartialConfig = DeepPartial<Config>;
// { db?: { host?: string; port?: number }; cache?: { enabled?: boolean; ttl?: number } }

// --- 4. Tuple Operations ---

// Tuple length
type Length<T extends any[]> = T['length'];
type Len = Length<[1, 2, 3]>; // 3

// Concat two tuples
type Concat<T extends any[], U extends any[]> = [...T, ...U];
type C = Concat<[1, 2], [3, 4]>; // [1, 2, 3, 4]

// Drop first N elements
type Drop<T extends any[], N extends number, Acc extends any[] = []> =
  Acc['length'] extends N
    ? T
    : T extends [infer _, ...infer Rest]
      ? Drop<Rest, N, [...Acc, unknown]>
      : [];

type Dropped = Drop<[1, 2, 3, 4], 2>; // [3, 4]

// --- 5. String Manipulation Types ---

// Reverse a string type
type Reverse<S extends string> = S extends `${infer First}${infer Rest}`
  ? `${Reverse<Rest>}${First}`
  : S;
type Rev = Reverse<'hello'>; // 'olleh'

// Trim whitespace
type TrimLeft<S extends string> = S extends ` ${infer Rest}` ? TrimLeft<Rest> : S;
type TrimRight<S extends string> = S extends `${infer Rest} ` ? TrimRight<Rest> : S;
type Trim<S extends string> = TrimRight<TrimLeft<S>>;

type Trimmed = Trim<'  hello  '>; // 'hello'

// Replace
type Replace<S extends string, From extends string, To extends string> =
  From extends ''
    ? S
    : S extends `${infer Before}${From}${infer After}`
      ? `${Before}${To}${After}`
      : S;

type Replaced = Replace<'hello world', 'world', 'TS'>; // 'hello TS'

// CamelCase
type CamelCase<S extends string> = S extends `${infer First}-${infer Rest}`
  ? `${First}${Capitalize<CamelCase<Rest>>}`
  : S;

type Camel = CamelCase<'my-component-name'>; // 'myComponentName'

// --- 6. Object Type Operations ---

// Pick by value type
type PickByType<T, V> = {
  [K in keyof T as T[K] extends V ? K : never]: T[K];
};

interface Mixed {
  name: string;
  age: number;
  email: string;
  score: number;
}

type StringProps = PickByType<Mixed, string>; // { name: string; email: string }

// Omit by value type
type OmitByType<T, V> = {
  [K in keyof T as T[K] extends V ? never : K]: T[K];
};

// Make specific keys required
type RequireKeys<T, K extends keyof T> = T & Required<Pick<T, K>>;

// Optional specific keys
type PartialKeys<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;

// --- 7. Type-level Boolean ---
type Bool = { true: true; false: false };
type Not<B extends keyof Bool> = B extends 'true' ? false : true;
type N = Not<'true'>; // false

// --- 8. Practical: Typed Event System ---
type EventMap = {
  click: { x: number; y: number };
  keydown: { key: string; code: string };
  resize: { width: number; height: number };
};

type EventHandler<T extends keyof EventMap> = (payload: EventMap[T]) => void;

function on<T extends keyof EventMap>(event: T, handler: EventHandler<T>): void {
  console.log(`Registered ${event} handler`);
}

on('click', (payload) => {
  console.log(payload.x, payload.y); // fully typed!
});

console.log('Type gymnastics examples compiled successfully!');
```

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| 递归深度有限制 | TS递归类型大约支持~45层 |
| 类型体操不产生运行时代码 | 纯编译时，不影响bundle大小 |
| 不要过度设计 | 实用性优先，炫技次之 |
| infer可以嵌套 | 但可读性下降，注意注释 |
| 分布式只在裸类型参数上触发 | `[T] extends [U]`阻止分发 |

## 六、避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-------------|-----------|
| 忘记约束泛型参数 | `T extends string`等约束防止无意义类型 |
| 递归没有终止条件 | 空字符串`''`或空元组`[]`作为base case |
| 滥用any做中间类型 | 用`unknown`或具体约束更安全 |
| 条件类型没有考虑分发 | 理解分发，需要时用`[T]`阻止 |
| 类型太复杂无法调试 | 分步定义中间类型，逐步验证 |

## 七、练习题

### 🟢 基础
1. 实现`If<C, T, F>`：C为true返回T，否则返回F。
2. 实现`Flatten<T>`：将嵌套数组`T[][]`变为一维`T[]`。

### 🟡 进阶
3. 实现`TupleToUnion<T>`：`[1, 2, 3]` → `1 | 2 | 3`。
4. 实现`AppendArgument<Fn, Arg>`：给函数类型追加一个参数。

### 🔴 挑战
5. 实现`DeepMerge<A, B>`：递归合并两个对象类型，冲突时B优先。

## 十一、代码演进

### v1 — 简单条件类型
```typescript
type IsString<T> = T extends string ? true : false;
```

### v2 — infer提取
```typescript
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
```

### v3 — 递归+模式匹配
```typescript
type DeepAwaited<T> = T extends Promise<infer U>
  ? DeepAwaited<U>
  : T extends object
    ? { [K in keyof T]: DeepAwaited<T[K]> }
    : T;
```

**总结：** 类型体操是TypeScript类型系统的极致运用。不是为了炫技，而是为了在编译时捕获更多错误。从简单条件类型开始，逐步掌握infer、递归和映射类型，你的类型编程能力将大幅提升。
