# 声明文件与类型声明

> **难度：** ⭐⭐⭐ 进阶级 | **阅读时间：** 约15分钟 | **前置知识：** TypeScript基础、模块系统

## 一、概念讲解

当你在TypeScript项目中使用JavaScript第三方库时，TypeScript编译器需要知道这些库的类型信息。**声明文件（.d.ts）** 就是用来描述JavaScript代码的类型信息的。

### 为什么需要声明文件？

```typescript
// Using lodash without types
import _ from "lodash";
_.groupBy(arr, fn); // TypeScript has no idea what this returns!

// After installing @types/lodash
import _ from "lodash";
const grouped = _.groupBy(arr, fn); // TypeScript knows the exact return type
```

### 声明文件的来源

1. **内置类型**：库自带 `.d.ts` 文件（如 axios、zod）
2. **DefinitelyTyped**：社区维护的 `@types/xxx` 包
3. **自己编写**：为无类型的库手写声明文件

## 三、完整TypeScript代码

```typescript
// ============================================
// Declaration Files & Type Declarations
// ============================================

// --- 1. Global Declaration (global.d.ts) ---
// Declare global variables that exist at runtime
declare const __APP_VERSION__: string;
declare const __DEV__: boolean;
declare const ga: (command: string, ...args: unknown[]) => void;

// --- 2. Module Declaration (modules.d.ts) ---
// Declare types for untyped JS modules

// Simple module with default export
declare module "legacy-utils" {
  function format(str: string, ...args: unknown[]): string;
  function parse(json: string): unknown;
  export { format, parse };
}

// Module with default export
declare module "math-helpers" {
  const helpers: {
    add(a: number, b: number): number;
    multiply(a: number, b: number): number;
  };
  export default helpers;
}

// Wildcard module declaration (for assets)
declare module "*.png" {
  const src: string;
  export default src;
}

declare module "*.svg" {
  const src: string;
  export default src;
}

declare module "*.css" {
  const classes: Record<string, string>;
  export default classes;
}

// --- 3. Ambient Namespace ---
declare namespace NodeJS {
  interface ProcessEnv {
    readonly NODE_ENV: "development" | "production" | "test";
    readonly DATABASE_URL: string;
    readonly API_KEY: string;
    readonly PORT?: string;
  }
}

// --- 4. Augmenting Existing Types ---
// Add properties to existing interfaces (module augmentation)
declare module "express" {
  interface Request {
    user?: {
      id: string;
      name: string;
      role: "admin" | "user";
    };
  }
}

// Augment global types
declare global {
  interface Array<T> {
    // Custom array method
    last(): T | undefined;
    first(): T | undefined;
  }

  interface String {
    toTitleCase(): string;
  }
}

// --- 5. Writing Declaration Files for a Library ---
// types/my-library/index.d.ts
interface MyLibraryOptions {
  theme: "light" | "dark";
  locale: string;
  debug?: boolean;
  onInit?: () => void;
  onError?: (error: Error) => void;
}

interface MyLibrary {
  init(options: MyLibraryOptions): void;
  destroy(): void;
  getVersion(): string;
  on(event: string, handler: (...args: unknown[]) => void): void;
  off(event: string, handler: (...args: unknown[]) => void): void;
}

declare class MyLibrary {
  constructor(container: HTMLElement);
}

export = MyLibrary;
export as namespace MyLib;

// --- 6. Type-only Imports ---
// Use 'import type' for type-only imports
import type { Request, Response } from "express";

// This compiles to nothing in JS output
type Handler = (req: Request, res: Response) => void;

// --- 7. Utility: Extract types from untyped code ---
// Given a JS function, create its typed wrapper
function typedWrapper<TArgs extends unknown[], TReturn>(
  fn: (...args: TArgs) => TReturn
): (...args: TArgs) => TReturn {
  return fn;
}

// --- 8. Declaration Merging ---
// Interfaces with same name are automatically merged
interface Config {
  host: string;
  port: number;
}

interface Config {
  database: string;
  username: string;
}

// Result: Config has all 4 properties
const config: Config = {
  host: "localhost",
  port: 5432,
  database: "myapp",
  username: "admin",
};

// --- 9. Practical: Type-safe env variables ---
function getEnv(key: keyof NodeJS.ProcessEnv): string | undefined {
  return process.env[key];
}

const dbUrl = getEnv("DATABASE_URL"); // string | undefined
const nodeEnv = getEnv("NODE_ENV"); // string | undefined

// --- 10. Generating .d.ts from TypeScript ---
// When publishing a library, enable in tsconfig:
// {
//   "compilerOptions": {
//     "declaration": true,
//     "declarationMap": true,
//     "outDir": "./dist"
//   }
// }
```

## 五、注意事项

| 要点 | 说明 | 建议 |
|------|------|------|
| 查找顺序 | 先库自带，再@types，最后手写 | `npm info <pkg> types` 检查是否自带类型 |
| 声明合并 | 同名接口自动合并 | 注意不要意外合并 |
| 全局污染 | `declare global` 修改全局类型 | 谨慎使用，优先模块化 |
| any导出 | `declare module "xxx"` 默认返回any | 尽量写详细的类型声明 |
| 三斜线 | `/// <reference types="..." />` | 现代项目用import替代 |

## 六、避坑指南

```typescript
// ❌ 使用 any 作为模块声明
declare module "unknown-lib"; // Everything is any
// ✅ 至少写出常用API的类型
declare module "unknown-lib" {
  export function main(config: unknown): Promise<unknown>;
}

// ❌ 在 .d.ts 中使用 const/let（不会真正创建变量）
// ✅ 使用 declare const 声明已存在的全局变量

// ❌ 忘记 export 导致全局污染
declare function helper(): void; // 污染全局
// ✅ 使用 export 限定作用域
export declare function helper(): void;

// ❌ 修改第三方类型没有用 declare module
// ✅ 正确的模块增强
declare module "express" {
  interface Request { customProp: string; }
}
```

## 八、知识点总结

```
声明文件体系
├── .d.ts → 纯类型声明，无运行时代码
├── declare → 告诉TS"这个已存在"
├── 三种来源 → 库自带 / @types / 手写
├── 模块声明 → declare module "name"
├── 全局声明 → declare global / declare var
├── 声明合并 → 同名接口自动合并
└── 模块增强 → 扩展第三方类型
```

## 九、举一反三

| 场景 | 声明方式 | 示例 |
|------|---------|------|
| 全局变量 | `declare const` | `declare const __DEV__: boolean` |
| 图片/CSS导入 | `declare module "*.png"` | webpack资源类型 |
| 环境变量 | ProcessEnv接口 | 类型安全的环境变量 |
| 第三方扩展 | `declare module` | 扩展Express Request |
| 库发布 | `declaration: true` | 自动生成.d.ts |

## 十一、代码演进

### v1 → 简单模块声明

```typescript
declare module "my-lib" {
  export function doSomething(input: string): number;
}
```

### v2 → 带泛型和接口

```typescript
declare module "my-lib" {
  interface Options<T> { data: T; verbose: boolean; }
  export function process<T>(options: Options<T>): Promise<T>;
  export default process;
}
```

### v3 → 完整的库声明文件

```typescript
// types/my-lib/index.d.ts
export class Client {
  constructor(config: ClientConfig);
  request<T>(endpoint: string): Promise<ApiResponse<T>>;
}
export interface ClientConfig { baseUrl: string; timeout?: number; }
export interface ApiResponse<T> { data: T; status: number; }
```
