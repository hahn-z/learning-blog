# 模块系统与命名空间

> **难度：** ⭐⭐⭐ 进阶级 | **阅读时间：** 约15分钟 | **前置知识：** ES Modules、TypeScript基础

## 一、概念讲解

TypeScript的模块系统建立在ES Modules之上，并添加了类型导入/导出的能力。理解模块解析机制、路径映射和命名空间的正确使用场景，是构建大型TypeScript项目的关键。

### ES Modules vs CommonJS vs Namespaces

```typescript
// ES Module (推荐) - static analysis, tree-shakeable
import { User } from "./user";
export function createUser() {}

// CommonJS (Node.js legacy)
const { User } = require("./user");
module.exports = { createUser };

// Namespace (legacy, avoid in new projects)
namespace App {
  export function createUser() {}
}
```

### 模块解析策略

TypeScript有两种模块解析策略：
- **Classic**：旧的解析方式，主要用于AMD/System
- **Node**：模拟Node.js的解析逻辑（推荐）

## 三、完整TypeScript代码

```typescript
// ============================================
// Module System & Namespaces
// ============================================

// --- 1. Named Exports (math.ts) ---
export const PI = 3.14159;

export function add(a: number, b: number): number {
  return a + b;
}

export function multiply(a: number, b: number): number {
  return a * b;
}

export interface MathResult {
  value: number;
  operation: string;
}

// --- 2. Type-only Exports ---
export type { MathResult };
export type Operation = "add" | "subtract" | "multiply" | "divide";

// --- 3. Default Export (calculator.ts) ---
export default class Calculator {
  private history: string[] = [];

  compute(a: number, op: Operation, b: number): number {
    const ops: Record<Operation, (a: number, b: number) => number> = {
      add: (x, y) => x + y,
      subtract: (x, y) => x - y,
      multiply: (x, y) => x * y,
      divide: (x, y) => (y !== 0 ? x / y : NaN),
    };
    const result = ops[op](a, b);
    this.history.push(`${a} ${op} ${b} = ${result}`);
    return result;
  }

  getHistory(): readonly string[] {
    return Object.freeze([...this.history]);
  }
}

// --- 4. Barrel File (index.ts) ---
// Re-export everything from submodules
// export { Calculator } from "./calculator";
// export { PI, add, multiply } from "./math";
// export type { MathResult, Operation } from "./math";

// --- 5. Namespace (legacy pattern) ---
namespace Validator {
  export function isEmail(value: string): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
  }

  export function isUrl(value: string): boolean {
    try {
      new URL(value);
      return true;
    } catch {
      return false;
    }
  }

  export function isInRange(value: number, min: number, max: number): boolean {
    return value >= min && value <= max;
  }

  // Nested namespace
  export namespace String {
    export function minLength(value: string, min: number): boolean {
      return value.length >= min;
    }

    export function maxLength(value: string, max: number): boolean {
      return value.length <= max;
    }
  }
}

// Usage
Validator.isEmail("test@example.com"); // true
Validator.String.minLength("hello", 3); // true

// --- 6. Module with Namespace Pattern ---
// Modern approach: one module per file, no namespaces
// user.types.ts
export interface User {
  id: string;
  name: string;
  email: string;
  role: UserRole;
}

export type UserRole = "admin" | "editor" | "viewer";

// user.service.ts
import type { User, UserRole } from "./user.types";

export class UserService {
  private users = new Map<string, User>();

  create(data: Omit<User, "id">): User {
    const user: User = {
      ...data,
      id: crypto.randomUUID(),
    };
    this.users.set(user.id, user);
    return user;
  }

  findByRole(role: UserRole): User[] {
    return Array.from(this.users.values()).filter((u) => u.role === role);
  }
}

// --- 7. Dynamic Imports ---
async function loadChartLibrary(): Promise<void> {
  // Dynamic import for code splitting
  const { Chart } = await import("./calculator");
  // Use Chart...
}

// --- 8. Path Mapping ---
// tsconfig.json:
// {
//   "compilerOptions": {
//     "baseUrl": ".",
//     "paths": {
//       "@/*": ["src/*"],
//       "@utils/*": ["src/utils/*"],
//       "@services/*": ["src/services/*"]
//     }
//   }
// }

// With path aliases:
// import { UserService } from "@services/user.service";
// import type { User } from "@types/user";

// --- 9. Conditional Exports (package.json) ---
// Modern packages can define different entry points:
// {
//   "exports": {
//     ".": { "import": "./dist/esm/index.js", "require": "./dist/cjs/index.js" },
//     "./utils": { "import": "./dist/esm/utils.js" }
//   }
// }

// --- 10. Circular Dependency Handling ---
// a.ts imports from b.ts and vice versa
// Solution: extract shared types to a third file

// shared.types.ts
export interface SharedConfig {
  name: string;
  version: string;
}

// moduleA.ts
import type { SharedConfig } from "./shared.types";
export function createA(config: SharedConfig) { return config; }

// moduleB.ts
import type { SharedConfig } from "./shared.types";
export function createB(config: SharedConfig) { return config; }
```

## 五、注意事项

| 要点 | 说明 | 建议 |
|------|------|------|
| import type | 纯类型导入不产生运行时代码 | 始终用 `import type` 导入类型 |
| barrel files | index.ts统一导出可能影响tree-shaking | 只在必要时使用 |
| 命名空间 | 旧模式，不推荐新项目使用 | 用ES Modules替代 |
| 路径映射 | 需要 bundler 或 tsconfig-paths 支持 | 确保运行时也能解析 |
| 循环依赖 | 模块间互相导入 | 提取共享类型到独立文件 |

## 六、避坑指南

```typescript
// ❌ namespace 和 module 混用
namespace App { export class User {} }
import { User } from "some-module";
// ✅ 统一使用 ES Modules

// ❌ barrel file 导致循环依赖
// a/index.ts re-exports from b, which imports from a
// ✅ 避免在 barrel file 中导入有依赖关系的模块

// ❌ 忘记设置 moduleResolution
// "moduleResolution": "classic" (outdated)
// ✅ 使用 "moduleResolution": "node" 或 "bundler"

// ❌ 路径别名在运行时无法解析
import { foo } from "@utils/foo"; // Runtime error!
// ✅ 使用 tsconfig-paths 或 bundler 的 alias 功能
```

## 八、知识点总结

```
模块系统
├── ES Modules → import/export (推荐)
│   ├── import type → 纯类型导入
│   └── dynamic import → 代码分割
├── 模块解析 → Node策略
│   ├── 路径映射 → @/* 别名
│   └── 条件导出 → package.json exports
├── 命名空间 → 旧模式，避免使用
├── Barrel Files → 统一导出入口
└── 循环依赖 → 提取共享类型解决
```

## 九、举一反三

| 场景 | 方案 | 关键点 |

## 十一、代码演进

### v1 → 命名空间

```typescript
namespace App {
  export class User { constructor(public name: string) {} }
  export function create(name: string) { return new User(name); }
}
```

### v2 → ES Modules + 类型分离

```typescript
// types.ts
export interface User { name: string; }
// service.ts
import type { User } from "./types";
export function create(name: string): User { return { name }; }
```

### v3 → 路径别名 + 模块化架构

```typescript
import type { User } from "@/types";
import { UserService } from "@/services/user.service";
```
