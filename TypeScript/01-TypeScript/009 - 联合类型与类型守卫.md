# 036-联合类型与类型守卫

> **难度：** 🟡 进阶 | **阅读时间：** 20分钟 | **前置知识：** 泛型基础

---

## 一、概念讲解

### 联合类型（Union Types）

联合类型表示一个值可以是**多种类型之一**：

```typescript
let id: string | number;
id = 'abc';  // OK
id = 123;    // OK
```

### 类型收窄（Type Narrowing）

TypeScript编译器通过分析代码的控制流，自动将联合类型**收窄**到更具体的类型：

```typescript
function process(value: string | number) {
  // Here: value is string | number
  if (typeof value === 'string') {
    // Here: value is string (narrowed!)
    console.log(value.toUpperCase());
  } else {
    // Here: value is number
    console.log(value.toFixed(2));
  }
}
```

### 类型守卫（Type Guards）

类型守卫是**返回类型谓词**的函数，告诉编译器如何收窄：

```typescript
function isString(val: unknown): val is string {
  return typeof val === 'string';
}
```

### 可辨识联合（Discriminated Unions）

通过一个**共同的字面量属性**来区分联合成员：

```typescript
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'square'; size: number }
  | { kind: 'triangle'; base: number; height: number };

function area(shape: Shape): number {
  switch (shape.kind) {  // 'kind' is the discriminant
    case 'circle': return Math.PI * shape.radius ** 2;
    case 'square': return shape.size ** 2;
    case 'triangle': return 0.5 * shape.base * shape.height;
  }
}
```

## 三、完整代码示例

```typescript
// ============================================
// Union Types and Type Guards
// ============================================

// --- 1. Basic Union Types ---
type ID = string | number;

function displayId(id: ID): string {
  if (typeof id === 'string') {
    return id.toUpperCase(); // narrowed to string
  }
  return id.toFixed(0);     // narrowed to number
}

// --- 2. typeof Guards ---
function padLeft(value: string, padding: string | number): string {
  if (typeof padding === 'number') {
    return ' '.repeat(padding) + value;
  }
  return padding + value;
}

console.log(padLeft('hello', 4));    // "    hello"
console.log(padLeft('hello', '>>')); // ">>hello"

// --- 3. instanceof Guards ---
class Dog {
  bark() { return 'Woof!'; }
}
class Cat {
  meow() { return 'Meow!'; }
}

type Pet = Dog | Cat;

function speak(pet: Pet): string {
  if (pet instanceof Dog) return pet.bark();
  return pet.meow();
}

// --- 4. 'in' Operator Guard ---
interface Fish {
  swim(): void;
  waterType: string;
}

interface Bird {
  fly(): void;
  wingspan: number;
}

type Animal = Fish | Bird;

function move(animal: Animal): void {
  if ('swim' in animal) {
    animal.swim(); // narrowed to Fish
  } else {
    animal.fly();  // narrowed to Bird
  }
}

// --- 5. Discriminated Unions ---

type Result<T> =
  | { status: 'success'; data: T }
  | { status: 'error'; error: string }
  | { status: 'loading' };

function handleResult<T>(result: Result<T>): string {
  switch (result.status) {
    case 'success':
      return `Got: ${JSON.stringify(result.data)}`;
    case 'error':
      return `Error: ${result.error}`;
    case 'loading':
      return 'Loading...';
  }
}

console.log(handleResult({ status: 'success', data: [1, 2, 3] }));
console.log(handleResult({ status: 'error', error: 'Not found' }));

// --- 6. Custom Type Guards ---

// String guard
function isNonEmptyString(value: unknown): value is string {
  return typeof value === 'string' && value.length > 0;
}

// Array element guard
function isArrayOf<T>(guard: (val: unknown) => val is T) {
  return (arr: unknown): arr is T[] =>
    Array.isArray(arr) && arr.every(guard);
}

const isNumberArray = isArrayOf((v: unknown): v is number => typeof v === 'number');

const maybeNumbers: unknown = [1, 2, 3];
if (isNumberArray(maybeNumbers)) {
  console.log(maybeNumbers.reduce((a, b) => a + b, 0)); // 6, type: number[]
}

// --- 7. Assertion Functions ---
function assertString(value: unknown, msg?: string): asserts value is string {
  if (typeof value !== 'string') {
    throw new TypeError(msg ?? `Expected string, got ${typeof value}`);
  }
}

let input: unknown = 'hello';
assertString(input);
console.log(input.toUpperCase()); // TS knows input is string

// --- 8. Exhaustive Check with never ---
type TrafficLight = 'red' | 'yellow' | 'green';

function handleTraffic(light: TrafficLight): string {
  switch (light) {
    case 'red': return 'Stop';
    case 'yellow': return 'Caution';
    case 'green': return 'Go';
    default: {
      const exhaustive: never = light; // Error if a case is missed
      return exhaustive;
    }
  }
}

// --- 9. satisfies operator (TS 4.9+) ---
const colors = {
  red: '#FF0000',
  green: '#00FF00',
  blue: '#0000FF'
} satisfies Record<string, string>;
// typeof colors = { red: string; green: string; blue: string }
// NOT Record<string, string> - keeps literal keys!

console.log('All union & guard examples completed!');
```

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| typeof只能检查原始类型 | `typeof []`是`'object'`，不区分数组 |
| instanceof需要class | 对接口/类型别名无效，它们编译后不存在 |
| 联合类型是OR关系 | `A \| B`表示A**或**B，不是A**和**B |
| 穷尽检查很重要 | `default`分支用`never`确保处理了所有情况 |
| 类型谓词要准确实现 | `is`声明和实际检查逻辑要一致 |

## 六、避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-------------|-----------|
| 用typeof检查数组 | 用`Array.isArray()` |
| 忘记穷尽检查 | `default`分支用`never`兜底 |
| 可辨识联合属性名不一致 | 统一用`type`或`kind`做判别属性 |
| 自定义守卫返回boolean | 用`val is Type`谓词语法 |
| `in`操作符检查假值 | 先检查`!= null`再`in` |

## 十一、代码演进

### v1 — 运行时检查，无类型安全
```typescript
function process(value: any) {
  if (typeof value === 'string') return value.toUpperCase();
  return value.toFixed(2);
}
```

### v2 — 联合类型+typeof收窄
```typescript
function process(value: string | number): string {
  if (typeof value === 'string') return value.toUpperCase();
  return value.toFixed(2);
}
```

### v3 — 可辨识联合+穷尽检查
```typescript
type Action =
  | { type: 'SET_NAME'; name: string }
  | { type: 'SET_AGE'; age: number }
  | { type: 'RESET' };

type State = { name: string; age: number };

function reducer(state: State, action: Action): State {
  switch (action.type) {
    case 'SET_NAME': return { ...state, name: action.name };
    case 'SET_AGE': return { ...state, age: action.age };
    case 'RESET': return { name: '', age: 0 };
    default: {
      const _: never = action;
      return _;
    }
  }
}
```

**总结：** 联合类型+类型守卫是TypeScript类型安全的基石。掌握收窄机制和可辨识联合模式，你就能处理几乎所有「不同类型走不同分支」的场景。
