# 037-枚举与字面量类型

> **难度：** 🟡 进阶 | **阅读时间：** 16分钟 | **前置知识：** 基础类型、联合类型

---

## 一、概念讲解

### 数字枚举（Numeric Enums）

```typescript
enum Direction {
  Up,     // 0 (auto-increment)
  Down,   // 1
  Left,   // 2
  Right   // 3
}

// Reverse mapping: Direction[0] === 'Up'
```

数字枚举有**双向映射**：`Direction.Up === 0` 和 `Direction[0] === 'Up'`。

### 字符串枚举（String Enums）

```typescript
enum Status {
  Active = 'ACTIVE',
  Inactive = 'INACTIVE'
}
// No reverse mapping for string enums
```

### const enum

```typescript
const enum Color {
  Red = '#FF0000',
  Green = '#00FF00'
}
let c = Color.Red; // Inlined to '#FF0000' in JS output
```

`const enum`编译时直接内联，**不生成JS代码**。

### 字面量联合替代枚举

```typescript
// Instead of enum
type Status = 'active' | 'inactive' | 'pending';

// Benefits: tree-shakeable, no runtime code, simpler
```

### as const 对象模式

```typescript
const Role = {
  Admin: 'ADMIN',
  Editor: 'EDITOR',
  Viewer: 'VIEWER'
} as const;

type Role = typeof Role[keyof typeof Role]; // 'ADMIN' | 'EDITOR' | 'VIEWER'
```

## 二、思维导图

```
枚举与字面量类型
├── Enum
│   ├── 数字枚举（自增+反向映射）
│   ├── 字符串枚举（单向映射）
│   ├── 混合枚举
│   ├── const enum（内联）
│   └── 计算成员
├── 字面量联合
│   ├── type Status = 'a' | 'b'
│   ├── 无运行时代码
│   └── tree-shakeable
├── as const 模式
│   ├── 对象 + as const
│   ├── typeof + keyof 提取类型
│   └── 运行时+类型都可用
└── 选择策略
    ├── 简单选项 → 字面量联合
    ├── 需要运行时对象 → as const
    └── 需要反向映射 → 数字枚举
```

## 三、完整代码示例

```typescript
// ============================================
// Enums and Literal Types
// ============================================

// --- 1. Numeric Enum ---
enum HttpStatus {
  OK = 200,
  BadRequest = 400,
  Unauthorized = 401,
  NotFound = 404,
  InternalError = 500
}

function getStatusText(code: HttpStatus): string {
  switch (code) {
    case HttpStatus.OK: return 'OK';
    case HttpStatus.BadRequest: return 'Bad Request';
    case HttpStatus.NotFound: return 'Not Found';
    default: return 'Unknown';
  }
}

console.log(getStatusText(200));        // "OK"
console.log(HttpStatus[200]);           // "OK" (reverse mapping)
console.log(HttpStatus['OK']);          // 200

// --- 2. String Enum ---
enum UserRole {
  Admin = 'ADMIN',
  Editor = 'EDITOR',
  Viewer = 'VIEWER'
}

function canEdit(role: UserRole): boolean {
  return role === UserRole.Admin || role === UserRole.Editor;
}

console.log(canEdit(UserRole.Admin));   // true
// UserRole['ADMIN']; // Error: no reverse mapping

// --- 3. Const Enum ---
const enum LogLevel {
  DEBUG,
  INFO,
  WARN,
  ERROR
}

function log(msg: string, level: LogLevel): void {
  const prefix = ['DEBUG', 'INFO', 'WARN', 'ERROR'][level];
  console.log(`[${prefix}] ${msg}`);
}

log('Hello', LogLevel.INFO); // Compiles to: console.log("[INFO] Hello")

// --- 4. Mixed Enum ---
enum Color {
  Red = '#FF0000',
  Green = '#00FF00',
  Blue = '#0000FF',
  Alpha = 0xFF  // numeric mixed with string - Error!
}
// Note: Mixing string and numeric in same enum is NOT allowed
// This would be an error. Keep enums homogeneous.

// --- 5. Literal Union (enum alternative) ---
type Theme = 'light' | 'dark' | 'system';

function applyTheme(theme: Theme): void {
  console.log(`Applying ${theme} theme`);
}

applyTheme('dark');
// applyTheme('blue'); // Error

// --- 6. as const Object Pattern (Best of both worlds) ---
const ErrorCode = {
  NetworkError: 'NETWORK_ERROR',
  Timeout: 'TIMEOUT',
  AuthFailed: 'AUTH_FAILED',
  NotFound: 'NOT_FOUND'
} as const;

// Extract the type
type ErrorCode = typeof ErrorCode[keyof typeof ErrorCode];
// Type: 'NETWORK_ERROR' | 'TIMEOUT' | 'AUTH_FAILED' | 'NOT_FOUND'

function handleError(code: ErrorCode): string {
  // Can use both runtime value and type
  const messages: Record<ErrorCode, string> = {
    NETWORK_ERROR: 'Network connection failed',
    TIMEOUT: 'Request timed out',
    AUTH_FAILED: 'Authentication failed',
    NOT_FOUND: 'Resource not found'
  };
  return messages[code];
}

console.log(handleError(ErrorCode.Timeout));    // "Request timed out"
console.log(handleError('AUTH_FAILED'));         // "Authentication failed"

// --- 7. Template Literal Types ---
type EventName = 'click' | 'focus' | 'blur';
type EventHandler = `on${Capitalize<EventName>}`;
// Type: 'onClick' | 'onFocus' | 'onBlur'

const handler: Record<EventHandler, () => void> = {
  onClick: () => console.log('clicked'),
  onFocus: () => console.log('focused'),
  onBlur: () => console.log('blurred')
};

// --- 8. Enum from array ---
const FRUITS = ['Apple', 'Banana', 'Cherry'] as const;
type Fruit = typeof FRUITS[number]; // 'Apple' | 'Banana' | 'Cherry'

function getFruit(index: number): Fruit {
  return FRUITS[index];
}

console.log(getFruit(1)); // "Banana"

console.log('All enum & literal examples completed!');
```

## 五、注意事项

| 注意点 | 说明 |
|--------|------|
| 数字枚举有反向映射 | `Enum[key]`和`Enum[value]`都可以 |
| 字符串枚举没有反向映射 | 只有`Enum.key = value`方向 |
| const enum和isolatedModules冲突 | 某些构建工具不支持const enum |
| 枚举生成运行时代码 | 字面量联合不生成任何JS |
| as const对象是最佳实践 | 同时有运行时值和类型安全 |

## 六、避坑指南

| ❌ 常见错误 | ✅ 正确做法 |
|-------------|-----------|
| 混合字符串和数字枚举 | 保持枚举类型一致 |
| 用数字枚举做API状态码 | 字符串枚举或字面量联合更安全 |
| 在库中使用const enum | 库用普通enum或as const模式 |
| 用enum但不需要反向映射 | 字面量联合更轻量 |
| 忽略enum的tree-shaking问题 | as const模式对bundler更友好 |

|------|----------|

| 用户角色 | 字符串枚举或字面量联合 |

## 十一、代码演进

### v1 — 硬编码字符串
```typescript
function setTheme(theme: string) {
  document.body.className = theme;
}
setTheme('darkk'); // typo, no error
```

### v2 — 使用枚举
```typescript
enum Theme { Light = 'LIGHT', Dark = 'DARK' }
function setTheme(theme: Theme) { /* ... */ }
setTheme(Theme.Dark);
```

### v3 — as const模式
```typescript
const Theme = { Light: 'light', Dark: 'dark', System: 'system' } as const;
type Theme = typeof Theme[keyof typeof Theme];

function setTheme(theme: Theme): void {
  console.log(`Setting theme: ${theme}`);
}

setTheme(Theme.Dark);     // OK
setTheme('system');        // OK
// setTheme('darkk');      // Error
```

---

**总结：** 在现代TypeScript中，`as const`对象模式是枚举的最佳替代。它同时提供运行时值和编译时类型安全，且对构建工具友好。
