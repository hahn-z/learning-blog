# 条件判断与三元表达式

> **难度标注：** ⭐⭐ 初级
>
> 适合有基本 Python 语法基础，刚开始学习流程控制的读者。

## 1. 概念讲解

条件判断是编程中最基础的流程控制机制。Python 提供了 `if`/`elif`/`else` 语句来进行条件分支，以及三元表达式（conditional expression）来实现简洁的条件赋值。

**核心概念：**

- **布尔表达式**：条件判断的本质是求值一个布尔表达式，结果为 `True` 或 `False`
- **代码块与缩进**：Python 用缩进（4个空格）来定义代码块，而不是花括号
- **链式判断**：`elif` 可以串联多个条件分支
- **三元表达式**：`value_if_true if condition else value_if_false`，一行搞定简单条件赋值
- **真值测试**（Truthiness）：任何对象都可以在布尔上下文中求值，不只有 `True`/`False`

**为什么条件判断重要？**

几乎所有程序都需要根据不同情况执行不同逻辑。用户登录成功跳首页，失败显示错误；购物车金额超 200 打折，否则原价；文件存在就读取，不存在就创建——这些全靠条件判断驱动。

## 3. 完整代码示例

```python
"""
Python 条件判断与三元表达式 - 完整示例
Covering: if/elif/else, ternary, logical ops, truthiness, match-case
"""

# --- 1. Basic if/elif/else ---
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Score: {score}, Grade: {grade}")  # Score: 85, Grade: B


# --- 2. Ternary expression (conditional expression) ---
age = 20
status = "adult" if age >= 18 else "minor"
print(f"Age {age}: {status}")  # Age 20: adult

# Nesting ternary (readability tip: avoid deep nesting)
label = "high" if score >= 90 else "medium" if score >= 70 else "low"
print(f"Label: {label}")  # Label: medium


# --- 3. Logical operators and short-circuit evaluation ---
x = 10

# Short-circuit: if first is False, skip second
result = x > 0 and print("positive!")  # prints "positive!"
result = x < 0 and print("negative!")  # prints nothing

# Common pattern: default value via 'or'
name = ""
display_name = name or "Anonymous"
print(f"Display: {display_name}")  # Display: Anonymous


# --- 4. Truthiness - what counts as False ---
falsy_values = [None, False, 0, 0.0, "", [], {}, (), set()]
for val in falsy_values:
    print(f"{repr(val):10s} -> {bool(val)}")

# Everything else is truthy
truthy_values = [1, -1, "hello", [0], {"a": 1}]
for val in truthy_values:
    print(f"{repr(val):15s} -> {bool(val)}")


# --- 5. Practical example: user permission check ---
def check_access(user_role, is_authenticated, resource_owner, current_user):
    """Check if user can access a resource."""
    if not is_authenticated:
        return "deny: not logged in"
    if user_role == "admin":
        return "allow: admin access"
    if current_user == resource_owner:
        return "allow: owner access"
    return "deny: insufficient permissions"

print(check_access("admin", True, "alice", "bob"))    # allow: admin access
print(check_access("user", True, "alice", "alice"))   # allow: owner access
print(check_access("user", True, "alice", "bob"))     # deny: insufficient permissions
print(check_access("user", False, "alice", "bob"))    # deny: not logged in


# --- 6. match-case (Python 3.10+) ---
command = "quit"

match command:
    case "start":
        print("Starting...")
    case "stop":
        print("Stopping...")
    case "quit" | "exit":
        print("Goodbye!")
    case _:
        print(f"Unknown command: {command}")

# Pattern matching with guards
point = (3, 0)

match point:
    case (0, 0):
        print("Origin")
    case (x, 0):
        print(f"On x-axis at {x}")  # This matches
    case (0, y):
        print(f"On y-axis at {y}")
    case (x, y):
        print(f"Point at ({x}, {y})")


# --- 7. Walrus operator in conditions (Python 3.8+) ---
import re

text = "Hello, Python 3.10!"
# Assign and check in one line
if match := re.search(r"Python (\d+\.\d+)", text):
    print(f"Found Python version: {match.group(1)}")  # Found Python version: 3.10


# --- 8. Common pattern: EAFP vs LBYL ---
data = {"name": "Alice", "age": 30}

# LBYL (Look Before You Leap) - check first
if "email" in data:
    email = data["email"]
else:
    email = "N/A"

# EAFP (Easier to Ask Forgiveness than Permission) - try first
try:
    email = data["email"]
except KeyError:
    email = "N/A"

# Best of both: dict.get()
email = data.get("email", "N/A")
print(f"Email: {email}")  # Email: N/A
```

## 5. 注意事项

| 注意点 | 说明 |
|--------|------|
| 缩进必须一致 | 同一代码块必须使用相同缩进，混用 Tab 和空格会报 `TabError` |
| `elif` 不是 `else if` | Python 用 `elif`，不是 JavaScript 的 `else if` |
| 比较用 `==`，不用 `is` | `is` 比较对象身份（内存地址），`==` 比较值 |
| 三元表达式没有 `switch` | Python 3.10 之前没有 switch，3.10+ 用 `match-case` |
| 条件表达式有返回值 | 三元表达式是表达式，能用在赋值右侧；if 语句是语句，不能 |
| 避免多层嵌套 | 超过 3 层嵌套考虑用 early return 或重构 |
| `match-case` 需要 3.10+ | 低版本 Python 不支持，用 `if/elif` 替代 |

## 6. 避坑指南

| ❌ 错误写法 | ✅ 正确写法 | 原因 |
|------------|-----------|------|
| `if x = 5:` | `if x == 5:` | `=` 是赋值，`==` 是比较 |
| `if x > 5 and < 10:` | `if 5 < x < 10:` | Python 支持链式比较 |
| `if type(x) == list:` | `if isinstance(x, list):` | `isinstance` 支持继承 |
| `if x == True:` | `if x:` | 直接真值测试更 Pythonic |
| `if x != None:` | `if x is not None:` | `None` 是单例，用 `is` 比较 |
| `if len(lst) > 0:` | `if lst:` | 空容器本身就是 falsy |
| `not x == y` | `not (x == y)` 或 `x != y` | `not` 优先级高于 `==`，容易误解 |

def classify_temperature(temp):
    if temp < 0:

    elif temp <= 15:

def safe_divide(a, b):
    if b == 0:
        if a == 0:
            return "Error: 0/0 is undefined"
        return "Error: division by zero"
    return f"{a} / {b} = {a / b}"

# Q4
display = name or "Anonymous"

# Q5
def calculator(expr):
    match expr:
        case ("+", a, b):
            return a + b
        case ("-", a, b):
            return a - b
        case ("*", a, b):
            return a * b
        case ("/", a, b) if b != 0:
            return a / b
        case ("/", _, 0):
            return "Error: division by zero"
        case _:
            return "Error: unknown operation"

## 9. 举一反三

| 场景 | 条件判断方式 | 示例 |
|------|-------------|------|
| 简单二选一赋值 | 三元表达式 | `x = a if cond else b` |
| 多分支固定值匹配 | `match-case`（3.10+） | `match cmd: case "start": ...` |
| 多分支范围判断 | `if/elif/else` | `if score >= 90: ...` |
| 检查键是否存在 | `dict.get(key, default)` | `data.get("name", "N/A")` |
| 类型判断 | `isinstance()` | `if isinstance(x, int): ...` |
| 空值检查 | `is not None` | `if result is not None: ...` |
| 多条件组合 | `any()` / `all()` | `if any(x > 0 for x in nums): ...` |
| 异常处理替代 | `try/except` (EAFP) | `try: x = d[k]; except KeyError: ...` |

## 11. 代码演进

### v1 — 基础 if/else（刚入门）

```python
score = 85

if score >= 60:
    print("pass")
else:
    print("fail")
```

这段代码能跑，但只能判断及格/不及格两个状态。

### v2 — 多分支 + 三元表达式（进阶）

```python
score = 85

# Multi-branch for grade
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "D"

# Ternary for simple pass/fail
status = "pass" if score >= 60 else "fail"
print(f"Grade: {grade}, Status: {status}")
```

用上了 `elif` 做多分支判断，三元表达式做简单的二选一。

### v3 — 函数封装 + match-case + 防御性编程（实战）

```python
def classify_score(score: int) -> dict:
    """Classify a score with full metadata."""
    if not isinstance(score, (int, float)):
        raise TypeError(f"Score must be numeric, got {type(score).__name__}")
    if not 0 <= score <= 100:
        raise ValueError(f"Score must be 0-100, got {score}")

    grade = (
        "A" if score >= 90 else
        "B" if score >= 80 else
        "C" if score >= 70 else
        "D" if score >= 60 else
        "F"
    )

    match grade:
        case "A" | "B":
            level = "excellent"
        case "C":
            level = "average"
        case "D":
            level = "below_average"
        case _:
            level = "failing"

    return {
        "score": score,
        "grade": grade,
        "level": level,
        "passed": score >= 60,
    }

# Usage
result = classify_score(85)
print(f"{result['grade']} - {result['level']}")  # B - excellent
```

v3 版本加入了类型检查、范围校验、函数封装，用链式三元做等级判断，用 `match-case` 做级别映射，返回结构化数据——这才是生产级代码的样子。
