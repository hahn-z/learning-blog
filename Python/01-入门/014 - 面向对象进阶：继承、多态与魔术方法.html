1. 概念讲解

继承（Inheritance）

继承让子类复用父类的代码，并在其基础上扩展或修改。

现实类比：
  父类 Animal：有 name、能 eat()
  子类 Dog：继承 Animal，新增 bark()
  子类 Cat：继承 Animal，新增 meow()

多态（Polymorphism）

多态 = 同一接口，不同实现。不同类的对象响应相同的方法名，但行为各不同。

# Both Dog and Cat have speak(), but sound different
animals = [Dog("Rex"), Cat("Whiskers")]
for a in animals:
    print(a.speak())  # Each class responds differently

魔术方法（Dunder Methods）

以双下划线包围的方法（__xxx__），让自定义类支持 Python 内置操作：

魔术方法 触发场景 示例
__len__ len(obj) len(my_list)
__getitem__ obj[key] my_list[0]
__contains__ x in obj 3 in my_set
__add__ a + b Vector(1,2) + Vector(3,4)
__eq__ a == b 自定义相等比较
__call__ obj() 把对象当函数调用

3. 代码演进

v1：基本继承

# v1: Basic inheritance

class Animal:
    def __init__(self, name: str):
        self.name = name

    def speak(self) -> str:
        return f"{self.name} makes a sound"

class Dog(Animal):
    def speak(self) -> str:              # override
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self) -> str:
        return f"{self.name} says Meow!"

# Polymorphism in action
animals = [Dog("Rex"), Cat("Whiskers"), Animal("Thing")]
for a in animals:
    print(a.speak())
# Rex says Woof!
# Whiskers says Meow!
# Thing makes a sound

v2：继承 + super + 多态

# v2: Inheritance with super(), working with parent state

class Employee:
    def __init__(self, name: str, salary: float):
        self.name = name
        self.salary = salary

    def __repr__(self):
        return f"Employee({self.name!r}, salary={self.salary})"

    def annual_cost(self) -> float:
        return self.salary * 12

class Manager(Employee):
    def __init__(self, name: str, salary: float, bonus: float):
        super().__init__(name, salary)   # call parent __init__
        self.bonus = bonus

    def annual_cost(self) -> float:      # override with extension
        return super().annual_cost() + self.bonus

    def __repr__(self):
        return f"Manager({self.name!r}, salary={self.salary}, bonus={self.bonus})"

e = Employee("Alice", 10000)
m = Manager("Bob", 15000, 50000)
print(e.annual_cost())  # 120000
print(m.annual_cost())  # 230000

v3：完整版（魔术方法 + ABC + 自定义容器）

# v3: Full featured - magic methods, ABC, custom container

from abc import ABC, abstractmethod
from typing import Iterator

# --- Abstract base class ---
class Shape(ABC):
    @abstractmethod
    def area(self) -> float: ...

    @abstractmethod
    def __repr__(self) -> str: ...

    def __eq__(self, other):
        if not isinstance(other, Shape):
            return NotImplemented
        return abs(self.area() - other.area()) < 1e-9

    def __lt__(self, other):
        if not isinstance(other, Shape):
            return NotImplemented
        return self.area() < other.area()


class Rectangle(Shape):
    def __init__(self, width: float, height: float):
        self.width = width
        self.height = height

    def area(self) -> float:
        return self.width * self.height

    def __repr__(self):
        return f"Rectangle({self.width}x{self.height})"


class Circle(Shape):
    def __init__(self, radius: float):
        self.radius = radius

    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2

    def __repr__(self):
        return f"Circle(r={self.radius})"


# --- Custom container with magic methods ---
class Drawing:
    """A collection of shapes — behaves like a list."""

    def __init__(self):
        self._shapes: list[Shape] = []

    def add(self, shape: Shape):
        self._shapes.append(shape)
        return self

    # Container protocol
    def __len__(self):
        return len(self._shapes)

    def __getitem__(self, index):
        return self._shapes[index]

    def __contains__(self, shape):
        return shape in self._shapes

    def __iter__(self) -> Iterator[Shape]:
        return iter(self._shapes)

    # Arithmetic: merge two drawings
    def __add__(self, other):
        result = Drawing()
        result._shapes = self._shapes + other._shapes
        return result

    # Callable: sort shapes by area
    def __call__(self, reverse=False):
        self._shapes.sort(key=lambda s: s.area(), reverse=reverse)
        return self

    def __repr__(self):
        return f"Drawing({self._shapes})"


# --- Demo ---
if __name__ == "__main__":
    d = Drawing()
    d.add(Rectangle(3, 4)).add(Circle(2)).add(Rectangle(2, 6))

    print(len(d))           # 3
    print(d[0])             # Rectangle(3x4)

    # Polymorphism: each shape computes area differently
    for s in d:
        print(f"  {s} → area = {s.area():.2f}")

    # Comparison via magic methods
    print(Rectangle(3, 4) == Rectangle(2, 6))  # True (same area)
    print(Rectangle(3, 4) < Circle(3))          # True (12 < ~28.27)

    # Callable: sort by area
    d(reverse=True)          # sorts in-place, largest first
    print(d)

5. 注意事项

项目 说明
super() 在哪用 __init__ 中调父类初始化，方法重写中扩展父类逻辑
MRO 顺序 ClassName.__mro__ 查看方法解析顺序，多重继承时尤为重要
NotImplemented vs NotImplementedError 前者用于比较魔术方法返回，后者用于抽象方法
鸭子类型优先 不需要显式继承才能多态，只要有同名方法就行
__slots__ 限制实例属性，节省内存，但会阻止动态添加属性

6. 避坑指南

❌ 错误 ✅ 正确 原因
class Dog: 不调父 __init__ super().__init__(name) 父类属性不会自动初始化
多重继承菱形问题不管 用 super() 遵循 MRO 避免重复调用顶层 __init__
__eq__ 返回 False 对非同类 返回 NotImplemented 让 Python 尝试反向比较
忘记实现所有 @abstractmethod 确保每个抽象方法都有实现 否则子类也无法实例化
在 __init__ 外用 super() super() 可在任何方法中使用 不限于 __init__，方法重写也可以用

7. 练习题

🟢 入门：继承链

创建 Vehicle → Car → ElectricCar 三层继承，每层新增属性和方法。

参考答案

class Vehicle:
    def __init__(self, brand: str):
        self.brand = brand

    def describe(self):
        return f"A {self.brand} vehicle"

class Car(Vehicle):
    def __init__(self, brand: str, seats: int):
        super().__init__(brand)
        self.seats = seats

    def describe(self):
        return f"A {self.brand} car with {self.seats} seats"

class ElectricCar(Car):
    def __init__(self, brand: str, seats: int, range_km: int):
        super().__init__(brand, seats)
        self.range_km = range_km

    def describe(self):
        return f"An electric {self.brand} with {self.range_km}km range, {self.seats} seats"

print(ElectricCar("Tesla", 5, 500).describe())
# An electric Tesla with 500km range, 5 seats

🟡 中级：自定义 Vector 类（魔术方法）

实现 Vector 类支持 +、-、==、abs()、repr 和 len（维度数）。

参考答案

import math

class Vector:
    def __init__(self, *components):
        self.components = list(components)

    def __add__(self, other):
        if len(self) != len(other):
            raise ValueError("Dimension mismatch")
        return Vector(*(a + b for a, b in zip(self.components, other.components)))

    def __sub__(self, other):
        if len(self) != len(other):
            raise ValueError("Dimension mismatch")
        return Vector(*(a - b for a, b in zip(self.components, other.components)))

    def __abs__(self):
        return math.sqrt(sum(c ** 2 for c in self.components))

    def __eq__(self, other):
        if not isinstance(other, Vector):
            return NotImplemented
        return self.components == other.components

    def __len__(self):
        return len(self.components)

    def __repr__(self):
        return f"Vector({', '.join(map(str, self.components))})"

v1 = Vector(1, 2, 3)
v2 = Vector(4, 5, 6)
print(v1 + v2)    # Vector(5, 7, 9)
print(abs(v1))     # 3.7416...
print(len(v1))     # 3

8. 知识点总结

面向对象进阶
├── 继承 Inheritance
│   ├── class Child(Parent) → 单继承
│   ├── class Child(A, B) → 多重继承（注意 MRO）
│   ├── super() → 调用父类方法
│   └── isinstance() / issubclass() → 类型检查
├── 多态 Polymorphism
│   ├── 鸭子类型 → 不需要共同父类
│   ├── 方法重写 → 子类覆盖父类方法
│   └── 运行时绑定 → 根据实际类型调用
├── 魔术方法 Dunder
│   ├── 比较: __eq__, __lt__, __le__, __gt__, __ge__
│   ├── 算术: __add__, __sub__, __mul__, __matmul__
│   ├── 容器: __len__, __getitem__, __setitem__, __contains__
│   ├── 字符串: __repr__, __str__, __format__
│   ├── 可调用: __call__
│   └── 上下文: __enter__, __exit__ (with语句)
└── 抽象基类 ABC
    ├── from abc import ABC, abstractmethod
    ├── @abstractmethod → 强制子类实现
    └── 不能直接实例化
