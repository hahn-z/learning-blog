日志标准化与 ELK 集成

难度： ⭐⭐⭐
前置知识： Hyperf 日志组件、Docker 基础、JSON 格式
预估用时： 45-60 分钟

1. 概念讲解

ELK Stack（Elasticsearch + Logstash + Kibana）是业界最流行的日志收集、存储和可视化方案。Elasticsearch 负责存储和搜索，Logstash 负责日志收集和转换，Kibana 负责可视化展示。

类比理解：ELK 就像一个大型图书馆系统。Logstash 是图书管理员，负责收集各处送来的书（日志）并贴标签分类；Elasticsearch 是图书馆的索引柜，可以秒级找到任何一本书；Kibana 是前台的搜索终端，输入关键词就能看到结果。

在用户服务中，我们将日志格式标准化为 JSON，通过 Filebeat 采集发送到 Elasticsearch，用 Kibana 进行搜索和可视化。

3.2 TraceId 处理器 app/Log/TraceIdProcessor.php

<?php
declare(strict_types=1);

namespace App\Log;

use Monolog\LogRecord;
use Monolog\Processor\ProcessorInterface;
use Hyperf\Context\Context;
use Hyperf\Coroutine\Coroutine;

/**
 * ✅ Add trace_id and request context to every log entry
 */
class TraceIdProcessor implements ProcessorInterface
{
    public function __invoke(LogRecord $record): LogRecord
    {
        // ✅ Generate or reuse trace_id within request lifecycle
        $traceId = Context::get('trace_id', uniqid('tr_', true));
        Context::set('trace_id', $traceId);

        $record->extra['trace_id'] = $traceId;
        $record->extra['coroutine_id'] = Coroutine::id();

        return $record;
    }
}

3.3 结构化日志帮助类 app/Log/StructuredLogger.php

<?php
declare(strict_types=1);

namespace App\Log;

use Psr\Log\LoggerInterface;
use Hyperf\Di\Annotation\Inject;

/**
 * ✅ Structured logger with consistent field naming for ELK
 */
class StructuredLogger
{
    #[Inject]
    private LoggerInterface $logger;

    /**
     * ✅ Log user action with structured context
     */
    public function logUserAction(string $action, int $userId, array $context = []): void
    {
        $this->logger->info("User action: {$action}", array_merge([
            'action' => $action,
            'user_id' => $userId,
            'timestamp' => date('c'),
        ], $context));
    }

    /**
     * ✅ Log saga event
     */
    public function logSagaEvent(string $sagaId, string $step, string $event, array $context = []): void
    {
        $this->logger->info("Saga {$event}: {$step}", array_merge([
            'saga_id' => $sagaId,
            'saga_step' => $step,
            'saga_event' => $event,
        ], $context));
    }

    /**
     * ✅ Log error with full context for debugging
     */
    public function logError(string $message, \Throwable $e, array $context = []): void
    {
        $this->logger->error($message, array_merge([
            'exception_class' => get_class($e),
            'exception_message' => $e->getMessage(),
            'exception_file' => $e->getFile() . ':' . $e->getLine(),
        ], $context));
    }
}

3.4 中间件：为每个请求添加 TraceId app/Middleware/TraceIdMiddleware.php

<?php
declare(strict_types=1);

namespace App\Middleware;

use Hyperf\Context\Context;
use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;
use Psr\Http\Server\MiddlewareInterface;
use Psr\Http\Server\RequestHandlerInterface;

/**
 * ✅ Assign trace_id to every request for distributed tracing
 */
class TraceIdMiddleware implements MiddlewareInterface
{
    public function process(ServerRequestInterface $request, RequestHandlerInterface $handler): ResponseInterface
    {
        // ✅ Use X-Trace-Id header or generate new one
        $traceId = $request->getHeaderLine('X-Trace-Id') ?: uniqid('tr_', true);
        Context::set('trace_id', $traceId);

        $response = $handler->handle($request);

        // ✅ Return trace_id in response header for debugging
        return $response->withHeader('X-Trace-Id', $traceId);
    }
}

5. 注意事项

约束 说明 违反后果
日志必须用 JSON 格式 ELK 需要结构化数据 无法解析，搜索困难
每条日志含 trace_id 分布式追踪的基础 无法关联同一请求的日志
敏感数据不入日志 密码、token 等 安全泄露
日志文件做轮转 防止磁盘满 磁盘爆满服务崩溃
Elasticsearch 索引按天分割 便于清理和查询 单索引过大查询慢

6. 避坑指南

错误 现象 正确做法
日志用纯文本 无法按字段搜索 用 JSON Formatter
不加 trace_id 无法追踪请求链路 中间件注入 + Context 传递
大量 DEBUG 日志 ES 存储暴增 生产用 INFO 级别
Filebeat 路径错误 采集不到日志 确认容器内路径映射
不配索引生命周期 ES 磁盘持续增长 配 ILM 策略保留 30 天
