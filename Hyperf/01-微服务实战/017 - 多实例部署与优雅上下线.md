# 017 - 多实例部署与优雅上下线

> **难度：** ⭐⭐⭐⭐ | **前置知识：** Docker 基础、健康检查（016）、Nacos 服务注册（014）| **预估用时：** 60 分钟

## 概念讲解

**一句话定义：** 多实例部署是指同一服务运行多个副本提高可用性和吞吐量，优雅上下线则确保实例在停止前处理完所有请求，避免请求丢失或错误。

**现实类比：** 就像餐厅高峰期——多名服务员同时上岗（多实例），下班时不直接关门，而是接待完最后一位顾客、清理完桌子再打烊（优雅停机）。顾客不会遇到"服务员突然消失"的情况。

**技术场景：** 用户服务从 1 个实例扩展到 5 个实例应对流量高峰。更新版本时，先启动新实例，新实例就绪后再下线旧实例，整个过程用户无感知。旧实例停止时先拒绝新请求，处理完现有请求后再退出。

┌──────────────────────────────────────────────────────┐
│ 1. Receive SIGTERM (K8s / Docker stop command)       │

```

### 优雅停机处理器 — GracefulShutdownHandler.php

```php
<?php

declare(strict_types=1);

namespace App\Handler;

use Hyperf\Contract\ConfigInterface;
use Hyperf\Server\Event\MainServerStart;
use Psr\Log\LoggerInterface;
use Swoole\Coroutine as Co;
use Swoole\Process;
use Swoole\Timer;

class GracefulShutdownHandler
{
    private bool $shuttingDown = false;
    private int $inFlightRequests = 0;
    private int $shutdownTimeout = 30;  // 🟡 seconds to wait for in-flight requests

    public function __construct(
        private LoggerInterface $logger,
        private ConfigInterface $config
    ) {
        $this->shutdownTimeout = (int) $this->config->get('server.settings.graceful_shutdown_timeout', 30);
    }

    /**
     * ✅ Register signal handlers for graceful shutdown
     */
    public function register(): void
    {
        // 🟢 Listen for SIGTERM (sent by Docker/K8s)
        Process::signal(SIGTERM, [$this, 'handleShutdown']);

        // 🟡 Also handle SIGINT (Ctrl+C)
        Process::signal(SIGINT, [$this, 'handleShutdown']);

        $this->logger->info('Graceful shutdown handlers registered');
    }

    /**
     * ✅ Handle shutdown signal
     */
    public function handleShutdown(int $signo): void
    {
        if ($this->shuttingDown) {
            $this->logger->warning('Shutdown already in progress, ignoring signal');
            return;
        }

        $this->shuttingDown = true;
        $signal = $signo === SIGTERM ? 'SIGTERM' : 'SIGINT';

        $this->logger->info("Received {$signal}, starting graceful shutdown...");

        // 🟡 Step 1: Set readiness to DOWN (stop accepting new traffic)
        $this->setReadinessDown();

        // 🟢 Step 2: Wait for in-flight requests
        $this->waitForInFlightRequests();

        // 🟡 Step 3: Deregister from Nacos
        $this->deregisterFromNacos();

        // 🟡 Step 4: Close connections
        $this->closeConnections();

        // 🟢 Step 5: Stop the server
        $this->stopServer();
    }

    /**
     * ✅ Increment in-flight request counter
     */
    public function incrementInFlightRequests(): void
    {
        $this->inFlightRequests++;
    }

    /**
     * ✅ Decrement in-flight request counter
     */
    public function decrementInFlightRequests(): void
    {
        $this->inFlightRequests--;
    }

    /**
     * ✅ Set readiness to DOWN to stop accepting new traffic
     */
    private function setReadinessDown(): void
    {
        $this->logger->info('Setting readiness to DOWN...');

        // Update local health check state
        $ready = false;  // This flag will be read by /health/ready endpoint

        // Deregister from Nacos immediately to stop receiving new traffic
        $this->deregisterFromNacos();

        // Wait a moment for the deregistration to propagate
        Co::sleep(2);

        $this->logger->info('Readiness set to DOWN');
    }

    /**
     * ✅ Wait for in-flight requests to complete
     */
    private function waitForInFlightRequests(): void
    {
        $startTime = time();

        while ($this->inFlightRequests > 0) {
            $elapsed = time() - $startTime;

            if ($elapsed >= $this->shutdownTimeout) {
                $this->logger->warning("Shutdown timeout reached, forcing exit with {$this->inFlightRequests} in-flight requests");
                break;
            }

            $this->logger->info("Waiting for {$this->inFlightRequests} in-flight requests... ({$elapsed}s elapsed)");
            Co::sleep(1);
        }

        $this->logger->info('All in-flight requests completed');
    }

    /**
     * ✅ Deregister from Nacos
     */
    private function deregisterFromNacos(): void
    {
        $nacosHost = $this->config->get('nacos.host', '127.0.0.1');
        $nacosPort = $this->config->get('nacos.port', 8848);
        $serviceName = $this->config->get('app_name', 'user-service');
        $ip = $this->getLocalIp();
        $port = $this->config->get('server_port', 9501);

        $url = "http://{$nacosHost}:{$nacosPort}/nacos/v1/ns/instance";
        $data = http_build_query([
            'serviceName' => $serviceName,
            'ip' => $ip,
            'port' => $port,
            'ephemeral' => 'true',
        ]);

        $ctx = stream_context_create([
            'http' => [
                'method' => 'DELETE',
                'header' => 'Content-Type: application/x-www-form-urlencoded',
                'content' => $data,
            ],
        ]);

        $response = @file_get_contents($url, false, $ctx);

        if ($response === 'ok') {
            $this->logger->info('Deregistered from Nacos');
        } else {
            $this->logger->error('Failed to deregister from Nacos', ['response' => $response]);
        }
    }

    /**
     * ✅ Close database and Redis connections
     */
    private function closeConnections(): void
    {
        $this->logger->info('Closing connections...');

        // Close DB connections
        $dbPool = $this->container->get(\Hyperf\DbConnection\Pool\PoolFactory::class);
        // DB connections will be closed automatically when the process exits

        // Close Redis connections
        $redisPool = $this->container->get(\Hyperf\Redis\Pool\RedisPoolFactory::class);

        $this->logger->info('Connections closed');
    }

    /**
     * ✅ Stop the server
     */
    private function stopServer(): void
    {
        $this->logger->info('Stopping server...');

        // Stop the Swoole server
        $server = $this->container->get(\Swoole\Server::class);
        $server->shutdown();

        $this->logger->info('Server stopped gracefully');
    }

    private function getLocalIp(): string
    {
        // Get the IP address that should be used for Nacos registration
        return env('APP_INSTANCE_IP', gethostbyname(gethostname()));
    }
}
```

### 中间件 — 计数器 — InFlightMiddleware.php

```php
<?php

declare(strict_types=1);

namespace App\Middleware;

use App\Handler\GracefulShutdownHandler;
use Hyperf\Di\Annotation\Inject;
use Hyperf\HttpServer\Contract\RequestInterface;
use Hyperf\HttpServer\Contract\ResponseInterface;
use Psr\Container\ContainerInterface;
use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;
use Psr\Http\Server\MiddlewareInterface;
use Psr\Http\Server\RequestHandlerInterface;

class InFlightMiddleware implements MiddlewareInterface
{
    #[Inject]
    private GracefulShutdownHandler $shutdownHandler;

    public function process(ServerRequestInterface $request, RequestHandlerInterface $handler): ResponseInterface
    {
        // 🟡 Increment in-flight counter
        $this->shutdownHandler->incrementInFlightRequests();

        try {
            $response = $handler->handle($request);
            return $response;
        } finally {
            // 🟢 Always decrement in-flight counter
            $this->shutdownHandler->decrementInFlightRequests();
        }
    }
}

### 更新 HealthController 支持优雅停机
```php
<?php

declare(strict_types=1);

namespace App\Controller;

use App\Handler\GracefulShutdownHandler;
use Hyperf\Di\Annotation\Inject;
use Hyperf\HttpServer\Annotation\Controller;
use Hyperf\HttpServer\Annotation\GetMapping;

#[Controller(prefix: '/health')]
class HealthController
{
    private bool $ready = true;
    #[Inject]
    private GracefulShutdownHandler $shutdownHandler;

    /**
     * ✅ Readiness check — returns DOWN during graceful shutdown
     */
    #[GetMapping(path: '/ready')]
    public function readiness(): array
    {
        return [
            'status' => $this->ready ? 'UP' : 'DOWN',
            'timestamp' => time(),
            'reason' => $this->ready ? null : 'graceful_shutdown_in_progress',
        ];
    }

    /**
     * ✅ Allow external trigger for graceful shutdown
     */
    #[GetMapping(path: '/shutdown')]
    public function triggerShutdown(): array
    {
        // Only allow local requests
        if ($_SERVER['REMOTE_ADDR'] !== '127.0.0.1') {
            return ['code' => 403, 'message' => 'Forbidden'];
        }

        $this->ready = false;
        $this->shutdownHandler->handleShutdown(SIGTERM);

        return ['code' => 0, 'message' => 'Graceful shutdown triggered'];
    }
}
```

### 滚动更新脚本 — rolling-update.sh

```bash
#!/bin/bash
# ✅ Rolling update script for user-service

set -e

NACOS_HOST="localhost"
NACOS_PORT="8848"
SERVICE_NAME="user-service"
NEW_VERSION="2.0.0"
MAX_DOWNTIME=5  # Maximum instances allowed to be down

echo "🚀 Starting rolling update to version {$NEW_VERSION}..."

# Get current instances
INSTANCES=$(curl -s "http://${NACOS_HOST}:${NACOS_PORT}/nacos/v1/ns/instance/list?serviceName=${SERVICE_NAME}" | \
  jq -r '.hosts[] | "\(.ip):\(.port)"' 2>/dev/null || echo "")

if [ -z "$INSTANCES" ]; then
  echo "❌ No instances found in Nacos"
  exit 1
fi

INSTANCE_COUNT=$(echo "$INSTANCES" | wc -l)
echo "📋 Found {$INSTANCE_COUNT} instances"

# Update each instance one by one
INDEX=0
for INSTANCE in $INSTANCES; do
  INDEX=$((INDEX + 1))
  IP=$(echo "$INSTANCE" | cut -d: -f1)
  PORT=$(echo "$INSTANCE" | cut -d: -f2)

  echo ""
  echo "🔄 Updating instance {$INDEX}/{$INSTANCE_COUNT}: {$IP}:{$PORT}"

  # 1. Stop accepting new traffic (readiness = DOWN)
  echo "   Setting readiness to DOWN..."
  curl -s "http://${IP}:${PORT}/health/shutdown" > /dev/null

  # 2. Wait for in-flight requests
  echo "   Waiting for in-flight requests..."
  sleep 5

  # 3. Deploy new version
  echo "   Deploying new version..."
  ssh root@${IP} "docker pull user-service:${NEW_VERSION} && docker stop user-service && docker run -d --name user-service -p 9501:9501 user-service:${NEW_VERSION}"

  # 4. Wait for health check
  echo "   Waiting for health check..."
  for i in {1..30}; do
    if curl -sf "http://${IP}:${PORT}/health/ready" | grep -q '"status":"UP"'; then
      echo "   ✅ Instance healthy"
      break
    fi
    echo "   Waiting... ({$i}/30)"
    sleep 2
  done

  echo "   ✅ Instance updated"
done

echo ""
echo "🎉 Rolling update complete!"
```

## 注意事项

| 约束 | 说明 | 违反后果 |
|------|------|----------|
| 🔴 先停流量再停机 | 先设置 readiness=DOWN，等待流量清空 | 新请求被拒绝，请求丢失 |
| 🟡 停机超时 | 设置最大等待时间（30s），避免永久阻塞 | 请求挂起导致实例无法退出 |
| 🟢 注销 Nacos | 停机前主动注销，不要依赖心跳超时 | 其他服务继续调用已停止实例 |
| 🔴 关闭连接 | 停机前关闭 DB/Redis 连接，释放资源 | 连接泄露 |
| 🟡 滚动更新 | 每次只更新一部分实例（1/N），保留容量 | 全量更新导致服务不可用 |

## 避坑指南

| 错误 | 现象 | 正确做法 |
|------|------|----------|
| kill -9 强制停止 | 进程直接退出，请求丢失 | 发送 SIGTERM，等待优雅停机 |
| 未注销 Nacos | 实例停了但仍在注册中心 | 停机前主动注销 |
| 滚动更新太快 | 同时更新多个实例 | 每次只更新 1 个，等待就绪 |
| 未关闭连接 | DB/Redis 连接未释放 | 停机前显式关闭连接池 |
| 忘记超时设置 | 永久等待无法完成的请求 | 设置最大停机时间（30s） |
