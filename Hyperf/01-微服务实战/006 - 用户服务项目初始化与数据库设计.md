# 用户服务项目初始化与数据库设计

> **难度：** ⭐⭐
> **前置知识：** Hyperf 入门（001）、配置管理（005）、MySQL 基础
> **预估用时：** 50 分钟

---

## 一、概念讲解

**一句话定义：** 数据库设计是围绕业务需求抽象出实体、定义表结构、建立关系和索引的过程，是应用系统的数据基础。

**现实类比：** 设计用户服务就像规划一个图书馆——用户是读者，订单是借阅记录，积分是信用分。你需要设计读者卡（user 表）、借阅登记簿（order 表）、积分流水账（point_log 表），并建立索引让检索快速完成。

**技术场景：** 用户服务是整个微服务体系的"登录门户"，承载用户注册、登录、信息管理、权限验证。良好的数据库设计需要考虑：密码安全（hash + salt）、数据一致性（事务）、查询性能（索引）、扩展性（分表预留）。

## 三、完整代码

### 3.1 数据库迁移文件

```php
<?php
// ✅ database/migrations/2026_05_01_000001_create_users_table.php

declare(strict_types=1);

use Hyperf\Database\Schema\Blueprint;
use Hyperf\Database\Schema\Schema;

Schema::create('users', function (Blueprint $table) {
    // ✅ Primary key with BIGINT for scalability
    $table->bigIncrements('id')->comment('User ID');

    // 🟢 Authentication fields (indexed for login queries)
    $table->string('email', 100)->unique()->nullable()->comment('User email (login key)');
    $table->string('phone', 20)->unique()->nullable()->comment('User phone (login key)');
    $table->string('username', 50)->unique()->nullable()->comment('Username (display name)');

    // ✅ Password hash (never store plain text!)
    $table->string('password', 255)->nullable()->comment('Password hash (bcrypt)');

    // 🟢 User status and metadata
    $table->enum('status', ['active', 'inactive', 'banned'])->default('active');
    $table->timestamp('last_login_at')->nullable();
    $table->string('last_login_ip', 50)->nullable();

    // 🔴 Soft delete - don't actually delete user records
    $table->timestamp('deleted_at')->nullable();

    // ✅ Standard timestamps
    $table->timestamp('created_at')->useCurrent();
    $table->timestamp('updated_at')->useCurrentOnUpdate();

    // ✅ Indexes
    $table->index(['email', 'status'])->name('idx_email_status');
    $table->index('last_login_at')->name('idx_last_login');
});
```

```php
<?php
```
```php
// ✅ database/migrations/2026_05_01_000003_create_user_tokens_table.php

Schema::create('user_tokens', function (Blueprint $table) {
    $table->bigIncrements('id');
    
    // 🟢 Token belongs to a user
    $table->bigInteger('user_id')->unsigned()->comment('Foreign key to users.id');
    $table->foreign('user_id')->references('id')->on('users')->onDelete('cascade');

    // 🔴 Security: Store token hash, not plain token
    $table->string('token_hash', 255)->comment('Hashed token value');
    $table->enum('type', ['access', 'refresh'])->comment('Token type');

    // 🟢 Token expiration
    $table->timestamp('expires_at')->nullable()->comment('Token expiration time');
    
    // 🟢 For cleanup: delete expired tokens
    $table->boolean('revoked')->default(false)->comment('Whether token is revoked');

    // ✅ Index for fast token lookup
    $table->index(['user_id', 'type', 'revoked'])->name('idx_user_type_revoked');
    $table->index('expires_at')->name('idx_expires_at');
    
    $table->timestamp('created_at')->useCurrent();
});
```

### 3.2 User Model

```php
<?php
// ✅ app/Model/User.php

declare(strict_types=1);

namespace App\Model;

use Hyperf\DbConnection\Model\Model;

class User extends Model
{
    protected ?string $table = 'users';

    // ✅ Mass assignable fields (whitelist for security)
    protected array $fillable = [
        'email',
        'phone',
        'username',
        'password',
        'status',
        'last_login_at',
        'last_login_ip',
    ];

    // ✅ Hidden fields (never exposed in JSON serialization)
    protected array $hidden = [
        'password',
    ];

    // 🟢 Cast to native types
    protected array $casts = [
        'last_login_at' => 'datetime',
        'deleted_at' => 'datetime',
    ];

    // 🟢 One-to-one relationship with profile
    public function profile()
    {
        return $this->hasOne(UserProfile::class, 'user_id', 'id');
    }

    // 🟢 One-to-many relationship with tokens
    public function tokens()
    {
        return $this->hasMany(UserToken::class, 'user_id', 'id');
    }

    // 🔴 Helper: Check if user is active
    public function isActive(): bool
    {
        return $this->status === 'active';
    }

    // 🔴 Helper: Verify password
    public function verifyPassword(string $password): bool
    {
        return password_verify($password, $this->password);
    }

    // 🔴 Helper: Hash password before save
    public function setPasswordAttribute(string $password): void
    {
        $this->attributes['password'] = password_hash($password, PASSWORD_BCRYPT);
    }
}
```

### 3.4 UserToken Model

```php
<?php
// ✅ app/Model/UserToken.php

declare(strict_types=1);

namespace App\Model;

use Hyperf\DbConnection\Model\Model;

class UserToken extends Model
{
    protected ?string $table = 'user_tokens';

    protected array $fillable = [
        'user_id',
        'token_hash',
        'type',
        'expires_at',
        'revoked',
    ];

    protected array $casts = [
        'expires_at' => 'datetime',
        'revoked' => 'boolean',
    ];

    // 🔴 Create access token (expires in 2 hours)
    public static function createAccessToken(int $userId, string $token): self
    {
        return self::create([
            'user_id' => $userId,
            'token_hash' => hash('sha256', $token),
            'type' => 'access',
            'expires_at' => now()->addHours(2),
        ]);
    }

    // 🔴 Create refresh token (expires in 7 days)
    public static function createRefreshToken(int $userId, string $token): self
    {
        return self::create([
            'user_id' => $userId,
            'token_hash' => hash('sha256', $token),
            'type' => 'refresh',
            'expires_at' => now()->addDays(7),
        ]);
    }

    // 🔴 Check if token is expired
    public function isExpired(): bool
    {
        return $this->expires_at && $this->expires_at->isPast();
    }

    // 🔴 Verify token hash
    public function verifyToken(string $token): bool
    {
        return hash_equals($this->token_hash, hash('sha256', $token));
    }
}
```

## 五、注意事项

| 约束 | 说明 | 违反后果 |
|------|------|----------|
| 密码必须 hash 存储 | 明文密码泄露 = 所有用户被盗 | 数据库泄露后直接接管账号 |
| 外键约束必须设置 | 保证数据完整性 | 孤儿记录，数据不一致 |
| 索引要合理 | 查询字段建索引 | 登录查询慢，数据库负载高 |
| 软删除优于硬删除 | deleted_at 标记删除 | 历史数据丢失，无法恢复 |
| 字段长度要预留 | varchar 考虑未来扩展 | 数据截断，报错 |

---

## 六、避坑指南

| 典型错误 | 现象 | 正确做法 |
|----------|------|----------|
| ❌ 直接保存密码明文 | 用户密码泄露到 DB | 始终 `password_hash()` 存储 |
| ❌ 删除用户用 `DELETE` | 数据永久丢失 | 用 `deleted_at` 软删除 |
| ❌ 忘记建索引 | 登录查询全表扫描 | email/phone 登录字段建索引 |
| ❌ 用 `text` 存手机号 | 占用空间大 | 用 `varchar(20)` 足够 |
| ❌ 表名用复数 | Hyperf 约定是单数 | Model 自动加 `s`，表用复数 |

## 十一、代码演进

### v1 ❌ 简单粗暴的表设计

```sql
-- ❌ No indexes, no foreign keys, plain password
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(255),
  password VARCHAR(255),
  name VARCHAR(100)
);
```

### v2 ✅ 规范化设计

```sql
-- ✅ With indexes, foreign keys, soft delete
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(100) UNIQUE,
  password VARCHAR(255),
  status ENUM('active', 'banned') DEFAULT 'active',
  deleted_at TIMESTAMP NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_email (email)
);
```

### v3 🟢 生产级设计（含安全 + 性能）

```sql
-- 🟢 Full design with relationships, token management, and security
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(100) UNIQUE COMMENT 'User email for login',
  phone VARCHAR(20) UNIQUE COMMENT 'User phone for login',
  username VARCHAR(50) UNIQUE COMMENT 'Display username',
  password VARCHAR(255) COMMENT 'Password hash (bcrypt)',
  status ENUM('active', 'inactive', 'banned') DEFAULT 'active',
  last_login_at TIMESTAMP NULL COMMENT 'Last successful login time',
  last_login_ip VARCHAR(50) NULL COMMENT 'Last login IP address',
  deleted_at TIMESTAMP NULL COMMENT 'Soft delete timestamp',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_email_status (email, status),
  INDEX idx_last_login (last_login_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='User master table';
```
