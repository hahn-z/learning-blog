链路上下文传播与跨服务调用追踪

难度标注

⭐⭐⭐⭐（中高级）

前置知识： 分布式链路追踪原理、Jaeger 部署、Hyperf HTTP 客户端

预估用时： 50-60 分钟

概念讲解

链路上下文传播（Context Propagation） 是指在微服务调用链中，将当前请求的追踪信息（Trace ID、Span ID 等）通过协议头或消息体传递给下游服务，使得所有服务产生的 Span 能够串联成一条完整的追踪链路。

现实类比： 你去医院看病，挂号时给你一个病历号（Trace ID）。你拿着病历号去看内科，内科医生在病历上记录（Span）后写上"转外科"。外科医生看到病历号，接着记录。如果每个人不传病历号，每个科室各记各的，你的看病过程就无法追溯。

技术场景： 用户服务通过 HTTP 调用订单服务、通过 gRPC 调用认证服务、通过 Kafka 发送消息给通知服务。三种协议、三种传输方式，但都需要传播同一个 Trace Context，才能在 Jaeger 中看到完整的调用链。

完整代码

1. HTTP 上下文传播中间件

<?php
// app/Middleware/TracePropagationMiddleware.php

declare(strict_types=1);

namespace App\Middleware;

use Hyperf\Context\Context;
use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;
use Psr\Http\Server\MiddlewareInterface;
use Psr\Http\Server\RequestHandlerInterface;

class TracePropagationMiddleware implements MiddlewareInterface
{
    // ✅ W3C Trace Context header
    private const TRACE_PARENT = 'traceparent';

    public function process(
        ServerRequestInterface $request,
        RequestHandlerInterface $handler
    ): ResponseInterface {
        // ✅ Extract traceparent from incoming request
        $traceParent = $request->getHeaderLine(self::TRACE_PARENT);

        if ($traceParent) {
            $context = $this->parseTraceParent($traceParent);
            if ($context) {
                // 🟢 Propagate existing trace
                Context::set('trace.trace_id', $context['trace_id']);
                Context::set('trace.parent_span_id', $context['parent_span_id']);
            }
        }

        // ✅ Generate new span ID for this service
        Context::set('trace.span_id', bin2hex(random_bytes(8)));

        // If no trace_id, create one (root span)
        if (!Context::has('trace.trace_id')) {
            Context::set('trace.trace_id', bin2hex(random_bytes(16)));
        }

        // ✅ Continue processing
        $response = $handler->handle($request);

        // 🟢 Add trace headers to response for debugging
        return $response->withHeader(
            'X-Trace-Id',
            Context::get('trace.trace_id', '')
        );
    }

    /**
     * ✅ Parse W3C traceparent: version-traceid-spanid-flags
     */
    private function parseTraceParent(string $header): ?array
    {
        $parts = explode('-', $header);
        if (count($parts) !== 4) {
            return null;
        }
        return [
            'trace_id' => $parts[1],
            'parent_span_id' => $parts[2],
            'flags' => $parts[3],
        ];
    }
}

2. HTTP 客户端传播（出站）

<?php
// app/Service/HttpClient/TracePropagationPlugin.php

declare(strict_types=1);

namespace App\Service\HttpClient;

use GuzzleHttp\Psr7\Request;
use Hyperf\Context\Context;
use Hyperf\Guzzle\CoroutineHandler;

class TracePropagationPlugin
{
    /**
     * ✅ Inject trace context into outgoing HTTP requests
     */
    public static function injectTraceHeaders(array $options): array
    {
        $traceId = Context::get('trace.trace_id');
        $spanId = Context::get('trace.span_id');

        if ($traceId) {
            // ✅ Build W3C traceparent for downstream
            $traceParent = sprintf(
                '00-%s-%s-01',
                $traceId,
                $spanId ?? bin2hex(random_bytes(8))
            );

            $options['headers'] = array_merge($options['headers'] ?? [], [
                'traceparent' => $traceParent,
                'X-Trace-Id' => $traceId,
            ]);
        }

        return $options;
    }
}

// app/Service/OrderClientService.php
<?php

declare(strict_types=1);

namespace App\Service;

use App\Service\HttpClient\TracePropagationPlugin;
use Hyperf\Guzzle\ClientFactory;

class OrderClientService
{
    public function __construct(
        private ClientFactory $clientFactory
    ) {}

    public function getUserOrders(int $userId): array
    {
        $client = $this->clientFactory->create([
            'base_uri' => 'http://order-service:9502',
        ]);

        // ✅ Propagate trace context to order service
        $options = TracePropagationPlugin::injectTraceHeaders([
            'headers' => ['Accept' => 'application/json'],
        ]);

        $response = $client->get("/api/orders/user/{$userId}", $options);

        return json_decode($response->getBody()->getContents(), true);
    }
}

3. gRPC 上下文传播

<?php
// app/GRPC/Client/GrpcTraceInterceptor.php

declare(strict_types=1);

namespace App\GRPC\Client;

use Hyperf\Context\Context;
use Hyperf\GrpcClient\BaseClient;

class GrpcTraceInterceptor
{
    /**
     * ✅ Create gRPC metadata with trace context
     */
    public static function injectTraceMetadata(): array
    {
        $traceId = Context::get('trace.trace_id');
        $spanId = Context::get('trace.span_id');

        if (!$traceId) {
            return [];
        }

        // ✅ gRPC uses metadata (similar to HTTP headers)
        return [
            'traceparent' => [sprintf('00-%s-%s-01', $traceId, $spanId ?? bin2hex(random_bytes(8)))],
            'x-trace-id' => [$traceId],
        ];
    }
}

4. Kafka 消息传播

<?php
// app/Producer/TraceAwareProducer.php

declare(strict_types=1);

namespace App\Producer;

use Hyperf\Context\Context;
use Hyperf\Kafka\Producer;

class TraceAwareProducer
{
    public function __construct(
        private Producer $producer
    ) {}

    /**
     * ✅ Send Kafka message with trace context in headers
     */
    public function send(string $topic, string $key, array $payload): void
    {
        $traceId = Context::get('trace.trace_id');
        $spanId = Context::get('trace.span_id');

        // ✅ Embed trace context in Kafka message headers
        $headers = [];
        if ($traceId) {
            $headers['traceparent'] = sprintf(
                '00-%s-%s-01',
                $traceId,
                $spanId ?? bin2hex(random_bytes(8))
            );
        }

        $this->producer->send($topic, [
            new \Hyperf\Kafka\Message(
                topicName: $topic,
                key: $key,
                value: json_encode($payload),
                headers: $headers,
            ),
        ]);
    }
}

// app/Consumer/TraceAwareConsumer.php
<?php

declare(strict_types=1);

namespace App\Consumer;

use Hyperf\Context\Context;
use Hyperf\Kafka\Annotation\Consumer;
use Hyperf\Kafka\Message\ConsumerMessage;

#[Consumer(topic: 'user-events', groupId: 'notification-service')]
class UserEventConsumer extends ConsumerMessage
{
    public function consume(\Hyperf\Kafka\Message $message): void
    {
        // ✅ Extract trace context from Kafka message headers
        $headers = $message->getHeaders();
        $traceParent = $headers['traceparent'] ?? null;

        if ($traceParent) {
            $parts = explode('-', $traceParent);
            if (count($parts) === 4) {
                Context::set('trace.trace_id', $parts[1]);
                Context::set('trace.parent_span_id', $parts[2]);
            }
        }

        // 🟢 New span ID for consumer processing
        Context::set('trace.span_id', bin2hex(random_bytes(8)));

        $payload = json_decode($message->getValue(), true);
        logger()->info('Processing user event', [
            'trace_id' => Context::get('trace.trace_id'),
            'event' => $payload,
        ]);

        // Process event...
    }
}

注意事项

约束 说明 违反后果
三种协议都要传播 HTTP/gRPC/Kafka 各有传播方式 链路在某一步断裂
traceparent 格式严格 W3C 规范：00-{32hex}-{16hex}-{2hex} 解析失败，创建新 Trace
消费端必须重新生成 Span ID 否则多个服务共享同一个 Span ID Jaeger 中 Span 冲突
异步消息的 Trace 生命周期长 Kafka 消息可能延迟消费 注意 Span 的 parent 关系用 FollowsFrom
不要在日志中遗漏 Trace ID 所有日志关联 Trace ID 无法通过日志追踪问题

避坑指南

典型错误 现象 正确做法
HTTP 客户端不传 traceparent 下游服务看到新 Trace 封装统一的 HTTP 客户端插件
gRPC 用 HTTP header 传追踪 Metadata 中找不到 gRPC 用独立的 Metadata 机制
Kafka 不传 headers 消费端无法关联 在 Producer 中注入 traceparent
Context 在协程间泄漏 Swoole 协程切换后 Trace 错乱 用 Hyperf Context 而非全局变量
Span ID 不重新生成 父子关系错误 每个服务/操作生成新的 Span ID

练习题

🟢 基础题

  1. 实现 HTTP 出站传播：封装一个 Guzzle Middleware，自动在所有出站请求中注入 traceparent。
    1. 验证传播正确性：用两个 Hyperf 服务，在 Jaeger 中确认跨服务 Span 的父子关系。

      2. 🟡 进阶题

      1. 实现 baggage 传播：除了 traceparent，额外传播一个自定义的 x-tenant-id baggage，在所有服务中可用。

        2. 🔴 开放题

        1. 设计统一传播层：抽象一个 PropagationInterface，支持 HTTP、gRPC、Kafka、Redis 命令等多种载体，一处配置全局生效。

          举一反三

          场景 变种说明 关键差异
          同步 HTTP → 异步事件 同步用 ChildOf，异步用 FollowsFrom 关系类型不同，影响 Jaeger 展示
          W3C → B3 传播格式 兼容旧 Zipkin 系统 格式不同但内容相同
          单语言 → 多语言传播 PHP 调 Go 调 Java 协议一致即可，语言无关
