# 服务治理组合实战（熔断+降级+限流联动）

## 难度标注

⭐⭐⭐⭐⭐（高级）

**前置知识：** 熔断器、降级策略、限流算法、Hyperf 中间件

**预估用时：** 60-70 分钟

---

## 概念讲解

**服务治理组合** 是将熔断、降级、限流三种机制协同工作，形成多层次的系统保护网。限流是第一道防线（控制进入的流量），熔断是第二道（快速失败不等待），降级是第三道（提供替代方案）。三者联动才能实现从"预防"到"容灾"的完整保护。

**现实类比：** 大坝防洪系统。限流 = 上游分洪闸（控制来水量），熔断 = 自动泄洪闸（水位太高自动开闸放水），降级 = 应急预案（水淹了农田但保住了城区）。三者配合：先控水（限流），撑不住就泄洪（熔断），最坏情况保核心（降级）。

**技术场景：** 用户服务在秒杀活动中突然涌入大量请求。首先限流器按 500 QPS 截流，超过的直接返回 429。进入的 500 QPS 中调用订单服务的部分，如果订单服务开始超时，熔断器 5 次失败后打开，后续请求快速返回。熔断触发的同时，降级策略生效，返回缓存的订单数据。三层防护确保核心功能不挂。

### 2. 组合治理中间件

```php
<?php
// app/Middleware/GovernanceMiddleware.php

declare(strict_types=1);

namespace App\Middleware;

use App\Service\Governance\GovernanceManager;
use App\Service\RateLimit\DistributedRateLimiter;
use Hyperf\HttpMessage\Exception\TooManyRequestsHttpException;
use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;
use Psr\Http\Server\MiddlewareInterface;
use Psr\Http\Server\RequestHandlerInterface;

class GovernanceMiddleware implements MiddlewareInterface
{
    public function __construct(
        private DistributedRateLimiter $limiter,
        private GovernanceManager $governance
    ) {}

    public function process(
        ServerRequestInterface $request,
        RequestHandlerInterface $handler
    ): ResponseInterface {
        $path = $request->getUri()->getPath();

        // ✅ Layer 1: Rate limiting
        $rateLimitResult = $this->checkRateLimit($path, $request);
        if (!$rateLimitResult->allowed) {
            throw new TooManyRequestsHttpException(
                $rateLimitResult->retryAfter > 0 ? $rateLimitResult->retryAfter : null,
                'Rate limit exceeded'
            );
        }

        // ✅ Layer 2 & 3 handled in service layer via annotations
        $response = $handler->handle($request);

        // 🟢 Add governance headers for observability
        return $response
            ->withHeader('X-RateLimit-Remaining', (string)$rateLimitResult->remaining)
            ->withHeader('X-Governance-Level', $this->governance->getCurrentLevel()->value);
    }

    private function checkRateLimit(string $path, ServerRequestInterface $request)
    {
        $config = config('governance.rate_limit.routes.' . $path)
            ?? config('governance.rate_limit.global');

        if ($config === null) {
            return new \App\Service\RateLimit\RateLimitResult(true, 999, 999, 0);
        }

        $ip = $request->getServerParams()['remote_addr'] ?? 'unknown';
        return $this->limiter->slidingWindow($path, $ip, $config['limit'], $config['window']);
    }
}
```

### 3. 治理管理器

```php
<?php
// app/Service/Governance/GovernanceManager.php

declare(strict_types=1);

namespace App\Service\Governance;

use App\Enum\DegradationLevel;
use App\Service\DegradationManager;
use Hyperf\Context\Context;
use Hyperf\Di\Annotation\Inject;

class GovernanceManager
{
    #[Inject]
    private DegradationManager $degradationManager;

    /**
     * ✅ Get current degradation level
     */
    public function getCurrentLevel(): DegradationLevel
    {
        return $this->degradationManager->getCurrentLevel();
    }

    /**
     * ✅ Check if feature is available at current governance level
     */
    public function isFeatureAvailable(string $feature): bool
    {
        return $this->degradationManager->isFeatureAvailable($feature);
    }

    /**
     * ✅ Execute call with full governance protection
     * Order: check degradation → check circuit breaker → execute → fallback
     */
    public function protectedCall(
        string $service,
        string $feature,
        callable $primary,
        ?callable $fallback = null
    ): mixed {
        // ✅ Check if feature is available (degradation layer)
        if (!$this->isFeatureAvailable($feature)) {
            return $fallback ? $fallback() : null;
        }

        try {
            return $primary();
        } catch (\Throwable $e) {
            // 🟡 Circuit breaker state managed by @CircuitBreaker annotation
            // This is the degradation layer fallback
            logger()->warning('Governance protected call failed', [
                'service' => $service,
                'feature' => $feature,
                'error' => $e->getMessage(),
            ]);

            return $fallback ? $fallback() : null;
        }
    }

    /**
     * ✅ Get governance dashboard data
     */
    public function getDashboard(): array
    {
        return [
            'degradation_level' => $this->getCurrentLevel()->value,
            'features' => [
                'user_profile' => $this->isFeatureAvailable('user_profile'),
                'order_stats' => $this->isFeatureAvailable('order_stats'),
                'points_query' => $this->isFeatureAvailable('points_query'),
                'recommendations' => $this->isFeatureAvailable('recommendations'),
            ],
        ];
    }
}
```

### 4. 用户服务完整治理集成

```php
<?php
// app/Service/UserGovernanceService.php

declare(strict_types=1);

namespace App\Service;

use App\Enum\DegradationLevel;
use App\Service\Governance\GovernanceManager;
use App\Service\RateLimit\DistributedRateLimiter;
use Hyperf\CircuitBreaker\Annotation\CircuitBreaker;
use Hyperf\Di\Annotation\Inject;
use Hyperf\Redis\Redis;

class UserGovernanceService
{
    #[Inject]
    private GovernanceManager $governance;

    #[Inject]
    private OrderClientService $orderClient;

    #[Inject]
    private CacheService $cacheService;

    #[Inject]
    private Redis $redis;

    /**
     * ✅ Full governance: rate limit + circuit breaker + degradation
     * Rate limit handled by middleware
     * Circuit breaker via annotation
     * Degradation via governance manager
     */
    #[CircuitBreaker(
        name: 'order-service',
        failCounter: 5,
        successCounter: 3,
        duration: 30,
        fallback: 'getUserOrdersGovernance'
    )]
    public function getUserOrders(int $userId): array
    {
        // ✅ Check degradation level first
        if (!$this->governance->isFeatureAvailable('order_stats')) {
            return $this->getFallbackOrders($userId);
        }

        return $this->orderClient->getUserOrders($userId);
    }

    /**
     * ✅ Fallback: circuit breaker + degradation combined
     */
    public function getUserOrdersGovernance(int $userId, ?\Throwable $e = null): array
    {
        // Log the fallback trigger reason
        logger()->warning('Order fetch fallback triggered', [
            'user_id' => $userId,
            'circuit_breaker' => $e ? 'open' : 'degraded',
            'error' => $e?->getMessage(),
        ]);

        // ✅ Try cache fallback first
        $cached = $this->cacheService->getCachedOrderStats($userId);
        if ($cached && !isset($cached['no_data'])) {
            $cached['fallback'] = true;
            $cached['fallback_reason'] = 'circuit_open';
            return $cached;
        }

        // ✅ Safe default response
        return [
            'user_id' => $userId,
            'orders' => [],
            'total' => 0,
            'fallback' => true,
            'fallback_reason' => 'service_unavailable',
            'message' => 'Order service temporarily unavailable',
        ];
    }

    private function getFallbackOrders(int $userId): array
    {
        return [
            'user_id' => $userId,
            'orders' => [],
            'fallback' => true,
            'fallback_reason' => 'feature_degraded',
        ];
    }
}
```

### 5. 治理监控接口

```php
<?php
// app/Controller/GovernanceController.php

declare(strict_types=1);

namespace App\Controller;

use App\Enum\DegradationLevel;
use App\Service\CircuitBreakerManager;
use App\Service\DegradationManager;
use App\Service\Governance\GovernanceManager;
use Hyperf\Di\Annotation\Inject;
use Hyperf\HttpServer\Annotation\Controller;
use Hyperf\HttpServer\Annotation\GetMapping;
use Hyperf\HttpServer\Annotation\PutMapping;

#[Controller(prefix: '/api/governance')]
class GovernanceController
{
    #[Inject]
    private GovernanceManager $governance;

    #[Inject]
    private DegradationManager $degradationManager;

    #[Inject]
    private CircuitBreakerManager $cbManager;

    // ✅ Full governance dashboard
    #[GetMapping(path: '')]
    public function dashboard(): array
    {
        return [
            'governance' => $this->governance->getDashboard(),
            'circuit_breakers' => $this->cbManager->getAllStates(),
        ];
    }

    // ✅ Update degradation level
    #[PutMapping(path: '/degradation/{level}')]
    public function setDegradation(string $level): array
    {
        $degLevel = DegradationLevel::tryFrom($level);
        if ($degLevel === null) {
            return ['error' => 'Invalid level'];
        }
        $this->degradationManager->setLevel($degLevel);
        return ['success' => true, 'level' => $degLevel->value];
    }

    // ✅ Reset circuit breaker
    #[PutMapping(path: '/circuit-breaker/{name}/reset')]
    public function resetBreaker(string $name): array
    {
        return ['success' => $this->cbManager->reset($name)];
    }
}
```

## 注意事项

| 约束 | 说明 | 违反后果 |
|------|------|----------|
| 三层顺序不能乱 | 限流→熔断→降级，从外到内 | 内层保护被流量冲破 |
| 状态要共享 | 多实例共享 Redis 中的限流和降级状态 | 各实例行为不一致 |
| 监控要全面 | 每层都要有指标和告警 | 不知道哪层在起作用 |
| 回复要有标识 | 响应中标注哪层触发了保护 | 前端无法正确展示 |
| 降级可以手动触发 | 知道大促要来了提前降级 | 来不及自动降级就挂了 |

## 避坑指南

| 典型错误 | 现象 | 正确做法 |
|----------|------|----------|
| 三层独立配置不协调 | 限流放行但熔断阈值太低 | 统一在 governance.php 配置 |
| 熔断 fallback 又调原服务 | 死循环 | fallback 只返回缓存/默认值 |
| 降级忘了通知前端 | 用户看到空白页 | 响应加 degraded 标识 |
| 限流 Key 粒度不对 | 限了所有用户 | 按 user_id 或 IP 限 |
| 没有 admin 接口 | 出问题只能改代码重启 | 提供动态配置 API |

## 练习题

### 🟢 基础题

1. **配置三层防护**：为一个新接口配置限流（100 QPS）+ 熔断（3 次失败）+ 降级（缓存兜底）。

2. **测试联动效果**：模拟下游服务不可用，观察三层防护的触发顺序和恢复过程。

### 🟡 进阶题

3. **自动化运维脚本**：写一个脚本，监控 governance dashboard，当 circuit breaker 频繁打开时自动提升降级级别。

### 🔴 开放题

4. **设计自适应服务治理**：基于 Prometheus 指标（错误率、延迟 P99、CPU），自动调整限流阈值、熔断参数和降级级别，实现闭环自动治理。

## 知识点总结

```
服务治理组合
├── 三层防护体系
│   ├── Layer 1: Rate Limiting（流量入口）
│   │   ├── 滑动窗口限流
│   │   ├── 令牌桶限流
│   │   └── 分级限流（IP/用户/接口）
│   ├── Layer 2: Circuit Breaker（调用保护）
│   │   ├── 三态模型
│   │   ├── Fallback 机制
│   │   └── 自动恢复
│   └── Layer 3: Degradation（降级兜底）
│       ├── 功能降级
│       ├── 数据降级（缓存）
│       └── 分级降级
├── 配置中心
│   ├── governance.php 统一配置
│   ├── 动态配置（Redis）
│   └── Admin API 热更新
└── 可观测性
    ├── Dashboard 接口
    ├── 响应 Headers
    └── 日志与指标
```
