# 用户服务实战：注册+初始化的分布式事务

> **难度：** ⭐⭐⭐⭐⭐
> **前置知识：** Saga 模式（033/034篇）、Hyperf 路由和控制器、Kafka
> **预估用时：** 60-90 分钟

## 1. 概念讲解

本篇将分布式事务理论落地到真实场景：**用户注册**。在微服务架构中，注册涉及多个服务：用户服务创建账号、积分服务初始化、通知服务发邮件。任何环节失败都需要回滚已完成的操作。

类比理解：就像开银行卡——填表（创建用户）、开户（初始化积分）、送礼品（欢迎邮件）。如果礼品送不了，柜员会把表撕了、户销了，不留半成品。

## 2. 实时脑图

```
┌─────────────────────────────────────────────────────┐
│      User Registration Saga - Full Flow              │
├─────────────────────────────────────────────────────┤
│                                                      │
│  POST /api/users/register                            │
│           |                                          │
│           v                                          │
│  [Validation] -- invalid -> 400                      │
│           |                                          │
│           v valid                                    │
│  [Saga: user_registration]                           │
│    |-- Step1: CreateUser (local DB tx)               │
│    |   +-- compensate: soft-delete                   │
│    |-- Step2: InitPoints (local DB tx)               │
│    |   +-- compensate: delete points record          │
│    +-- Step3: PublishEvent (Kafka send)              │
│        +-- compensate: send cancel event             │
│           |                                          │
│           v                                          │
│  [Kafka: user.registered]                            │
│    |-- EmailConsumer -> send welcome email           │
│    +-- StatsConsumer -> record registration stats    │
│                                                      │
│  State Flow:                                         │
│  RUNNING -> COMPLETED -> user.status = 'active'     │
│  RUNNING -> COMPENSATING -> user.status = 'cancelled'│
└─────────────────────────────────────────────────────┘
```

## 3. 完整代码

### 3.1 注册请求 DTO `app/Dto/RegisterRequest.php`

```php
<?php
declare(strict_types=1);

namespace App\Dto;

class RegisterRequest
{
    public function __construct(
        public readonly string $username,
        public readonly string $email,
        public readonly string $password,
    ) {}

    /** ✅ Validate input */
    public function validate(): array
    {
        $errors = [];
        if (strlen($this->username) < 3 || strlen($this->username) > 50) {
            $errors[] = 'Username must be 3-50 characters';
        }
        if (!filter_var($this->email, FILTER_VALIDATE_EMAIL)) {
            $errors[] = 'Invalid email format';
        }
        if (strlen($this->password) < 8) {
            $errors[] = 'Password must be at least 8 characters';
        }
        return $errors;
    }

    public static function fromArray(array $data): self
    {
        return new self(
            username: $data['username'] ?? '',
            email: $data['email'] ?? '',
            password: $data['password'] ?? '',
        );
    }

    public function toContext(): array
    {
        return ['username' => $this->username, 'email' => $this->email, 'password' => $this->password];
    }
}
```

### 3.2 注册控制器 `app/Controller/RegisterController.php`

```php
<?php
declare(strict_types=1);

namespace App\Controller;

use App\Dto\RegisterRequest;
use App\Saga\SagaOrchestrator;
use App\Saga\Steps\CreateUserStep;
use App\Saga\Steps\InitPointsStep;
use App\Saga\Steps\PublishRegistrationEventStep;
use App\Repository\UserRepository;
use Hyperf\Di\Annotation\Inject;
use Hyperf\HttpServer\Annotation\Controller;
use Hyperf\HttpServer\Annotation\PostMapping;
use Hyperf\HttpServer\Contract\RequestInterface;

#[Controller(prefix: '/api')]
class RegisterController
{
    #[Inject] private SagaOrchestrator $sagaOrchestrator;
    #[Inject] private CreateUserStep $createUserStep;
    #[Inject] private InitPointsStep $initPointsStep;
    #[Inject] private PublishRegistrationEventStep $publishEventStep;
    #[Inject] private UserRepository $userRepository;

    #[PostMapping('/register')]
    public function register(RequestInterface $request): array
    {
        $dto = RegisterRequest::fromArray($request->all());

        // ✅ Input validation
        $errors = $dto->validate();
        if (!empty($errors)) {
            return ['code' => 400, 'message' => 'Validation failed', 'errors' => $errors];
        }

        // ✅ Check duplicate
        if ($this->userRepository->existsByEmail($dto->email)) {
            return ['code' => 409, 'message' => 'Email already registered'];
        }

        // ✅ Execute registration saga
        $result = $this->sagaOrchestrator->execute('user_registration', [
            $this->createUserStep,
            $this->initPointsStep,
            $this->publishEventStep,
        ], $dto->toContext());

        if ($result->success) {
            $this->userRepository->activate($result->data['user_id']);
            return ['code' => 0, 'message' => 'Registration successful',
                'data' => ['user_id' => $result->data['user_id'], 'username' => $dto->username]];
        }

        return ['code' => 500, 'message' => 'Registration failed, please try again'];
    }
}
```

### 3.3 Kafka 事件发布步骤 `app/Saga/Steps/PublishRegistrationEventStep.php`

```php
<?php
declare(strict_types=1);

namespace App\Saga\Steps;

use App\Kafka\Producer\ReliableProducer;
use App\Saga\SagaStepInterface;
use Hyperf\Di\Annotation\Inject;

class PublishRegistrationEventStep implements SagaStepInterface
{
    #[Inject] private ReliableProducer $producer;

    public function getName(): string { return 'publish_registration_event'; }

    public function execute(array $context): array
    {
        // ✅ Publish user.registered event
        $this->producer->reliableSend('user.registered', [
            'user_id' => $context['user_id'],
            'email' => $context['email'],
            'username' => $context['username'],
            'registered_at' => date('c'),
        ], (string) $context['user_id']);
        return $context;
    }

    public function compensate(array $context): void
    {
        // ✅ Publish cancellation event for downstream consumers
        if (isset($context['user_id'])) {
            $this->producer->reliableSend('user.registration.cancelled', [
                'user_id' => $context['user_id'],
                'cancelled_at' => date('c'),
            ], (string) $context['user_id']);
        }
    }
}
```

## 5. 注意事项

| 约束 | 说明 | 违反后果 |
|------|------|----------|
| email 唯一索引 | 防止重复注册 | 同一邮箱注册多个账号 |
| 用户状态生命周期 | pending -> active -> cancelled | 状态混乱 |
| context 不传密码明文 | 安全风险 | 密码泄露到日志 |
| 补偿考虑并发 | 同 email 可能并发注册 | 误删其他请求的数据 |
| 注册接口限流 | 防止批量注册 | 系统被刷 |

## 6. 避坑指南

| 错误 | 现象 | 正确做法 |
|------|------|----------|
| 不检查 email 重复 | 同一邮箱注册两次 | 先查后插 + 唯一索引 |
| 密码明文存 context | 日志泄露密码 | 只传必要字段 |
| 不激活用户 | status 还是 pending | Saga 成功后显式 activate |
| 补偿硬删除 | 审计丢失 | 软删除 status=cancelled |
| 没有请求限流 | 接口被刷爆 | Redis 限流 |

## 8. 知识点总结

```
User Registration Saga
├── Input: RegisterRequest DTO + validation
├── Steps: CreateUser -> InitPoints -> PublishEvent
├── Post-Saga: activate user
├── Failure: auto-compensation + cancel event
└── DB: users, user_points, saga_instances, saga_steps
```

## 9. 举一反三

| 场景 | Saga 步骤 | 关键差异 |
|------|-----------|----------|
| 用户注销 | 标记注销->清积分->发通知 | 需冷却期 |
| 密码重置 | 验证token->改密码->清缓存 | Token 有效期 |
| 升级VIP | 扣费->更新等级->开通特权 | 涉及支付需强一致 |

```php
$user = User::create($data);

    Points::create(['user_id' => $user->id]);
```

```php
    throw $e;
}
```
